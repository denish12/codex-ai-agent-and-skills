# Skill: UX Discovery

## Цель
Собрать недостающий UX контекст перед оформлением UX Spec.

## Когда использовать
- Сразу после получения PRD
- При изменении требований/ролей/потоков

## Выход
- Список уточняющих вопросов (5–15), отсортированных по критичности
- Допущения, если без них нельзя двигаться
- Черновой список экранов/потоков

## Вопросы (задавай релевантное)
### Платформа и layout
- Это desktop-first, mobile-first или равнозначно responsive?
- Ключевые брейкпоинты/минимальные размеры?

### Роли и права
- Какие роли пользователей? Что запрещено каждой роли?
- Нужна ли админка/модерация?

### Навигация и структура
- Какие основные разделы?
- Есть ли deep linking/шаринг ссылок?

### Бренд/визуальные ожидания
- Есть ли дизайн-система/референсы?
- Если нет — используем “системный UI” (например, стандартные паттерны выбранного фреймворка)?

### Локализация
- Один язык или несколько?
- Форматы дат/времени/валют?

### UX edge cases
- Что показываем при пустых данных?
- Что делаем при ошибках сети/валидации?
- Какие операции потенциально долгие (нужны прогресс/скелетоны)?

## Правило
Если ответ критичен для архитектуры/роутинга/API — обязательно уточнить.
Если можно двигаться дальше с безопасным допущением — делай допущение и отмечай.
