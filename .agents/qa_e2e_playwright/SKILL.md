# Skill: QA E2E Playwright

## Purpose
Спроектировать и выполнить надежную end-to-end проверку критичных пользовательских сценариев с помощью Playwright.

## When to use
- Нужен end-to-end coverage для критичного user flow.
- Нужно воспроизвести регрессию на уровне UI и интеграции.
- Для релизного гейта требуется E2E smoke или regression suite.

## Workflow
1. Определи критичные сценарии, риски и ожидаемые результаты.
2. Зафиксируй preconditions: окружение, тестовые данные, роли и feature flags.
3. Спроектируй стабильные селекторы и шаги, не привязывая тесты к хрупкой разметке.
4. Реализуй happy path, error path и ключевые edge cases.
5. Собери артефакты прогона: trace, screenshot, video и console/network evidence при падениях.
6. Верни результат с покрытием, найденными сбоями, риском flaky-поведения и влиянием на релиз.

## Deliverables
- Playwright test cases для критичных сценариев.
- Описание preconditions и настройки test data.
- Run report с артефактами, сбоями и рисками.

## Boundaries / Non-goals
- Не подменяй E2E-покрытие unit или integration тестами.
- Не опирайся на хрупкие селекторы, если доступны роли, labels или test ids.
- Не скрывай flaky-поведение; явно зафиксируй его в результате.

## Done criteria
- Критичные сценарии воспроизводимы и надежно проверяются.
- Для каждого сбоя хватает артефактов для диагностики.
- Итоговый вывод достаточно надежен для решения о релизе.
