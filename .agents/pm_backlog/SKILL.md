# Skill: Backlog (эпики/фичи/задачи)

## Цель
Превратить PRD в планируемую работу: что делать и в каком порядке.

## Выход
Backlog в формате:

### Epics (E-xx)
- E-01: ...
  - Feature (F-xx): ...
    - Task (T-xx): ... (коротко, проверяемо)

Для каждой Feature:
- Приоритет (P0/P1/P2)
- Зависимости (например: UX Spec, архитектура, интеграции)
- Риск/неясность (если есть)

## Правила декомпозиции
- Делай “вертикальные” фичи (end-to-end), а не слой за слоем.
- MVP сначала: P0 только то, без чего продукт не работает.
- Если есть неопределённость — создай Spike задачу (time-box) отдельным пунктом.

## Предложение итераций
- Итерация 1 (MVP slice 1): ...
- Итерация 2 (MVP slice 2): ...
- Итерация 3 (hardening / v1): ...
