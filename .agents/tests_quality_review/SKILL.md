# Skill: Tests Quality Review

## Проверить
- Тесты проверяют поведение, а не детали реализации
- Есть happy + edge + error paths
- Интеграционные тесты реально проверяют интеграцию (API/DB/контракты), а не “пере-моканы”
- Тесты независимы (нет скрытого порядка)
- Флейки: тайминги/рандом/внешняя сеть → устранены
- Названия тестов читаемые, структура понятная

## Выход
- Недостающие кейсы (списком)
- Хрупкие тесты (что исправить)
