# Skill: Observability Review

## Проверить
- Структурированные логи (ключи, а не голые строки)
- Корреляция request_id/trace_id (если предусмотрено)
- Уровни логирования корректны (error только для реальных ошибок)
- Не логируется PII/секреты
- Ключевые события (audit) логируются при необходимости

## Выход
- Несоответствия плану
- Конкретные рекомендации
