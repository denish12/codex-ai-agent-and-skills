# Skill: Testing Strategy (JS/TS)

## Unit
- Чистые функции, утилиты, хуки/логика компонентов
- Для UI: тестировать пользовательское поведение

## Integration
- API routes/handlers
- Репозитории/БД (или тестовый контейнер/эмулятор)
- Интеграции через моки/контракты

## E2E (Playwright)
- Только критичные потоки (логин/ключевые CRUD/оплаты/поиск) — по задаче

## Организация
- Тесты рядом с кодом (unit)
- Интеграция рядом с маршрутом/модулем
- E2E в отдельной папке (e2e/)

## Моки
- Мокать внешние сервисы (OpenAI/Redis/Supabase/HTTP)
- Не мокать всё подряд: сохранять смысл интеграционных тестов
