# Skill: Performance Review Baseline

## Проверить
- N+1 запросы / неоптимальные выборки
- Нет пагинации там, где список может быть большим
- Лишние round-trips (можно batch)
- Тяжёлые операции в синхронном request-path (нужна async/queue?) — только если архитектура допускает
- Кэширование применено там, где это было предусмотрено (и безопасно)

## Выход
- Bottlenecks (P1/P2 обычно, P0 если ломает SLA/UX)
- Рекомендации оптимизации без premature optimization
