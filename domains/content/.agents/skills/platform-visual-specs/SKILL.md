# Platform Visual Specs — Спецификации визуалов по платформам

> **Дата актуализации:** 2026-Q1. Платформы регулярно меняют спецификации — при сомнениях проверять актуальные данные через веб-поиск.

## Когда использовать
- При **подготовке визуального контента** для конкретной платформы.
- При **создании `$visual-brief`** — указать точные размеры и ограничения.
- При **`$image-prompt-engineering`** — выбрать ratio для AI-генерации.
- При **`$platform-compliance`** — технические спеки для проверки.
- При **кросс-постинге** — адаптация размеров под другую платформу.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Платформа | ✅ | Где будет опубликовано |
| Тип контента | ✅ | Пост / Stories / Reels / Carousel / видео / обложка / реклама |
| Organic / Paid | ✅ | Рекламные форматы имеют отдельные требования |
| Текст на изображении | ⬚ | Если да — учесть правило 20% для рекламы и safe zones |

## Accessibility и Dark Mode

### Accessibility

| Требование | Описание | Как проверить |
|-----------|----------|---------------|
| **Alt-текст** | Описание изображения для screen readers | Заполнить при публикации (все платформы поддерживают) |
| **Контраст текста** | Коэффициент контраста ≥ 4.5:1 (AA) для текста на изображении | Инструменты: WebAIM Contrast Checker, Figma plugins |
| **Размер текста** | Текст на изображении ≥ 16px (читаем на мобильном) | Визуальная проверка на телефоне |
| **Не только цветом** | Информация не должна передаваться только цветом | Иконки, паттерны, подписи дополняют цвет |

### Dark Mode

| Элемент | Проблема | Решение |
|---------|---------|---------|
| Логотип на белом фоне | На тёмном фоне — белый прямоугольник | Использовать прозрачный фон (PNG) или тёмную версию лого |
| Светлые изображения | Слепят в тёмной теме | Добавить тонкую рамку (1px border) или мягкий drop shadow |
| Текст на изображении | Белый текст на светлом изображении не виден в light mode (и наоборот) | Добавлять подложку / overlay под текст |
| Инфографика | Цвета могут сливаться с тёмным фоном | Проверять preview в обоих режимах |

## Протокол

### Шаг 1 — Определение контекста
1. Определить платформу.
2. Определить тип контента (пост / видео / stories / carousel / реклама).
3. Определить organic / paid.

### Шаг 2 — Выбор спецификаций
1. Найти спеки в справочнике по платформе и типу.
2. Зафиксировать: размер, ratio, формат, макс. файл, safe zone.
3. Если видео — добавить: codec, FPS, bitrate, длительность.
4. Если реклама — проверить рекламные спеки (правило 20% текста, CTA-зоны).

### Шаг 3 — Safe zone проверка
1. Убедиться, что ключевой контент (текст, лицо, лого) не попадает в safe zone.
2. Для Reels/Stories — учесть UI элементы платформы.
3. Для YouTube thumbnail — учесть перекрытие длительности.

### Шаг 4 — Accessibility и Dark Mode
1. Проверить контраст текста (≥ 4.5:1).
2. Подготовить alt-текст.
3. Проверить визуал в dark mode (если применимо).

### Шаг 5 — Кросс-платформенные адаптации
Если контент для нескольких платформ — подготовить размеры для каждой:

| Платформа | Рекомендуемый формат для AI-генерации | MJ параметр |
|-----------|--------------------------------------|-------------|
| Instagram Feed | 1080×1350 (4:5) | `--ar 4:5` |
| Instagram Stories/Reels | 1080×1920 (9:16) | `--ar 9:16` |
| LinkedIn | 1200×627 (1.91:1) | `--ar 191:100` |
| YouTube Thumbnail | 1280×720 (16:9) | `--ar 16:9` |
| Telegram | 800×400 (2:1) | `--ar 2:1` |
| Pinterest | 1000×1500 (2:3) | `--ar 2:3` |
| OG Image | 1200×630 (1.91:1) | `--ar 191:100` |

## Валидация (Quality Gate)

Спецификации считаются корректно применёнными если:

- [ ] Размер визуала соответствует спекам платформы
- [ ] Ratio правильный
- [ ] Формат файла поддерживается
- [ ] Вес файла в лимите
- [ ] Safe zone учтён (ключевой контент не обрезается UI)
- [ ] Текст на изображении читаем (≥ 16px, контраст ≥ 4.5:1)
- [ ] Alt-текст подготовлен
- [ ] Dark mode проверен (если применимо)
- [ ] Рекламные ограничения учтены (если paid)

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$visual-brief` | **Потребитель:** размеры → visual brief |
| `$image-prompt-engineering` | **Потребитель:** ratio → промпт |
| `$platform-strategy` | **Источник:** платформы → какие спеки нужны |
| `$platform-compliance` | **Комплемент:** visual specs = размеры, compliance = правила |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Один размер для всех платформ | Обрезается, растягивается, выглядит непрофессионально | Свой размер для каждой платформы |
| Игнорирование safe zone | Текст скрыт под UI платформы | Проверять safe zone для каждого формата |
| Текст мелкий (< 16px) | Нечитаем на мобильном — 60%+ трафика | Минимум 16px, лучше 24px+ |
| Не проверен dark mode | Лого/текст сливается с фоном | Проверять в обоих режимах |
| Тяжёлые файлы | Медленная загрузка, отклонение платформой | Оптимизировать: WebP для веба, сжатие для соцсетей |
| Спеки 2023 года | Платформы обновили требования | Проверять актуальность, следить за обновлениями |
| Рекламные спеки для organic | Правило 20% текста — для рекламы, не для organic | Разделять organic и paid |
| Нет alt-текста | Недоступно для screen readers, хуже SEO | Всегда добавлять alt-текст |
