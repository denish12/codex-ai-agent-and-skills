# Email Copywriting — Email-копирайтинг

## Когда использовать
- При создании **одиночных email** — newsletter, promo, product update, announcement.
- При проектировании **email-последовательностей** — onboarding, nurturing, retention, win-back.
- При **оптимизации существующих** — низкий open rate, низкий CTR, высокий unsubscribe.
- Как часть **контент-стратегии** — email как канал из `$platform-strategy`.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Тип письма | ✅ | Newsletter / Promo / Product Update / Onboarding / Nurturing / Retention / Win-back / Announcement / Transactional |
| Цель | ✅ | Конкретное действие: open → read → click → convert |
| Персона ЦА | ✅ | Из `$audience-analysis` — язык, боли, триггеры |
| Сегмент базы | ✅ | Кому отправляем (новые / активные / спящие / все) |
| Этап воронки | ✅ | TOFU / MOFU / BOFU — определяет агрессивность CTA |
| Бренд-гайдлайны | ✅ | Из `$brand-guidelines` — ToV, визуальный стиль |
| Оффер / ценность | ⬚ | Что предлагаем (для promo / nurturing) |
| Позиция в последовательности | ⬚ | Если часть серии — какое по счёту, что было до и после |
| Текущие метрики | ⬚ | Open rate, CTR, unsubscribe rate — если оптимизация |

> Если тип или сегмент не указаны — **запросить у пользователя**. Email без сегмента = спам.

## Типы писем и их структуры

### Newsletter (дайджест)
**Цель:** удержание, привычка открывать. **Частота:** регулярная.
```
Subject → Прехедер → Приветствие → Подводка (1-2 предл.) → 
Блок 1 [тема + описание + ссылка] → Блок 2 → Блок 3 → 
Бонус/рекомендация → Подпись → Footer
```
- 3-5 блоков контента, каждый с отдельной ссылкой.
- CTA распределён: каждый блок = отдельная ссылка.
- Длина: 300-500 слов.

### Promo (промо / оффер)
**Цель:** конверсия, покупка. **Частота:** по кампаниям.
```
Subject (urgency/value) → Прехедер → Hook (боль или выгода) → 
Оффер (что получает) → Social proof (отзыв/цифры) → 
CTA-кнопка → Ограничение (urgency/scarcity) → P.S. с повтором CTA → Footer
```
- Один оффер, один CTA (повторяется 2-3 раза).
- Длина: 150-300 слов.

### Product Update
**Цель:** информирование, adoption новой фичи.
```
Subject → Прехедер → «Что нового» (1 предл.) → 
Фича 1 [описание + выгода] → Фича 2 → 
CTA «Попробовать» → Подпись → Footer
```
- Фокус на выгоде, не на техническом описании.
- Длина: 200-400 слов.

### Onboarding (серия)
**Цель:** активация, first value. **Обычно 3-7 писем.**
```
Письмо 1: Welcome — кто мы, что ожидать, первый шаг
Письмо 2: Quick win — самый простой способ получить ценность
Письмо 3: Core feature — главная функция, как использовать
Письмо 4: Social proof — кейсы, отзывы
Письмо 5: CTA upgrade / next step — конверсия
```
- Каждое письмо — одно действие. Не перегружать.
- Интервал: день 0, день 1, день 3, день 5, день 7.

### Nurturing (серия)
**Цель:** прогрев, доведение до решения. **MOFU → BOFU.**
```
Письмо 1: Ценный контент (без продажи)
Письмо 2: Кейс / история успеха
Письмо 3: FAQ / возражения
Письмо 4: Оффер с social proof
Письмо 5: Последний шанс / urgency
```
- Градиент от ценности к продаже.
- Интервал: 2-4 дня между письмами.

### Retention
**Цель:** реактивация спящих пользователей.
```
Subject (персональный / «мы скучаем») → Напоминание о ценности → 
Что нового (с момента ухода) → Стимул вернуться (бонус/скидка) → CTA → Footer
```
- Короткое, персональное, с конкретным стимулом.

### Win-back
**Цель:** вернуть тех, кто отписался / перестал покупать.
```
Subject (интрига / прямой вопрос) → «Мы заметили, что вы ушли» → 
Причины вернуться (1-3) → Эксклюзивный оффер → CTA → 
P.S. «Если не интересно — отпишитесь» → Footer
```
- Последний шанс — уважительный тон, без давления.

## Формулы Subject Line

| Формула | Описание | Пример | Когда использовать |
|---------|----------|--------|-------------------|
| **Число + выгода** | Конкретика привлекает | «5 способов сократить затраты на 30%» | Newsletter, образовательные |
| **Вопрос** | Вовлекает, запускает мышление | «Почему ваш email-маркетинг не работает?» | Nurturing, TOFU |
| **Персонализация** | Имя / компания в subject | «[Имя], ваш персональный отчёт готов» | Все типы, увеличивает OR на 10-20% |
| **Urgency** | Ограничение по времени | «Последние 24 часа: -40% на всё» | Promo, win-back |
| **Curiosity gap** | Незавершённость, интрига | «Мы нашли то, что вы искали» | Retention, nurturing |
| **How-to** | Обещание научить | «Как получить первых 100 подписчиков за неделю» | Newsletter, onboarding |
| **Social proof** | Авторитет / цифры | «10 000 маркетологов уже используют этот метод» | Promo, BOFU |
| **Негатив (осторожно)** | Страх потери | «Вы теряете клиентов из-за этой ошибки» | Nurturing, MOFU |

### Правила subject line
- **≤ 50 символов** — оптимально для мобильных (60% открытий).
- **Нет спам-слов** — БЕСПЛАТНО, СРОЧНО, $$$ (спам-фильтры).
- **Не обманывать** — subject должен соответствовать содержанию.
- **Прехедер дополняет** — не повторяет subject, а расширяет.
- **A/B тест** — всегда 2 варианта с разными формулами.

## Технические требования

| Параметр | Требование | Почему |
|----------|-----------|--------|
| **Ширина** | 600px максимум | Корректное отображение в большинстве клиентов |
| **Вес HTML** | ≤ 100 KB | Gmail обрезает письма > 102 KB |
| **Изображения** | Alt-текст обязателен | 40% пользователей блокируют картинки |
| **Соотношение текст/картинки** | 60/40 минимум текста | Спам-фильтры штрафуют image-heavy |
| **Шрифт** | Web-safe: Arial, Helvetica, Georgia, Verdana | Кастомные шрифты не везде работают |
| **CTA-кнопка** | HTML-кнопка (не картинка), контрастный цвет, ≤ 5 слов | Картинка-кнопка не видна без загрузки изображений |
| **Plain text** | Всегда включать plain text версию | Fallback для клиентов без HTML |
| **Responsive** | Mobile-first: однополосная вёрстка | 60%+ открытий на мобильных |
| **Тёмная тема** | Проверить: логотип, цвета, фон | Всё больше пользователей используют dark mode |
| **Прехедер** | 45-85 символов, заполнить осознанно | Если пустой — подтянется мусор из body |

## Юридические требования

| Требование | Стандарт | Обязательность |
|-----------|----------|---------------|
| **Ссылка отписки** | CAN-SPAM, GDPR | 🔴 Обязательно в каждом письме |
| **Физический адрес** | CAN-SPAM | 🔴 Обязательно (US-рассылки) |
| **Идентификация отправителя** | CAN-SPAM, GDPR | 🔴 «От кого» — не вводить в заблуждение |
| **Согласие получателя** | GDPR (double opt-in для EU) | 🔴 Обязательно |
| **Обработка отписки** | CAN-SPAM (10 дней), GDPR (без задержки) | 🔴 Отписка должна работать |
| **Маркировка рекламы** | Законодательство РФ | 🟡 Если промо — пометить «Реклама» |

> Юридический блок обязателен в **каждом** письме. Нет отписки = штраф.

## Метрики и бенчмарки

| Метрика | Формула | Бенчмарк (средний) | Хороший | Проблема если |
|---------|---------|--------------------|---------|--------------| 
| **Open Rate** | Opens / Delivered × 100 | 20-25% | > 30% | < 15% — subject line или deliverability |
| **CTR** | Clicks / Delivered × 100 | 2-3% | > 5% | < 1% — CTA или контент |
| **CTOR** | Clicks / Opens × 100 | 10-15% | > 20% | < 8% — контент не релевантен |
| **Unsubscribe Rate** | Unsubs / Delivered × 100 | 0.1-0.3% | < 0.1% | > 0.5% — слишком часто или нерелевантно |
| **Bounce Rate** | Bounces / Sent × 100 | < 2% | < 0.5% | > 5% — проблема с базой |
| **Spam Complaint** | Complaints / Delivered × 100 | < 0.1% | < 0.05% | > 0.1% — риск блокировки домена |

> Бенчмарки варьируются по индустриям. Указанные — средние по рынку.

## Протокол

### Шаг 1 — Контекст и сегмент
1. Определить тип письма и цель.
2. Определить сегмент базы и персону.
3. Определить этап воронки и позицию в последовательности (если серия).
4. Проверить: когда было последнее письмо этому сегменту? (не спамить).

### Шаг 2 — Subject Line + Прехедер
1. Выбрать **2-3 формулы** из таблицы.
2. Написать **3 варианта** subject line (≤ 50 символов).
3. Для каждого — написать прехедер (45-85 символов, дополняет, не повторяет).
4. Выбрать пару для основной отправки + пару для A/B теста.
5. Проверить на спам-слова.

### Шаг 3 — Структура и тело
1. Выбрать **структуру по типу письма** (см. выше).
2. Написать тело письма.
3. Проверить scanability: заголовки, буллеты, жирный текст, короткие абзацы.
4. Проверить длину — соответствует типу.

### Шаг 4 — CTA
1. Сформулировать CTA с помощью `$cta-optimization` (или базовых правил).
2. Один основной CTA-кнопка + максимум один вторичный (текстовая ссылка).
3. Повторить CTA: above the fold + конец письма.
4. CTA-кнопка: контрастный цвет, ≤ 5 слов, HTML (не картинка).

### Шаг 5 — Персонализация
1. Имя в subject / приветствии (если есть в базе).
2. Динамический контент по сегменту (если платформа позволяет).
3. Поведенческие триггеры (если автоматизация — последнее действие, просмотренный товар).

### Шаг 6 — Технические и юридические проверки
1. Пройти по таблице технических требований.
2. Добавить обязательные юридические элементы (отписка, адрес, идентификация).
3. Проверить вес HTML (≤ 100 KB).

### Шаг 7 — A/B тест
1. Определить переменную теста: subject line (чаще всего), CTA, время отправки.
2. Подготовить вариант B.
3. Указать размер тестовой выборки (10-20% базы) и критерий победителя (open rate / CTR).

## Валидация (Quality Gate)

Email считается готовым если:

- [ ] Тип письма определён, структура соответствует типу
- [ ] Subject line ≤ 50 символов, нет спам-слов
- [ ] Прехедер 45-85 символов, дополняет subject
- [ ] Тело scannable: заголовки, буллеты, короткие абзацы
- [ ] CTA — один основной, конкретный, ≤ 5 слов на кнопке
- [ ] Персонализация добавлена (хотя бы имя)
- [ ] Alt-текст у всех изображений
- [ ] Ссылка отписки присутствует
- [ ] Plain text версия подготовлена
- [ ] A/B вариант subject line готов
- [ ] Вес HTML ≤ 100 KB
- [ ] Проверено на мобильном (responsive)

## Email-последовательности

### Проектирование серии

| Параметр | Описание |
|----------|----------|
| **Триггер** | Что запускает серию (регистрация, покупка, N дней без активности) |
| **Количество писем** | 3-7 для onboarding, 4-6 для nurturing, 2-3 для win-back |
| **Интервал** | Onboarding: 1-2 дня. Nurturing: 2-4 дня. Win-back: 3-7 дней |
| **Градиент** | От ценности (первые письма) к конверсии (последние) |
| **Exit conditions** | Когда прекращать: конвертировался, отписался, достиг конца серии |

### Шаблон последовательности
```
| # | День | Тема письма | Тип | Цель | CTA |
|---|------|-------------|-----|------|-----|
| 1 | 0 | [subject] | Welcome | Первое впечатление | [действие] |
| 2 | 1 | [subject] | Quick win | Первая ценность | [действие] |
| 3 | 3 | [subject] | Core feature | Adoption | [действие] |
| 4 | 5 | [subject] | Social proof | Доверие | [действие] |
| 5 | 7 | [subject] | Upgrade/CTA | Конверсия | [действие] |
```

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Нет сегментации | Одно письмо для всех = нерелевантный контент | Сегмент + персона для каждого письма |
| Subject > 50 символов | Обрезается на мобильных | ≤ 50 символов, самое важное в начале |
| Несколько равных CTA | Распыление: ничего не кликают | Один основной CTA, повторить 2 раза |
| Image-only email | Не видно без загрузки картинок, спам-фильтры | 60%+ текста, alt-текст обязателен |
| Нет отписки | Штраф CAN-SPAM до $46K за письмо | Ссылка отписки в каждом письме |
| Отправка без A/B | Не оптимизируешь → не растёшь | Минимум A/B на subject line |
| Стена текста | Не читают, удаляют | Scanability: заголовки, буллеты, ≤ 3 предложения на абзац |
| Слишком частые | > 3/нед → unsubscribe | Newsletter 1/нед, promo ≤ 2/мес для холодных |
| Нет plain text | Часть клиентов не видит HTML | Всегда plain text fallback |
