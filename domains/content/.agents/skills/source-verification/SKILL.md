# Source Verification — Верификация источников

## Когда использовать
- После **`$topic-research`** — перед передачей данных Copywriter'у.
- Параллельно с **`$fact-checking`** — на этапе Reviewer.
- При **сомнении в источнике** — проверка любого источника перед использованием.
- При **data-driven контенте** — все источники данных верифицируются обязательно.

> **Разграничение с `$fact-checking`:** Source verification проверяет **источник** (кто, когда, насколько надёжен). Fact checking проверяет **утверждение** (правда ли то, что написано). Source verification → вход для fact checking.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Список источников | ✅ | URL, названия, ссылки — то, что проверяем |
| Тип контента | ✅ | Определяет уровень строгости |
| Тема | ✅ | Контекст: в какой области используется источник |
| Откуда источники | ⬚ | Из `$topic-research` / от пользователя / из SERP / из конкурентов |

## Уровни строгости

| Уровень | Типы контента | Минимальный рейтинг | Особенности |
|---------|--------------|--------------------| ------------|
| 🔴 **Максимальный** | Медицина, финансы, юридика, научные темы (YMYL) | ⭐⭐⭐⭐ и выше | Только peer-reviewed, гос. статистика. Конфликт интересов = отклонение |
| 🟡 **Стандартный** | Статьи, кейсы, аналитика, email, лонгриды | ⭐⭐⭐ и выше | Авторитетные СМИ, отраслевые отчёты допустимы |
| 🔵 **Базовый** | Соцсети, короткие посты, развлекательный | ⭐⭐ и выше (с перекрёстной проверкой) | Блоги допустимы, если факт подтверждён вторым источником |

## CRAAP-фреймворк

Каждый источник проверяется по 5 критериям:

| Критерий | Вопросы для проверки | Red flags |
|----------|---------------------|-----------|
| **C — Currency (Актуальность)** | Когда опубликовано? Обновлялось ли? Актуальны ли данные для нашей темы? | Старше 2 лет для tech/бизнеса. Старше 5 лет для науки. Ссылки на ещё более старые источники |
| **R — Relevance (Релевантность)** | Источник по нашей теме? Нужная глубина? Целевая аудитория источника совпадает? | Смежная тема выдаётся за прямую. Научная статья для популярного поста (избыточно) или блог для научной темы (недостаточно) |
| **A — Authority (Авторитетность)** | Кто автор? Квалификация? Репутация издания? Есть ли рецензирование? | Автор неизвестен. Издание без редакции. Self-published без подтверждения |
| **A — Accuracy (Точность)** | Данные подтверждены другими источниками? Методология описана? Есть ссылки на первоисточники? | Нет методологии. Нет первоисточников. Данные противоречат другим надёжным источникам |
| **P — Purpose (Цель)** | Зачем написано? Информировать? Продать? Убедить? Есть ли скрытая повестка? | Рекламный контент замаскирован под исследование. Спонсируемое исследование без disclosure |

### Scoring по CRAAP

Каждый критерий: **1 (плохо) — 3 (средне) — 5 (отлично)**

| Итого (из 25) | Интерпретация |
|---------------|---------------|
| 21-25 | Отличный источник — использовать без ограничений |
| 16-20 | Хороший — использовать, но перекрёстно проверить ключевые данные |
| 11-15 | Сомнительный — использовать только с перекрёстным подтверждением ≥ 2 других источников |
| ≤ 10 | Ненадёжный — не использовать |

## Типы источников — специфические критерии

### Научное исследование / Peer-reviewed journal
| Что проверять | Как |
|--------------|-----|
| Peer review | Журнал с рецензированием (не preprint без review) |
| Выборка | N достаточно для выводов? |
| Методология | Описана, воспроизводима? |
| Финансирование | Кто спонсировал? Указан ли disclosure? |
| Retraction | Не отозвана ли статья? (проверить Retraction Watch) |

### Новостная статья / СМИ
| Что проверять | Как |
|--------------|-----|
| Издание | Крупное / независимое / таблоид? |
| Автор | Журналист с track record или «редакция»? |
| Источники в статье | Есть цитаты экспертов? Ссылки на оригинал? |
| Дата | Актуальна? Не обновлялась ли с новыми данными? |
| Bias | Политическая / коммерческая предвзятость издания? |

### Корпоративный отчёт / Пресс-релиз
| Что проверять | Как |
|--------------|-----|
| Конфликт интересов | Компания продвигает свой продукт через «исследование»? |
| Методология | Описана ли? Или только выводы? |
| Выборка | Только клиенты компании = предвзятая выборка |
| Перекрёстная проверка | Подтверждают ли независимые источники? |

### Блог / Пост эксперта
| Что проверять | Как |
|--------------|-----|
| Автор | Реальный эксперт с подтверждённой квалификацией? |
| Источники | Ссылается на первичные данные или «по моему опыту»? |
| Bias | Аффилиат? Продвигает продукт? |
| Перекрёстная проверка | Обязательна — блог не самодостаточный источник |

### Социальные сети / Reddit / Quora
| Что проверять | Как |
|--------------|-----|
| Автор | Верифицированный аккаунт? Реальный эксперт? |
| Доказательства | Есть ли ссылки, скриншоты, данные? |
| Consensus | Мнение одного человека или подтверждено другими? |
| **Правило** | Не использовать как единственный источник. Только как иллюстрация + перекрёстное подтверждение |

## Проверка Bias (предвзятости)

| Тип bias | Как обнаружить | Пример |
|----------|---------------|--------|
| **Коммерческий** | Источник продаёт продукт, связанный с выводами | «Наше исследование показало, что наш продукт лучший» |
| **Спонсируемый** | Исследование оплачено заинтересованной стороной | Фарм-компания спонсирует исследование своего лекарства |
| **Selection bias** | Выборка не репрезентативна | Опрос только среди клиентов = предвзятые результаты |
| **Confirmation bias** | Данные подобраны под вывод | Cherry-picking: показаны только подтверждающие факты |
| **Survivorship bias** | Учтены только успешные кейсы | «Все успешные стартапы делают X» (а неуспешные тоже делали X) |
| **Идеологический** | Издание имеет политическую позицию | Политически ангажированное СМИ о бизнес-регуляции |

> При обнаружении bias — **не автоматически отклонять**, а понизить рейтинг + указать bias в комментарии + требовать перекрёстное подтверждение.

## Шкала рейтинга (расширенная)

| Рейтинг | Тип источника | Примеры | Допустимость |
|---------|--------------|---------|-------------|
| ⭐⭐⭐⭐⭐ | Первичный / академический | Peer-reviewed journals (Nature, Lancet, JAMA), гос. статистика (Росстат, BLS, Eurostat), SEC filings, патенты | Все уровни строгости |
| ⭐⭐⭐⭐ | Авторитетный вторичный | Крупные СМИ с редакцией (Reuters, Bloomberg, BBC), отчёты аналитических компаний (McKinsey, Gartner, Statista), официальные блоги крупных компаний | Все уровни строгости |
| ⭐⭐⭐ | Надёжный вторичный | Отраслевые СМИ (TechCrunch, The Verge), книги экспертов, корпоративные отчёты с методологией, крупные блоги с редакцией | 🟡 Стандартный и ниже |
| ⭐⭐ | Пользовательский контент | Личные блоги экспертов, подкасты, YouTube-каналы специалистов, Reddit (верифицированные), Quora (with credentials) | 🔵 Базовый (с перекрёстной проверкой) |
| ⭐ | Ненадёжный | Анонимные источники, контент-фермы, affiliate-сайты, жёлтые СМИ, непроверяемые данные | ❌ Не использовать |

### Пограничные случаи

| Ситуация | Решение |
|----------|---------|
| Корпоративный отчёт (McKinsey) vs корпоративный блог (NoName SaaS) | McKinsey = ⭐⭐⭐⭐ (методология, репутация). NoName SaaS = ⭐⭐⭐ (может быть bias) |
| Wikipedia | ⭐⭐ — НЕ самостоятельный источник. Использовать ссылки из Wikipedia → найти и проверить первоисточники |
| Preprint (не рецензирован) | ⭐⭐⭐ — снизить на 1 звезду от журнала. Пометить «preprint, не рецензирован» |
| Опрос с маленькой выборкой (N < 100) | Снизить на 1 звезду. Пометить «малая выборка: N = [число]» |

## Протокол

### Шаг 1 — Подготовка
1. Получить список источников и тип контента.
2. Определить уровень строгости (🔴 / 🟡 / 🔵).
3. Определить минимальный допустимый рейтинг.

### Шаг 2 — CRAAP-оценка
Для каждого источника:
1. Пройти **5 критериев CRAAP** (Currency, Relevance, Authority, Accuracy, Purpose).
2. Оценить каждый 1-5 баллов.
3. Подсчитать итого (из 25).

### Шаг 3 — Тип источника
1. Классифицировать: научное / СМИ / корпоративное / блог / соцсети.
2. Применить **специфические критерии** для типа.
3. Для научных: проверить retraction, методологию, финансирование.
4. Для корпоративных: проверить конфликт интересов.

### Шаг 4 — Bias-проверка
1. Проверить на 6 типов bias.
2. Если bias обнаружен — понизить рейтинг + пометить + требовать перекрёстное подтверждение.

### Шаг 5 — Рейтинг и решение
1. Присвоить **рейтинг** (⭐ — ⭐⭐⭐⭐⭐).
2. Сравнить с минимально допустимым для уровня строгости.
3. Решение: ✅ Принят / ⚠️ Условно (с перекрёстной проверкой) / ❌ Отклонён.

### Шаг 6 — Первоисточники
1. Для каждого вторичного источника — **найти первоисточник**.
2. Если первоисточник недоступен — пометить и понизить рейтинг.

## Валидация (Quality Gate)

Верификация считается завершённой если:

- [ ] Все источники из списка проверены (нет пропущенных)
- [ ] CRAAP-оценка проведена для каждого источника
- [ ] Тип каждого источника определён, специфические критерии применены
- [ ] Bias-проверка выполнена
- [ ] Рейтинг присвоен и соответствует уровню строгости
- [ ] Для вторичных источников — найдены первоисточники (или помечено «не найден»)
- [ ] Отклонённые источники имеют обоснование + рекомендация замены
- [ ] Все принятые источники ≥ минимального рейтинга для уровня строгости

## Handoff

| Результат | Маршрут |
|-----------|---------|
| Все источники верифицированы | → Copywriter (с рейтингами) + `$fact-checking` (для проверки фактов из источников) |
| Часть отклонена, замена найдена | → Researcher (заменить) → повторная верификация замен |
| Часть отклонена, замена не найдена | → Researcher (найти альтернативы) или → пользователь (решение: удалить факт или найти источник) |

При передаче — использовать `$handoff` с приложением полного отчёта.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Wikipedia как источник | Wikipedia = ⭐⭐, может содержать ошибки | Использовать ссылки из Wikipedia → найти первоисточник |
| «По данным исследований» без ссылки | Непроверяемо, нет доверия | Конкретный источник: кто, когда, методология |
| Один авторитетный источник = достаточно | Даже Nature может быть retracted | Перекрёстная проверка для ключевых данных |
| Корпоративный отчёт = объективен | Конфликт интересов | Проверить bias, найти независимое подтверждение |
| Пересказ вместо первоисточника | Искажение данных через цепочку пересказов | Всегда искать первоисточник |
| Все источники одного рейтинга | Нет дифференциации | CRAAP-scoring для каждого, рейтинг ⭐-⭐⭐⭐⭐⭐ |
| Отклонение за любой bias | Bias ≠ ложь, иногда предвзятый источник содержит верные данные | Понизить рейтинг + перекрёстная проверка, не автоотклонение |
| Старые данные без проверки | Мир изменился | Актуальность: ≤ 2 лет (tech), ≤ 5 лет (наука) |
