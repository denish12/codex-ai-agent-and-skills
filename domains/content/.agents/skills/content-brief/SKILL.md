# Content Brief — Бриф на создание контента

## Когда использовать
- При **передаче задачи** от Strategist к Researcher / Copywriter / Visual Concept.
- При **планировании контента** — каждая единица из `$content-calendar` получает свой бриф.
- При **пересоздании / обновлении** существующего контента (rewrite, repurpose).

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Тема / идея | ✅ | О чём контент — из `$content-calendar`, задания пользователя или Strategist |
| Персона ЦА | ✅ | Конкретная персона из `$audience-analysis` |
| Платформа | ✅ | Из `$platform-strategy` — где будет опубликовано |
| Формат | ✅ | Пост, статья, reel, карусель, email, видео и т.д. |
| Бренд-гайдлайны | ✅ | Из `$brand-guidelines` — ToV, визуальные стандарты |
| Дата публикации | ⬚ | Из `$content-calendar` или от пользователя |
| SEO-данные | ⬚ | Ключевые слова, search intent (для текстовых форматов: статья, блог, лендинг) |
| Референсы | ⬚ | Примеры контента (свои / конкурентные / из других ниш) |

> Если персона или платформа не указаны — **запросить у пользователя**. Бриф без ЦА и канала — бесполезен.

## Протокол

### Шаг 1 — Цель и контекст
1. Зафиксировать **цель контента** — что должен сделать читатель после потребления (подписаться, купить, перейти, запомнить бренд).
2. Определить **этап воронки**: TOFU (awareness) / MOFU (consideration) / BOFU (decision).
3. Указать **контекст публикации** — почему именно сейчас (тренд, сезон, запуск, актуальное событие).

### Шаг 2 — Аудитория и тон
1. Указать конкретную **персону** из `$audience-analysis`.
2. Зафиксировать **тон** для этой персоны (из `$tone-of-voice` / `$brand-guidelines`).
3. Указать **язык клиента** — слова и фразы, которые резонируют с этой персоной.

### Шаг 3 — Контентное ядро
1. Сформулировать **точную тему** (не общую, а конкретный угол).
2. Определить **ключевые тезисы** (3-5 обязательных пунктов).
3. Указать **запрещённые темы** и формулировки.
4. Перечислить **обязательные элементы** — факты, цитаты, данные, статистика.

### Шаг 4 — SEO (для текстовых форматов)
> Пропустить для форматов без SEO-компонента (сторис, reels, email).

1. **Основное ключевое слово** + 3-5 дополнительных.
2. **Search intent** — информационный / навигационный / транзакционный.
3. **Конкуренты в SERP** — топ-3 результата по запросу (для понимания уровня).
4. **Мета-заголовок и мета-описание** — если применимо.

### Шаг 5 — Формат и структура
1. Указать **формат** (пост, карусель, статья, reel, email, thread и т.д.).
2. Задать **объём** — символы / слова / слайды / секунды.
3. Описать **структуру** — порядок секций, заголовки, hook.
4. Указать **платформенные ограничения** — лимит символов, соотношение сторон, длительность.

### Шаг 6 — CTA
1. **Основной CTA** — одно конкретное действие (не «подпишись и купи и поделись»).
2. **Вторичный CTA** — если применимо (сохрани, отправь другу).
3. CTA должен соответствовать этапу воронки из шага 1.

### Шаг 7 — Визуальные требования
1. Описать **визуальную концепцию** — что на картинке/видео.
2. Указать **стиль** — из `$brand-guidelines` (фото / иллюстрация / типографика / микс).
3. Указать **размеры** — под платформу.
4. Если нужен полный визуальный бриф → ссылка на `$visual-brief`.

### Шаг 8 — Референсы
1. **Свои лучшие** — 1-2 примера собственного контента, который сработал.
2. **Конкурентные** — 1-2 примера из `$competitor-content-analysis`.
3. **Вдохновение** — из других ниш / платформ (если применимо).
4. Для каждого референса — что именно взять (структуру, тон, hook, визуал).

### Шаг 9 — Distribution
1. **Основной канал** — где публикуем.
2. **Кросс-постинг** — адаптации для других платформ (указать какие изменения).
3. **Промо** — organic / paid / оба. Бюджет, если известен.
4. **Партнёрства** — коллаборации, инфлюенсеры, репосты (если планируются).

### Шаг 10 — Метрики и дедлайн
1. **Метрики успеха** — 2-3 конкретных KPI с целевыми значениями (ER > 5%, охват > 10K, клики > 200).
2. **Дедлайн производства** — когда контент должен быть готов.
3. **Дата публикации** — когда выходит.

## Валидация (Quality Gate)

Бриф считается готовым к передаче если:

- [ ] Цель конкретна и измерима (не «привлечь внимание», а «подписки на вебинар»)
- [ ] Указана конкретная персона (не «все»)
- [ ] Платформа и формат определены
- [ ] Ключевые тезисы конкретны (3-5 штук)
- [ ] CTA — одно основное действие, соответствует этапу воронки
- [ ] Объём и структура зафиксированы
- [ ] Визуальные требования указаны (хотя бы минимальные)
- [ ] Метрики имеют целевые значения
- [ ] SEO-блок заполнен (для текстовых форматов) или помечен N/A
- [ ] Нет конфликтов (цель TOFU + CTA «купи сейчас» = конфликт)

## Handoff

| Итог | Маршрут |
|------|---------|
| Бриф готов, текст нужен | → **Copywriter** (или Researcher → Copywriter если тема требует исследования) |
| Бриф готов, визуал нужен | → **Visual Concept** (с `$visual-brief` или визуальной секцией из этого брифа) |
| Бриф готов, текст + визуал | → **Copywriter** + **Visual Concept** параллельно |

При передаче — использовать `$handoff` с приложением полного брифа и указанием статуса.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$content-calendar` | Источник: тема, дата, ID единицы контента |
| `$audience-analysis` | Источник: персона, боли, триггеры, язык клиента |
| `$platform-strategy` | Источник: платформа, формат, время публикации |
| `$brand-guidelines` | Источник: ToV, визуальные стандарты |
| `$tone-of-voice` | Источник: детальный тон для персоны |
| `$competitor-content-analysis` | Источник: референсы, gaps |
| `$visual-brief` | Потребитель: детальное ТЗ на визуал создаётся на основе брифа |
| `$seo-copywriting` | Потребитель: использует SEO-блок из брифа |
| `$brand-compliance` | Проверяет финальный контент на соответствие данным из брифа |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| «Тема: маркетинг» | Слишком широко — копирайтер не знает угол | Конкретный угол: «5 ошибок email-маркетинга для SaaS в 2026» |
| Нет персоны | Текст ни для кого = текст для никого | Всегда конкретная персона из `$audience-analysis` |
| 3 CTA в одном посте | Распыление внимания, конверсия падает | Один основной CTA |
| Цель TOFU + CTA «купи» | Конфликт этапа воронки и действия | CTA соответствует этапу: TOFU = подпишись/узнай |
| Бриф без дедлайна | Задача без срока = задача без приоритета | Всегда дедлайн + дата публикации |
| Без референсов | Копирайтер угадывает уровень | 2-3 референса с пометкой «что взять» |
| Объём не указан | Получите 200 слов или 2000 | Конкретный диапазон: «800-1000 слов» |

## Статусы брифа

| Статус | Значение |
|--------|----------|
| `Draft` | Бриф в работе, не все поля заполнены |
| `Ready` | Бриф прошёл валидацию, готов к передаче |
| `In Production` | Контент создаётся по брифу |
| `Review` | Контент вернулся на проверку |
| `Done` | Контент опубликован |
