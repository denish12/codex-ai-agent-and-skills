# Content Review Checklist — Чеклист ревью контента

## Когда использовать
- На этапе **Reviewer** — основная проверка качества контента перед `$content-release-gate`.
- При **повторном ревью** — после доработки по предыдущим замечаниям.
- При **самопроверке** копирайтером — до передачи на ревью (сокращает итерации).

## Зона ответственности

Этот скил проверяет **качество контента** — текст, визуал, ссылки, SEO, юридику, accessibility.

> **Разграничение с `$brand-compliance`:** Brand compliance проверяет соответствие бренд-гайдлайнам (ToV, палитра, лого, позиционирование). Этот чеклист проверяет качество исполнения (грамматика, факты, читаемость, ссылки, SEO). Пункты пересекаются минимально — при совпадении ссылаться на результат `$brand-compliance`.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Контент для проверки | ✅ | Финальный текст + визуал от Copywriter / Visual Concept |
| Контент-бриф | ✅ | Из `$content-brief` — для сверки цели, тезисов, CTA, персоны, объёма |
| Бренд-гайдлайны | ✅ | Из `$brand-guidelines` — для проверки ToV и стоп-листа |
| Результат `$brand-compliance` | ⬚ | Если уже пройден — ссылаться на него, не дублировать |
| Результат `$fact-checking` | ⬚ | Если уже пройден — ссылаться, не перепроверять |
| Платформенные спеки | ⬚ | Из `$platform-strategy` / `$platform-compliance` — размеры, лимиты |

> Если контент-бриф не предоставлен — **запросить у пользователя**. Ревью без эталона = субъективное мнение.

## Severity — маппинг по умолчанию

Каждый пункт чеклиста имеет **severity по умолчанию**. Ревьюер может повысить или понизить severity с обоснованием.

| Severity | Определение | Влияние |
|----------|-------------|---------|
| 🔴 **Blocker** | Публикация невозможна. Фактическая ошибка, юридический риск, полное несоответствие брифу. | REQUIRES CHANGES |
| 🟡 **Major** | Публикация нежелательна. Заметная проблема качества. | REQUIRES CHANGES (при ≥ 3) |
| 🔵 **Minor** | Публикация возможна. Косметическое улучшение. | Не блокирует |

### Порог решения

| Решение | Условие |
|---------|---------|
| **APPROVED** | 0 Blocker + ≤ 2 Major + любое количество Minor |
| **REQUIRES CHANGES** | ≥ 1 Blocker ИЛИ ≥ 3 Major |

## Протокол

### Шаг 1 — Подготовка
1. Получить все обязательные входные данные.
2. Если `$brand-compliance` и/или `$fact-checking` уже пройдены — загрузить результаты, не дублировать проверку.
3. Открыть контент-бриф — зафиксировать: цель, персону, тезисы, CTA, объём, формат.

### Шаг 2 — Прохождение чеклиста
1. Пройти **все пункты** последовательно по категориям (текст → визуал → SEO → ссылки → юридика → accessibility).
2. Для каждого пункта отметить: ✅ / ❌ / N/A.
3. Для каждого ❌ — **обязательно**:
   - Указать severity (по умолчанию из маппинга, можно скорректировать с обоснованием).
   - Привести конкретную цитату / пример проблемы.
   - Предложить **suggested fix** — конкретное исправление, а не «переделать».

### Шаг 3 — Формирование решения
1. Подсчитать 🔴 / 🟡 / 🔵.
2. Применить порог решения.
3. Сформировать список действий (если REQUIRES CHANGES).

## Полный чеклист с severity-маппингом

### Категория: Текст — соответствие брифу

| # | Критерий | Default severity |
|---|----------|-----------------|
| 1 | Тема соответствует брифу | 🔴 Blocker |
| 2 | Цель контента достигается (читатель понимает, что делать) | 🔴 Blocker |
| 3 | Персона ЦА — текст говорит на её языке | 🟡 Major |
| 4 | Все ключевые тезисы из брифа раскрыты | 🟡 Major |
| 5 | Запрещённые темы не затронуты | 🔴 Blocker |
| 6 | Обязательные элементы присутствуют (факты, цитаты, данные) | 🟡 Major |

### Категория: Текст — качество

| # | Критерий | Default severity |
|---|----------|-----------------|
| 7 | Нет орфографических ошибок | 🟡 Major |
| 8 | Нет грамматических ошибок | 🟡 Major |
| 9 | Читаемость — предложения не перегружены, абзацы ≤ 3-4 предложения | 🔵 Minor |
| 10 | Заголовок / hook захватывает внимание | 🟡 Major |
| 11 | Объём соответствует брифу и платформе | 🟡 Major |
| 12 | Нет запрещённых слов / клише (из стоп-листа) | 🟡 Major |
| 13 | Нет повторов, воды, бессмысленных фраз | 🔵 Minor |
| 14 | Логичная структура — от hook к CTA | 🔵 Minor |

### Категория: Текст — CTA

| # | Критерий | Default severity |
|---|----------|-----------------|
| 15 | CTA присутствует | 🔴 Blocker |
| 16 | CTA конкретен (одно действие, не «подпишись и купи и поделись») | 🟡 Major |
| 17 | CTA соответствует этапу воронки из брифа | 🟡 Major |

### Категория: Факты

| # | Критерий | Default severity |
|---|----------|-----------------|
| 18 | Все факты и цифры верифицированы (или ссылка на `$fact-checking`) | 🔴 Blocker |
| 19 | Источники указаны где требуется | 🟡 Major |

### Категория: Визуал

| # | Критерий | Default severity |
|---|----------|-----------------|
| 20 | Визуал соответствует тексту и теме | 🟡 Major |
| 21 | Размеры соответствуют платформе | 🔴 Blocker |
| 22 | Текст на изображении читаем (контраст, размер шрифта) | 🟡 Major |
| 23 | Визуальный стиль соответствует бренду (или ссылка на `$brand-compliance`) | 🟡 Major |
| 24 | Лого размещено корректно (если требуется) | 🔵 Minor |
| 25 | Нет артефактов AI-генерации (лишние пальцы, текст-мусор, искажения) | 🔴 Blocker |

### Категория: SEO (для текстовых форматов)

| # | Критерий | Default severity |
|---|----------|-----------------|
| 26 | Основное ключевое слово присутствует в заголовке | 🟡 Major |
| 27 | Ключевые слова распределены естественно по тексту | 🔵 Minor |
| 28 | Мета-заголовок ≤ 60 символов | 🔵 Minor |
| 29 | Мета-описание ≤ 160 символов, содержит CTA | 🔵 Minor |
> N/A для форматов без SEO (сторис, reels, email).

### Категория: Ссылки и мета

| # | Критерий | Default severity |
|---|----------|-----------------|
| 30 | Все ссылки рабочие (нет 404) | 🔴 Blocker |
| 31 | UTM-метки добавлены (если есть ссылки) | 🔵 Minor |
| 32 | Хэштеги бренда использованы | 🔵 Minor |

### Категория: Юридика

| # | Критерий | Default severity |
|---|----------|-----------------|
| 33 | Disclaimer присутствует (если контент рекламный / финансовый / медицинский) | 🔴 Blocker |
| 34 | Копирайт © указан (если требуется) | 🟡 Major |
| 35 | Нет нарушения авторских прав (чужие фото, текст, музыка) | 🔴 Blocker |

### Категория: Accessibility

| # | Критерий | Default severity |
|---|----------|-----------------|
| 36 | Alt-текст добавлен к изображениям | 🔵 Minor |
| 37 | Контраст текста достаточен для чтения | 🔵 Minor |
| 38 | Субтитры / текстовая версия для видео (если применимо) | 🟡 Major |

## Валидация (Quality Gate)

Ревью считается завершённым если:

- [ ] Все пункты чеклиста пройдены (✅ / ❌ / N/A — ни одного пустого)
- [ ] Для каждого ❌ указаны: severity, цитата/пример, suggested fix
- [ ] Решение соответствует порогу (не субъективное, а по правилам)
- [ ] Если REQUIRES CHANGES — список действий сформирован с приоритетами
- [ ] Если есть результат `$brand-compliance` / `$fact-checking` — сослались, не дублировали

## Handoff

| Решение | Маршрут |
|---------|---------|
| **APPROVED** | → `$content-release-gate` с приложением чеклиста |
| **REQUIRES CHANGES** | → обратно исполнителю с конкретным списком правок |

Маршрут возврата по типу замечания:

| Категория замечаний | Куда возврат |
|--------------------|-------------|
| Текст (тема, тезисы, CTA, качество) | → Copywriter |
| Факты | → Researcher → `$fact-checking` |
| Визуал | → Visual Concept |
| SEO | → Copywriter (или SEO-специалист) |
| Юридика | → Эскалация пользователю |

При передаче — использовать `$handoff` с приложением чеклиста и списка правок.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$content-brief` | **Эталон:** сверка цели, тезисов, CTA, персоны, объёма |
| `$brand-guidelines` | **Эталон:** ToV, стоп-лист |
| `$brand-compliance` | **Комплемент:** бренд-проверка — если пройдена, не дублировать |
| `$fact-checking` | **Комплемент:** верификация фактов — если пройдена, не дублировать |
| `$readability-scoring` | **Комплемент:** детальная оценка читаемости |
| `$platform-compliance` | **Комплемент:** проверка правил платформы |
| `$content-release-gate` | **Потребитель:** результат чеклиста — вход для Release Gate |
| `$board` | Обновить статус гейта Reviewer |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Ревью без брифа | Субъективное «мне не нравится» вместо проверки по критериям | Всегда сверять с `$content-brief` |
| ❌ без suggested fix | Автор не знает, что исправлять | Каждое замечание = конкретное предложение |
| Всё — blocker | Автор демотивирован, не понимает приоритеты | Severity по маппингу, корректировка с обоснованием |
| Дублирование `$brand-compliance` | Двойная работа, возможные противоречия | Ссылаться на результат, не перепроверять |
| Пропуск категорий | «Текст ок» — а ссылки битые, disclaimer нет | Пройти все категории, отметить N/A если не применимо |
| Ревью только текста | Визуал с артефактами AI проходит незамеченным | Визуал — обязательная категория |
| Субъективные замечания | «Заголовок слабый» — а почему? | Конкретика: «Заголовок не содержит выгоду для ЦА, предлагаю: [вариант]» |
