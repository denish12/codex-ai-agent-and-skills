# Brand Compliance — Соответствие бренду

## Когда использовать
- При **ревью контента** перед публикацией (гейт Reviewer).
- При **проверке визуалов** — баннеры, креативы, сторис, видео-обложки.
- При **аудите** существующего контента на соответствие обновлённым гайдлайнам.
- При **онбординге нового копирайтера / дизайнера** — калибровка первых материалов.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Контент для проверки | ✅ | Текст, визуал или полный пост — то, что проверяем |
| Бренд-гайдлайны | ✅ | Результат `$brand-guidelines` или файл/URL гайдлайнов бренда |
| ToV-документ | ✅ | Результат `$tone-of-voice` или описание тона бренда |
| Платформа | ✅ | Где будет опубликовано (влияет на формат-спецификации) |
| Тип контента | ⬚ | Пост / статья / email / баннер / видео / сторис |
| Предыдущие замечания | ⬚ | Если это повторная проверка — список предыдущих нарушений |

> Если бренд-гайдлайны или ToV не предоставлены — **запросить у пользователя**. Без эталона проверка невозможна.

## Severity — Уровни критичности

| Уровень | Иконка | Определение | Примеры |
|---------|--------|-------------|---------|
| **Critical** | 🔴 | Публикация невозможна. Юридический риск, грубое искажение бренда, запрещённый контент. | Неправильное лого, юридический disclaimer отсутствует, оскорбительная формулировка, нарушение копирайта |
| **Major** | 🟡 | Публикация нежелательна. Заметное отклонение от бренда, требует исправления. | Неверный ToV, использование запрещённых слов, цвета за пределами палитры, messaging не совпадает с позиционированием |
| **Minor** | 🔵 | Публикация возможна, но рекомендуется исправить. Косметические отклонения. | Немного другой оттенок цвета, стиль фото чуть отличается, мелкие стилистические расхождения |

### Порог прохождения

| Результат | Условие |
|-----------|---------|
| **COMPLIANT** | 0 Critical + 0 Major |
| **CONDITIONALLY COMPLIANT** | 0 Critical + 1-2 Major (публикация после исправления Major) |
| **NON-COMPLIANT** | ≥ 1 Critical ИЛИ ≥ 3 Major |

## Протокол

### Шаг 0 — Подготовка
1. Получить все обязательные входные данные.
2. Загрузить бренд-гайдлайны (`$brand-guidelines`) и ToV-документ (`$tone-of-voice`).
3. Если это повторная проверка — загрузить список предыдущих нарушений.

### Шаг 1 — Проверка Tone of Voice
1. Сверить формальность / неформальность текста с ToV-документом.
2. Проверить эмоциональную окраску — соответствует ли характеру бренда.
3. Проверить на запрещённые слова, клише, стоп-фразы.
4. Оценить единообразие тона на протяжении всего материала.
5. Проверить язык клиента — использованы ли слова/фразы, понятные ЦА (из `$audience-analysis`).

### Шаг 2 — Проверка Visual Identity
1. Цветовая палитра — все цвета в рамках утверждённой палитры (HEX-коды).
2. Типографика — шрифты, размеры, начертания соответствуют гайдлайнам.
3. Лого — размер, охранное поле, допустимый вариант, запрещённые трансформации.
4. Визуальный стиль — фотографии, иллюстрации, иконки соответствуют стилю бренда.
5. Композиция — отступы, выравнивание, иерархия элементов.
6. Платформенные спецификации — размеры, соотношения сторон, safe zones для платформы.

### Шаг 3 — Проверка Messaging
1. Ценностное предложение — передано верно, не искажено.
2. Позиционирование — соответствует стратегии бренда.
3. CTA — соответствует утверждённым формулировкам, не противоречит бренду.
4. Непротиворечивость — нет конфликтов с другим контентом бренда на том же канале.
5. Ключевые сообщения — приоритетные тезисы бренда присутствуют (если применимо).

### Шаг 4 — Проверка юридических и комплаенс-требований
1. Юридический disclaimer — присутствует, если требуется (реклама, финансы, здоровье).
2. Копирайт — © и год указаны корректно.
3. Отказ от ответственности — если есть обещания/гарантии.
4. Пользовательский контент — если используется чужой контент, есть ли разрешение.
5. Хэштеги бренда — использованы утверждённые, нет запрещённых.

### Шаг 5 — Формирование отчёта
1. Для каждого нарушения: severity + конкретная цитата/пример + рекомендация по исправлению.
2. Рассчитать итог по порогу прохождения.
3. Сформировать сводку и список действий.

## Валидация (Quality Gate)

Проверка считается завершённой если:

- [ ] Все 4 слоя проверены (ToV, Visual, Messaging, Legal)
- [ ] Для каждого нарушения указан severity, цитата и suggested fix
- [ ] Итог соответствует порогу прохождения (не субъективный, а по правилам)
- [ ] Если `NON-COMPLIANT` или `CONDITIONALLY COMPLIANT` — список действий сформирован
- [ ] Если повторная проверка — предыдущие нарушения проверены на исправление

## Handoff

| Итог | Маршрут |
|------|---------|
| **COMPLIANT** | → `$content-release-gate` (финальный гейт перед публикацией) |
| **CONDITIONALLY COMPLIANT** | → обратно исполнителю (Copywriter / Visual Concept) с конкретным списком Major-исправлений → повторная проверка |
| **NON-COMPLIANT** | → обратно исполнителю с полным списком → обновить `$board` статус на `[↩] Возврат` → повторная проверка |

При передаче — использовать `$handoff` с приложением полного отчёта.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$brand-guidelines` | **Источник эталона** — визуальная идентичность, лого, палформа, типографика |
| `$tone-of-voice` | **Источник эталона** — тон, стиль, запрещённые слова |
| `$audience-analysis` | Язык клиента — проверка что текст говорит на языке ЦА |
| `$content-review-checklist` | Комплементарный скил — `$brand-compliance` проверяет бренд, `$content-review-checklist` — качество контента |
| `$platform-compliance` | Комплементарный скил — `$brand-compliance` проверяет бренд, `$platform-compliance` — правила платформы |
| `$board` | Обновить статус гейта по результату проверки |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Проверка без гайдлайнов | Субъективное мнение, а не compliance | Всегда сверять с конкретным эталоном |
| Все нарушения одного уровня | Мелочи блокируют публикацию | Использовать severity: Critical / Major / Minor |
| «Нарушение есть» без цитаты | Исполнитель не понимает, что исправлять | Всегда цитировать конкретное место |
| Нарушение без suggested fix | Обратная связь без решения | Для каждого нарушения — рекомендация |
| Проверка только текста | Визуал может нарушать бренд сильнее | Проверять все 4 слоя |
| Пропуск юридики | Штрафы, блокировки, иски | Всегда проверять disclaimer, копирайт, разрешения |
