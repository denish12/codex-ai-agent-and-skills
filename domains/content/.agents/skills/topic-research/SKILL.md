# Topic Research — Глубокое исследование темы

## Когда использовать
- При создании **экспертного контента** — статьи, лонгриды, карусели с данными.
- При **заполнении `$content-brief`** — сбор фактов, статистики, цитат для конкретной единицы контента.
- При **входе в новую тему** — первичное погружение перед планированием контент-стратегии.
- При **обновлении старого контента** — проверка актуальности данных, поиск новых.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Тема | ✅ | Конкретная тема исследования (не «маркетинг», а «ROI email-маркетинга в B2B SaaS 2025-2026») |
| Контент-бриф | ⬚ | Из `$content-brief` — если исследование для конкретной единицы контента |
| Тип исследования | ✅ | Quick / Standard / Deep Dive (см. ниже) |
| Персона ЦА | ✅ | Из `$audience-analysis` — определяет глубину и угол исследования |
| Платформа | ⬚ | Влияет на глубину: Instagram-пост vs лонгрид |
| Ключевые вопросы | ⬚ | Если пользователь / Strategist уже определили, что нужно узнать |

> Если тема сформулирована слишком широко — **уточнить** конкретный угол до начала. «Email-маркетинг» — не тема. «Почему open rate падает в B2B SaaS» — тема.

## Типы исследований

| Тип | Когда | Глубина | Время | Минимумы |
|-----|-------|---------|-------|----------|
| **Quick** | Короткий пост, сторис, мем с данными | Поверхностное: 2-3 ключевых факта | Быстро | 3 факта, 2 источника, 0 экспертов |
| **Standard** | Пост, карусель, email, Telegram-пост | Среднее: 5-10 фактов, 1-2 эксперта | Среднее | 7 фактов, 5 источников, 1 эксперт |
| **Deep Dive** | Статья, лонгрид, видео-эссе, отчёт | Глубокое: 10+ фактов, 3+ экспертов, все углы | Долго | 12 фактов, 8 источников, 3 эксперта, дискуссионные точки |

## Методология поиска

### Где искать

| Источник | Что найти | Запросы/инструменты |
|----------|----------|---------------------|
| **Научные базы** | Peer-reviewed исследования, мета-анализы | Google Scholar, PubMed, Semantic Scholar |
| **Отраслевые отчёты** | Статистика, тренды, прогнозы | Statista, Gartner, McKinsey, Deloitte, HubSpot Research |
| **Новостные СМИ** | Свежие события, реакции | Google News, Reuters, Bloomberg, отраслевые СМИ |
| **Блоги экспертов** | Мнения, кейсы, практический опыт | Отраслевые блоги, Medium, Substack, личные сайты экспертов |
| **Соцсети** | Мнения, тренды, дискуссии, hot takes | LinkedIn (посты лидеров мнений), Twitter/X, Reddit |
| **YouTube** | Экспертные разборы, конференции, подкасты | YouTube Search, подкасты в нише |
| **Данные компаний** | Кейсы, бенчмарки, product data | Официальные блоги, case studies, press releases |
| **Гос. статистика** | Демография, экономика, рынки | Росстат, BLS, Eurostat, World Bank |

### Стратегия поиска

1. **Начать с обзорных** — найти 2-3 обзорные статьи/отчёта по теме для понимания ландшафта.
2. **Углубиться в данные** — из обзорных → найти первоисточники статистики.
3. **Найти экспертов** — кто пишет/говорит на эту тему? Какие у них позиции?
4. **Найти противоречия** — есть ли альтернативные точки зрения? Споры?
5. **Проверить свежесть** — самые свежие данные по теме.

### Формулировка поисковых запросов

| Цель | Шаблон запроса |
|------|---------------|
| Статистика | `"[тема] statistics 2025"`, `"[тема] data report"` |
| Тренды | `"[тема] trends 2026"`, `"future of [тема]"` |
| Экспертные мнения | `"[тема]" site:linkedin.com`, `"[эксперт]" "[тема]"` |
| Исследования | `"[тема]" site:scholar.google.com`, `"[тема] study research"` |
| Кейсы | `"[тема] case study"`, `"[тема] results"` |
| Противоречия | `"[тема] myth"`, `"[тема] wrong"`, `"[тема] debate"` |

## Адаптивная актуальность

Не хардкод «2 года» — зависит от темы:

| Тип темы | Макс. возраст данных | Примеры |
|----------|---------------------|---------|
| **Tech / Digital marketing** | ≤ 1 год | AI, social media алгоритмы, SaaS-метрики |
| **Бизнес / Рынки** | ≤ 2 года | Размер рынка, потребительское поведение |
| **Наука / Медицина** | ≤ 5 лет (для основ), ≤ 2 года (для прорывов) | Медицинские рекомендации, научные открытия |
| **История / Культура** | Без ограничений (но интерпретации — свежие) | Исторические факты, культурные тренды |
| **Юридика** | Действующее законодательство (проверить дату) | Законы, регуляции — могли измениться |

> Если данные старше порога — **искать свежие**. Если свежих нет — использовать с пометкой: «Данные [год], свежее не найдено».

## Content Gap Analysis

Перед написанием — проверить, что уже существует:

| Шаг | Действие | Зачем |
|-----|----------|-------|
| 1 | Загуглить тему → посмотреть топ-10 | Что уже написано? |
| 2 | Проанализировать структуру 3-5 лучших | Какие подтемы покрыты? |
| 3 | Найти **пробелы** — что не покрыто / покрыто плохо | Наш уникальный вклад |
| 4 | Проверить PAA (People Also Ask) | Вопросы, на которые нет хороших ответов |
| 5 | Проверить Reddit / Quora | Какие вопросы задают реальные люди? |

> **Правило:** Если тема полностью покрыта — найти уникальный угол: свежие данные, другая аудитория, практический опыт, нестандартный формат.

## Протокол

### Шаг 1 — Подготовка
1. Получить тему, определить тип исследования (Quick / Standard / Deep Dive).
2. Уточнить угол: не широко, а конкретно.
3. Сформулировать **5-10 ключевых вопросов**, на которые исследование должно ответить.
4. Определить персону → влияет на глубину и угол.

### Шаг 2 — Content Gap Analysis
1. Проверить, что уже существует по теме (топ-10 Google, PAA, Reddit).
2. Найти пробелы — наш уникальный вклад.
3. Зафиксировать: что мы сделаем лучше / иначе / глубже.

### Шаг 3 — Сбор фактов и данных
1. По методологии поиска — собирать факты, статистику, данные.
2. Для каждого факта — фиксировать: значение + источник + год.
3. Соблюдать порог актуальности по типу темы.
4. Собрать минимум по требованиям типа исследования.

### Шаг 4 — Экспертные мнения
1. Найти **экспертов** по теме — кто пишет, выступает, исследует.
2. Собрать **цитаты** с атрибуцией (имя, должность, источник).
3. Для Deep Dive — минимум 3 эксперта с разными позициями.

### Шаг 5 — Дискуссионные точки и противоречия
1. Есть ли **альтернативные мнения** по теме?
2. Есть ли **мифы**, которые можно развенчать?
3. Зафиксировать: позиция A vs позиция B — с аргументами обеих сторон.

### Шаг 6 — Структурирование
1. Сгруппировать данные по **тезисам** (3-7 ключевых тезисов).
2. К каждому тезису — поддерживающие факты / статистика / цитата.
3. Определить **самый сильный факт** — крючок для заголовка / hook.
4. Сформировать **структуру для копирайтера**.

### Шаг 7 — Передача на верификацию
1. Передать все источники в `$source-verification`.
2. Дождаться результата верификации.
3. Заменить отклонённые источники или удалить неподтверждённые факты.

## Валидация (Quality Gate)

Исследование считается завершённым если:

- [ ] Ключевые вопросы сформулированы и на все есть ответы
- [ ] Content Gap Analysis проведён (что уже есть, наш уникальный вклад)
- [ ] Минимумы по типу исследования выполнены (факты, источники, эксперты)
- [ ] Все данные в пороге актуальности (по типу темы)
- [ ] Экспертные мнения с полной атрибуцией (имя, должность, источник)
- [ ] Дискуссионные точки выявлены (для Standard и Deep Dive)
- [ ] Данные структурированы по тезисам + крючок-факт выделен
- [ ] Источники переданы в `$source-verification`
- [ ] Отклонённые источники заменены или факты удалены

## Handoff

| Потребитель | Как использует |
|-------------|---------------|
| `$source-verification` | **Обязательно:** верификация всех найденных источников |
| `$data-storytelling` | Факты и данные → нарратив |
| `$content-brief` | Факты, тезисы, крючок → заполнение брифа |
| Copywriter (агент) | Структура + факты → написание текста |
| `$fact-checking` | На этапе ревью — проверка фактов из исследования |
| `$seo-copywriting` | LSI-слова, PAA-вопросы из gap analysis |

При передаче — использовать `$handoff` с полным отчётом.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$source-verification` | **Потребитель:** все источники → на верификацию |
| `$data-storytelling` | **Потребитель:** факты → нарратив |
| `$content-brief` | **Потребитель:** данные исследования → бриф |
| `$competitor-content-analysis` | **Комплемент:** конкуренты по контенту (не по бизнесу) — overlap при content gap |
| `$audience-analysis` | **Источник:** персона → угол и глубина исследования |
| `$trend-research` | **Комплемент:** topic research = факты по теме, trend research = что трендит сейчас |
| `$fact-checking` | **Потребитель:** проверяет факты из исследования на этапе ревью |
| `$seo-copywriting` | **Потребитель:** LSI-слова, PAA → SEO |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Тема слишком широкая | «Маркетинг» → 1 млн результатов, нет фокуса | Уточнить угол: «ROI email-маркетинга в B2B SaaS 2026» |
| Только первая страница Google | 80% статей по теме — рерайт друг друга | Искать глубже: Scholar, Reddit, эксперты, первоисточники |
| Факты без источников | Непроверяемо, нет доверия | Каждый факт = источник + год |
| Старые данные без пометки | Читатель принимает за актуальные | Пометить год, искать свежие |
| Нет Content Gap Analysis | Пишем то же, что уже написано | Проверить топ-10, найти пробелы |
| Только «за» — нет противоречий | Однобокий контент, нет глубины | Показать обе стороны, найти мифы |
| Deep Dive для Instagram-поста | Избыточная глубина, потрачено время | Тип исследования = тип контента |
| Нет передачи на верификацию | Непроверенные факты попадают в текст | Всегда → `$source-verification` |
