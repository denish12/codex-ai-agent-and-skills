# Marketing Psychology — Психология маркетинга

## Когда использовать
- При **разработке стратегии** — выбор позиционирования, messaging, воронки на основе психологии.
- При **формировании брифа** — какие триггеры и модели встроить в контент.
- При **написании текста** — применение конкретных приёмов убеждения.
- При **оптимизации CTA** — выбор психологического рычага для конверсии.
- При **проектировании email-серий** — последовательность триггеров по этапам.
- При **анализе конкурентов** — какие психологические приёмы используют они.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Персона ЦА | ✅ | Из `$audience-analysis` — боли, триггеры, барьеры, JTBD |
| Бизнес-цель | ✅ | Awareness / лиды / конверсия / retention |
| Этап воронки | ✅ | TOFU / MOFU / BOFU — определяет набор моделей |
| Тип контента | ✅ | Пост / email / лендинг / реклама / карусель |
| Продукт / оффер | ⬚ | Что продвигаем — для привязки моделей к конкретике |
| Ограничения | ⬚ | Юридические, этические, бренд — что нельзя |

> Психология без персоны = манипуляция вслепую. Всегда начинать с понимания аудитории.

## Психологические модели и триггеры

### Каталог моделей

#### 1. Reciprocity (Взаимность)
**Суть:** Когда человек получает ценность бесплатно — он чувствует обязательство ответить.
**Как применять:** Дать ценность ДО просьбы: бесплатный гайд, инструмент, чеклист, консультация.
**Leverage:** Высокий (TOFU/MOFU)
**Пример:** «Скачай бесплатный чеклист из 50 пунктов» → потом «Попробуй наш инструмент»

#### 2. Social Proof (Социальное доказательство)
**Суть:** Люди следуют за действиями других, особенно похожих на себя.
**Типы:**
- **Числовой:** «10 000+ подписчиков», «500+ компаний используют»
- **Авторитетный:** «Рекомендует [эксперт]», «По версии Forbes»
- **Peer:** «Маркетологи из [ваша ниша] уже перешли»
- **Отзыв:** Конкретная цитата клиента с результатом
- **Case study:** Детальная история успеха
**Leverage:** Очень высокий (MOFU/BOFU)
**Пример:** «847 SaaS-компаний увеличили retention на 23% за первый квартал»

#### 3. Scarcity (Дефицит)
**Суть:** Ограниченное кажется более ценным.
**Типы:**
- **По количеству:** «Осталось 12 мест»
- **По времени:** «До пятницы», «Последние 24 часа»
- **По доступу:** «Только для первых 100», «Закрытый доступ»
**Leverage:** Высокий (BOFU)
**⚠️ Правило:** Scarcity должен быть **реальным**. Фейковый дефицит = потеря доверия навсегда.
**Пример:** «Набор закрывается в пятницу — осталось 7 мест из 50»

#### 4. Loss Aversion (Неприятие потерь)
**Суть:** Боль потери в 2× сильнее радости приобретения (Kahneman & Tversky).
**Как применять:** Показать что теряет человек, если НЕ действует.
**Leverage:** Очень высокий (MOFU/BOFU)
**Пример:** «Каждый день без автоматизации — 3 часа потерянного времени. Это 60 часов в месяц»
**⚠️ Баланс:** Не переборщить — избыток страха → отторжение. Показать потерю → тут же дать решение.

#### 5. Anchoring (Якорение)
**Суть:** Первая цифра / факт определяет рамку для оценки всего последующего.
**Как применять:** Сначала назвать высокую цену / большую проблему → потом показать альтернативу.
**Leverage:** Высокий (BOFU, pricing)
**Пример:** «Агентство берёт $5000/мес. Наш инструмент — $99/мес. Тот же результат»

#### 6. Authority (Авторитет)
**Суть:** Люди доверяют экспертам, лидерам мнений, институциям.
**Как применять:** Показать экспертность: исследования, цитаты экспертов, сертификаты, упоминания в медиа.
**Leverage:** Высокий (TOFU/MOFU)
**Пример:** «По данным McKinsey (2026)...», «Рекомендует [известный эксперт]»

#### 7. Commitment & Consistency (Последовательность)
**Суть:** Люди стремятся быть последовательными: если сказал «да» малому — скажет «да» большому.
**Как применять:** Начать с малой просьбы (подпишись) → потом большая (купи). Foot-in-the-door.
**Leverage:** Средний-Высокий (воронка целиком)
**Пример:** «Скачай бесплатный чеклист» → «Запишись на вебинар» → «Попробуй trial» → «Оформи подписку»

#### 8. Curiosity Gap (Разрыв любопытства)
**Суть:** Мозг не терпит незавершённости — стремится закрыть информационный пробел.
**Как применять:** Начать историю / факт, но не договорить → CTA = узнать дальше.
**Leverage:** Очень высокий (TOFU — hooks, subject lines)
**Пример:** «87% маркетологов делают эту ошибку. Проверь — ты тоже?»
**⚠️ Правило:** Curiosity gap должен быть **закрыт** в контенте. Не закрыть = clickbait → потеря доверия.

#### 9. Framing Effect (Эффект фрейминга)
**Суть:** Одна и та же информация, поданная по-разному, вызывает разные решения.
**Типы:**
- **Positive frame:** «95% клиентов довольны» (подчёркивает успех)
- **Negative frame:** «Не теряйте 5 часов в неделю на ручную работу» (подчёркивает потерю)
- **Temporal frame:** «За 15 минут в день» vs «90 часов в год»
**Leverage:** Высокий (любой этап)
**Пример:** «Сэкономь 30%» (positive) vs «Не переплачивай 30%» (negative) — второе сильнее.

#### 10. Endowment Effect (Эффект обладания)
**Суть:** Люди ценят то, что уже «имеют», выше, чем то, чего не имеют.
**Как применять:** Дать попробовать бесплатно → человек не хочет потерять то, что уже использует.
**Leverage:** Высокий (BOFU — trials, freemium)
**Пример:** «14 дней бесплатно. Настрой за 5 минут» → после trial: «Не потеряй свои настройки и данные — продолжи подписку»

#### 11. Paradox of Choice (Парадокс выбора)
**Суть:** Слишком много вариантов → паралич решения → ничего не выбирают.
**Как применять:** Ограничить выбор: 2-3 варианта, один рекомендованный. Один CTA.
**Leverage:** Высокий (BOFU — pricing, CTA)
**Пример:** Pricing page: 3 плана, средний = «Популярный». Не 7 планов.

#### 12. Bandwagon Effect (Эффект подражания)
**Суть:** Чем больше людей делает X — тем больше хочется делать X.
**Как применять:** Показывать рост: «Присоединились 500 человек за последнюю неделю», «Тренд 2026 года».
**Leverage:** Средний-Высокий (TOFU/MOFU)
**Пример:** «2000+ маркетологов подписались за этот месяц — присоединяйся»

#### 13. Zeigarnik Effect (Эффект Зейгарник)
**Суть:** Незавершённые задачи запоминаются лучше завершённых.
**Как применять:** Open loops в контенте, серии, «часть 1 из 5», progress bars.
**Leverage:** Средний (retention, серии)
**Пример:** «В следующем письме — 3-й шаг, который удвоил конверсию. Не пропусти»

#### 14. IKEA Effect (Эффект IKEA)
**Суть:** Люди ценят то, что создали сами, выше объективной стоимости.
**Как применять:** Вовлечь в создание: «Настрой свой план», «Собери свой набор», калькуляторы, квизы.
**Leverage:** Средний (MOFU — interactive content)
**Пример:** «Пройди квиз за 2 минуты → получи персональный план»

#### 15. Peak-End Rule (Правило пика и конца)
**Суть:** Люди запоминают пиковый момент опыта и конец. Середина забывается.
**Как применять:** Поставить WOW-момент в середине контента + сильный финал (CTA).
**Leverage:** Средний (контент-структура)
**Пример:** Карусель: slide 5 = шокирующий факт (пик), slide 10 = мощный CTA (конец)

## Scoring System — Leverage × Feasibility

### Как выбрать модели для конкретной задачи

Для каждой модели оценить по 2 осям (1-5):

| Ось | 1 | 3 | 5 |
|-----|---|---|---|
| **Leverage** (сила воздействия) | Слабое влияние на ЦА | Умеренное | Прямое влияние на решение |
| **Feasibility** (реализуемость) | Сложно реализовать (нужны данные, бюджет, time) | Умеренно | Легко встроить в контент |

**Score = Leverage × Feasibility** (макс 25)

### Матрица приоритизации

| Score | Приоритет | Действие |
|:-----:|-----------|----------|
| 20-25 | 🟢 **Must Use** | Обязательно применить |
| 12-19 | 🔵 **Should Use** | Рекомендуется, если есть ресурс |
| 6-11 | 🟡 **Could Use** | По возможности |
| 1-5 | ⬚ **Skip** | Не стоит усилий |

## Комбинации моделей по задачам

### Лендинг (конверсия)
**Рекомендуемый стек:** Authority + Social Proof + Loss Aversion + Scarcity + Anchoring
```
[Authority] «По данным McKinsey...»
[Loss Aversion] «Вы теряете 60 часов в месяц на ручную работу»
[Social Proof] «500+ компаний уже автоматизировали»
[Anchoring] «Агентство = $5000/мес. Мы = $99/мес»
[Scarcity] «Цена растёт через 3 дня»
```

### Email-серия (nurturing)
**Рекомендуемый стек:** Reciprocity → Commitment → Social Proof → Loss Aversion → Scarcity
```
Письмо 1: [Reciprocity] Бесплатный гайд
Письмо 2: [Commitment] Квиз / чеклист (малое действие)
Письмо 3: [Social Proof] Кейс клиента
Письмо 4: [Loss Aversion] Что теряете без решения
Письмо 5: [Scarcity] Оффер с ограничением
```

### Пост в соцсетях (awareness)
**Рекомендуемый стек:** Curiosity Gap + Authority + Bandwagon
```
[Curiosity Gap] Hook: «87% маркетологов делают эту ошибку»
[Authority] Данные из исследования
[Bandwagon] «Присоединяйся к 10K+ подписчиков»
```

### Карусель (education + engagement)
**Рекомендуемый стек:** Curiosity Gap + Peak-End + Zeigarnik + Commitment
```
Slide 1: [Curiosity Gap] Hook-заголовок с интригой
Slide 2-4: [Commitment] Контент (малые «да»: «Знал? → Нет? → Вот:»)
Slide 5: [Peak-End] Пиковый инсайт / шокирующий факт
Slide 9-10: [Peak-End] Сильный финал + CTA
[Zeigarnik] «Часть 2 — завтра» (если серия)
```

### Pricing page
**Рекомендуемый стек:** Anchoring + Paradox of Choice + Social Proof + Endowment
```
[Anchoring] Показать самый дорогой план первым
[Paradox of Choice] 3 плана (не 7), средний = «Популярный»
[Social Proof] «Выбор 70% клиентов»
[Endowment] «14 дней бесплатно — попробуй прямо сейчас»
```

## Протокол

### Шаг 1 — Контекст
1. Определить персону, бизнес-цель, этап воронки, тип контента.
2. Определить ограничения (этика, юридика, бренд).

### Шаг 2 — Выбор моделей
1. По таблице scoring по воронке — выбрать **3-5 моделей** с наивысшим leverage для этапа.
2. Для каждой — оценить **feasibility** (есть ли данные для social proof? реальный ли scarcity?).
3. Рассчитать **Score = Leverage × Feasibility**.
4. Отсортировать. Взять **топ-3** (Must Use + Should Use).

### Шаг 3 — Применение
1. Определить **где в контенте** применять каждую модель (hook / body / CTA / P.S.).
2. Для каждой — написать **конкретную формулировку** (не абстрактно, а текст).
3. Проверить **этичность** (см. ниже).

### Шаг 4 — Комбинация
1. Проверить что модели **не конфликтуют** (Scarcity + «мы для всех» = конфликт).
2. Проверить что **не больше 3-4** моделей в одной единице контента (перегруз → манипулятивно).
3. Выстроить **последовательность**: от soft (Curiosity, Authority) к hard (Scarcity, Loss Aversion).

### Шаг 5 — Этическая проверка
Пройти чеклист:
- [ ] Scarcity реальный (не фейковый дедлайн)?
- [ ] Social proof правдивый (не выдуманные цифры)?
- [ ] Loss aversion не чрезмерный (показывает потерю, но не запугивает)?
- [ ] Curiosity gap закрывается в контенте (не clickbait)?
- [ ] Человек после принятия решения **не пожалеет** (ценность реальна)?

> **Правило:** Если модель работает только через обман — не использовать. Этичная психология = показать **реальную** ценность эффективнее.

## Этические границы

| ✅ Этично | ❌ Неэтично |
|-----------|------------|
| Реальный scarcity (мест действительно мало) | Фейковый таймер, который обнуляется |
| Social proof с реальными числами | Выдуманные отзывы и цифры |
| Loss aversion: показать реальную цену бездействия | Fear-mongering: запугивание без решения |
| Curiosity gap, закрытый в контенте | Clickbait: hook без содержания |
| Anchoring с реальными ценами конкурентов | Завышение «было» для иллюзии скидки |
| Reciprocity: ценность бесплатно | Guilt-tripping: «мы дали, теперь ты должен» |
| Framing: выбор угла подачи | Искажение данных через cherry-picking |

## Валидация (Quality Gate)

Применение психологии считается готовым если:

- [ ] Персона определена (модели привязаны к конкретным болям/триггерам)
- [ ] Выбрано 2-4 модели (не больше — перегруз)
- [ ] Для каждой модели: Score (Leverage × Feasibility) рассчитан
- [ ] Для каждой модели: конкретная формулировка (не абстракция)
- [ ] Модели не конфликтуют между собой
- [ ] Последовательность: soft → hard
- [ ] Этическая проверка пройдена (5 пунктов)
- [ ] Curiosity gaps закрываются в контенте
- [ ] Scarcity реальный (если используется)

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| 7+ моделей в одном посте | Манипулятивно, перегруз, отторжение | 2-4 модели максимум |
| Фейковый scarcity | Потеря доверия навсегда | Только реальные ограничения |
| Curiosity без closure | Clickbait → unfollow | Всегда закрывать gap в контенте |
| Только negative framing | Страх → отторжение | Баланс: показать потерю → дать решение |
| Психология без персоны | Приёмы мимо аудитории | Модели привязаны к конкретным болям ЦА |
| Loss aversion для тира A | VIP не нуждаются в запугивании | Тир A = exclusivity и reciprocity |
| Scarcity для TOFU | Холодная аудитория не купит под давлением | Scarcity = BOFU. TOFU = Curiosity, Authority |
| Копирование без адаптации | «Осталось 3 места» для SaaS с безлимитной подпиской | Scarcity должен быть правдоподобен |
