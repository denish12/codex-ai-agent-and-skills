# Visual Brief — Визуальный бриф

## Когда использовать
- При создании **любого визуального контента** — пост, баннер, Stories, обложка, карусель, thumbnail.
- Как **конкретизация `$moodboard`** — мудборд задаёт стиль, visual brief описывает конкретную задачу.
- Как **вход для `$image-prompt-engineering`** — бриф → промпт для AI-генерации.
- Как **ТЗ для дизайнера** — полная спецификация без необходимости переспрашивать.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Контент-бриф | ✅ | Из `$content-brief` — тема, тезисы, CTA, персона, платформа |
| Бренд-гайдлайны | ✅ | Из `$brand-guidelines` — палитра, типографика, лого, визуальный стиль |
| Мудборд | ⬚ | Из `$moodboard` — если создан для кампании / серии (настроение, референсы, расширенная палитра) |
| Платформа и формат | ✅ | Из `$platform-visual-specs` — размеры, ratio, safe zones |
| Текст для визуала | ⬚ | Если на изображении будет текст — готовый заголовок, CTA |
| Исполнитель | ⬚ | Дизайнер / AI-генерация / шаблон Canva — влияет на детализацию |

> Если бренд-гайдлайны не предоставлены — **запросить**. Visual brief без бренда = случайный визуал.

## Связь с другими скилами

```
$moodboard (стиль кампании)
       ↓
$visual-brief (ТЗ на конкретный визуал) ← $content-brief (тема, CTA)
       ↓                                 ← $brand-guidelines (палитра, лого)
       ↓                                 ← $platform-visual-specs (размеры)
       ↓
$image-prompt-engineering (промпт для AI)
       ↓
Visual Concept (агент) → генерация / создание
       ↓
$content-review-checklist (#20-25) → проверка
```

## Протокол

### Шаг 1 — Контекст и цель
1. Загрузить `$content-brief` — понять тему, персону, CTA.
2. Определить **цель визуала**:
   - **Привлечь внимание** — hook, остановить скролл.
   - **Передать информацию** — инфографика, схема, данные.
   - **Вызвать эмоцию** — настроение, атмосфера, вдохновение.
   - **Усилить CTA** — кнопка, стрелка, фокус на действии.
3. Определить **что зритель должен понять за 1-3 секунды** (главная мысль визуала).

### Шаг 2 — Центральный образ
1. Описать **главный элемент** визуала — что находится в фокусе.
2. Описать **детали субъекта** — конкретно, не абстрактно:
   - ❌ «Человек работает»
   - ✅ «Молодая женщина в тёмно-синем блейзере, работает за ноутбуком в светлом офисе, улыбается»
3. Если несколько элементов — описать **иерархию**: что главное, что вторичное, что фоновое.

### Шаг 3 — Композиция
Определить расположение элементов:

| Элемент композиции | Описание | Когда использовать |
|-------------------|----------|-------------------|
| **Focal point** | Где взгляд зрителя должен остановиться первым | Всегда — определить главный якорь |
| **Rule of thirds** | Субъект на пересечении третей | Динамичные, editorial фото |
| **Центральная** | Субъект в центре, симметрия | Минимализм, продукт, лого |
| **Negative space** | Пустое пространство для текста или воздуха | Если есть текст на изображении |
| **Text placement** | Где будет текст: верх / низ / лево / право / overlay | Если заголовок/CTA на визуале |
| **Z-pattern / F-pattern** | Движение взгляда по визуалу | Сложные визуалы с несколькими элементами |
| **Grid** | Карусель или серия — единая сетка | Карусели, серии постов |

### Шаг 4 — Бренд-элементы
Из `$brand-guidelines`:
1. **Цвета** — конкретные HEX. Роли: основной, акцент, фон.
2. **Типографика** — шрифт, размер, начертание (для текста на визуале).
3. **Лого** — нужно ли? Какой вариант? Позиция? Размер?
4. **Графические приёмы** — фирменные рамки, overlay, паттерны (если есть в гайдлайне).

Из `$moodboard` (если есть):
5. **Стиль** — медиум (фото / иллюстрация / 3D), обработка, текстуры.
6. **Настроение** — эмоция из мудборда.

### Шаг 5 — Текст на изображении
Если визуал содержит текст:

| Элемент | Текст | Размер | Позиция | Стиль |
|---------|-------|--------|---------|-------|
| **Заголовок** | [текст или «нет»] | [крупный / средний] | [где на визуале] | [начертание, цвет] |
| **Подзаголовок** | [текст или «нет»] | [средний / мелкий] | [где] | [стиль] |
| **CTA** | [текст или «нет»] | [размер] | [где] | [кнопка / текст / стрелка] |
| **Водяной знак / ©** | [текст или «нет»] | [мелкий] | [угол] | [полупрозрачный] |

**Правила текста:**
- Читаемость: контраст ≥ 4.5:1 (используя подложку если нужно).
- Размер: ≥ 16px на мобильном (≈ 24px+ на исходнике).
- Объём: заголовок ≤ 7 слов, подзаголовок ≤ 15 слов.

### Шаг 6 — Ограничения (Do / Don't)

| ✅ Обязательно | ❌ Запрещено |
|---------------|------------|
| [что должно быть] | [что нельзя] |
| [элемент / стиль] | [элемент / стиль — почему] |

### Шаг 7 — Референсы
1. Из `$moodboard` — выбрать **2-3 наиболее релевантных** для этой задачи.
2. Для каждого — **что конкретно взять** (палитру? композицию? стиль?).
3. Если нет мудборда — описать стиль словами + 1-2 референса.

### Шаг 8 — Технические спеки
Из `$platform-visual-specs`:
1. Размер (W×H px).
2. Ratio.
3. Формат файла.
4. Safe zones.
5. Для AI-генерации: Midjourney `--ar` параметр.

### Шаг 9 — Варианты (A/B)
Если нужно тестировать визуалы:
1. **Вариант A** — основной (по брифу).
2. **Вариант B** — альтернатива (другая композиция / цветовая акцентуация / другой образ).
3. Указать **переменную теста**: что именно отличается.

### Шаг 10 — Accessibility
1. **Alt-текст** — описание изображения для screen readers.
2. **Контраст** — текст на изображении ≥ 4.5:1.
3. **Dark mode** — как визуал будет выглядеть на тёмном фоне?
4. **Не только цветом** — информация не передаётся только цветом (иконки, подписи).

## Валидация (Quality Gate)

Visual brief считается готовым если:

- [ ] Цель визуала определена (что зритель поймёт за 1-3 секунды)
- [ ] Центральный образ описан конкретно (не «человек», а детальное описание)
- [ ] Композиция определена (focal point, placement, negative space для текста)
- [ ] Бренд-элементы из `$brand-guidelines` (цвета HEX, шрифт, лого — позиция и вариант)
- [ ] Текст на изображении: содержание + размер + позиция (или «нет текста»)
- [ ] Технические спеки из `$platform-visual-specs` (размер, ratio, формат, safe zone)
- [ ] Ограничения Do/Don't зафиксированы
- [ ] Референсы указаны (из `$moodboard` или описательно)
- [ ] Alt-текст подготовлен
- [ ] Контраст текста ≥ 4.5:1 (если текст на визуале)
- [ ] Если A/B — переменная теста указана

## Handoff

| Потребитель | Как использует |
|-------------|---------------|
| `$image-prompt-engineering` | **Основной:** бриф → промпт для AI-генерации |
| Visual Concept (агент) | ТЗ для создания визуала (дизайнер или AI) |
| `$content-review-checklist` | Ревьюер проверяет визуал по пунктам #20-25 |
| `$content-release-gate` | Визуал как часть пакета публикации |

При передаче — использовать `$handoff` с приложением полного брифа.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$content-brief` | **Источник:** тема, CTA, персона → цель и содержание визуала |
| `$brand-guidelines` | **Источник:** палитра, типографика, лого, визуальный стиль |
| `$moodboard` | **Источник:** настроение, референсы, расширенная палитра |
| `$platform-visual-specs` | **Источник:** размеры, ratio, safe zones |
| `$image-prompt-engineering` | **Потребитель:** бриф → промпт |
| `$content-review-checklist` | **Потребитель:** проверка визуала |
| `$headline-formulas` | **Источник:** заголовок для текста на визуале |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| «Красивая картинка про маркетинг» | Абстрактно — дизайнер/AI не знает что делать | Конкретный субъект, композиция, настроение |
| Бриф без бренд-элементов | Визуал не узнаваем как бренд | Всегда: цвета HEX, шрифт, лого из `$brand-guidelines` |
| Текст без указания позиции | Текст закрывает лицо / ключевой элемент | Text placement + negative space в композиции |
| Нет safe zones | Текст скрыт под UI платформы | Safe zones из `$platform-visual-specs` |
| Один вариант без альтернативы | Нет выбора — если не подойдёт, начинать заново | A/B: минимум 2 варианта для важных визуалов |
| Бриф для AI = бриф для дизайнера | Разный уровень конкретики нужен | Для AI → `$image-prompt-engineering` с техническими параметрами |
| Нет alt-текста | Недоступно для screen readers, хуже SEO | Всегда подготовить alt-текст |
| Мелкий текст на визуале | Нечитаем на мобильном | ≥ 24px на исходнике, проверить на телефоне |
