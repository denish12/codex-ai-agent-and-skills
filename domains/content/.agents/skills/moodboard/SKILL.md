# Moodboard — Мудборд

## Когда использовать
- При создании **визуальной концепции** для кампании, серии контента, ребрендинга.
- При **старте работы Visual Concept** — до `$visual-brief` и `$image-prompt-engineering`.
- При **расхождении видения** — синхронизация между пользователем и агентом «как это должно выглядеть».
- При **выходе на новую платформу** — адаптация визуального стиля под канал.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Контекст задачи | ✅ | Для чего мудборд: кампания, серия постов, ребрендинг, конкретная единица контента |
| Бренд-гайдлайны | ✅ | Из `$brand-guidelines` — палитра, стиль, типографика. Мудборд расширяет, не противоречит |
| Персона ЦА | ✅ | Из `$audience-analysis` — визуальные ожидания аудитории |
| Платформа | ✅ | Влияет на визуальный стиль и форматы |
| Настроение / эмоция | ⬚ | Если не указано — определить из контекста контента и бренда |
| Референсы от пользователя | ⬚ | URL, описания, скриншоты — если пользователь имеет видение |
| Результат `$competitor-content-analysis` | ⬚ | Визуальный стиль конкурентов — для дифференциации |

> Если бренд-гайдлайны не предоставлены — **запросить** или направить на `$brand-guidelines`. Мудборд без бренда = произвольный стиль.

## Связь с `$brand-guidelines`

Мудборд **не заменяет** бренд-гайдлайны — он **расширяет** их для конкретного контекста:

| Аспект | `$brand-guidelines` | Мудборд |
|--------|---------------------|---------|
| Палитра | Фиксированная палитра бренда | Расширенная палитра для кампании (в рамках бренда) |
| Типографика | Утверждённые шрифты | Акцентные приёмы (размеры, расположение) |
| Стиль фото | Общие правила стиля | Конкретные референсы для этой задачи |
| Настроение | Характер бренда (архетип) | Настроение конкретной кампании |

> **Правило:** Если мудборд противоречит `$brand-guidelines` — мудборд неправильный. Бренд-гайдлайны приоритетнее.

## Протокол

### Шаг 1 — Контекст и ограничения
1. Определить **для чего** мудборд (кампания / серия / единица контента).
2. Загрузить `$brand-guidelines` — зафиксировать палитру, стиль, типографику как ограничения.
3. Определить персону → какой визуальный стиль резонирует с ЦА.
4. Определить платформу → визуальные ожидания канала.

### Шаг 2 — Настроение и эмоция
1. Определить **основную эмоцию** (1 слово): уверенность, вдохновение, спокойствие, энергия, теплота, провокация, премиальность, игривость.
2. Определить **вторичную эмоцию** (оттенок): профессионализм + теплота, энергия + минимализм.
3. Описать **атмосферу** в 2-3 предложениях — как если бы описывали кадр из фильма.

### Шаг 3 — Цветовая палитра
1. Взять **основные цвета бренда** из `$brand-guidelines`.
2. Расширить для кампании: добавить 1-2 **дополнительных цвета**, гармоничных с брендом.
3. Определить **роли**: доминирующий (60%), вторичный (30%), акцент (10%).
4. Указать HEX + описательное название.

### Шаг 4 — Референсы
1. Отобрать **6-10 референсов** (описать текстом, не прикреплять файлы).
2. Категоризировать каждый — что конкретно берём:

| Категория | Что берём | Пример |
|-----------|----------|--------|
| **Палитра** | Цветовая гамма | «Приглушённые земляные тона из этого shot'а» |
| **Композиция** | Расположение элементов, кадр | «Много воздуха, субъект смещён вправо» |
| **Стиль** | Медиум, обработка | «Матовая обработка, soft light, editorial feel» |
| **Типографика** | Шрифтовое решение | «Bold sans-serif на контрастном фоне» |
| **Настроение** | Общее ощущение | «Спокойствие + уверенность, как у бренда Aesop» |
| **Текстура / Паттерн** | Фоновые элементы | «Тонкая зернистость плёнки, grain effect» |

3. Для каждого референса — **что НЕ берём** (если что-то в референсе не подходит).

### Шаг 5 — Типографика
1. **Заголовки** — шрифт, начертание, размер, особенности (caps, tracking).
2. **Основной текст** — шрифт, начертание.
3. **Акценты** — цитаты, выноски, CTA.
4. Всё в рамках `$brand-guidelines` (или с обоснованием расширения).

### Шаг 6 — Тип изображений и стиль
1. Определить **основной медиум**: фото / иллюстрация / 3D / flat / mixed.
2. Описать **стилевые характеристики** подробно (освещение, обработка, фильтры, текстуры).
3. Определить **визуальные паттерны** — повторяющиеся элементы (геометрия, градиенты, рамки, overlays).

### Шаг 7 — Do / Don't
Для каждого ключевого аспекта (палитра, стиль, композиция, типографика):
- ✅ **Do:** конкретный пример того, что подходит.
- ❌ **Don't:** конкретный пример того, что запрещено, и почему.

### Шаг 8 — Платформенная адаптация
Если мудборд для нескольких платформ — указать различия:

| Платформа | Визуальный акцент | Особенности |
|-----------|------------------|-------------|
| **Instagram** | Яркий, вертикальный, mobile-first | Текст крупный, читаем на телефоне, визуал > текст |
| **LinkedIn** | Профессиональный, сдержанный | Инфографика, данные, минимализм |
| **Telegram** | Информативный, компактный | Горизонтальные картинки, не перегруженные |
| **YouTube** | Кликабельный, контрастный | Thumbnails: лицо + текст + эмоция |

## Валидация (Quality Gate)

Мудборд считается готовым если:

- [ ] Контекст задачи определён (для чего мудборд)
- [ ] Настроение и атмосфера описаны (основная + вторичная эмоция)
- [ ] Палитра: 3-5 цветов с ролями, не противоречит `$brand-guidelines`
- [ ] Референсы: 6-10 штук, каждый категоризирован (что берём + что не берём)
- [ ] Типографика определена (в рамках `$brand-guidelines`)
- [ ] Тип изображений и стиль описаны подробно
- [ ] Do / Don't: минимум по 2 для палитры, стиля, композиции
- [ ] Платформенные адаптации указаны (если несколько платформ)
- [ ] Нет противоречий с `$brand-guidelines`
- [ ] Согласован с пользователем

## Handoff

| Потребитель | Как использует |
|-------------|---------------|
| `$visual-brief` | **Основной потребитель** — мудборд определяет стиль, visual brief конкретизирует задачу |
| `$image-prompt-engineering` | Стиль, палитра, настроение → ключевые слова для промпта |
| Visual Concept (агент) | Общее визуальное направление для создания креативов |
| `$brand-guidelines` | Если мудборд удачный — элементы можно встроить в гайдлайны бренда |

При передаче — использовать `$handoff` с приложением полного мудборда.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$brand-guidelines` | **Источник + ограничение:** палитра, стиль, типографика — мудборд расширяет, не противоречит |
| `$audience-analysis` | **Источник:** персона → визуальные ожидания ЦА |
| `$competitor-content-analysis` | **Источник:** визуальный стиль конкурентов → дифференциация |
| `$platform-strategy` | **Источник:** платформа → визуальные ожидания канала |
| `$visual-brief` | **Потребитель:** мудборд → visual brief для конкретной задачи |
| `$image-prompt-engineering` | **Потребитель:** стиль и палитра → промпт для AI-генерации |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Мудборд противоречит бренду | Визуал не узнаваем как бренд | Всегда проверять на совместимость с `$brand-guidelines` |
| Референсы без категоризации | «Мне нравится» — но что конкретно? | Для каждого: что берём (палитру? стиль? композицию?) |
| Одна эмоция на всё | «Красиво» — не настроение | Основная + вторичная эмоция, описание атмосферы |
| Палитра из 10 цветов | Хаос, нет узнаваемости | 3-5 цветов с чёткими ролями (60/30/10) |
| Нет «запрещённых» стилей | Дизайнер может пойти не туда | Do / Don't обязательны |
| Мудборд без персоны | Визуал для себя, не для ЦА | Визуальные ожидания персоны из `$audience-analysis` |
| Одинаковый мудборд для всех платформ | Instagram ≠ LinkedIn визуально | Платформенные адаптации |
| Копирование конкурента | Нет дифференциации | Референсы конкурентов — для понимания, не для копирования |
