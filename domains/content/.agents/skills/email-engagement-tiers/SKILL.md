# Email Engagement Tiers — Сегментация по активности

## Когда использовать
- При **планировании email-стратегии** — определить тиры до запуска рассылок.
- При **создании email-контента** — адаптировать текст, тон, CTA, urgency под тир.
- При **падении метрик** — open rate / CTR снижаются → пересмотреть сегментацию.
- При **чистке базы** — определить кого реактивировать, а кого удалить.
- При **планировании email-последовательностей** — разные серии для разных тиров.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Данные email-платформы | ✅ | Open rate, CTR, дата последнего открытия/клика — по подписчикам или сегментам |
| Размер базы | ✅ | Общее количество подписчиков |
| Частота рассылок | ✅ | Сколько писем/нед отправляется сейчас |
| Бизнес-модель | ✅ | B2B / B2C / SaaS / E-commerce — влияет на пороги тиров |
| Текущая сегментация | ⬚ | Если уже есть — что используется |
| Цели email-канала | ⬚ | Retention / конверсия / nurturing / awareness |
| ESP (платформа рассылок) | ⬚ | Mailchimp / Brevo / ConvertKit / HubSpot / кастом — влияет на возможности сегментации |

> Если данных по активности нет (нет аналитики) — скил **не применим**. Сначала собрать данные минимум за 60 дней.

## Определение тиров

### Стандартная модель (5 тиров)

| Тир | Название | Определение | Доля базы (типичная) |
|-----|----------|-------------|:--------------------:|
| **🟢 A — VIP / Champions** | Самые активные | Открывали **и кликали** в последние **30 дней**. Регулярно взаимодействуют | 10-20% |
| **🔵 B — Engaged** | Активные | Открывали **хотя бы 1 раз** в последние **60 дней** | 20-30% |
| **🟡 C — Passive** | Пассивные | Открывали **хотя бы 1 раз** в последние **61-120 дней** | 15-25% |
| **🟠 D — At Risk** | Под угрозой | **Не открывали** **120-180 дней** | 10-20% |
| **🔴 E — Dormant / Dead** | Спящие | **Не открывали** **> 180 дней** | 10-30% |

### Адаптация порогов по бизнес-модели

| Бизнес-модель | Тир A | Тир B | Тир C | Тир D | Тир E |
|---------------|-------|-------|-------|-------|-------|
| **SaaS (высокая частота)** | 14 дней, open + click | 30 дней, open | 31-90 дней | 91-150 дней | > 150 дней |
| **E-commerce** | 30 дней, open + click + purchase | 60 дней, open + click | 61-120 дней, open | 121-180 дней | > 180 дней |
| **B2B (низкая частота)** | 30 дней, open + click | 90 дней, open | 91-180 дней | 181-270 дней | > 270 дней |
| **Newsletter / Media** | 14 дней, open | 45 дней, open | 46-90 дней | 91-150 дней | > 150 дней |

> Пороги адаптируются под бизнес. Если рассылка 1/мес — окна шире. Если 3/нед — окна уже.

### Дополнительные сигналы (уточняющие)

| Сигнал | Повышает тир | Понижает тир |
|--------|:------------:|:------------:|
| Клик по CTA | ✅ +1 тир | — |
| Покупка через email | ✅ +1 тир (до A) | — |
| Ответ на письмо (reply) | ✅ → сразу A | — |
| Forward / share | ✅ → сразу A | — |
| Unsubscribe-попытка (но остался) | — | ⚠️ -1 тир |
| Spam complaint | — | 🔴 → сразу E + удалить |
| Bounce (soft) | — | ⚠️ после 3 soft bounces → E |
| Bounce (hard) | — | 🔴 → удалить из базы |

## Стратегия по тирам

### 🟢 Тир A — VIP / Champions

| Аспект | Стратегия |
|--------|----------|
| **Цель** | Удержать, увеличить LTV, превратить в адвокатов бренда |
| **Частота** | Максимальная из допустимой (3-5/нед для SaaS, 2-3/нед для e-com) |
| **Контент** | Эксклюзив: ранний доступ, VIP-офферы, behind-the-scenes, закрытый контент |
| **Тон** | Персональный, тёплый, «мы ценим тебя» |
| **CTA** | Прямой, конверсионный: купи, обнови, порекомендуй |
| **Urgency** | Низкий — они и так лояльны. Urgency = обесценивание |
| **Personalization** | Максимальная: имя, история покупок, рекомендации |
| **Subject line** | Персональный, без clickbait: «[Имя], ваш эксклюзив», «Для вас — раньше всех» |

**Ключевые метрики тира A:**
- Open Rate: > 50%
- CTR: > 10%
- Unsubscribe: < 0.05%

### 🔵 Тир B — Engaged

| Аспект | Стратегия |
|--------|----------|
| **Цель** | Перевести в тир A (увеличить частоту взаимодействия и глубину) |
| **Частота** | Средняя (2-3/нед для SaaS, 1-2/нед для e-com) |
| **Контент** | Ценный: how-to, кейсы, полезные инструменты, дайджесты |
| **Тон** | Дружеский-экспертный |
| **CTA** | Вовлекающий: узнай больше, попробуй, скачай |
| **Urgency** | Умеренный — «только на этой неделе» допустим |
| **Personalization** | Средняя: имя, тематические предпочтения |
| **Subject line** | Value-driven: «5 способов...», «Как мы увеличили...», число + результат |

**Ключевые метрики:**
- Open Rate: 30-50%
- CTR: 3-10%
- Цель: конвертировать в A через клик + дополнительное действие

### 🟡 Тир C — Passive

| Аспект | Стратегия |
|--------|----------|
| **Цель** | Реактивировать: вернуть интерес, напомнить о ценности |
| **Частота** | Сниженная (1/нед для SaaS, 1/2 нед для e-com). **Не спамить** — это ускорит уход |
| **Контент** | Лучшее из лучшего: топ-контент, который работал для тира A/B |
| **Тон** | «Мы тут, с ценностью» — не навязчивый |
| **CTA** | Мягкий: «Посмотри что нового», «Мы подготовили для тебя» |
| **Urgency** | Минимальный — давление отпугивает пассивных |
| **Personalization** | Высокая — показать что помним: «С момента вашей последней покупки мы добавили...» |
| **Subject line** | Re-engagement: «Мы скучаем», «Что нового у нас за [период]», «Специально для вас» |

**Ключевые метрики:**
- Open Rate: 15-30%
- Цель: вернуть в B в течение 30 дней
- Если не реагирует 30 дней → переход в D

### 🟠 Тир D — At Risk

| Аспект | Стратегия |
|--------|----------|
| **Цель** | Последняя попытка реактивации перед переводом в E |
| **Частота** | Минимальная (1-2 письма в мес). Win-back серия |
| **Контент** | Win-back серия из 3-4 писем (см. протокол ниже) |
| **Тон** | Прямой, честный: «Мы заметили, что вы не читаете. Вот что изменилось» |
| **CTA** | Чёткий выбор: «Хочу остаться» / «Отписаться» |
| **Urgency** | Допустим: «Если не ответите — мы перестанем писать через [N] дней» |
| **Personalization** | Максимальная — последний шанс показать ценность |
| **Subject line** | Direct: «Нам продолжать?», «Последний шанс: ваш эксклюзив», «Мы скоро перестанем писать» |

**Win-back серия (Тир D):**

| # | День | Тема | Подход |
|---|------|------|--------|
| 1 | 0 | «Мы скучаем» | Напомнить о ценности: что классного пропустил |
| 2 | 5 | «Что нового» | Показать 3-5 лучших обновлений/постов с момента ухода |
| 3 | 12 | «Эксклюзив для вас» | Стимул: скидка, бонус, бесплатный контент — только для returning |
| 4 | 20 | «Последнее письмо» | Прямой вопрос: «Хотите остаться?» + ссылка «Да, остаюсь» / «Нет, отпишите» |

**Ключевые метрики:**
- Win-back open rate: 5-15%
- Win-back conversion (D→B): 3-8%
- Если после серии не открыл → тир E

### 🔴 Тир E — Dormant / Dead

| Аспект | Стратегия |
|--------|----------|
| **Цель** | Гигиена базы: sunset → удаление |
| **Частота** | **0 писем** (прекратить рассылку) |
| **Контент** | Одно финальное «sunset email», затем тишина |
| **Действие** | Sunset policy → удалить из активной базы |

**Sunset Email (одно, последнее):**
> Тема: «Мы прощаемся (если вы не против)»
> Тело: «Мы заметили, что вы давно не читаете наши письма. Мы уважаем ваше время и перестанем писать. Если хотите остаться — нажмите [кнопку]. Если нет — ничего делать не нужно.»
> CTA: «Остаюсь!» (re-confirm opt-in)

**После sunset:**
- Нажал «Остаюсь» → тир C (пассивный) → re-engagement
- Не нажал за 14 дней → **удалить из базы**

### Почему удалять тир E

| Причина | Влияние |
|---------|---------|
| **Deliverability** | Мёртвые адреса → ISP считает вас спамером → inbox placement падает для ВСЕХ |
| **Open Rate** | Тир E тянет средний OR вниз → алгоритмы ESP пессимизируют |
| **Стоимость** | ESP берёт плату за размер базы. Мёртвые адреса = деньги в никуда |
| **Spam traps** | Старые заброшенные адреса могут стать spam traps ISP |

## Протокол

### Шаг 1 — Аудит текущей базы
1. Получить данные из ESP: open rate, last open date, last click date — по каждому подписчику или сегменту.
2. Распределить подписчиков по 5 тирам (стандартная модель или адаптированная).
3. Зафиксировать **размер каждого тира** (абсолютные числа и %).
4. Если тир E > 30% базы — это 🔴 проблема с гигиеной.

### Шаг 2 — Адаптация порогов
1. Определить бизнес-модель (SaaS / e-com / B2B / media).
2. Адаптировать пороги по таблице.
3. Если есть дополнительные сигналы (покупки, replies) — учесть.
4. Согласовать пороги с пользователем.

### Шаг 3 — Стратегия по тирам
Для каждого тира определить:
1. **Цель** — чего добиваемся.
2. **Частота** — сколько писем.
3. **Контент** — что отправляем (тип, глубина, эксклюзив).
4. **Тон и CTA** — как говорим и к чему призываем.
5. **Subject line стратегия** — какие формулы работают.
6. **Персонализация** — уровень и данные.

### Шаг 4 — Win-back серия (Тир D)
1. Спроектировать серию из 3-4 писем.
2. Определить интервалы.
3. Определить стимул (скидка / бонус / контент).
4. Определить exit condition: открыл → тир B/C, не открыл → тир E.

### Шаг 5 — Sunset Policy (Тир E)
1. Определить sunset email.
2. Определить срок ожидания (14 дней).
3. Определить действие: re-confirm → C, молчание → удалить.

### Шаг 6 — Регулярный пересмотр
1. **Еженедельно:** мониторинг метрик по тирам (автоматизировать в ESP).
2. **Ежемесячно:** перераспределение подписчиков по тирам (автоматическое или ручное).
3. **Ежеквартально:** пересмотр порогов, чистка тира E, анализ win-back конверсии.

## Адаптация контента по тирам (для Copywriter)

### Как меняется текст

| Элемент | 🟢 Тир A | 🔵 Тир B | 🟡 Тир C | 🟠 Тир D |
|---------|----------|----------|----------|----------|
| **Subject line** | Персональный, эксклюзив | Value-driven, числа | Re-engagement, «мы тут» | Direct, urgency |
| **Hook** | «Для вас первым» | «Вот что работает» | «Давно не виделись» | «Мы скоро перестанем» |
| **Тон** | Тёплый, инсайдерский | Экспертный, дружеский | Мягкий, не навязчивый | Прямой, честный |
| **CTA** | Купи, обнови, порекомендуй | Узнай, попробуй, скачай | Посмотри что нового | Хочу остаться / Отпишите |
| **Urgency** | Низкий (не нужен) | Умеренный | Минимальный | Допустим |
| **Длина** | Любая (читают) | Средняя | Короткая (не перегружать) | Минимальная (1 CTA) |
| **Персонализация** | Полная: имя + история + рекомендации | Средняя: имя + предпочтения | Высокая: «с момента вашего...» | Максимальная: последний шанс |
| **Контент** | Эксклюзив, VIP, ранний доступ | Полезный: how-to, кейсы | Лучшее из лучшего (proven) | Win-back серия |

### Пример: один оффер → 4 версии

**Оффер:** Новый гайд «Email-маркетинг 2026»

**🟢 Тир A:**
> Subject: «[Имя], ваш эксклюзивный доступ к гайду»
> Body: «Как наш VIP-подписчик, вы получаете гайд на 48 часов раньше всех. Внутри: 15 стратегий...»
> CTA: «Скачать раньше всех»

**🔵 Тир B:**
> Subject: «15 email-стратегий, которые работают в 2026»
> Body: «Мы собрали результаты 500+ кампаний в один гайд. Вот 3 инсайта, которые удивили нас...»
> CTA: «Скачать бесплатно»

**🟡 Тир C:**
> Subject: «Мы подготовили кое-что для вас»
> Body: «Привет, [Имя]! С момента вашего последнего визита мы опубликовали 12 статей. Но самое ценное — в этом гайде...»
> CTA: «Посмотреть что внутри»

**🟠 Тир D:**
> Subject: «[Имя], мы скоро перестанем писать. Но сначала — подарок»
> Body: «Мы заметили, что давно не виделись. Перед прощанием хотим оставить вам наш лучший гайд...»
> CTA: «Скачать и остаться» / «Отписаться»

## Валидация (Quality Gate)

Сегментация считается готовой если:

- [ ] Тиры определены (5 уровней с конкретными порогами)
- [ ] Пороги адаптированы под бизнес-модель
- [ ] Размер каждого тира зафиксирован (абсолютные числа и %)
- [ ] Стратегия для каждого тира: цель, частота, контент, тон, CTA
- [ ] Win-back серия спроектирована для тира D (3-4 письма)
- [ ] Sunset policy определена для тира E
- [ ] Метрики по тирам определены с целевыми значениями
- [ ] Регулярный пересмотр запланирован (еженедельно / ежемесячно / ежеквартально)
- [ ] Согласовано с пользователем

## Handoff

| Потребитель | Как использует |
|-------------|---------------|
| `$email-copywriting` | **Основной:** адаптация текста, тона, CTA по тирам |
| `$content-calendar` | Email-рассылки в календаре с пометкой тира |
| `$content-brief` | Тир как обязательное поле в брифе email-контента |
| `$platform-strategy` | Email-сегментация как часть канальной стратегии |
| `$cta-optimization` | CTA адаптируется по тирам (разная агрессивность) |
| `$audience-analysis` | Тиры как дополнительный слой сегментации (поведенческий) |

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$email-copywriting` | **Потребитель:** тиры определяют тип письма, тон, CTA, frequency |
| `$audience-analysis` | **Комплемент:** персоны = кто (демография + JTBD), тиры = как себя ведёт (активность) |
| `$platform-strategy` | **Источник:** email как канал → тиры как стратегия внутри канала |
| `$content-calendar` | **Потребитель:** разные рассылки для разных тиров в одном календаре |
| `$content-brief` | **Потребитель:** тир указывается в брифе → Copywriter адаптирует |
| `$cta-optimization` | **Потребитель:** агрессивность CTA зависит от тира |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Одно письмо для всех | Тир A получает то же, что тир D → неэффективно | Адаптация контента по тирам |
| Спамить тир C/D | Ускоряет отписку и спам-жалобы | Снизить частоту, повысить ценность |
| Не удалять тир E | Тянет deliverability вниз для всей базы | Sunset → удалить через 14 дней |
| Urgency для тира A | VIP не нуждаются в давлении, это обесценивает | Эксклюзив > urgency для лояльных |
| Одинаковый тон для всех | «Мы скучаем» для тира A = странно | Тон адаптируется: тёплый (A), мягкий (C), прямой (D) |
| Пороги без адаптации | SaaS и B2B = разная активность | Адаптировать пороги под бизнес-модель |
| Win-back без стимула | «Вернись» без причины — не работает | Стимул: скидка, бонус, эксклюзивный контент |
| Нет регулярного пересмотра | Подписчики мигрируют между тирами | Ежемесячное перераспределение |
| Hard bounce = продолжать слать | Портит sender reputation | Hard bounce = немедленное удаление |
