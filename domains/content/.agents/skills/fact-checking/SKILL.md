# Fact Checking — Факт-чекинг

## Когда использовать
- На этапе **Reviewer** — перед финальным одобрением контента с данными и утверждениями.
- При **создании data-driven контента** — параллельно с `$data-storytelling`.
- При **аудите существующего контента** — проверка актуальности старых публикаций.
- При **любом сомнении** в достоверности факта — лучше проверить один раз.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Текст для проверки | ✅ | Контент с фактическими утверждениями |
| Тип контента | ✅ | Определяет уровень строгости (см. ниже) |
| Источники автора | ⬚ | Если автор указал свои источники — проверить их |
| Результат `$source-verification` | ⬚ | Если уже пройден — использовать, не дублировать |

> Если текст не содержит фактических утверждений (чисто мнение, юмор) — скил **не применяется**. Отметить N/A.

## Уровни строгости

Строгость проверки зависит от типа контента и рисков:

| Уровень | Типы контента | Требования | Пример |
|---------|--------------|------------|--------|
| 🔴 **Максимальный** | Медицина, финансы, юридика, научные утверждения | Только peer-reviewed / официальные источники. 0 неверифицированных. | Статья о лекарствах, инвестиционный обзор |
| 🟡 **Стандартный** | Статьи, кейсы, аналитика, email, лонгриды | Надёжные источники (Tier 1-2). ≤ 1 неверифицированное (Minor). | Блог-пост с данными, карусель со статистикой |
| 🔵 **Базовый** | Соцсети, короткие посты, развлекательный контент | Здравый смысл + проверка ключевых цифр. Допустимы общеизвестные факты без ссылки. | Мем с цифрой, пост с «по данным исследований» |

> Уровень определяет, сколько проверок нужно и какие источники допустимы.

## Типология фактов

| Тип факта | Что проверять | Метод проверки | Пример |
|-----------|--------------|----------------|--------|
| **Статистика / цифры** | Число, выборка, год, методология | Найти оригинальное исследование | «87% маркетологов используют AI» |
| **Цитата** | Точность, контекст, авторство | Найти первоисточник цитаты | «Стив Джобс сказал: ...» |
| **Дата / хронология** | Корректность даты, порядок событий | Несколько независимых источников | «Компания основана в 2015 году» |
| **Имя / должность** | Правильность написания, актуальность должности | Официальный сайт, LinkedIn | «CEO Google Сундар Пичай» |
| **Научное утверждение** | Научный консенсус, не опровергнуто ли | Peer-reviewed journals, мета-анализы | «Кофе снижает риск диабета» |
| **Юридическое утверждение** | Действующее законодательство, юрисдикция | Официальные правовые базы | «По закону РФ реклама должна быть маркирована» |
| **Причинно-следственная связь** | Корреляция ≠ причинность | Проверить методологию исследования | «Контент-маркетинг увеличивает продажи на 30%» |
| **Общеизвестный факт** | Проверить, действительно ли общеизвестен | Быстрая проверка, если есть сомнения | «Земля вращается вокруг Солнца» |

## Иерархия источников

| Tier | Тип источника | Надёжность | Примеры |
|------|--------------|------------|---------|
| **Tier 1** | Первичные / академические | Максимальная | Peer-reviewed journals, государственная статистика (Росстат, BLS), SEC filings, патенты |
| **Tier 2** | Авторитетные вторичные | Высокая | Отраслевые отчёты (McKinsey, Gartner, Statista), крупные СМИ (Reuters, Bloomberg), официальные блоги компаний |
| **Tier 3** | Надёжные вторичные | Средняя | Отраслевые СМИ, книги экспертов, TED-выступления, крупные блоги с редакцией |
| **Tier 4** | Пользовательский контент | Низкая | Личные блоги, посты в соцсетях, Reddit, Quora, Wikipedia (без проверки первоисточника) |
| **Tier 5** | Ненадёжные | Минимальная | Анонимные источники, контент-фермы, жёлтые СМИ, affiliate-сайты |

### Правила по уровням строгости
- 🔴 **Максимальный:** только Tier 1-2. Tier 3 допустим для контекста, но не для ключевых утверждений.
- 🟡 **Стандартный:** Tier 1-3. Tier 4 допустим для иллюстраций, но не для цифр.
- 🔵 **Базовый:** Tier 1-4. Tier 5 не допустим ни при каком уровне.

## Severity — уровни ошибок

| Severity | Определение | Примеры | Влияние |
|----------|-------------|---------|---------|
| 🔴 **Critical** | Фактическая ошибка, которая вводит в заблуждение, юридический/медицинский риск | Неверная дозировка лекарства, ложная статистика, несуществующий закон | FAIL |
| 🟡 **Major** | Заметная неточность, которая подрывает доверие | Устаревшая цифра (2019 вместо 2025), неверная должность, искажённая цитата | FAIL при ≥ 2 |
| 🔵 **Minor** | Мелкая неточность, не влияющая на суть | Округление (87% вместо 86.7%), неточная дата (март вместо апреля) | Не блокирует |

### Порог PASS / FAIL

| Результат | Условие |
|-----------|---------|
| **VERIFIED** | 0 Critical + 0 Major + 0 неверифицированных (🔴 уровень строгости) |
| **PASS** | 0 Critical + ≤ 1 Major + ≤ 1 неверифицированных Minor |
| **PASS WITH NOTES** | 0 Critical + ≤ 1 Major (с исправлением) + ≤ 2 неверифицированных Minor |
| **FAIL** | ≥ 1 Critical ИЛИ ≥ 2 Major ИЛИ ≥ 3 неверифицированных |

## Методы верификации

| Метод | Когда использовать | Как |
|-------|-------------------|-----|
| **Поиск первоисточника** | Статистика, научные утверждения | Найти оригинальное исследование/отчёт, не пересказ |
| **Перекрёстная проверка** | Даты, имена, события | ≥ 2 независимых источника совпадают |
| **Обратный поиск цитаты** | Цитаты, приписанные известным людям | Веб-поиск точной цитаты + имя → проверить атрибуцию |
| **Wayback Machine** | Данные с удалённых/изменённых страниц | web.archive.org → снимок на дату |
| **Проверка методологии** | Статистика с необычными выводами | Кто проводил? Какая выборка? Кто спонсировал? |
| **Проверка актуальности** | Любые данные | Дата публикации источника ≤ 2 лет (для быстрых рынков ≤ 1 год) |
| **Проверка контекста** | Цитаты, вырванные из контекста | Прочитать оригинал шире — 2-3 абзаца вокруг цитаты |
| **Калькулятор / здравый смысл** | Числа, которые «не бьются» | Пересчитать: проценты, соотношения, порядок величин |

## Протокол

### Шаг 0 — Подготовка
1. Получить текст и определить тип контента.
2. Определить уровень строгости (🔴 / 🟡 / 🔵).
3. Загрузить результат `$source-verification`, если есть.

### Шаг 1 — Извлечение фактов
1. Прочитать текст целиком.
2. Выделить **все** фактические утверждения — каждое в отдельную строку.
3. Классифицировать каждое по типологии (статистика / цитата / дата / имя / научное / юридическое / причинность / общеизвестное).
4. Определить **критичность** каждого факта для сути контента (ключевой / поддерживающий / иллюстративный).

### Шаг 2 — Верификация
Для каждого утверждения:
1. Выбрать метод верификации по типу факта.
2. Найти источник (соблюдая иерархию по уровню строгости).
3. Проверить точность: число, дата, имя, контекст.
4. Проверить актуальность: данные не устарели?
5. Присвоить статус: ✅ Верно / ❌ Неверно / ⚠️ Не верифицировано.

### Шаг 3 — Оценка ошибок
Для каждого ❌ и ⚠️:
1. Присвоить severity (Critical / Major / Minor).
2. Для ❌ — указать правильную версию с источником.
3. Для ⚠️ — рекомендация: найти источник / удалить / заменить на верифицированный факт.

### Шаг 4 — Вердикт
1. Подсчитать по severity.
2. Применить порог PASS / FAIL.
3. Сформировать список обязательных правок и рекомендуемых удалений.

## Валидация (Quality Gate)

Факт-чекинг считается завершённым если:

- [ ] Все фактические утверждения извлечены из текста
- [ ] Каждое утверждение классифицировано по типу
- [ ] Каждое утверждение проверено (✅ / ❌ / ⚠️ — нет пустых)
- [ ] Для каждого ❌ указаны: severity, правильная версия, источник
- [ ] Для каждого ⚠️ указана рекомендация (найти / удалить / заменить)
- [ ] Источники соответствуют уровню строгости
- [ ] Вердикт соответствует порогу (не субъективный)
- [ ] Список обязательных правок сформирован

## Handoff

| Результат | Маршрут |
|-----------|---------|
| **VERIFIED / PASS** | → `$content-review-checklist` (пункт #18 — ссылка на этот результат) → `$content-release-gate` |
| **PASS WITH NOTES** | → Copywriter (исправить Major + заменить/удалить ⚠️) → повторный факт-чекинг |
| **FAIL** | → Copywriter / Researcher (исправить Critical/Major) → обязательный повторный факт-чекинг |

При передаче — использовать `$handoff` с приложением полного отчёта.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$source-verification` | **Комплемент:** верификация источников → используется как вход для факт-чекинга |
| `$data-storytelling` | **Источник:** данные из data story проверяются факт-чекингом |

| `$content-review-checklist` | **Потребитель:** пункт #18 ссылается на результат факт-чекинга |
| `$content-release-gate` | **Потребитель:** VERIFIED/PASS обязателен для Release Gate |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Проверять только «подозрительные» факты | Очевидные факты тоже бывают неверными | Проверять все фактические утверждения |
| Wikipedia как единственный источник | Wikipedia — Tier 4, могут быть ошибки | Wikipedia → найти первоисточник из сносок |
| «По данным исследований» без ссылки | Непроверяемое утверждение | Конкретный источник: кто, когда, где, выборка |
| Старые данные без проверки | «В 2020 году рынок составлял...» → а в 2026? | Проверять актуальность, искать свежие данные |
| Корреляция = причинность | «Компании с блогом растут быстрее» ≠ блог → рост | Явно указывать: корреляция, не доказанная причинность |
| Проверка по пересказу вместо оригинала | Пересказ мог исказить данные | Всегда искать первоисточник |
| Один источник = верифицировано | Источник может ошибаться | Для ключевых фактов — ≥ 2 независимых источника |
| Факт верный, но устаревший | Мир изменился с момента публикации | Актуальность: ≤ 2 лет, для быстрых рынков ≤ 1 год |
