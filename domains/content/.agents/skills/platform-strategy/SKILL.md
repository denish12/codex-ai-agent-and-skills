# Platform Strategy — Стратегия по платформам

## Когда использовать
- При **старте контент-стратегии** — после `$audience-analysis`, до `$content-calendar`.
- При **масштабировании** — добавление новых платформ к существующим.
- При **пересмотре стратегии** — метрики не достигаются, нужна перебалансировка.
- При **ограниченных ресурсах** — выбор: где фокусироваться, а от чего отказаться.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Персоны ЦА | ✅ | Из `$audience-analysis` — кто, где проводит время, платформенные предпочтения |
| Бизнес-цели | ✅ | Awareness / лиды / продажи / комьюнити — что приоритет |
| Бюджет | ✅ | $0 (только organic) / конкретная сумма (paid + organic). Влияет на стратегию |
| Ресурсы на контент | ✅ | Сколько единиц контента в неделю реально производить (1 человек vs команда) |
| Текущие каналы | ⬚ | Где уже есть присутствие, аудитория, метрики |
| Результат `$competitor-content-analysis` | ⬚ | Где конкуренты, где gaps |
| Гео | ⬚ | Целевые рынки (влияет на выбор: WeChat vs Instagram, VK vs Telegram) |

> Если бюджет и ресурсы не указаны — **запросить**. Стратегия «5 платформ × 5 постов/нед» без ресурсов = нереалистичная.

## Платформы и их характеристики

| Платформа | Сильная сторона | Воронка | Аудитория (общая) | Органический потенциал (2025-2026) |
|-----------|----------------|---------|-------------------|-----------------------------------|
| **Instagram** | Визуальный контент, brand awareness, комьюнити | TOFU / MOFU | 18-44, B2C, lifestyle | Средний (Reels > Feed) |
| **TikTok** | Вирусность, охват, молодая аудитория | TOFU | 16-34, B2C | Высокий |
| **LinkedIn** | B2B, экспертность, нетворкинг | MOFU / BOFU | 25-55, B2B, профессионалы | Высокий (текст > видео) |
| **Telegram** | Лояльная аудитория, глубина контента | MOFU / BOFU | 18-45, tech-savvy, СНГ | Высокий |
| **YouTube** | Длинный контент, SEO, evergreen | TOFU / MOFU | 18-54, все | Средний (требует продакшна) |
| **Twitter/X** | Новости, мнения, real-time | TOFU | 20-45, tech, медиа | Низкий (алгоритм непредсказуем) |
| **Email** | Конверсия, retention, персонализация | MOFU / BOFU | Подписчики (любой возраст) | Высокий (своя база) |

## Протокол

### Шаг 1 — Аудитория → Платформы
1. Для каждой персоны из `$audience-analysis` — определить где она проводит время (платформенные предпочтения).
2. Сопоставить с таблицей платформ.
3. Выписать **все потенциальные платформы** (длинный список).

### Шаг 2 — Фильтрация по целям и воронке
1. Для каждой бизнес-цели — определить этап воронки:
   - Awareness → TOFU платформы (Instagram, TikTok, YouTube, Blog).
   - Лиды → MOFU/BOFU (LinkedIn, Telegram, Email).
   - Продажи → BOFU (Email, Telegram, ретаргетинг).
   - Комьюнити → MOFU (Telegram, Discord, группы).
2. Отфильтровать платформы, не соответствующие целям.

### Шаг 3 — Оценка ресурсов
1. Определить **контент-мощность**: сколько единиц контента реально производить в неделю.
2. Оценить требования каждой платформы:

| Платформа | Минимальная частота | Усилие на единицу | Необходимый контент/нед |
|-----------|--------------------|--------------------|------------------------|
| Instagram | 3-5/нед | Среднее (текст + визуал) | 3-5 |
| TikTok | 3-7/нед | Среднее (видео) | 3-7 |
| LinkedIn | 2-3/нед | Среднее (текст) | 2-3 |
| Telegram | 3-5/нед | Низкое (текст) | 3-5 |
| YouTube | 1-2/нед | Высокое (видео-продакшн) | 1-2 |
| Email | 1-2/нед | Среднее | 1-2 |
| Blog | 1-2/мес | Высокое (лонгрид + SEO) | 0.25-0.5 |

3. **Правило:** Суммарный контент/нед по всем платформам ≤ контент-мощности. Если не хватает — сокращать платформы.

### Шаг 4 — Приоритизация
Ранжировать платформы по 4 критериям (1-5 баллов):

| Критерий | Описание |
|----------|----------|
| **Аудитория** | Наша ЦА там присутствует? (из `$audience-analysis`) |
| **Цели** | Платформа решает нашу бизнес-задачу? |
| **Конкуренция** | Есть gap у конкурентов? (из `$competitor-content-analysis`) |
| **Ресурсоёмкость** | Мы можем производить контент с нужной частотой? |

Результат:
- **Приоритет 1 (Core):** 1-2 платформы — основной фокус, максимум ресурсов.
- **Приоритет 2 (Growth):** 1-2 платформы — развитие, умеренные ресурсы.
- **Приоритет 3 (Experimental):** 0-1 платформа — тест, минимум ресурсов.
- **Отложенные:** остальные — не сейчас, вернуться при росте ресурсов.

### Шаг 5 — Стратегия по каждой платформе
Для каждой выбранной платформы:
1. **Роль в воронке** — TOFU / MOFU / BOFU.
2. **Форматы** — какие типы контента (reels, посты, карусели, видео, статьи).
3. **Частота** — публикаций/неделю.
4. **Лучшее время** — на основе данных платформы или `$audience-analysis`.
5. **Тон** — адаптация ToV под платформу (LinkedIn формальнее Instagram).
6. **Paid / Organic** — стратегия продвижения.

### Шаг 6 — Paid vs Organic
Для каждой приоритетной платформы:

| Тип | Когда | Бюджет | Цель |
|-----|-------|--------|------|
| **Organic** | Всегда — базовый контент | $0 | Awareness, engagement, комьюнити |
| **Paid boost** | Лучшие органические посты | 10-20% бюджета | Масштабирование того, что уже работает |
| **Paid ads** | Конкретные кампании | 80-90% бюджета | Лиды, конверсии, ретаргетинг |

> Если бюджет = $0: только organic. Фокус на 1-2 платформах с максимальным органическим потенциалом.

### Шаг 7 — Кросс-постинг
1. Определить **контент-ядро**: основная платформа, где создаётся оригинал.
2. Определить **адаптации**: что и как менять для других платформ.

| Оригинал → Адаптация | Что меняется |
|-----------------------|-------------|
| Instagram Reel → TikTok | Удалить водяной знак, адаптировать caption |
| Instagram Carousel → LinkedIn | Сделать текстовый пост + PDF carousel |
| Blog → Telegram | Выжимка ключевых тезисов |
| YouTube → Instagram Reels | Нарезать лучшие моменты (≤ 60 сек) |
| Email → Telegram | Адаптировать формат, убрать HTML |

3. **Правило:** Кросс-постинг ≠ копипаст. Каждая платформа требует адаптации.

### Шаг 8 — Roadmap роста
Фазированный план по кварталам:

| Фаза | Период | Платформы | Фокус |
|------|--------|-----------|-------|
| **Launch** | Q1 | 1-2 Core платформы | Набрать первую аудиторию, найти работающие форматы |
| **Grow** | Q2 | Core + 1 Growth | Масштабировать то, что работает. Тестировать новую платформу |
| **Scale** | Q3-Q4 | Core + Growth + Experimental | Оптимизировать, добавить paid, автоматизировать кросс-постинг |

> Roadmap адаптируется по результатам метрик каждого квартала.

## Метрики и бенчмарки

| Платформа | Ключевая метрика | Бенчмарк (средний) | Хороший | Что измеряет |
|-----------|-----------------|--------------------|---------|--------------| 
| **Instagram** | ER (Engagement Rate) | 1-3% | > 3% | Вовлечённость |
| **Instagram** | Reach Rate | 10-20% подписчиков | > 30% | Охват |
| **TikTok** | Views / Followers | 10-30% | > 50% | Виральность |
| **LinkedIn** | Impressions / Followers | 5-15% | > 20% | Охват |
| **Telegram** | Views / Subscribers | 30-50% | > 60% | Readthrough |
| **YouTube** | CTR thumbnail | 2-5% | > 5% | Кликабельность |
| **YouTube** | Avg view duration | 40-50% | > 50% | Удержание |
| **Email** | Open Rate | 20-25% | > 30% | Доставляемость + subject |
| **Email** | CTR | 2-3% | > 5% | Конверсия |
| **Blog** | Organic traffic / мес | Зависит от ниши | Рост 10-20% MoM | SEO-рост |

## Валидация (Quality Gate)

Стратегия считается готовой если:

- [ ] Персоны сопоставлены с платформами (где ЦА проводит время)
- [ ] Платформы приоритизированы (Core / Growth / Experimental / Отложенные)
- [ ] Суммарная частота ≤ контент-мощности (реалистичность)
- [ ] Для каждой платформы: роль в воронке, форматы, частота, время
- [ ] Paid/Organic стратегия определена (с учётом бюджета)
- [ ] Кросс-постинг спланирован (что адаптировать, не копипастить)
- [ ] Метрики и целевые значения зафиксированы
- [ ] Roadmap роста определён (фазы, квартальный план)
- [ ] Нет платформ без обоснования (каждая — зачем она тут)
- [ ] Согласовано с пользователем

## Handoff

Результат этого скила является входом для:

| Потребитель | Как использует |
|-------------|---------------|
| `$content-calendar` | **Основной потребитель:** платформы, частота, форматы → план публикаций |
| `$content-brief` | Платформа + формат для конкретной единицы контента |
| `$audience-analysis` | Обратная связь: если ЦА нет на платформе — пересмотреть |
| `$platform-compliance` | Требования платформы → compliance-проверка |
| `$platform-visual-specs` | Визуальные спеки для каждой платформы |
| `$tone-of-voice` | Адаптация тона под платформу |
| `$cta-optimization` | Формат CTA зависит от платформы |

При передаче — использовать `$handoff` с приложением стратегии и приоритизации.

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$audience-analysis` | **Источник:** персоны → где они есть → выбор платформ |
| `$competitor-content-analysis` | **Источник:** где конкуренты, где gaps |
| `$content-calendar` | **Потребитель:** платформы + частота → календарь |
| `$platform-compliance` | **Потребитель:** правила платформы |
| `$platform-visual-specs` | **Потребитель:** визуальные спеки |
| `$brand-guidelines` | **Источник:** ToV → адаптация тона по платформам |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| 5+ платформ без ресурсов | Размазанное присутствие, 0 результата | Лучше 2 платформы хорошо, чем 5 плохо |
| Все платформы = TOFU | Нет конверсии, только охват | Минимум 1 платформа MOFU/BOFU (email, Telegram) |
| Кросс-постинг = копипаст | Водяные знаки, неадаптированный формат, penalty | Каждая платформа = адаптация |
| Нет бюджетного контекста | Стратегия с paid для бюджета $0 | Явно указать paid/organic для каждой платформы |
| Частота без ресурсов | «7 постов/нед» → выгорание через 2 недели | Частота ≤ реальная контент-мощность |
| Выбор платформы «потому что модно» | ЦА нет на TikTok, но все говорят постить | Выбор по данным: ЦА + цели + ресурсы |
| Одинаковый тон везде | LinkedIn-тон в TikTok = неестественно | Адаптировать ToV под платформу |
| Нет roadmap | Попытка запустить всё сразу | Фазированный рост: Launch → Grow → Scale |
