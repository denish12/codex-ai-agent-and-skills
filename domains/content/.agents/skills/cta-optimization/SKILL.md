# CTA Optimization — Оптимизация CTA

## Когда использовать
- При **создании контента** — формулировка CTA для поста, email, лендинга, статьи, видео.
- При **оптимизации существующего** — CTA не конвертирует, нужна переработка.
- При **A/B тестировании** — генерация вариантов для сравнения.
- Как **часть `$content-brief`** — секция CTA формулируется с помощью этого скила.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Целевое действие | ✅ | Что должен сделать пользователь (одно конкретное действие) |
| Персона ЦА | ✅ | Из `$audience-analysis` — для кого пишем CTA |
| Этап воронки | ✅ | TOFU / MOFU / BOFU — определяет тип CTA |
| Платформа | ✅ | Где будет размещён CTA (влияет на формат и размещение) |
| Контекст контента | ✅ | О чём текст/пост — CTA должен логически вытекать из контента |
| Тип контента | ⬚ | Пост / email / лендинг / статья / видео / сторис |
| Текущий CTA | ⬚ | Если оптимизация существующего — текущая формулировка и её метрики |
| Ограничения | ⬚ | Юридические, бренд-стоп-лист, платформенные лимиты |

> Если этап воронки не указан — **определить** по контексту контента или запросить у пользователя. CTA без воронки = CTA наугад.

## CTA по этапам воронки

| Этап | Цель CTA | Типы действий | Примеры |
|------|----------|---------------|---------|
| **TOFU** (Awareness) | Привлечь, заинтересовать, подписать | Подписаться, сохранить, поделиться, узнать больше | «Подпишись, чтобы не пропустить», «Сохрани на потом» |
| **MOFU** (Consideration) | Углубить доверие, получить контакт | Скачать, зарегистрироваться, посмотреть демо, записаться | «Скачай бесплатный гайд», «Смотри полный разбор» |
| **BOFU** (Decision) | Конвертировать в клиента | Купить, оформить, попробовать бесплатно, забронировать | «Попробуй 14 дней бесплатно», «Забронируй место» |

> **Правило:** CTA должен соответствовать этапу. TOFU + «Купи сейчас» = конфликт. BOFU + «Подпишись» = упущенная конверсия.

## Психологические триггеры

| Триггер | Описание | Когда использовать | Пример |
|---------|----------|-------------------|--------|
| **Value** | Явная выгода от действия | Всегда — базовый триггер | «Получи чеклист из 50 пунктов» |
| **Urgency** | Ограничение по времени | Запуски, промо, события | «До пятницы — бесплатно» |
| **Scarcity** | Ограничение по количеству | Лимитированные офферы | «Осталось 12 мест» |
| **Social proof** | Другие уже сделали это | Подписки, покупки, регистрации | «Присоединяйся к 10 000+ подписчиков» |
| **Loss aversion** | Страх упустить / потерять | MOFU/BOFU, высокая конкуренция | «Не упусти возможность» |
| **Curiosity gap** | Незавершённость, интрига | TOFU, вовлекающий контент | «Узнай, какой из 5 способов работает лучше» |
| **Reciprocity** | Отдаём ценность бесплатно → просим действие | Лид-магниты, freemium | «Скачай бесплатно — мы уже собрали всё за тебя» |
| **Identity** | Связь с самоидентификацией | Ниши с сильным комьюнити | «Для тех, кто строит бизнес серьёзно» |

> Комбинируй 1-2 триггера. Больше 2 — перегруз, CTA звучит манипулятивно.

## Формулы CTA

### Формула 1: Глагол + Результат
```
[Действие] + [что получит пользователь]
→ «Скачай гайд по email-маркетингу»
→ «Получи персональный план»
```

### Формула 2: Ценность → Действие
```
[Что пользователь получит] → [минимальное действие]
→ «50 идей для контента — скачай бесплатно»
→ «Готовый шаблон — забери в один клик»
```

### Формула 3: Действие + Urgency/Scarcity
```
[Действие] + [ограничение]
→ «Забронируй место — осталось 5»
→ «Попробуй бесплатно до конца недели»
```

### Формула 4: Social Proof + Действие
```
[Доказательство] + [действие]
→ «10 000 маркетологов уже используют — попробуй и ты»
→ «Присоединяйся к 500+ студентам курса»
```

### Формула 5: Вопрос + Ответ-действие
```
[Вопрос к боли] → [действие как решение]
→ «Устал планировать контент вручную? Попробуй [продукт]»
→ «Не знаешь, с чего начать? Скачай roadmap»
```

### Формула 6: Identity + Действие
```
[Кто ты] → [действие для таких как ты]
→ «Для фаундеров, которые ценят время — автоматизируй»
→ «Если ты серьёзно про рост — подпишись»
```

## Платформенная адаптация

| Платформа | Формат CTA | Размещение | Ограничения | Особенности |
|-----------|-----------|------------|-------------|-------------|
| **Instagram (пост)** | Текст в caption | Конец caption (после ценности) | Нет кликабельных ссылок в caption | «Ссылка в bio», стрелка ↑ к bio |
| **Instagram (сторис)** | Стикер-ссылка, свайп | Последний слайд или поверх визуала | Стикер ограничен по размеру | Крупный текст + стрелка к стикеру |
| **Telegram** | Текст + inline-кнопка / ссылка | Конец поста | Markdown-ссылки, inline-кнопки бота | Кнопки работают лучше текстовых ссылок |
| **Email** | Кнопка (основной) + текстовая ссылка (резервный) | Above the fold + повтор в конце | Кнопка ≤ 4-5 слов, контрастный цвет | Один основной CTA, кнопка > ссылки по CTR |
| **Лендинг** | Кнопка (hero) + повтор по странице | Hero section + после каждого блока ценности + финал | Кнопка ≤ 5 слов | Цвет кнопки контрастен к фону, повторять 3-5 раз |
| **YouTube** | Голосовой + текст в описании + карточки | Голос: в начале (тизер) + в конце. Описание: первые 2 строки | Первые 2 строки описания видны без «ещё» | «Ссылка в описании», end screen |
| **LinkedIn** | Текст в посте | Конец поста | Ссылки снижают органический охват | CTA без ссылки в посте, ссылка в первом комментарии |

## Протокол

### Шаг 1 — Контекст
1. Получить входные данные: персона, воронка, платформа, цель.
2. Определить, какие триггеры уместны для этой персоны и этапа воронки.
3. Проверить ограничения (юридика, стоп-лист, платформа).

### Шаг 2 — Генерация вариантов
1. Выбрать **2-3 формулы** из списка выше, подходящие под контекст.
2. Для каждой формулы — сформулировать CTA с использованием выбранных триггеров.
3. Итого: **3-5 вариантов** CTA.

### Шаг 3 — Оценка вариантов
Для каждого варианта проверить:

| Критерий | Описание |
|----------|----------|
| Конкретность | Понятно ли, что произойдёт после клика/действия? |
| Ценность | Есть ли явная выгода для пользователя? |
| Краткость | ≤ 5-7 слов для кнопки, ≤ 15 слов для текстового CTA? |
| Соответствие воронке | CTA подходит этапу TOFU/MOFU/BOFU? |
| Соответствие персоне | CTA говорит на языке ЦА? |
| Соответствие платформе | Формат CTA подходит для платформы? |
| Нет манипуляции | Не больше 2 триггеров, нет обмана? |

### Шаг 4 — Размещение
1. Определить **основное размещение** — где в контенте ставить CTA.
2. Определить **повторное размещение** — если формат позволяет (лендинг, длинная статья, email).
3. Зафиксировать формат: кнопка / ссылка / текст / голос / стикер.

### Шаг 5 — Рекомендация
1. Выбрать **рекомендованный вариант** с обоснованием.
2. Предложить **вариант для A/B теста** (отличающийся по триггеру или формуле).
3. Указать метрики для оценки эффективности.

## Валидация (Quality Gate)

CTA считается оптимизированным если:

- [ ] Одно основное целевое действие (не два равнозначных)
- [ ] CTA соответствует этапу воронки
- [ ] CTA содержит явную ценность для пользователя
- [ ] CTA конкретен и понятен (читатель знает, что произойдёт)
- [ ] CTA ≤ 5-7 слов (кнопка) или ≤ 15 слов (текстовый)
- [ ] Формат и размещение адаптированы под платформу
- [ ] Использовано не больше 2 психологических триггеров
- [ ] Есть минимум 2 варианта для A/B теста
- [ ] Нет конфликта с юридическими ограничениями и стоп-листом бренда

## Handoff

| Потребитель | Как использует |
|-------------|---------------|
| `$content-brief` | CTA вставляется в секцию CTA брифа |
| `$email-copywriting` | CTA для email-кампаний |
| `$seo-copywriting` | CTA для статей и лендингов |
| `$social-media-formats` | CTA адаптированный под формат платформы |
| `$content-review-checklist` | Reviewer проверяет CTA по пунктам #15-17 |

## Интеграция с другими скилами

| Скил | Взаимодействие |
|------|----------------|
| `$audience-analysis` | **Источник:** персона, боли, триггеры, язык клиента |
| `$content-brief` | **Потребитель:** CTA как часть брифа |
| `$tone-of-voice` | **Источник:** тон CTA должен соответствовать ToV бренда |
| `$platform-strategy` | **Источник:** платформа → формат и размещение CTA |
| `$content-review-checklist` | Ревьюер проверяет CTA на соответствие брифу и воронке |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Несколько равнозначных CTA | Распыление внимания → ни один не работает | Один основной CTA + максимум один вторичный |
| CTA до ценности | Читатель ещё не мотивирован | CTA после создания ценности |
| TOFU + «Купи сейчас» | Конфликт воронки — холодная аудитория не покупает | TOFU = подпишись/сохрани, BOFU = купи |
| «Подробнее» / «Нажми сюда» | Нет ценности, нет конкретики | Конкретный глагол + результат |
| 3+ триггера в одном CTA | Звучит манипулятивно, вызывает недоверие | Максимум 2 триггера |
| Одинаковый CTA везде | Не учитывает платформу и контекст | Адаптация под платформу и формат |
| CTA не на языке ЦА | «Оформите заявку» для зумеров | Использовать язык клиента из `$audience-analysis` |
| Нет A/B вариантов | Невозможно оптимизировать без данных | Минимум 2 варианта для сравнения |
