# Brand Guidelines — Гайдлайны бренда

## Когда использовать
- При **создании бренда с нуля** — формирование полного гайдлайна.
- При **онбординге** — передача гайдлайнов новому копирайтеру, дизайнеру, подрядчику.
- При **ребрендинге** — обновление существующего гайдлайна с фиксацией изменений.
- При **аудите** — проверка полноты и актуальности действующего гайдлайна.
- Как **эталон** для `$brand-compliance` — без этого документа compliance-проверка невозможна.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Название бренда | ✅ | Юридическое и коммуникационное название |
| Ниша / индустрия | ✅ | В какой отрасли работает бренд |
| Существующие материалы | ✅ | Figma, PDF-гайдлайн, сайт, соцсети — любые источники визуала и стиля |
| Миссия / ценности | ⬚ | Если уже сформулированы — предоставить; если нет — будут определены в шаге 1 |
| Целевая аудитория | ⬚ | Результат `$audience-analysis` или описание ЦА |
| Конкуренты | ⬚ | Для позиционирования и дифференциации |

> Если существующих материалов нет (бренд с нуля) — агент формирует гайдлайн через интервью с пользователем. Не придумывать без подтверждения.

## Источники данных
1. **Существующие активы бренда** — сайт, соцсети, презентации, упаковка.
2. **Figma / Sketch / PDF** — дизайн-система, если есть.
3. **Интервью с пользователем** — миссия, ценности, позиционирование, тон (если не задокументированы).
4. **`$audience-analysis`** — кому говорим, на каком языке.
5. **`$competitor-content-analysis`** — от чего дифференцируемся.

## Протокол

### Шаг 1 — Brand Identity (ядро)
1. Зафиксировать или сформулировать **миссию** (зачем бренд существует).
2. Определить **ценности** (3-5 ключевых принципов).
3. Сформулировать **позиционирование** (для кого, что делает, чем отличается).
4. Определить **архетип бренда** (если применимо: Герой, Мудрец, Бунтарь и т.д.).
5. Зафиксировать **ключевые сообщения** (3-5 тезисов, которые бренд транслирует всегда).

### Шаг 2 — Naming & Verbal Identity
1. Правила написания названия бренда (регистр, язык, допустимые сокращения).
2. Слоган / tagline (если есть).
3. Тональность названий продуктов / услуг.
4. Правила транслитерации (для мультиязычных брендов).

### Шаг 3 — Tone of Voice
1. Определить характеристики тона (4 оси: формальность, эмоциональность, юмор, экспертность).
2. Для каждой оси — шкала от 1 до 5 с текущей позицией бренда.
3. Запрещённые слова и клише (стоп-лист).
4. Обязательные речевые паттерны (если есть).
5. Связать с `$tone-of-voice` — этот скил даёт подробную разработку тона.

> Секция ToV в гайдлайне — краткая выжимка. Полная проработка — через `$tone-of-voice`.

### Шаг 4 — Визуальная идентичность
1. **Цветовая палитра** — основные, акцентные, нейтральные цвета (HEX, RGB, CMYK).
2. **Типографика** — шрифты для заголовков, основного текста, CTA, подписей.
3. **Лого** — варианты, минимальные размеры, охранное поле, запрещённые трансформации.
4. **Визуальный стиль** — фотография, иллюстрации, иконки, паттерны/текстуры.
5. **Композиция** — сетка, отступы, иерархия элементов.
6. **Графические приёмы** — фирменные рамки, формы, оверлеи (если есть).

### Шаг 5 — Do / Don't
Для каждого ключевого элемента (лого, цвета, тон, фото) — конкретные примеры:
- ✅ **Do:** правильное использование с описанием.
- ❌ **Don't:** неправильное использование с объяснением, почему нельзя.

> Минимум 2 Do и 2 Don't для каждого элемента: лого, цвета, типографика, ToV.

### Шаг 6 — Платформенные адаптации
Для каждой приоритетной платформы (из `$platform-strategy`):
- Размеры и соотношения сторон для визуалов.
- Допустимый вариант лого.
- Адаптации типографики (мобильные размеры, web-safe шрифты).
- Особенности тона (если платформа требует адаптации — LinkedIn формальнее TikTok).

### Шаг 7 — Юридика и Trademark
1. Символ trademark (™ / ®) — когда и где использовать.
2. Копирайт — формат, год, обязательность.
3. Правила использования бренда третьими сторонами (партнёры, медиа, инфлюенсеры).
4. Ограничения: что категорически запрещено делать с элементами бренда.

### Шаг 8 — Версионирование
1. Присвоить версию гайдлайна.
2. Зафиксировать дату и автора.
3. Если обновление — описать что изменилось (changelog).

## Валидация (Quality Gate)

Гайдлайн считается полным если:

- [ ] Brand Identity заполнена (миссия, ценности, позиционирование, ключевые сообщения)
- [ ] Verbal Identity зафиксирована (написание названия, слоган)
- [ ] ToV определён (4 оси + стоп-лист) или есть ссылка на `$tone-of-voice`
- [ ] Визуальная идентичность полная (палитра, типографика, лого, стиль)
- [ ] Do / Don't присутствуют для: лого, цвета, типографика, ToV (минимум по 2)
- [ ] Платформенные адаптации есть для всех приоритетных платформ
- [ ] Юридическая секция заполнена (trademark, копирайт, правила для третьих лиц)
- [ ] Версия и дата зафиксированы
- [ ] Гайдлайн согласован с пользователем

## Handoff

Результат этого скила является **эталоном** для:

| Потребитель | Как использует |
|-------------|---------------|
| `$brand-compliance` | Сверяет контент с гайдлайном — **основной потребитель** |
| `$tone-of-voice` | Использует ToV-секцию как отправную точку для детальной проработки |
| `$visual-brief` | Берёт визуальные спецификации для конкретных задач |
| `$content-brief` | Ссылается на гайдлайн в секции «Стиль» |
| Copywriter (агент) | Следует ToV и messaging при создании текстов |
| Visual Concept (агент) | Следует визуальной идентичности при создании креативов |

При передаче — использовать `$handoff` с указанием версии гайдлайна.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Гайдлайн без миссии/ценностей | Визуал без ядра — косметика без стратегии | Всегда начинать с Brand Identity |
| Только визуал, нет текста | Копирайтер не знает как писать | ToV + messaging обязательны |
| Нет Do / Don't | Абстрактные правила — каждый интерпретирует по-своему | Конкретные примеры правильного и неправильного |
| Один размер для всех платформ | Лого 1000px в сторис выглядит иначе чем в хедере | Адаптации под каждую платформу |
| Нет стоп-листа | Копирайтер использует слова, чуждые бренду | Явный список запрещённых слов и клише |
| Нет версии | При обновлении непонятно что актуально | Всегда версия + дата + changelog |
| Выдуманные гайдлайны | Агент придумал палитру без пользователя | Всё подтверждать с пользователем |
