# Storytelling Framework — Фреймворк сторителлинга

## Когда использовать
- При создании **контента с нарративом** — статьи, длинные посты, email, видео, кейсы.
- При **продающем контенте** — лендинги, promo, офферы.
- При **вовлекающем контенте** — personal stories, behind the scenes, transformations.
- Как часть **`$content-brief`** — выбор нарративной структуры для единицы контента.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Цель контента | ✅ | Убедить / вдохновить / обучить / развлечь / продать |
| Персона ЦА | ✅ | Из `$audience-analysis` — боли, триггеры, язык |
| Платформа | ✅ | Определяет длину и глубину нарратива |
| Тип контента | ✅ | Пост / статья / email / reel / лендинг / кейс / видео |
| Ключевые данные / факты | ⬚ | Из `$topic-research` / `$data-storytelling` — если data-driven |
| Продукт / оффер | ⬚ | Если продающий контент — что продвигаем |

> Если цель не указана — **запросить**. Фреймворк без цели = случайный выбор.

## Decision Tree — выбор фреймворка

| Цель | Тип контента | Рекомендуемые фреймворки |
|------|-------------|------------------------|
| **Продать** (конверсия) | Лендинг, promo, email | PAS, PASTOR, AIDA, FAB |
| **Убедить** (мнение) | Пост, статья, LinkedIn | BAB, PAS, StoryBrand |
| **Обучить** (know-how) | Карусель, статья, видео | 4P, STAR, How-to (AIDA-адаптация) |
| **Вдохновить** (мотивация) | Пост, reel, email | Hero's Journey, BAB, Pixar Formula |
| **Кейс** (результат) | Пост, статья, видео | STAR, BAB, Hero's Journey |
| **Развлечь** (engagement) | Reel, пост, TikTok | Pixar Formula, Open Loop |

> Выбрать **1 фреймворк** для одной единицы контента. Не смешивать.

## Фреймворки

### 1. PAS — Problem → Agitate → Solution

**Суть:** Показать проблему, усилить боль, предложить решение.
**Лучше всего для:** Продающий, MOFU/BOFU, email, лендинг.

| Этап | Что делает | Emotional arc |
|------|-----------|---------------|
| **P — Problem** | Описать проблему, близкую аудитории | Узнавание: «это про меня» |
| **A — Agitate** | Усилить боль: что будет, если не решить? Последствия | Напряжение: тревога, frustration |
| **S — Solution** | Показать решение (продукт/метод) + CTA | Облегчение: «вот как это исправить» |

**Пример (Instagram, email-маркетинг):**
> **P:** Тратишь 3 часа на одну email-рассылку?
> **A:** Пока ты вручную собираешь письма, конкуренты автоматизировали весь процесс. Каждый день промедления — потерянные лиды.
> **S:** [Продукт] позволяет создать email-кампанию за 15 минут. Попробуй бесплатно →

---

### 2. AIDA — Attention → Interest → Desire → Action

**Суть:** Классическая рекламная воронка в одном тексте.
**Лучше всего для:** Реклама, promo, лендинг, объявления.

| Этап | Что делает | Emotional arc |
|------|-----------|---------------|
| **A — Attention** | Hook — захватить внимание | Любопытство |
| **I — Interest** | Почему это важно для читателя | Интерес: «расскажи подробнее» |
| **D — Desire** | Выгоды, трансформация, social proof | Желание: «я тоже хочу» |
| **A — Action** | Конкретный CTA | Решимость: «делаю прямо сейчас» |

**Пример (лендинг, SaaS):**
> **A:** 87% маркетологов тратят больше 5 часов в неделю на ручное планирование контента.
> **I:** А что если бы AI делал это за вас — и лучше?
> **D:** [Продукт] создаёт контент-план, генерирует тексты и публикует по расписанию. 500+ компаний уже перешли.
> **A:** Попробуй 14 дней бесплатно →

---

### 3. BAB — Before → After → Bridge

**Суть:** Показать контраст «было/стало» и мостик — как перейти.
**Лучше всего для:** Кейсы, мотивационный, трансформация, LinkedIn.

| Этап | Что делает | Emotional arc |
|------|-----------|---------------|
| **Before** | Ситуация «до» — боль, проблема | Узнавание |
| **After** | Ситуация «после» — результат, мечта | Вдохновение: «хочу так же» |
| **Bridge** | Как перейти от Before к After (продукт/метод/действие) | Решение: «вот мой путь» |

**Пример (LinkedIn, кейс):**
> **Before:** Год назад наш Telegram-канал имел 200 подписчиков и 30 просмотров на пост.
> **After:** Сегодня — 15 000 подписчиков, 8 000 просмотров, и канал приносит 40% лидов.
> **Bridge:** Что изменилось: контент-календарь по формуле 40/30/20/10 + ежедневные посты в 9:00. Вот 3 принципа, которые сработали →

---

### 4. Hero's Journey (упрощённый)

**Суть:** История героя: обычный мир → вызов → трансформация → новый мир.
**Лучше всего для:** Brand stories, кейсы, вдохновляющий, видео.

| Этап | Что делает | Emotional arc |
|------|-----------|---------------|
| **Обычный мир** | Ситуация до — герой (клиент/бренд) в привычной среде | Идентификация |
| **Вызов** | Проблема, кризис, задача, которая заставляет действовать | Напряжение |
| **Трансформация** | Что изменилось — решение найдено, барьеры преодолены | Кульминация |
| **Новый мир** | Результат — герой изменился, жизнь улучшилась | Катарсис, вдохновение |

**Пример (видео, бренд-стори):**
> **Обычный мир:** Мария — маркетолог в стартапе. Одна на все задачи, выгорает.
> **Вызов:** CEO ставит цель ×3 по лидам за квартал. Без бюджета и команды.
> **Трансформация:** Мария внедряет AI-контент-стратегию, автоматизирует рутину, фокусируется на креативе.
> **Новый мир:** Лиды ×4, Мария получает повышение, нанимает команду.

---

### 5. STAR — Situation → Task → Action → Result

**Суть:** Структурированный кейс с цифрами.
**Лучше всего для:** Кейсы, портфолио, LinkedIn, B2B.

| Этап | Что делает | Emotional arc |
|------|-----------|---------------|
| **S — Situation** | Контекст: кто, где, когда | Понимание контекста |
| **T — Task** | Задача: что нужно было достичь | Фокус: «сложная задача» |
| **A — Action** | Конкретные действия: что сделали | Интерес: «как именно?» |
| **R — Result** | Результат с цифрами | Доверие: «вот доказательство» |

**Пример (LinkedIn, B2B):**
> **S:** Клиент — SaaS-стартап, 50 trial/мес, конверсия в paid 3%.
> **T:** Увеличить конверсию trial→paid до 10% за 3 месяца.
> **A:** Перестроили onboarding-серию email (5 писем), добавили in-app tooltips, запустили win-back для churned.
> **R:** Конверсия 12.4% за 2.5 месяца. LTV вырос на 35%.

---

### 6. PASTOR — Problem → Amplify → Story → Transformation → Offer → Response

**Суть:** Расширенный PAS с историей и оффером. Самый полный продающий фреймворк.
**Лучше всего для:** Длинные лендинги, sales pages, email-серии.

| Этап | Что делает |
|------|-----------|
| **P — Problem** | Описать проблему |
| **A — Amplify** | Усилить последствия (сильнее чем Agitate в PAS) |
| **S — Story** | Рассказать историю (свою или клиента) |
| **T — Transformation** | Показать трансформацию (before → after) |
| **O — Offer** | Представить предложение |
| **R — Response** | Призвать к конкретному действию |

---

### 7. Pixar Formula

**Суть:** Формула Pixar для простых и мощных историй.
**Лучше всего для:** Короткие истории, reels, посты, вдохновляющий контент.

```
Жил-был [герой].
Каждый день он [рутина].
Однажды [событие-перелом].
Из-за этого [последствие 1].
Из-за этого [последствие 2].
И наконец [развязка / новая реальность].
```

**Пример (Instagram reel):**
> Жил-был маркетолог, который тратил 6 часов на контент-план. Каждый день он вручную собирал темы, писал тексты, подбирал картинки. Однажды он попробовал AI-инструменты. Из-за этого план стал занимать 30 минут. Из-за этого появилось время на стратегию. И наконец — контент стал приносить лиды, а не только лайки.

---

### 8. FAB — Features → Advantages → Benefits

**Суть:** От характеристик продукта к выгодам для пользователя.
**Лучше всего для:** Продуктовые посты, описания, карточки товаров.

| Этап | Что делает | Пример (email-сервис) |
|------|-----------|----------------------|
| **F — Features** | Что есть (характеристика) | «Drag-and-drop редактор» |
| **A — Advantages** | Чем это лучше (преимущество) | «Не нужен дизайнер или код» |
| **B — Benefits** | Что это даёт пользователю (выгода) | «Создавай красивые письма за 10 минут сам» |

---

### 9. 4P — Promise → Picture → Proof → Push

**Суть:** Обещание → визуализация → доказательство → подталкивание к действию.
**Лучше всего для:** Образовательный, how-to, карусели.

| Этап | Что делает |
|------|-----------|
| **Promise** | Пообещать конкретный результат |
| **Picture** | Нарисовать картинку: как будет выглядеть жизнь после |
| **Proof** | Доказать: данные, кейс, social proof |
| **Push** | CTA: подтолкнуть к действию |

---

### 10. Open Loop

**Суть:** Начать историю, не закончить → создать интригу → закончить в конце (или в следующем контенте).
**Лучше всего для:** Серии, threads, reels (retention), email (open rate).

```
[Hook: незавершённый вопрос / начало истории без конца]
↓
[Контент: ценность, тезисы, развитие]
↓
[Закрытие loop'а: ответ на вопрос / конец истории]
```

> Open Loop в начале удерживает внимание до конца. Открытый loop в конце → мотивирует подписаться / прочитать часть 2.

## Emotional Arc — как строить напряжение

Любая хорошая история следует эмоциональной дуге:

```
Эмоция
  │
  │         ╱ Кульминация
  │        ╱
  │       ╱
  │      ╱
  │     ╱
  │    ╱ Нарастание напряжения
  │   ╱
  ├──╱─────────── Hook (захват)
  │ ╱
  │╱   Setup (контекст)
  └──────────────────────────── Время
          Конфликт    Разрешение → CTA
```

### Элементы эмоциональной дуги

| Элемент | Что делает | Где в тексте |
|---------|-----------|-------------|
| **Hook** | Захват внимания — неожиданный факт, вопрос, противоречие | Первое предложение / первые 3 сек видео |
| **Setup** | Контекст — кто герой, где он, что происходит | 2-3 предложения после hook |
| **Конфликт** | Напряжение — проблема, барьер, вызов | Центральная часть |
| **Нарастание** | Усиление — последствия, ставки растут | Перед кульминацией |
| **Кульминация** | Пиковый момент — переломная точка, инсайт, решение | 2/3 текста |
| **Разрешение** | Результат — что изменилось, урок, takeaway | Предпоследний абзац |
| **CTA** | Действие — естественно вытекает из разрешения | Последнее предложение |

> **Правило:** Нет конфликта = нет истории. Даже образовательный контент имеет конфликт: «незнание → знание», «ошибка → исправление».

## Платформенная адаптация

| Платформа | Длина нарратива | Глубина | Лучшие фреймворки |
|-----------|----------------|---------|-------------------|
| **Instagram (пост)** | Короткий (500-1500 символов) | 1-2 этапа фреймворка | PAS, BAB, Open Loop |
| **Instagram (карусель)** | Средний (1 слайд = 1 этап) | Полный фреймворк | AIDA, 4P, STAR |
| **Instagram (reel)** | 15-60 сек | Экспресс-нарратив | Pixar, BAB, Open Loop |
| **LinkedIn** | Средний-длинный (1000-2500 символов) | Полный фреймворк | STAR, BAB, Hero's Journey |
| **Telegram** | Средний (1000-3000 символов) | Полный фреймворк | PAS, STAR, BAB |
| **Email** | Средний (150-500 слов) | Зависит от типа email | PAS (promo), AIDA (launch), PASTOR (long-form sales) |
| **Блог / статья** | Длинный (1500-4000 слов) | Расширенный фреймворк | Hero's Journey, PASTOR, STAR |
| **YouTube** | 3-15 мин | Полный с визуальным storytelling | Hero's Journey, Open Loop, Pixar |
| **TikTok** | 15-60 сек | Экспресс | Pixar, Open Loop, BAB |

## Протокол

### Шаг 1 — Контекст
1. Определить цель, персону, платформу, тип контента.
2. По Decision Tree — выбрать **1 фреймворк**.
3. Определить длину и глубину нарратива по платформе.

### Шаг 2 — Данные для нарратива
1. Собрать факты, данные, цитаты (из `$topic-research`, `$data-storytelling`).
2. Определить **героя** истории: клиент? пользователь? бренд? абстрактная персона?
3. Определить **конфликт**: какая проблема / вызов / барьер?

### Шаг 3 — Заполнение фреймворка
1. Заполнить каждый этап фреймворка конкретными данными.
2. Проверить **emotional arc**: есть ли напряжение → кульминация → разрешение?
3. CTA вытекает из разрешения **естественно** (не пристёгнут).

### Шаг 4 — Проверка нарратива
Пройти чеклист:
1. Есть ли **конфликт**? (нет конфликта = нет истории)
2. Есть ли **трансформация**? (герой изменился)
3. Читатель **узнаёт себя** в герое? (идентификация)
4. История **ведёт к CTA** естественно? (не навязано)
5. **Hook** в первом предложении захватывает? (не начинать с контекста)

### Шаг 5 — Адаптация под платформу
1. Подогнать длину под платформу.
2. Если карусель — 1 этап = 1 слайд.
3. Если видео — визуализировать каждый этап.

## Валидация (Quality Gate)

Нарратив считается готовым если:

- [ ] Фреймворк выбран по Decision Tree (цель × платформа)
- [ ] Все этапы фреймворка заполнены конкретными данными
- [ ] Есть конфликт / напряжение (не плоский текст)
- [ ] Есть трансформация (герой изменился / читатель узнал новое)
- [ ] Hook в первом предложении (не начинается с контекста)
- [ ] CTA вытекает из разрешения естественно
- [ ] Emotional arc прослеживается
- [ ] Длина адаптирована под платформу
- [ ] Читатель может узнать себя в герое (идентификация с персоной ЦА)

## Handoff

| Потребитель | Как использует |
|-------------|---------------|
| `$content-brief` | Фреймворк как часть секции «Структура» |
| Copywriter (агент) | Пишет текст по заполненному фреймворку |
| `$email-copywriting` | PAS / AIDA / PASTOR для email |
| `$social-media-formats` | Нарратив адаптируется под формат платформы |
| `$headline-formulas` | Hook из нарратива → заголовок |
| `$cta-optimization` | CTA из разрешения нарратива |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Нет конфликта | Плоский текст, не вовлекает | Даже в how-to есть конфликт: незнание → знание |
| Начать с контекста вместо hook | Читатель уходит до сути | Hook в первом предложении, контекст — во втором |
| CTA пристёгнут | «И кстати, купите наш продукт» после истории о котиках | CTA вытекает из разрешения нарратива |
| Смешивать фреймворки | PAS + AIDA + Hero's Journey = хаос | Один фреймворк на одну единицу контента |
| Герой = бренд | Бренд хвалит себя, нет идентификации | Герой = клиент / читатель. Бренд = проводник/помощник |
| Нет трансформации | «Было плохо. Вот решение.» — плоско | Показать путь от Before к After |
| Длинная история в коротком формате | Hero's Journey в одном твите = скомканно | Платформа определяет фреймворк и глубину |
| Выдуманная история без пометки | Если story = фикция, аудитория почувствует обман | Реальные кейсы > выдумка. Или пометить «представим ситуацию» |
