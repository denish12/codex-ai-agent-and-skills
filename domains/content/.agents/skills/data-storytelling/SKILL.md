# Data Storytelling — Data-driven нарративы

## Когда использовать
- После **сбора фактов и статистики** (`$topic-research`, `$source-verification`) — структурировать в историю.
- При создании **аналитического контента** — статьи, карусели, инфографика, отчёты.
- При **обосновании позиции** — кейсы, сравнения, прогнозы.
- Для **усиления любого контента** данными — даже 1-2 цифры делают пост убедительнее.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Данные / факты | ✅ | Цифры, статистика, результаты исследований — то, из чего строим историю |
| Источники данных | ✅ | Откуда данные: исследование, аналитика, API, отчёт, `$source-verification` |
| Тема / контекст | ✅ | О чём рассказываем — привязка к теме контента |
| Персона ЦА | ✅ | Из `$audience-analysis` — определяет глубину и язык подачи |
| Платформа | ✅ | Влияет на объём, формат визуализации, глубину |
| Цель контента | ⬚ | Информировать / убедить / вовлечь / конвертировать |
| Тип data story | ⬚ | Если не указан — агент определяет по характеру данных |

> Если данные не предоставлены — **запросить у пользователя** или направить на `$topic-research` / `$source-verification`. Не выдумывать цифры.

## Типы Data Stories

| Тип | Когда использовать | Вопрос, на который отвечает | Пример |
|-----|-------------------|----------------------------|--------|
| **Тренд** | Данные показывают изменение во времени | «Что происходит и куда движется?» | «Рынок AI-агентов вырос на 340% за год» |
| **Сравнение** | Есть 2+ объекта или периода | «Кто лучше / что изменилось?» | «Email vs Telegram: ROI маркетинговых каналов» |
| **Аномалия** | Есть неожиданный факт, выброс, противоречие | «Почему это странно?» | «87% компаний используют AI, но только 12% видят ROI» |
| **Корреляция** | Связь между двумя факторами | «Что с чем связано?» | «Компании с контент-стратегией получают 6× больше лидов» |
| **Прогноз** | Данные позволяют экстраполировать | «Что будет дальше?» | «К 2027 году 50% контента будет генерироваться AI» |
| **Масштаб** | Нужно показать размер явления | «Насколько это много/мало?» | «Ежедневно публикуется 7.5M постов в блогах — это 87 постов в секунду» |

> Тип определяет нарративную структуру и выбор визуализации.

## Нарративные структуры по типам

### Тренд
```
Контекст (было) → Точка перелома → Тренд (данные) → Что это значит → Прогноз/CTA
```

### Сравнение
```
Общий контекст → Объект A (данные) → Объект B (данные) → Ключевое отличие → Вывод
```

### Аномалия
```
Ожидание (что все думают) → Реальность (данные) → Почему так → Инсайт → Вывод
```

### Корреляция
```
Наблюдение → Связь (данные A + данные B) → Причина/механизм → Что с этим делать
```

### Прогноз
```
Текущая ситуация (данные) → Факторы роста/падения → Прогноз → Что делать уже сейчас
```

### Масштаб
```
Абстрактная цифра → Масштабирование (перевод в понятное) → Почему это важно → Вывод
```

## Техники убедительности

| Техника | Описание | Пример |
|---------|----------|--------|
| **Масштабирование** | Перевод абстрактной цифры в понятную | «$3.5 трлн — это больше ВВП Великобритании» |
| **Сравнение** | Сопоставление с чем-то знакомым | «Это как если бы каждый третий житель Москвы...» |
| **Антитезис** | Контраст ожидания и реальности | «Все думают, что X. На самом деле — Y» |
| **Метафора** | Перевод данных в образ | «Email-рассылка — это рыбалка: 95% улова — это тема письма» |
| **Персонализация** | Перевод на уровень одного человека | «В среднем маркетолог тратит 6 часов в неделю на планирование контента» |
| **Before/After** | Показать состояние до и после | «До автоматизации: 3 дня. После: 15 минут» |
| **Rule of Three** | Три факта запоминаются лучше | «Быстрее. Дешевле. Эффективнее.» |
| **Вопрос-ответ** | Задать вопрос, ответить данными | «Сколько стоит один лид? $5 в Instagram vs $50 в LinkedIn» |

> Использовать 1-3 техники на одну data story. Больше — перегруз.

## Визуализация по типу данных

| Тип данных / story | Рекомендуемая визуализация | Когда НЕ использовать |
|--------------------|--------------------------|-----------------------|
| **Тренд** (время) | Line chart, Area chart | Pie chart (нет временной оси) |
| **Сравнение** (категории) | Bar chart (горизонтальный), Grouped bar | Line chart (нет непрерывности) |
| **Пропорции** (части целого) | Pie chart (≤ 5 сегментов), Stacked bar | Pie chart с 10+ сегментами |
| **Корреляция** (две переменные) | Scatter plot, Bubble chart | Bar chart (не показывает связь) |
| **Масштаб** (один большой факт) | Big number + контекст, Pictogram | Сложный график (убивает эффект) |
| **Рейтинг** (порядок) | Horizontal bar (отсортированный) | Pie chart (нет порядка) |
| **Процесс** (этапы) | Funnel chart, Sankey diagram | Pie chart |
| **Для соцсетей** | Карусель с 1 графиком на слайд, инфографика | Сложные dashboard'ы |

> Если данных мало (1-3 цифры) — визуализация не нужна. Достаточно крупного числа в тексте.

## Валидация данных (обязательная)

Перед построением нарратива — проверить данные:

| Критерий | Вопрос | Риск если не проверить |
|----------|--------|----------------------|
| **Актуальность** | Когда собраны данные? | Устаревшая статистика = ложные выводы |
| **Источник** | Кто провёл исследование? Есть ли методология? | Ненадёжный источник = нет доверия |
| **Выборка** | Сколько респондентов / точек данных? | Маленькая выборка = случайность |
| **Контекст** | Данные не вырваны из контекста? | Cherry-picking = манипуляция |
| **Причинность vs корреляция** | Связь = причина или совпадение? | Ложная каузальность = ошибочный вывод |
| **Полнота** | Показаны ли данные «за» И «против»? | Однобокость = bias |

> Если данные не проходят валидацию — не использовать. Лучше меньше цифр, но надёжных.

## Платформенная адаптация

| Платформа | Глубина | Количество фактов | Визуализация | Формат |
|-----------|---------|-------------------|-------------|--------|
| **LinkedIn** | Высокая — развёрнутый анализ | 5-10 фактов | Инфографика, карусель, скриншот графика | Длинный пост / статья |
| **Instagram** | Средняя — 1 инсайт + 2-3 факта | 2-4 факта | Карусель (1 факт = 1 слайд), инфографика | Карусель / reel |
| **Telegram** | Средне-высокая | 3-7 фактов | Встроенные таблицы, ссылка на полный отчёт | Пост с форматированием |
| **Twitter/X** | Низкая — 1 крючок-факт | 1-2 факта | Скриншот графика / big number | Thread или одиночный пост |
| **YouTube** | Высокая — визуальный нарратив | 5-15 фактов | Анимированные графики, B-roll с данными | Видео-эссе / разбор |
| **Блог / статья** | Максимальная | 10+ фактов | Встроенные графики, таблицы, интерактив | Лонгрид |
| **Email** | Средняя — 1 ключевой инсайт | 2-3 факта | Big number, мини-инфографика | Дайджест / рассылка |

## Протокол

### Шаг 0 — Сбор и валидация данных
1. Получить входные данные: факты, источники, тема, персона, платформа.
2. Пройти **валидацию данных** по 6 критериям.
3. Отсеять ненадёжные данные — работать только с проверенными.

### Шаг 1 — Определение типа data story
1. Проанализировать характер данных — тренд? сравнение? аномалия?
2. Выбрать тип (или комбинацию из 2 типов).
3. Выбрать нарративную структуру под тип.

### Шаг 2 — Центральный инсайт
1. Из всех данных — выбрать **один центральный инсайт** (главная мысль).
2. Сформулировать в одно предложение.
3. Проверка: инсайт **неочевиден**? Если все и так знают — это не инсайт.

### Шаг 3 — Крючок-факт
1. Выбрать **самый сильный / неожиданный факт** из данных.
2. Это будет заголовок, hook, первое предложение.
3. Проверка: факт вызывает реакцию «серьёзно?!» или «не знал»?

### Шаг 4 — Построение нарратива
1. Разложить данные по нарративной структуре выбранного типа.
2. Применить **1-3 техники убедительности**.
3. Для каждого факта — «что это значит» (не просто цифра, а интерпретация).
4. Финал — **вывод + CTA** (что делать с этим знанием).

### Шаг 5 — Визуализация
1. Определить, нужна ли визуализация (если 1-3 цифры — не нужна).
2. Выбрать тип визуализации по маппингу.
3. Описать визуализацию (для `$visual-brief` или дизайнера).

### Шаг 6 — Адаптация под платформу
1. Определить глубину и количество фактов по таблице.
2. Адаптировать формат (лонгрид vs карусель vs thread).
3. Адаптировать язык под персону.

### Шаг 7 — Финальная проверка
Прогнать через фильтр:
1. История понятна **без цифр**? (цифры подтверждают, не заменяют).
2. Цифры понятны **без контекста**? Если нет — добавить масштабирование.
3. Инсайт **неочевиден**?
4. Данные **не вводят в заблуждение**?

## Валидация (Quality Gate)

Data story считается готовой если:

- [ ] Все данные прошли валидацию (6 критериев)
- [ ] Определён тип data story
- [ ] Центральный инсайт сформулирован (1 предложение, неочевиден)
- [ ] Крючок-факт выбран (вызывает реакцию)
- [ ] Нарратив выстроен по структуре выбранного типа
- [ ] Использовано 1-3 техники убедительности
- [ ] Каждый факт имеет интерпретацию («что это значит»)
- [ ] Визуализация выбрана (или обоснованно не нужна)
- [ ] Адаптировано под платформу и персону
- [ ] Финальная проверка пройдена (4 вопроса)

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Цифры без контекста | «Рынок $4.2 трлн» — и что? Много это или мало? | Масштабирование: «Это больше ВВП Германии» |
| Данные вместо истории | Перечень фактов без нарратива | Сначала структура, потом данные подтверждают |
| Cherry-picking | Показаны только удобные данные | Показать полную картину, включая «неудобные» факты |
| Корреляция = причинность | «Мороженое → утопления» | Явно указать: корреляция, не причинность |
| Устаревшие данные | «Исследование 2019 года» в 2026 | Данные ≤ 2 лет, для быстрых рынков ≤ 1 год |
| Слишком много цифр | 15 фактов в одном посте | 2-5 фактов для соцсетей, больше — для лонгридов |
| Нет инсайта | Все и так знали этот факт | Инсайт = неочевидный вывод из данных |
| Выдуманные данные | Агент сгенерировал «статистику» | Только верифицированные источники |
