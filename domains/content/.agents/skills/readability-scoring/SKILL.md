# Readability Scoring — Оценка читаемости

## Когда использовать
- На этапе **Reviewer** — оценка читаемости перед одобрением.
- На этапе **Copywriter** (самопроверка) — до передачи на ревью.
- При **оптимизации существующего контента** — текст не вовлекает, дочитывание низкое.
- Для **длинных форматов** (лонгрид, статья, email) — особенно критично.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Текст для оценки | ✅ | Финальный текст контента |
| Платформа | ✅ | Определяет нормы читаемости (соцсети ≠ блог ≠ email) |
| Тип контента | ✅ | Пост / карусель / статья / email / лендинг / видео-скрипт |
| Персона ЦА | ✅ | Из `$audience-analysis` — уровень экспертизы определяет допустимую сложность |
| Язык | ⬚ | Русский / английский — разные формулы и нормы |

> Если персона не указана — **запросить**. Текст для CEO и для студента — разный уровень допустимой сложности.

## Адаптивные нормы по платформам

| Параметр | Соцсети (Instagram, TikTok, Twitter) | Telegram / LinkedIn | Email | Блог / Статья | Лендинг |
|----------|--------------------------------------|--------------------|----|---------------|---------|
| **Ср. длина предложения** | 8-15 слов | 12-20 слов | 12-18 слов | 15-22 слов | 10-15 слов |
| **Макс. предложений в абзаце** | 2-3 | 3-4 | 3-4 | 4-5 | 2-3 |
| **Макс. длина предложения** | 25 слов | 30 слов | 30 слов | 35 слов | 20 слов |
| **Подзаголовки** | Не обязательны (short form) | Желательны | Да (каждые 2-3 абзаца) | Обязательны (каждые 200-300 слов) | Обязательны |
| **Буллеты / списки** | Опционально | Рекомендуется | Рекомендуется | Обязательны для перечислений | Обязательны |
| **Активный залог** | > 90% | > 80% | > 85% | > 75% | > 90% |
| **Допустимый жаргон** | Минимум (0-1 термин) | Умеренно (для B2B) | Минимум | Умеренно (с пояснениями) | Минимум |

## Адаптация по уровню аудитории

| Уровень ЦА | Допустимая сложность | Примечания |
|-------------|---------------------|------------|
| **Новички** | Простой язык, нет жаргона, короткие предложения | Каждый термин — пояснять |
| **Средний** | Умеренная сложность, базовый отраслевой жаргон | Сложные термины — 1 пояснение при первом упоминании |
| **Эксперты** | Высокая, профессиональный жаргон допустим | Не упрощать до банальности — оскорбляет экспертизу |

## Метрики читаемости

### Основные метрики (считать для каждого текста)

| Метрика | Как считать | Что показывает |
|---------|------------|----------------|
| **Средняя длина предложения** | Всего слов / всего предложений | Общая сложность текста |
| **Макс. длина предложения** | Самое длинное предложение (слов) | «Узкие места» — где читатель теряется |
| **Средняя длина абзаца** | Предложений в абзацах (среднее) | Визуальная плотность |
| **Доля активного залога** | Предложения с активным залогом / всего × 100% | Энергичность текста |
| **Доля «воды»** | Стоп-слова и пустые конструкции / всего слов × 100% | Информационная плотность |
| **Scanability score** | Чеклист из 6 пунктов (см. ниже) | Можно ли «просканировать» текст за 10 сек |

### Индексы читаемости

#### Для русского языка

| Индекс | Формула (упрощённая) | Интерпретация |
|--------|---------------------|---------------|
| **Индекс Флеша (адаптированный)** | 206.835 − 1.3 × (слов/предложений) − 60.1 × (слогов/слов) | 60-70: нормальный текст. < 30: сложный. > 80: очень простой |
| **Средняя длина слова** | Всего символов / всего слов | > 7 символов: сложный текст. < 5: простой |

> Для русского языка формулы Flesch-Kincaid дают приблизительную оценку. Основной ориентир — **средняя длина предложения + средняя длина слова**.

#### Для английского языка

| Индекс | Формула | Целевое значение |
|--------|---------|-----------------|
| **Flesch Reading Ease** | 206.835 − 1.015 × ASL − 84.6 × ASW | 60-70 (standard), 70-80 (easy) |
| **Flesch-Kincaid Grade** | 0.39 × ASL + 11.8 × ASW − 15.59 | 7-9 (Grade level, для wide audience) |
| **Gunning Fog** | 0.4 × (ASL + % complex words) | < 12 для wide audience |

> ASL = Average Sentence Length. ASW = Average Syllables per Word.

## Scanability — проверка «сканируемости»

Большинство читателей **сканируют**, а не читают. Проверить:

| # | Критерий | Описание | Статус |
|---|----------|----------|--------|
| 1 | **F-pattern** | Первые слова каждого абзаца несут смысл (не «Однако», «Кроме того») | ✅/❌ |
| 2 | **Подзаголовки** | Каждые 200-300 слов (для лонгридов) — информативные, не декоративные | ✅/❌ |
| 3 | **Буллеты** | Перечисления оформлены списками, не текстовой «стеной» | ✅/❌ |
| 4 | **Выделение ключевого** | Жирный / курсив для ключевых мыслей (не больше 10% текста) | ✅/❌ |
| 5 | **Визуальное дыхание** | Короткие абзацы, пустые строки между секциями | ✅/❌ |
| 6 | **Hook → Main → CTA** | Текст следует паттерну: захват → ценность → действие | ✅/❌ |

**Scanability score:** [N/6] — ≥ 5 = хорошо, 3-4 = доработать, ≤ 2 = переструктурировать.

## Проверка на «воду»

### Что считается «водой»

| Категория | Примеры | Что делать |
|-----------|---------|-----------|
| **Канцеляризмы** | «осуществлять», «в настоящее время», «являться», «данный» | Заменить на простые слова: «делать», «сейчас», «быть», «этот» |
| **Вводные конструкции** | «Как известно», «Стоит отметить, что», «Не секрет, что» | Удалить — они ничего не добавляют |
| **Усилители-пустышки** | «Очень», «действительно», «абсолютно», «максимально» | Удалить или заменить на конкретику |
| **Тавтология** | «Самый лучший», «предварительный анонс», «свободная вакансия» | Убрать дублирование |
| **Отглагольные существительные** | «Осуществление процесса оптимизации» | → «Оптимизировать» |
| **Цепочки прилагательных** | «Инновационный, уникальный, качественный продукт» | Оставить 1 конкретное |

### Норма «воды»

| Платформа | Допустимая «вода» |
|-----------|------------------|
| Соцсети | ≤ 5% |
| Email / Telegram | ≤ 10% |
| Блог / статья | ≤ 15% |
| Академический текст | ≤ 20% |

## Оценка и порог

### Шкала оценки

| Оценка | Значение | Условие |
|--------|----------|---------|
| **A — Отлично** | Текст лёгкий, ясный, вовлекающий | Все метрики ✅, scanability ≥ 5/6, «вода» в норме |
| **B — Хорошо** | Текст читаемый, мелкие замечания | 1-2 метрики ⚠️, scanability ≥ 4/6 |
| **C — Удовлетворительно** | Текст тяжеловатый, нужна доработка | 3+ метрик ⚠️ или 1 ❌, scanability 3/6 |
| **D — Плохо** | Текст сложно читать | 2+ метрик ❌, scanability ≤ 2/6 |

### Порог решения

| Решение | Условие |
|---------|---------|
| **PASS** | Оценка A или B |
| **NEEDS IMPROVEMENT** | Оценка C или D |

## Протокол

### Шаг 1 — Подготовка
1. Получить текст, платформу, тип контента, персону.
2. Выбрать **адаптивные нормы** по платформе и аудитории.

### Шаг 2 — Метрики
1. Посчитать **все основные метрики** (средняя/макс. длина предложений, абзацы, активный залог).
2. Посчитать **индекс читаемости** (Flesch или средняя длина слова).
3. Сравнить с нормами — отметить ✅ / ⚠️ / ❌.

### Шаг 3 — Scanability
1. Пройти **чеклист из 6 пунктов**.
2. Подсчитать scanability score.

### Шаг 4 — «Вода»
1. Выявить **канцеляризмы, вводные, пустышки, тавтологию**.
2. Подсчитать **% «воды»**.
3. Сравнить с нормой платформы.

### Шаг 5 — Проблемные места
Для каждой проблемы:
1. **Цитата** — конкретное предложение/абзац.
2. **Проблема** — что не так (длинно / пассивный / жаргон / вода).
3. **Suggested fix** — конкретная переписанная версия.

### Шаг 6 — Оценка и решение
1. Определить оценку (A/B/C/D).
2. Применить порог (PASS / NEEDS IMPROVEMENT).
3. Если NEEDS IMPROVEMENT — приоритизировать исправления.

## Валидация (Quality Gate)

Оценка читаемости считается завершённой если:

- [ ] Все основные метрики посчитаны и сравнены с адаптивными нормами
- [ ] Индекс читаемости рассчитан
- [ ] Scanability чеклист пройден (6/6 пунктов)
- [ ] «Вода» проверена и посчитана в %
- [ ] Проблемные места выявлены с конкретными цитатами
- [ ] Для каждой проблемы — suggested fix (переписанная версия)
- [ ] Оценка (A-D) и решение (PASS/NEEDS IMPROVEMENT) соответствуют правилам

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Одни нормы для всех платформ | Пост Instagram ≠ лонгрид в блоге | Адаптивные нормы по платформам |
| «Текст сложный» без конкретики | Копирайтер не знает что исправлять | Конкретные цитаты + suggested fix |
| Упрощение до банальности | Эксперты чувствуют снисхождение | Адаптировать под уровень ЦА |
| Только длина предложений | Короткие предложения ≠ читаемый текст (может быть рубленый, сухой) | Баланс: метрики + scanability + ясность + стиль |
| Игнорирование «воды» | Текст формально читаемый, но пустой | Проверять % воды, удалять канцеляризмы |
| Нет проверки scanability | Текст «стена» — никто не дочитает | F-pattern, подзаголовки, буллеты, выделение |
| Один проход без итерации | Первое исправление может создать новые проблемы | После правок — повторная оценка |
