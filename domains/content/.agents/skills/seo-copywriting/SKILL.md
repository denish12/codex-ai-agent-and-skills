# SEO Copywriting — SEO-копирайтинг

## Когда использовать
- При создании **контента для блога, сайта, лендинга** — любого индексируемого материала.
- При **оптимизации существующего** — статья не ранжируется, нужна переработка.
- При **обновлении контента** — статья устарела, потеряла позиции.
- Как часть **`$content-brief`** — SEO-блок для текстовых форматов.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Основное ключевое слово | ✅ | Из keyword research или пользователя |
| Search volume | ✅ | Объём поиска/мес — определяет, стоит ли писать |
| Keyword Difficulty | ⬚ | Сложность (0-100). Влияет на шансы ранжирования |
| Поисковый интент | ✅ | Informational / Commercial / Transactional / Navigational |
| Персона ЦА | ✅ | Из `$audience-analysis` — для кого пишем |
| Текущий контент (если оптимизация) | ⬚ | URL, текущая позиция, текущий трафик |
| Конкуренты в SERP | ⬚ | Топ-3-5 URL из выдачи (если не предоставлены — агент анализирует) |

> Если ключевое слово не предоставлено — **запросить** или направить на keyword research. SEO без ключевого слова = текст в пустоту.

## SERP-анализ (обязательный шаг)

Перед написанием — проанализировать **топ-3-5 результатов** по целевому запросу:

| Параметр | Что смотреть | Зачем |
|----------|-------------|-------|
| **Объём** | Средний объём топ-5 (слов) | Ориентир для нашего текста (± 20%) |
| **Структура** | H2/H3 заголовки, секции | Какие темы покрывают |
| **Формат** | Listicle / how-to / guide / comparison | Какой формат Google предпочитает |
| **Featured Snippet** | Есть ли snippet? Какой тип? | Возможность его «перехватить» |
| **People Also Ask** | Вопросы в PAA | Дополнительные H2/FAQ для покрытия |
| **Gaps** | Что НЕ покрыто в топ-5 | Наше конкурентное преимущество |
| **E-E-A-T сигналы** | Автор, дата, источники, опыт | Что делают конкуренты для доверия |

> **Правило:** Наш контент должен быть **лучше** топ-3, а не копией. Лучше = полнее, свежее, с уникальным углом, лучше структурирован.

## E-E-A-T — Experience, Expertise, Authoritativeness, Trustworthiness

Google оценивает контент по E-E-A-T. Как встроить в текст:

| Сигнал | Как реализовать | Пример |
|--------|----------------|--------|
| **Experience** | Показать личный опыт, кейсы, first-hand данные | «Мы протестировали 15 инструментов за 3 месяца — вот результаты» |
| **Expertise** | Автор с квалификацией, глубина раскрытия темы | Биография автора, должность, ссылки на профили |
| **Authoritativeness** | Ссылки на авторитетные источники, цитаты экспертов | Ссылки на исследования, Tier 1-2 источники |
| **Trustworthiness** | Дата публикации/обновления, прозрачность, источники | «Обновлено: [дата]», ссылки на первоисточники |

### YMYL (Your Money Your Life)
Для тем здоровья, финансов, юридических — E-E-A-T **критически важен**:
- Автор должен быть квалифицированным специалистом.
- Все утверждения — с источниками.
- Дисклеймер обязателен.

## Поисковый интент — структура контента

| Intent | Что ищет пользователь | Оптимальный формат | CTA |
|--------|----------------------|-------------------|-----|
| **Informational** | Узнать, разобраться | How-to, guide, listicle, explainer | Подписка, скачивание, дочитывание |
| **Commercial** | Сравнить, выбрать | Comparison, review, «лучшие X» | Демо, trial, консультация |
| **Transactional** | Купить, оформить | Лендинг, product page, pricing | Покупка, оформление, заявка |
| **Navigational** | Найти конкретную страницу | About, pricing, login | Прямой переход |

> Intent определяет формат. Писать how-to guide под транзакционный запрос — мимо.

## Featured Snippets — оптимизация

| Тип snippet | Как попасть | Формат в тексте |
|-------------|------------|-----------------|
| **Paragraph** | Чёткий ответ в 40-60 слов сразу после H2 с вопросом | `## Что такое X?\n[Ответ 40-60 слов]` |
| **List** | Нумерованный или маркированный список | `## Как сделать X?\n1. Шаг 1\n2. Шаг 2` |
| **Table** | Структурированные данные в таблице | HTML `<table>` или Markdown-таблица |
| **Video** | YouTube видео с timestamp'ами | Встроить видео + добавить текстовый transcript |

### People Also Ask (PAA)
- Собрать 5-10 вопросов из PAA.
- Каждый вопрос → отдельный H2 или секция FAQ.
- Ответ — сразу под заголовком, 40-60 слов (формат paragraph snippet).

## On-Page SEO чеклист

### Мета-данные

| Элемент | Требование | Пример |
|---------|-----------|--------|
| **Title tag** | ≤ 60 символов. Ключевое слово в начале. Бренд в конце (опционально) | «SEO-копирайтинг: полный гайд 2026 — [Бренд]» |
| **Meta Description** | 120-155 символов. Ключевое слово + CTA + ценность | «Узнайте, как писать SEO-тексты, которые ранжируются в топ-3. Пошаговый гайд + чеклист.» |
| **URL slug** | Короткий, с ключевым словом, через дефисы | `/seo-copywriting-guide` |
| **H1** | Один на странице. Содержит ключевое слово. ≠ Title (можно длиннее) | «SEO-копирайтинг: пошаговый гайд по созданию текстов, которые ранжируются» |

### Ключевые слова

| Элемент | Рекомендация |
|---------|-------------|
| **Основное КС** | В Title, H1, первые 100 слов, 1-2 × в H2, заключение |
| **LSI-слова** (3-7) | Равномерно по тексту, в H2/H3 заголовках |
| **Плотность КС** | 1-2% для основного. Не форсировать — естественность важнее |
| **Synonym variations** | Использовать синонимы — Google понимает семантику |

### Структура контента

| Элемент | Требование |
|---------|-----------|
| **H2** | Каждые 200-300 слов. Информативные, содержат LSI-слова |
| **H3** | Подпункты внутри H2. Не пропускать уровни (нет H1 → H3) |
| **Абзацы** | ≤ 150 слов, 3-4 предложения |
| **Списки** | Для перечислений — нумерованные или маркированные |
| **Table of Contents** | Для статей > 1500 слов — оглавление с якорными ссылками |

## Internal Linking — стратегия

| Правило | Описание |
|---------|----------|
| **Количество** | 3-5 внутренних ссылок на 1000 слов |
| **Anchor text** | Описательный, с ключевыми словами целевой страницы. Не «нажмите здесь» |
| **Релевантность** | Ссылка должна быть полезна читателю, не просто для SEO |
| **Hub-and-spoke** | Pillar page (основная) → cluster pages (подтемы) → ссылки между собой |
| **Обновление старых** | При публикации нового контента — добавить ссылки на него из старых статей |

## Оптимизация изображений

| Элемент | Требование |
|---------|-----------|
| **Alt-текст** | Описательный, с ключевым словом (если естественно). ≤ 125 символов |
| **Имя файла** | `seo-copywriting-checklist.jpg`, не `IMG_2847.jpg` |
| **Формат** | WebP (лучшее сжатие) или JPG. PNG для графиков/инфографики |
| **Размер** | ≤ 200 KB для фото, ≤ 500 KB для инфографики |
| **Lazy loading** | `loading="lazy"` для изображений below the fold |
| **Responsive** | `srcset` для разных разрешений |

## Schema Markup — структурированные данные

| Тип | Когда использовать | Результат в SERP |
|-----|-------------------|-----------------|
| **Article** | Блог-посты, статьи | Дата, автор, thumbnail |
| **FAQ** | Страницы с вопросами-ответами | Раскрывающиеся вопросы под результатом |
| **HowTo** | Пошаговые инструкции | Шаги с изображениями |
| **Breadcrumb** | Все страницы | Навигация вместо URL |
| **Product** | Товарные страницы | Цена, рейтинг, наличие |
| **Review** | Обзоры | Звёзды рейтинга |

> Schema не влияет напрямую на ранжирование, но **увеличивает CTR** в SERP на 20-30%.

## Объём контента по интенту

| Intent | Рекомендуемый объём | Обоснование |
|--------|--------------------|----|
| **Informational (how-to)** | 1500-3000 слов | Глубокое раскрытие темы |
| **Informational (listicle)** | 1000-2500 слов | Зависит от количества пунктов |
| **Commercial (comparison)** | 2000-4000 слов | Детальное сравнение |
| **Transactional (landing)** | 500-1500 слов | Фокус на конверсии, не на длине |

> Ориентир: средний объём топ-5 ± 20%. Не раздувать ради длины — качество > количество.

## Протокол

### Шаг 1 — Keyword и Intent
1. Зафиксировать основное ключевое слово, search volume, KD.
2. Определить поисковый интент.
3. Подобрать 3-7 LSI-слов и синонимов.

### Шаг 2 — SERP-анализ
1. Проанализировать **топ-5 результатов** по 7 параметрам.
2. Определить формат, который Google предпочитает.
3. Собрать PAA-вопросы.
4. Найти gaps — что не покрыто конкурентами.

### Шаг 3 — Структура
1. Написать **outline**: H1 → H2 → H3.
2. Включить PAA-вопросы как H2/FAQ.
3. Определить объём (по SERP-анализу + интент).
4. Спланировать Featured Snippet блоки.

### Шаг 4 — Мета-данные
1. Title tag (≤ 60, КС в начале).
2. Meta Description (120-155, КС + CTA).
3. URL slug (короткий, с КС).
4. H1 (КС, ≠ Title).

### Шаг 5 — Написание текста
1. Основное КС в первых 100 словах.
2. LSI-слова в H2/H3 и по тексту.
3. Плотность 1-2%, естественно.
4. E-E-A-T сигналы: опыт, источники, дата, автор.
5. Featured Snippet блоки в формате paragraph/list/table.

### Шаг 6 — Internal Linking
1. 3-5 внутренних ссылок на 1000 слов.
2. Описательный anchor text.
3. Hub-and-spoke: связь с pillar page.

### Шаг 7 — Оптимизация изображений
1. Alt-текст с КС.
2. Описательные имена файлов.
3. WebP формат, ≤ 200 KB.
4. Lazy loading для below the fold.

### Шаг 8 — Schema Markup
1. Определить подходящий тип (Article, FAQ, HowTo).
2. Подготовить JSON-LD или рекомендации для разработчика.

### Шаг 9 — Технический чеклист
1. Mobile-friendly (responsive).
2. Page speed (Core Web Vitals: LCP < 2.5s, FID < 100ms, CLS < 0.1).
3. HTTPS.
4. Canonical URL.

## Валидация (Quality Gate)

SEO-оптимизация считается завершённой если:

- [ ] Основное КС и LSI определены, intent зафиксирован
- [ ] SERP-анализ проведён (топ-5 конкурентов)
- [ ] Title ≤ 60 символов, КС в начале
- [ ] Meta Description 120-155 символов, с CTA
- [ ] H1 содержит КС, один на странице
- [ ] КС в первых 100 словах
- [ ] Плотность КС 1-2%
- [ ] H2 каждые 200-300 слов, содержат LSI
- [ ] Internal links: 3-5 на 1000 слов, описательный anchor
- [ ] Alt-текст у всех изображений
- [ ] E-E-A-T сигналы присутствуют (автор, дата, источники)
- [ ] Featured Snippet оптимизация (если есть snippet в SERP)
- [ ] Schema markup определён
- [ ] Объём соответствует SERP-анализу (± 20% от среднего топ-5)

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Keyword stuffing | Google штрафует за переспам | Плотность 1-2%, естественное употребление |
| Писать для Google, не для людей | Низкий dwell time → потеря позиций | Сначала ценность для читателя, потом SEO |
| Игнорирование intent | How-to guide под транзакционный запрос — не ранжируется | Формат контента = формат, который Google показывает |
| Копирование структуры конкурентов | Нет уникальности, Google не будет показывать дубликат | Покрыть то же + добавить уникальный угол/данные |
| Тонкий контент (< 500 слов) | Недостаточно глубоко для ранжирования | Объём по SERP-анализу |
| Нет internal links | Страница изолирована, бот не находит | 3-5 ссылок на 1000 слов |
| Alt-текст = «image1» | Потерянная SEO-возможность + плохо для accessibility | Описательный alt с КС |
| Нет даты/автора | Низкий E-E-A-T, Google не доверяет | Автор с био + дата публикации/обновления |
| Старый контент без обновления | Теряет позиции со временем | Обновлять каждые 6-12 месяцев |
