# Competitor Content Analysis — Анализ контента конкурентов

## Когда использовать
- При **разработке контент-стратегии** — до `$content-calendar` и `$platform-strategy`.
- При **выходе на новую платформу** — понять, что работает у конкурентов на этом канале.
- При **падении метрик** — найти, что конкуренты делают лучше.
- При **квартальном/месячном пересмотре** — обновить картину конкурентного поля.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Бренд / продукт | ✅ | Чей контент анализируем в сравнении (наш бренд) |
| Ниша / индустрия | ✅ | Для поиска конкурентов, если не предоставлены |
| Список конкурентов | ⬚ | 3-5 конкурентов. Если не предоставлены — агент предлагает и согласовывает с пользователем |
| Платформы для анализа | ⬚ | Если не указаны — берутся из `$platform-strategy` или определяются по присутствию конкурентов |
| Период анализа | ⬚ | По умолчанию 30 дней. Адаптировать: сезонный бизнес → 90 дней, запуск → 14 дней |
| Существующая аналитика | ⬚ | Собственные метрики бренда для сравнения (ER, охват, частота) |

> Если конкуренты не предоставлены — агент предлагает список по критериям: **прямые** (тот же продукт, та же ЦА), **косвенные** (другой продукт, та же боль), **аспирационные** (лидеры ниши, на которых равняемся). Список **согласовать с пользователем** до начала анализа.

## Источники данных и методология

### Где собирать данные
1. **Соцсети напрямую** — профили конкурентов: лента, сторис, reels, highlights, закреплённые посты.
2. **Веб-поиск** — блоги конкурентов, гостевые публикации, PR, YouTube.
3. **Комментарии и отзывы** — что аудитория говорит под постами конкурентов.
4. **Инструменты** (если у пользователя есть доступ) — SimilarWeb, SEMrush, Social Blade, Brand24.
5. **Telegram-каналы** — контент, репосты, реакции.

### Формула ER (Engagement Rate)
Использовать единую формулу для корректного сравнения:

```
ER (%) = (Лайки + Комментарии + Репосты + Сохранения) / Подписчики × 100
```

- Если точные данные недоступны — использовать **оценку** с пометкой `≈` и указанием метода (визуальная оценка, выборка N постов).
- Для Telegram: `ER = Просмотры / Подписчики × 100` (реакции как бонус).
- Для YouTube: `ER = (Лайки + Комментарии) / Просмотры × 100`.

## Протокол

### Шаг 0 — Подготовка
1. Получить все обязательные входные данные.
2. Если конкуренты не указаны — предложить 3-5 по критериям (прямые / косвенные / аспирационные) и согласовать.
3. Определить период анализа (адаптировать под контекст).
4. Определить платформы для анализа.

### Шаг 1 — Профиль каждого конкурента
Для каждого конкурента собрать:
1. Платформы присутствия и размер аудитории на каждой.
2. Частота публикаций (публикаций/неделю по каждой платформе).
3. Топ-5 постов по вовлечённости за период — зафиксировать тему, формат, ER.
4. ToV и визуальный стиль — описание + 2-3 характерных примера.
5. ER — по формуле или оценка (указать метод).

### Шаг 2 — Контент-микс
Для каждого конкурента определить пропорции по типам контента:

| Тип | Описание |
|-----|----------|
| Образовательный | How-to, tips, разборы, гайды |
| Продающий | Офферы, кейсы, отзывы, демо |
| Развлекательный | Мемы, тренды, интерактив |
| Бренд / имиджевый | Миссия, команда, за кулисами, ценности |
| Комьюнити | UGC, вопросы, опросы, ответы |

Зафиксировать примерные % и какой тип даёт лучший ER.

### Шаг 3 — Анализ воронки
Оценить, как конкуренты покрывают этапы воронки контентом:

| Этап | Цель | Примеры контента |
|------|------|------------------|
| **TOFU** (Awareness) | Привлечь внимание | Вирусный контент, тренды, широкие темы |
| **MOFU** (Consideration) | Создать доверие | Кейсы, сравнения, экспертный контент |
| **BOFU** (Decision) | Конвертировать | Офферы, демо, отзывы, CTA |

Определить: какие этапы перенасыщены, какие — пустые (gap).

### Шаг 4 — Gaps и возможности
1. **Content gaps** — темы, которые ни один конкурент не освещает или освещает плохо.
2. **Format gaps** — форматы, которые не используют (подкасты, карусели, threads и т.д.).
3. **Funnel gaps** — этапы воронки без контента.
4. **Platform gaps** — платформы, где конкуренты отсутствуют, но ЦА есть.
5. **ToV gaps** — тональность, которую никто не занял (все серьёзные → возможность для юмора).

### Шаг 5 — Сравнительная матрица
Собрать side-by-side сравнение всех конкурентов + наш бренд (если данные есть):
- Аудитория, частота, ER, контент-микс, воронка, ToV.
- Визуально выделить, где наш бренд сильнее / слабее.

### Шаг 6 — Рекомендации по дифференциации
На основе gaps и матрицы — 3-5 конкретных actionable рекомендаций:
- Что делать (формат, тема, платформа).
- Почему это сработает (gap + данные).
- Приоритет (Высокий / Средний / Низкий).

## Валидация (Quality Gate)

Анализ считается завершённым если:

- [ ] Проанализировано 3-5 конкурентов (согласованных с пользователем)
- [ ] Для каждого конкурента: профиль, топ-5 постов, ER, ToV
- [ ] Контент-микс определён для каждого конкурента (% по типам)
- [ ] Воронка проанализирована (TOFU / MOFU / BOFU покрытие)
- [ ] Gaps выявлены минимум по 2 категориям (content, format, funnel, platform, ToV)
- [ ] Сравнительная матрица включает всех конкурентов
- [ ] Рекомендации конкретные и actionable (3-5 штук с приоритетами)
- [ ] Все ER рассчитаны по единой формуле или помечены как `≈ оценка`
- [ ] Нет confirmation bias — зафиксированы и сильные стороны конкурентов

## Handoff

Результат этого скила является входом для:

| Потребитель | Как использует |
|-------------|---------------|
| `$platform-strategy` | Gaps по платформам → выбор каналов |
| `$content-calendar` | Gaps по темам и форматам → планирование контента |
| `$audience-analysis` | Аудитория конкурентов → уточнение персон |
| `$tone-of-voice` | ToV-gaps → выбор незанятой тональности |
| `$content-brief` | Рекомендации → конкретные брифы на контент |

При передаче — использовать `$handoff` с приложением сравнительной матрицы и топ-3 рекомендаций.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Анализ без согласования списка конкурентов | Можно проанализировать нерелевантных | Согласовать с пользователем до начала |
| Confirmation bias | Искать только слабости конкурентов | Фиксировать и сильные стороны |
| Поверхностный скроллинг | «Они постят мемы» — не анализ | Топ-5 постов, ER, контент-микс в % |
| Сравнение разных периодов | Конкурент A за 30 дней vs B за 7 дней | Единый период для всех |
| ER без методики | «Вовлечённость высокая» — субъективно | Единая формула, числа, метод |
| Только один конкурент | Нет картины рынка | Минимум 3 для выявления паттернов |
| Рекомендации без данных | «Нужно больше reels» — а почему? | Каждая рекомендация привязана к конкретному gap |
