# Воркфлоу: /quick-post — Быстрый пайплайн создания поста

## Описание
Ускоренный пайплайн для создания простых публикаций без полного стратегического цикла. Минимум гейтов, максимум скорости — при гарантированном brand compliance.

## Когда использовать
- Простой пост с **blast radius ≈ 0** (см. чеклист ниже).
- Нет сложных фактических утверждений.
- Тема ясна, аудитория известна.
- Нужен результат быстро (дедлайн, инфоповод, тренд).
- Реакция на событие, которое теряет актуальность.

## Когда НЕ использовать
- Тема спорная, чувствительная или political → `/start-content`.
- Есть сложные фактические утверждения, требующие исследования → `/start-content`.
- Нужна стратегическая проработка (новая аудитория, новая платформа) → `/start-content`.
- Есть готовый текст, нужны правки → `/edit-content`.
- Пост > 30 минут работы → `/start-content`.

## Вход воркфлоу

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Тема / идея | ✅ | О чём пост — конкретно |
| Платформа | ✅ | Где публикуем |
| Бренд-гайдлайны | ✅ | Из `$brand-guidelines` — ToV, стоп-лист. Без них brand compliance невозможен |
| Цель | ⬚ | Engagement / awareness / traffic / мнение (если не указана — Conductor определяет) |
| Контекст | ⬚ | Тренд, инфоповод, дата — почему именно сейчас |
| Контент-календарь | ⬚ | Если пост из `$content-calendar` — ID единицы |

## Blast Radius чеклист

Conductor **обязан** пройти перед выбором `/quick-post`:

| # | Критерий | ✅ Quick OK | ❌ Не Quick |
|---|----------|:----------:|:-----------:|
| 1 | Тема **не** спорная / чувствительная / political | ✅ | → `/start-content` |
| 2 | **Нет** фактических утверждений, требующих проверки | ✅ | Если 1-2 простых факта — quick + `$fact-checking`. Если много → `/start-content` |
| 3 | Аудитория и платформа **известны** (не новые) | ✅ | → `/start-content` со Strategist |
| 4 | Пост **не** промо / оффер / реклама | ✅ | Промо требует более тщательной проработки |
| 5 | Blast radius при ошибке **минимален** | ✅ | Если ошибка → скандал/штраф → `/start-content` |
| 6 | Оценочное время ≤ 30 минут | ✅ | → `/start-content` или `/edit-content` |

**Результат:** Все ✅ → `/quick-post`. Хотя бы один ❌ → переключить режим.

## Типы quick-постов

| Тип | Описание | Шаблон структуры |
|-----|----------|-----------------|
| **Мнение / инсайт** | Личная позиция бренда по теме | Hook (провокация/вопрос) → Мнение (2-3 предложения) → CTA (вопрос к аудитории) |
| **Реакция на тренд** | Адаптация тренда под бренд | Тренд-формат + наш угол + CTA |
| **Реакция на инфоповод** | Комментарий к событию | Событие (1 предложение) → Наша позиция → CTA |
| **Цитата / факт дня** | Вдохновляющая цитата или интересный факт | Цитата/факт → Контекст (почему важно) → CTA |
| **Behind the scenes** | Закулисье бренда | Описание момента → Инсайт → CTA |
| **Вовлечение** | Опрос, вопрос, челлендж | Вопрос / условие → Варианты ответа / участия |
| **Репост с комментарием** | Шаринг чужого контента с добавленной ценностью | Наш комментарий (hook) → Ссылка/репост → CTA |

> Conductor определяет тип на Gate 1 → Copywriter использует соответствующий шаблон.

## Пайплайн

```
CONDUCTOR → COPYWRITER → REVIEWER → RELEASE CHECK (мини)
   G1            G2          G3           G4
```

### Правило параллелизма

| Режим | Когда | Как работает |
|-------|-------|-------------|
| **Последовательный** (по умолчанию) | Стандартный quick-post | Copywriter → Approved → Reviewer → Approved → Release |
| **Быстрый** (с разрешения пользователя) | Экстремально срочно, пользователь явно попросил | Copywriter → Reviewer **без паузы на Approved** между ними. Approved только после Reviewer |

> Быстрый режим = исключение из Правила 1. Только если пользователь **явно** запросил ускорение.

## Гейты и deliverables

### Gate 1: Conductor
**Вход:** Задача пользователя
**Действия:**
1. Пройти **Blast Radius чеклист** — подтвердить, что задача подходит для Quick Pipeline.
2. Определить **тип quick-поста** из таблицы.
3. Определить платформу и ключевые параметры (тема, ToV, CTA).
4. Сформулировать задачу для Copywriter в **3-5 предложениях**.
5. Инициализировать мини-доску (`$board` — Quick Pipeline).
**Обязательные скилы:** `$board`
**Deliverable:** Краткий бриф: тема + платформа + тип поста + ToV + CTA-цель
**Проверка:** Blast Radius чеклист пройден, тип определён
**→ Ждать "Approved"**

> **Автопереход:** если задача очевидна (пользователь дал тему + платформу) и пользователь явно сказал «быстро» — Conductor может предложить автопереход. Но решение за пользователем.

### Gate 2: Copywriter
**Вход:** Краткий бриф от Conductor
**Действия:**
1. Применить ToV (`$tone-of-voice`).
2. Выбрать шаблон структуры по **типу quick-поста**.
3. Написать пост под платформу (`$social-media-formats`).
4. Сформулировать hook / заголовок (`$headline-formulas`).
5. Добавить CTA (`$cta-optimization`).
6. Проверить лимиты платформы (символы, хэштеги).
7. Если есть фактические утверждения → `$fact-checking` (обязателен).
**Обязательные скилы:** `$tone-of-voice`, `$social-media-formats`, `$cta-optimization`
**Опциональные:** `$headline-formulas`, `$fact-checking` (если факты в тексте)
**Deliverable:** Готовый текст поста + хэштеги
**Проверка:** Текст полный, соответствует платформе, ToV, типу поста
**→ Ждать "Approved" (или immediate handoff в быстром режиме)**

### Gate 3: Reviewer
**Вход:** Текст от Copywriter
**Действия:**
1. **Быстрая проверка** по `$content-review-checklist` — **только Blockers и Critical**:
   - [ ] Тема соответствует задаче
   - [ ] CTA присутствует
   - [ ] ToV соответствует бренду
   - [ ] Нет запрещённых слов (стоп-лист)
   - [ ] Нет фактических ошибок (если есть факты)
   - [ ] Лимиты платформы соблюдены
2. Brand compliance (`$brand-compliance`) — **основные параметры** (не полный чеклист).
3. Platform compliance (`$platform-compliance`) — **технические требования**.
**Обязательные скилы:** `$brand-compliance`
**Опциональные:** `$platform-compliance`
**Deliverable:** APPROVED или **1-2 конкретные правки** (не списки, не развёрнутый чеклист)

**При APPROVED:** → Gate 4 (Release Check)
**При правках:** → Reverse handoff → Copywriter → исправление → повторный Gate 3

**Проверка:** Нет Blockers
**→ Ждать "Approved"**

### Gate 4: Release Check (мини)
**Вход:** APPROVED от Reviewer
**Действия:**
- Финальная сверка (30 секунд):
  - [ ] Текст = финальная версия (не черновик)
  - [ ] Хэштеги на месте
  - [ ] Время публикации оптимально (если важно)
  - [ ] Brand compliance ✅
- Обновить `$board` — все гейты `[✓]`
- Обновить `domains/content/CONTEXT.md` (краткая запись)
**Deliverable:** Пост готов к публикации
**→ Финальное "Approved" → Публикация немедленно**

## Reverse Handoff — протокол возврата

Если Reviewer дал **правки** (не APPROVED):

```
Reviewer (1-2 правки)
    │
    └── Reverse $handoff → Copywriter
        └── Copywriter вносит правки (< 5 мин)
            └── Forward $handoff → Reviewer (повторный G3)
```

**Правила для Quick Pipeline:**
1. Reviewer даёт **максимум 1-2 правки** — не развёрнутый чеклист. Quick = быстро.
2. Если правок > 3 → задача **не quick** → предложить `/edit-content`.
3. Максимум **2 итерации** (Copywriter → Reviewer → Copywriter → Reviewer). Если не APPROVED → эскалация пользователю.
4. Обновить `$board` при каждом возврате.

## Время выполнения

| Этап | Целевое время | Максимум |
|------|--------------|----------|
| Gate 1: Conductor | 1-2 мин | 5 мин |
| Gate 2: Copywriter | 3-8 мин | 15 мин |
| Gate 3: Reviewer | 1-3 мин | 5 мин |
| Gate 4: Release Check | 30 сек | 1 мин |
| **Итого** | **5-15 мин** | **30 мин** |

> Если > 30 минут → остановиться, предложить `/start-content` или `/edit-content`.

## Ограничения режима

- ❌ Не проводить исследование (`$trend-research`, `$topic-research`, `$source-verification`).
- ❌ Не создавать визуальный бриф (только текст). Если нужен визуал → `/start-content` или `/edit-content`.
- ❌ Не пересматривать стратегию (аудитория, платформа).
- ❌ Не запускать `$moodboard`, `$image-prompt-engineering`.
- ❌ Reviewer не делает полный `$content-review-checklist` — только Blockers.
- ✅ Если в тексте есть фактические утверждения → `$fact-checking` **обязателен** даже в Quick.
- ✅ `$brand-compliance` обязателен **всегда** (Правило 7 из pipeline-rules).

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Quick для спорной темы | Скандал, негатив, репутационные потери | Blast Radius чеклист. Спорное → `/start-content` |
| Quick для промо / оффера | Нет проработки CTA, оффера, воронки | Промо → `/start-content` или `/edit-content` |
| Пропустить brand compliance | «Это же quick» — но бренд нарушен | Brand compliance обязателен ВСЕГДА. Правило 7 |
| Факты без проверки | «Это же quick» — но факт неверный → потеря доверия | Факты → `$fact-checking` даже в Quick |
| Quick затянулся > 30 мин | Это уже не quick — scope creep | Остановиться, переключить режим |
| Reviewer пишет 10 замечаний | Это не quick review — задача слишком сложная | Максимум 1-2 правки. Если > 3 → `/edit-content` |
| Нет хэштегов / CTA | «Быстро = можно без CTA» — нет | CTA обязателен. Хэштеги по правилам платформы |
| Quick без `$board` | Потеря трекинга | Мини-доска обязательна |
