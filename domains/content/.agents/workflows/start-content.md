# Воркфлоу: /start-content — Полный пайплайн создания контента

## Описание
Полный пайплайн для создания контента с нуля. Максимальная проработка: стратегия, исследование, копирайтинг, визуал, ревью, release gate. Используется для новых публикаций, кампаний и контента, требующего глубокой подготовки.

## Когда использовать
- Новая публикация или кампания с нуля.
- Контент требует исследования данных / фактической базы.
- Нужна стратегическая проработка (аудитория, платформа, цели).
- Нет готового текста или концепции.
- Правки существующего контента > 50% (переключение из `/edit-content`).

## Когда НЕ использовать
- Есть готовый текст, нужны правки ≤ 50% → `/edit-content`.
- Простой пост, blast radius ≈ 0 → `/quick-post`.
- При сомнении — **использовать** `/start-content` (лучше перестраховаться).

## Вход воркфлоу

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Задача / тема | ✅ | Что нужно создать — от пользователя |
| Бренд-гайдлайны | ✅ | Из `$brand-guidelines`. Без них — запросить перед стартом |
| Бизнес-цели | ⬚ | Awareness / лиды / продажи / комьюнити |
| Целевая платформа | ⬚ | Если известна заранее. Если нет — определит Strategist |
| Персоны ЦА | ⬚ | Если уже есть `$audience-analysis`. Если нет — создаст Strategist |
| Бюджет | ⬚ | Если есть paid-компонент |
| Дедлайн | ⬚ | Влияет на приоритизацию и глубину исследования |

> Если бренд-гайдлайны не предоставлены — Conductor запрашивает **перед стартом** пайплайна. Без них brand compliance (Правило 7) невозможен.

## Пайплайн

```
CONDUCTOR → STRATEGIST → RESEARCHER → COPYWRITER → VISUAL CONCEPT → REVIEWER → RELEASE GATE
   G1            G2            G3           G4            G5             G6          G7
```

### Параллелизм

| Гейты | Параллельно? | Условие |
|-------|:------------:|---------|
| G1→G2→G3 | ❌ Строго последовательно | Каждый зависит от предыдущего |
| **G4 + G5** | ✅ Возможно | Если визуал создаётся по брифу (не по тексту). Conductor решает |
| G4→G5 | По умолчанию последовательно | Visual Concept часто зависит от финального текста |
| G6 | ❌ Только после G4+G5 | Reviewer проверяет и текст, и визуал |
| G7 | ❌ Только после G6 APPROVED | Release Gate = финальная проверка |

> Conductor на Gate 1 определяет, будут ли G4+G5 параллельными, и сообщает пользователю.

## Гейты и deliverables

### Gate 1: Conductor
**Вход:** Задача пользователя + вход воркфлоу
**Действия:**
1. Определить тип контента, платформу (если известна), scope.
2. Проверить наличие `$brand-guidelines`. Если нет → запросить.
3. Определить будут ли G4+G5 параллельными.
4. Инициализировать доску (`$board` — Full Pipeline).
5. Сформулировать задачу для Strategist.
6. Выполнить `$handoff` (forward) → Strategist.
**Обязательные скилы:** `$board`, `$handoff`
**Deliverable:** Handoff Envelope → Strategist (задача + scope + ограничения)
**Проверка:** `$gates` — задача конкретна, board создан, brand guidelines есть
**→ Ждать "Approved"**

### Gate 2: Strategist
**Вход:** `$handoff` от Conductor
**Действия:**
1. Анализ аудитории (`$audience-analysis`) — персоны, боли, триггеры.
2. Выбор платформ (`$platform-strategy`) — каналы, форматы, частота.
3. Анализ конкурентов (`$competitor-content-analysis`) — gaps, дифференциация.
4. Контент-план (`$content-calendar`) — если планируется серия / кампания.
5. Формирование ToV (`$tone-of-voice`) — если не определён ранее.
6. Формирование брифа (`$content-brief`) — полное ТЗ для Researcher + Copywriter.
7. Выполнить `$handoff` (forward) → Researcher.
**Обязательные скилы:** `$audience-analysis`, `$platform-strategy`, `$content-brief`
**Опциональные:** `$competitor-content-analysis`, `$content-calendar`, `$tone-of-voice`
**Deliverable:** Контент-бриф + контент-план (если серия)
**Проверка:** `$gates` — бриф содержит: тему, персону, платформу, формат, ToV, CTA, объём
**→ Ждать "Approved"**

### Gate 3: Researcher
**Вход:** `$handoff` от Strategist (бриф)
**Действия:**
1. Анализ трендов (`$trend-research`) — что актуально по теме.
2. Исследование темы (`$topic-research`) — факты, статистика, эксперты.
3. Верификация источников (`$source-verification`) — все источники проверены.
4. Формирование нарратива (`$data-storytelling`) — если data-driven контент.
5. Выполнить `$handoff` (forward) → Copywriter.
**Обязательные скилы:** `$topic-research`, `$source-verification`
**Опциональные:** `$trend-research`, `$data-storytelling`
**Deliverable:** Пакет данных — факты (верифицированные), статистика, цитаты, нарратив, структура для копирайтера
**Проверка:** `$gates` — все факты верифицированы, источники ≥ минимального рейтинга, нарратив сформирован
**→ Ждать "Approved"**

### Gate 4: Copywriter
**Вход:** `$handoff` от Researcher (данные) + бриф от Strategist (G2)
**Действия:**
1. Применение ToV (`$tone-of-voice`).
2. Выбор storytelling framework (`$storytelling-framework`).
3. Написание заголовков (`$headline-formulas`) — 3-5 вариантов.
4. Написание текста по брифу + данным.
5. Адаптация под платформу (`$social-media-formats` / `$email-copywriting`).
6. Оптимизация CTA (`$cta-optimization`).
7. SEO-оптимизация (`$seo-copywriting`) — если индексируемый контент.
8. Выполнить `$handoff` (forward) → Visual Concept (или Reviewer, если визуал параллельно).
**Обязательные скилы:** `$tone-of-voice`, `$headline-formulas`, `$cta-optimization`
**Опциональные:** `$storytelling-framework`, `$social-media-formats`, `$email-copywriting`, `$seo-copywriting`
**Deliverable:** Финальный текст + варианты заголовков + CTA
**Проверка:** `$gates` — текст соответствует брифу (тема, тезисы, персона), ToV, объёму, платформе
**→ Ждать "Approved"**

### Gate 5: Visual Concept
**Вход:** `$handoff` от Copywriter (текст) или от Conductor (бриф, если параллельно с G4)
**Действия:**
1. Применение бренд-гайдлайнов (`$brand-guidelines`).
2. Создание мудборда (`$moodboard`) — если новая кампания / серия.
3. Визуальный бриф (`$visual-brief`) — детальное ТЗ.
4. Спеки платформы (`$platform-visual-specs`) — размеры, ratio, safe zones.
5. AI-промпты (`$image-prompt-engineering`) — если AI-генерация.
6. Выполнить `$handoff` (forward) → Reviewer.
**Обязательные скилы:** `$visual-brief`, `$platform-visual-specs`
**Опциональные:** `$moodboard`, `$image-prompt-engineering`, `$brand-guidelines`
**Deliverable:** Визуальный бриф + AI-промпты (или ТЗ для дизайнера)
**Проверка:** `$gates` — бриф полный, спеки соответствуют платформе, стиль = бренд
**→ Ждать "Approved"**

### Gate 6: Reviewer
**Вход:** `$handoff` от Visual Concept (текст + визуал)
**Действия:**
1. Чеклист ревью (`$content-review-checklist`) — полный, 38 пунктов.
2. Бренд-compliance (`$brand-compliance`) — ToV, визуал, messaging, юридика.
3. Факт-чекинг (`$fact-checking`) — все фактические утверждения.
4. Читаемость (`$readability-scoring`) — если текстовый формат.
5. Платформа-compliance (`$platform-compliance`) — технические + правила + алгоритмические.

**При APPROVED:** → `$handoff` (forward) → Release Gate.
**При REQUIRES CHANGES:** → Reverse `$handoff` → Copywriter / Visual Concept (по типу замечаний).

**Обязательные скилы:** `$content-review-checklist`, `$brand-compliance`
**Опциональные:** `$fact-checking`, `$readability-scoring`, `$platform-compliance`
**Deliverable:** APPROVED или список правок с severity (Blocker / Major / Minor)
**Проверка:** `$gates` — все Blockers закрыты, Majors ≤ 2 (исправлены или приняты)
**→ Ждать "Approved"**

### Gate 7: Release Gate
**Вход:** APPROVED от Reviewer
**Действия:**
1. Финальный чеклист (`$content-release-gate`) — артефакты, мета, юридика.
2. Подготовка пакета для публикации: текст + визуал + хэштеги + ссылки + UTM + расписание.
3. Обновить `$board` — все гейты `[✓]`.
4. Обновить `domains/content/CONTEXT.md`.
**Обязательные скилы:** `$content-release-gate`
**Deliverable:** Полный пакет для публикации

**При READY TO PUBLISH:** → Финальное "Approved" → Публикация.
**При HOLD:** → Reverse `$handoff` по типу блокера (см. `$content-release-gate` rollback).

**→ Финальное "Approved" → Публикация**

## Reverse Handoff — протокол возврата

### Из Gate 6 (Reviewer)

```
Reviewer (REQUIRES CHANGES)
    │
    ├── Текстовые замечания → Reverse $handoff → Copywriter (G4)
    │   └── Copywriter исправляет → Forward $handoff → Reviewer (повторный G6)
    │
    ├── Визуальные замечания → Reverse $handoff → Visual Concept (G5)
    │   └── Visual Concept исправляет → Forward $handoff → Reviewer (повторный G6)
    │
    ├── Фактические ошибки → Reverse $handoff → Researcher (G3)
    │   └── Researcher исправляет/заменяет → Forward → Copywriter → Reviewer
    │
    └── Оба (текст + визуал) → Reverse $handoff → Copywriter + Visual Concept → Reviewer
```

### Из Gate 7 (Release Gate)

```
Release Gate (HOLD)
    │
    ├── Текст → Copywriter → Reviewer → Release Gate (повторный)
    ├── Визуал → Visual Concept → Reviewer → Release Gate (повторный)
    ├── Юридика → Эскалация пользователю
    └── Мета / ссылки → Conductor исправляет → Release Gate (повторный)
```

### Правила reverse handoff
1. Reverse `$handoff` содержит: **конкретные замечания** + **severity** + **что НЕ трогать**.
2. Обновить `$board`: текущий гейт → `[↩] Возврат`, целевой → `[→] В работе`.
3. После исправлений — **повторная** `$gates` проверка → forward `$handoff`.
4. Максимум **3 итерации** на гейт Reviewer. Если после 3 не APPROVED → эскалация пользователю.

## Эскалация

| Ситуация | Действие | Кто |
|----------|----------|-----|
| Блокер на любом гейте | `$board` → `[!]`, уведомить пользователя, не переходить | Текущий агент |
| Бренд-гайдлайны не определены | Остановить пайплайн, запросить у пользователя | Conductor (G1) |
| Недостаточно данных для Researcher | Reverse → Strategist для уточнения брифа, или запросить у пользователя | Researcher (G3) |
| Правки > 3 итераций на Reviewer | Эскалация пользователю — возможно, проблема в брифе | Conductor |
| Пользователь не отвечает | Ждать. Не переходить без Approved. Напомнить | Conductor |
| Дедлайн давит | Предложить сократить scope или переключить на `/quick-post` для части контента | Conductor |
| Конфликт между гейтами | Эскалировать пользователю с аргументами обеих сторон | Conductor |
| Задача оказалась слишком простой | Предложить переключиться на `/quick-post`. Решение за пользователем | Conductor (G1-G2) |

## Порог переключения на другой режим

| Индикатор | Направление | Решение |
|-----------|-------------|---------|
| Задача тривиальна (тема ясна, нет фактов, blast radius ≈ 0) | → `/quick-post` | Conductor предлагает на G1 |
| Strategist определил, что стратегия уже есть, нужен только текст | → `/edit-content` (если есть черновик) или продолжить без G2 | Conductor предлагает, пользователь решает |
| В процессе G3-G4 стало ясно, что задача проще, чем казалось | → Пропустить нельзя (Правило 2), но сократить глубину гейтов | Conductor |

> Переключение режима — **всегда решение пользователя**. Conductor предлагает, не переключает сам.

## Оркестрационные скилы (используются на каждом гейте)

| Скил | Когда | Описание |
|------|-------|----------|
| `$board` | G1 (создание), каждый переход (обновление), G7 (закрытие) | Единственный источник правды о статусе |
| `$gates` | Перед каждым переходом | Проверка deliverable: PASS / CONDITIONAL PASS / FAIL |
| `$handoff` | При каждом переходе (forward и reverse) | Передача артефактов + контекста + задачи |

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Пропустить Strategist «тема и так понятна» | Нет брифа → Copywriter работает вслепую | Правило 2: не пропускать гейты |
| Researcher без верификации | Непроверенные факты в тексте → потеря доверия | `$source-verification` обязателен |
| Copywriter игнорирует бриф | Текст не соответствует стратегии | `$gates` проверяет соответствие брифу |
| Visual Concept без brand guidelines | Визуал не узнаваем как бренд | `$brand-guidelines` обязателен |
| Reviewer проверяет только текст | Визуал с AI-артефактами проходит | Reviewer проверяет и текст, и визуал |
| Release Gate «и так всё ок» | Битые ссылки, нет UTM, нет disclaimer | `$content-release-gate` — полный чеклист |
| Параллельные G4+G5 когда визуал зависит от текста | Visual Concept переделывает после получения текста | Параллельно только если визуал по брифу, не по тексту |
| 5+ итераций на Reviewer | Бесконечный цикл, время тратится | Max 3 итерации → эскалация |
| Conductor переписывает текст вместо Copywriter | Нарушение ролей, нет качественного ревью | Правило 6: правки к исполнителю |
