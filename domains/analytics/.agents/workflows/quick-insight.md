# Воркфлоу: /quick-insight — Быстрый аналитический пайплайн (1 сессия)

## Описание
Упрощённый аналитический пайплайн в одной сессии. Только команда Аналитиков (без Критиков), Медиатор формирует заключение без состязательного анализа, верстальщики готовят PDF-отчёт. Подходит для простых вопросов и быстрых проверок.

## Когда использовать
- Простой аналитический вопрос, не требующий devil's advocate.
- Нужен быстрый ответ с визуализацией.
- Один фреймворк или ограниченный набор фреймворков.
- Предварительная проверка гипотезы перед полным `/analyze`.

## Когда НЕ использовать
- Сложный стратегический вопрос с высокими ставками → `/analyze`.
- Нужна проверка аргументов второй командой → `/analyze`.
- Решение влияет на критические бизнес-процессы → `/analyze`.
- При сомнении → `/analyze`.

## Вход воркфлоу

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Вопрос / тема | ✅ | Что нужно проанализировать |
| Данные пользователя | ⬚ | Имеющиеся данные, метрики |
| Отрасль / рынок | ⬚ | Если известно |

## Пайплайн (1 сессия)

```
CONDUCTOR → INTERVIEWER → RESEARCHER(α) → DATA_ANALYST(α) → STRATEGIST(α) → MEDIATOR → DESIGNER → LAYOUTER → RELEASE GATE
    G1           G2            G3               G4                G5             G6          G7         G8         G9
```

### Параллелизм

| Гейты | Параллельно? | Причина |
|-------|:------------:|---------|
| G1→G9 | ❌ Строго последовательно | Каждый зависит от предыдущего |

## Гейты и deliverables

### Gate 1: Conductor
**Действия:**
1. Подтвердить режим `/quick-insight`.
2. Инициализировать доску (`$board` — Quick Pipeline).
3. `$handoff` → Interviewer.
**Обязательные скилы:** `$board`, `$handoff`
**→ Ждать "Approved"**

### Gate 2: Interviewer
**Действия:**
1. Краткое интервью (3-5 вопросов — адаптивно, но быстрее чем в Full).
2. Определить вопрос, scope, метод оценки.
3. Сформировать Research Brief.
4. `$handoff` → Researcher.
**Deliverable:** Research Brief (краткий)
**→ Ждать "Approved"**

### Gate 3: Researcher (Alpha)
**Действия:**
1. Сбор фактов (`$web-research` — обязательно).
2. Верификация данных пользователя.
3. Формирование Research Package.
4. `$handoff` → Data Analyst.
**Обязательные скилы:** `$web-research`
**Deliverable:** Research Package
**→ Ждать "Approved"**

### Gate 4: Data Analyst (Alpha)
**Действия:**
1. Выбор 1-3 релевантных фреймворков.
2. Применение фреймворков к данным.
3. Формирование выводов.
4. `$handoff` → Strategist.
**Deliverable:** Analytical Report (compact)
**→ Ждать "Approved"**

### Gate 5: Strategist (Alpha)
**Действия:**
1. Синтез в стратегические рекомендации.
2. Краткая оценка рисков.
3. Plan действий (top-3 приоритета).
4. `$handoff` → Mediator.
**Deliverable:** Strategy Report (compact)
**→ Ждать "Approved"**

### Gate 6: Mediator
**Действия:**
1. Оценка качества анализа (без сравнения — только одна команда).
2. Формирование заключения и итоговых рекомендаций.
3. Проверка логической связности: данные → аналитика → стратегия.
4. `$handoff` → Designer.
**Deliverable:** Quick Conclusion (заключение + рекомендации)
**→ Ждать "Approved"**

### Gate 7: Designer
**Действия:**
1. Проектирование упрощённого отчёта (`$report-design`).
2. Выбор визуализаций.
3. Executive summary.
4. `$handoff` → Layouter.
**Deliverable:** Report Design Spec (compact)
**→ Ждать "Approved"**

### Gate 8: Layouter
**Действия:**
1. HTML-файл с inline CSS (`$html-pdf-report`).
2. Графики и таблицы.
3. Упрощённая структура (без Appendix B — нет Критиков).
4. `$handoff` → Conductor.
**Deliverable:** HTML файл (→ Print-to-PDF)
**→ Ждать "Approved"**

### Gate 9: Release Gate
**Проверка:**
- [ ] Все гейты `[✓]`
- [ ] PDF содержит: Title, TOC, Main Section, Appendix A (Analytics), Appendix C (Mediator reasoning)
- [ ] Данные верифицированы
- [ ] Визуализации корректны

**Решение:** GO ✅ / NO-GO ❌

## Структура Quick PDF-отчёта

```
Титульная страница
Оглавление
Основной раздел (Заключение Медиатора)
  ├─ Executive Summary
  ├─ Ключевые выводы
  ├─ Визуализации
  └─ Рекомендации
Апендикс А — Отчёт команды Аналитиков
Апендикс В — Рассуждения Медиатора
```

> Апендикс Б (Критики) и Апендикс Г (Синтез) отсутствуют в Quick Pipeline.
