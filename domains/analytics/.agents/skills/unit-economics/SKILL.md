# Юнит-экономика — Анализ экономики единицы

## Когда использовать
- При оценке жизнеспособности бизнес-модели — сходится ли экономика на единицу.
- При планировании роста — при каких условиях масштабирование выгодно.
- При привлечении инвестиций — инвесторы требуют unit economics.
- При оптимизации маркетинга — оценка эффективности каналов привлечения.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Бизнес-модель | ✅ | SaaS / E-commerce / Marketplace / Сервис |
| Ценовая модель | ✅ | Подписка / транзакция / лицензия / freemium |
| Средний чек (ACV/ARPU) | ✅ | Средний доход на клиента за период |
| Маркетинговые расходы | ⬚ | Бюджет по каналам (если известен) |
| Количество клиентов | ⬚ | Текущая база или прогноз |
| Churn rate | ⬚ | Процент оттока (если известен) |
| Gross margin | ⬚ | Валовая маржа продукта |
| Каналы привлечения | ⬚ | Какие каналы используются |
| Исторические данные | ⬚ | Метрики за прошлые периоды |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

> Если реальных данных нет — скил работает с **моделированными данными** на основе отраслевых бенчмарков, явно помеченными. Методология остаётся применимой при появлении реальных данных.

## Источники данных
1. **Веб-поиск** — отраслевые бенчмарки (SaaS Capital, OpenView, ProfitWell, Baremetrics).
2. **Финансовые данные** — публичная отчётность аналогичных компаний.
3. **Отраслевые отчёты** — средние CAC, LTV, churn по вертикалям.
4. **Данные от пользователя** — реальные метрики из CRM, аналитики, бухгалтерии.
5. **Маркетинговые платформы** — стоимость привлечения по каналам.

> Для бенчмарков — указать источник, год и применимость к данному бизнесу. Данные старше 12 месяцев — пометить ⚠️.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `cohort-analysis` | Когортный LTV, retention curves для точного расчёта LTV (шаг 3) | Если есть данные по когортам — предпочтительный метод LTV |
| `rfm-analysis` | LTV per segment (Champions vs At Risk) для дифференцированной экономики (шаг 3) | Если нужна unit economics по сегментам |

## Протокол

### Шаг 0 — Сбор контекста
1. Проверить наличие обязательных данных.
2. Определить тип бизнес-модели и «единицу» анализа:

   | Бизнес-модель | Единица | ARPU = | Churn = | Типичный цикл |
   |---------------|---------|--------|---------|----------------|
   | SaaS (подписка) | Аккаунт / seat | MRR per account | % аккаунтов, отменивших подписку | Месяц |
   | E-commerce | Клиент | Средний чек × частота | % не вернувшихся за 12 мес | Заказ |
   | Marketplace | Транзакция | Take rate × GMV | % продавцов/покупателей ушедших | Транзакция |
   | Сервис | Клиент | Средний чек × визиты | % не вернувшихся | Визит |

3. Выбрать валюту и период.

### Шаг 1 — Revenue Model (поток выручки)

Визуализировать поток unit economics:

```
Revenue ($ARPU/мес)
  └─ COGS (-$XX) ──────────────────── = Gross Margin ($XX, XX%)
       └─ Variable Costs (-$XX) ───── = Contribution Margin ($XX, XX%)
            └─ Allocated Fixed (-$XX)  = Net Margin ($XX, XX%)
```

| Компонент | Значение | % от Revenue | Описание |
|-----------|:--------:|:------------:|----------|
| Revenue (ARPU) | $XX/мес | 100% | [Источник дохода] |
| COGS | -$XX | XX% | [Хостинг, поддержка, инфраструктура] |
| **Gross Margin** | **$XX** | **XX%** | Revenue - COGS |
| Variable Costs | -$XX | XX% | [Onboarding, транзакционные] |
| **Contribution Margin** | **$XX** | **XX%** | Gross Margin - Variable |
| Allocated Fixed | -$XX | XX% | [R&D, G&A, аллоцированные на единицу] |
| **Net Margin** | **$XX** | **XX%** | Contribution - Fixed |

### Шаг 2 — CAC (Customer Acquisition Cost)

**Blended CAC:**
`CAC = (Маркетинг + Продажи + Инструменты) / Кол-во новых клиентов`

**CAC по каналам:**

| Канал | Бюджет | Клиенты | CAC | % от total | LTV/CAC канала | Вердикт |
|-------|:------:|:-------:|:---:|:----------:|:--------------:|:-------:|
| Paid (Google/FB) | $XX | N | $XX | XX% | X.Xx | Масштабировать / Оптимизировать / Отключить |
| Content/SEO | $XX | N | $XX | XX% | X.Xx | ... |
| Sales (outbound) | $XX | N | $XX | XX% | X.Xx | ... |
| Referral | $XX | N | $XX | XX% | X.Xx | ... |
| Organic | $0 | N | $0 | XX% | ∞ | Максимизировать |
| **Blended** | **$XX** | **N** | **$XX** | **100%** | **X.Xx** | |

Динамика CAC: растёт ↑ / стабилен → / снижается ↓ (по кварталам).

### Шаг 3 — LTV (Lifetime Value)

Рассчитать **минимум двумя методами** для перекрёстной проверки:

| Метод | Формула | Результат | Когда использовать | Допущения |
|-------|---------|:---------:|---------------------|-----------|
| **Простой** | `ARPU × Gross Margin × (1 / Churn)` | $XX | Нет когортных данных | Churn постоянен |
| **Когортный** | Σ(фактическая выручка когорты) / размер когорты | $XX | Есть данные `$cohort-analysis` | Зрелые когорты (≥12 мес) |
| **DCF** | Σ(ARPU × GM × retention_rate_month_i / (1+r)^i) | $XX | Для инвесторов, точный расчёт | Ставка дисконтирования |

**Сверка:** если разброс > 30% — зафиксировать причину, обосновать выбор финального значения.

> Если есть `$rfm-analysis` — рассчитать LTV per segment:

| Сегмент | ARPU | Churn | LTV | % базы | % revenue |
|---------|:----:|:-----:|:---:|:------:|:---------:|
| Champions | $XX | X% | $XX | X% | X% |
| Loyal | $XX | X% | $XX | X% | X% |
| At Risk | $XX | X% | $XX | X% | X% |
| **Blended** | **$XX** | **X%** | **$XX** | **100%** | **100%** |

### Шаг 4 — LTV/CAC Ratio

| LTV/CAC | Интерпретация | Действие |
|:-------:|---------------|----------|
| < 1.0 | 🔴 Убыток на каждом клиенте | Срочно: снизить CAC или увеличить LTV |
| 1.0-3.0 | 🟡 На грани окупаемости | Оптимизация: churn, pricing, каналы |
| 3.0-5.0 | 🟢 Здоровая экономика | Масштабировать, инвестировать в рост |
| > 5.0 | 🔵 Возможно, недоинвестируем | Увеличить marketing spend, тестировать новые каналы |

Сравнить с бенчмарками по отрасли и стадии компании.

### Шаг 5 — Payback Period

`Payback = CAC / (ARPU × Gross Margin) = X месяцев`

| Бизнес-модель | Здоровый Payback | Проблемный |
|---------------|:----------------:|:----------:|
| SaaS (SMB) | < 12 мес | > 18 мес |
| SaaS (Enterprise) | < 18 мес | > 24 мес |
| E-commerce | < 3 мес | > 6 мес |
| Marketplace | < 6 мес | > 12 мес |

Влияние на cash flow: `Cash needed = CAC × New Customers/мес × Payback (мес)`.

### Шаг 6 — Break-Even Analysis

Определить точку безубыточности:

**На уровне unit:**
`Break-even units = Fixed Costs / Contribution Margin per unit`

**На уровне бизнеса:**

| Метрика | Формула | Значение |
|---------|---------|:--------:|
| Fixed Costs (мес) | [R&D + G&A + Infra] | $XX |
| Contribution Margin per unit (мес) | [ARPU × GM - Variable] | $XX |
| **Break-even clients** | Fixed / CM per unit | **N клиентов** |
| Текущая база | | N клиентов |
| Gap до break-even | | ±N клиентов |
| При текущем росте — break-even через | | X мес |

### Шаг 7 — Sensitivity Analysis

| Параметр | Базовый | Оптимист. (+) | Пессимист. (-) |
|----------|:-------:|:-------------:|:--------------:|
| ARPU | $XX | $XX (+20%) | $XX (-20%) |
| Churn | X% | X% (-30%) | X% (+30%) |
| CAC | $XX | $XX (-25%) | $XX (+25%) |

| Сценарий | ARPU | Churn | CAC | LTV | LTV/CAC | Payback | Break-even |
|----------|:----:|:-----:|:---:|:---:|:-------:|:-------:|:----------:|
| Оптимистичный | $XX | X% | $XX | $XX | X.Xx | X мес | X мес |
| **Базовый** | **$XX** | **X%** | **$XX** | **$XX** | **X.Xx** | **X мес** | **X мес** |

**Kill-условие:** при каком сценарии LTV/CAC < 1.0 → модель нежизнеспособна.

### Шаг 8 — Бенчмарк и рекомендации

Сравнить все метрики с отраслевыми бенчмарками:

| Метрика | Наше | Бенчмарк (отрасль) | Δ | Оценка |

## Пример — B2B SaaS: HR-платформа, подписка per seat

**Контекст:** HR-платформа автоматизации рекрутинга, B2B SaaS, подписка $500/мес per company (средне 10 seats × $50). 200 клиентов, РФ.

### Revenue Model

| Компонент | Значение | % | Описание |
|-----------|:--------:|:-:|----------|
| Revenue (ARPU) | $500/мес | 100% | 10 seats × $50 |
| COGS | -$75 | 15% | Хостинг, API HH.ru, поддержка L1 |
| **Gross Margin** | **$425** | **85%** | |
| Variable Costs | -$50 | 10% | Onboarding нового клиента (аморт.) |
| **Contribution Margin** | **$375** | **75%** | |

### CAC по каналам

| Канал | Бюджет/мес | Клиенты/мес | CAC | LTV/CAC | Вердикт |
|-------|:----------:|:-----------:|:---:|:-------:|---------|
| Google Ads | $8K | 5 | $1,600 | 3.5x | 🟢 Масштабировать |
| LinkedIn Ads | $5K | 2 | $2,500 | 2.2x | 🟡 Оптимизировать |
| Outbound Sales | $12K | 4 | $3,000 | 1.9x | 🟡 Оптимизировать |
| Content/SEO | $3K | 3 | $1,000 | 5.6x | 🟢 Масштабировать |
| Referral | $1K | 2 | $500 | 11.2x | 🟢 Максимизировать |
| Organic | $0 | 4 | $0 | ∞ | 🟢 |
| **Blended** | **$29K** | **20** | **$1,450** | **3.9x** | 🟢 |

### LTV

| Метод | Расчёт | LTV |
|-------|--------|:---:|
| Простой | $500 × 85% × (1/3.5%) = $500 × 0.85 × 28.6 мес | **$12,143** |
| Когортный | Средняя когорта (12+ мес): $11,800 за lifetime | **$11,800** |
| **Финальная оценка** | Среднее (разброс 3%) | **$5,600** (3-year capped) |

> Capped at 3 years (36 мес) для консервативной оценки: $500 × 0.85 × Σ(retention_i) = **$5,600**.

### Ключевые метрики

| Метрика | Наше | Бенчмарк (SaaS SMB) | Оценка |
|---------|:----:|:--------------------:|:------:|
| LTV/CAC | 3.9x | 3-5x | 🟢 |
| Payback | 3.4 мес | < 12 мес | 🟢 |
| Gross Margin | 85% | 70-80% | 🟢 |
| Monthly Churn | 3.5% | 2-5% | 🟡 |
| Break-even | 160 клиентов ($60K fixed) | — | ✅ (200 > 160) |

### Sensitivity (SOM год 3)

| Сценарий | ARPU | Churn | CAC | LTV (3yr) | LTV/CAC | Payback |
|----------|:----:|:-----:|:---:|:---------:|:-------:|:-------:|
| Оптимистичный | $600 | 2.5% | $1,100 | $7,200 | 6.5x | 2.2 мес |
| **Базовый** | **$500** | **3.5%** | **$1,450** | **$5,600** | **3.9x** | **3.4 мес** |
| Пессимистичный | $400 | 5% | $1,800 | $3,400 | 1.9x | 5.3 мес |

**Kill-условие:** Churn > 7% → LTV/CAC < 1.0 → модель нежизнеспособна.

### Рекомендации

| # | Узкое место | Действие | Эффект | Приоритет |
|---|-------------|----------|:------:|:---------:|
| 1 | Churn 3.5% (верхняя граница) | Guided onboarding + health score alerts | Churn → 2.5%, LTV +29% | P1 |
| 2 | LinkedIn CAC $2,500 (высокий) | A/B тест креативов, сузить таргетинг | CAC → $1,800 | P2 |
| 3 | Referral только 10% клиентов | Запуск реферальной программы «месяц бесплатно» | +5 клиентов/мес, CAC $500 | P2 |

## Валидация (Quality Gate)

- [ ] Revenue model визуализирован: Revenue → COGS → Gross Margin → Variable → Contribution Margin
- [ ] CAC рассчитан (blended + по каналам с LTV/CAC per channel)
- [ ] LTV рассчитан минимум 2 методами с указанием допущений
- [ ] Разброс между методами LTV зафиксирован (< 30% или обоснован выбор)
- [ ] LTV/CAC ratio рассчитан и интерпретирован по шкале
- [ ] Payback period определён в месяцах, сравнён с бенчмарком
- [ ] Break-even рассчитан: кол-во клиентов и срок достижения
- [ ] Sensitivity analysis: 3 сценария + kill-условие (при каком LTV/CAC < 1.0)
- [ ] Метрики сравнены с отраслевыми бенчмарками (с источниками)
- [ ] Рекомендации привязаны к узким местам с ожидаемым эффектом
- [ ] Если данные моделированные — явно помечено с источниками бенчмарков
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$unit-economics` является входом для:
- **Стратег / Медиатор** — экономика единицы как фундамент для стратегии роста.
- **`tam-sam-som`** — revenue potential = SOM × ARPU.
- **`cohort-analysis`** — LTV по когортам для уточнения модели.
- **`rfm-analysis`** — LTV по сегментам для дифференцированного маркетинга.
- **`ansoff-matrix`** — unit economics новых рынков / продуктов.
- **`customer-journey-mapping`** — точки оттока → оптимизация churn и CAC.

Формат передачи: dashboard метрик + revenue model + CAC per channel + LTV (2 метода) + sensitivity + рекомендации. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| LTV без учёта churn | Завышение в разы | Всегда учитывать churn; или capped LTV (3 года) |
| CAC без продаж | Маркетинг — не весь CAC | Включать зарплаты sales, инструменты, overhead |
| Один метод LTV | Разные методы → разный результат | Минимум 2 метода, сверка, обоснование выбора |
| Без payback period | Не видно нагрузки на cash flow | Payback в мес + cash needed = CAC × new × payback |
| Без бенчмарков | Непонятно, хорошие ли цифры | Сравнить с отраслью (SaaS Capital, OpenView) |
| Статичная модель | Метрики меняются каждый квартал | Ежеквартальный пересчёт |
| Без sensitivity | Ложная уверенность в одном сценарии | 3 сценария + kill-условие (LTV/CAC < 1.0) |
| Blended CAC без каналов | Не видно, какой канал убыточен | CAC per channel + LTV/CAC per channel → вердикт |
| Без break-even | Непонятно, когда бизнес выйдет в плюс | Break-even clients + срок достижения |
