# Юнит-экономика — Анализ экономики единицы

## Когда использовать
- При оценке жизнеспособности бизнес-модели — сходится ли экономика на единицу.
- При планировании роста — при каких условиях масштабирование выгодно.
- При привлечении инвестиций — инвесторы требуют unit economics.
- При оптимизации маркетинга — оценка эффективности каналов привлечения.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Бизнес-модель | ✅ | SaaS / E-commerce / Marketplace / Сервис |
| Ценовая модель | ✅ | Подписка / транзакция / лицензия / freemium |
| Средний чек (ACV/ARPU) | ✅ | Средний доход на клиента за период |
| Маркетинговые расходы | ⬚ | Бюджет по каналам (если известен) |
| Количество клиентов | ⬚ | Текущая база или прогноз |
| Churn rate | ⬚ | Процент оттока (если известен) |
| Gross margin | ⬚ | Валовая маржа продукта |
| Каналы привлечения | ⬚ | Какие каналы используются |
| Исторические данные | ⬚ | Метрики за прошлые периоды |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — отраслевые бенчмарки (SaaS Capital, OpenView, ProfitWell, Baremetrics).
2. **Финансовые данные** — публичная отчётность аналогичных компаний.
3. **Отраслевые отчёты** — средние CAC, LTV, churn по вертикалям.
4. **Данные от пользователя** — реальные метрики из CRM, аналитики, бухгалтерии.
5. **Маркетинговые платформы** — стоимость привлечения по каналам.

> Для бенчмарков — указать источник, год и применимость к данному бизнесу.

## Протокол

### Шаг 0 — Сбор контекста
Проверить наличие обязательных данных. Определить тип бизнес-модели и «единицу» анализа (клиент / подписка / заказ). Выбрать валюту и период.

### Шаг 1 — Бизнес-модель и ценообразование
- Описать модель монетизации (подписка, транзакция, комиссия).
- Зафиксировать средний чек (ARPU / ACV).
- Определить структуру выручки: разовый платёж vs рекуррентный.
- Определить единицу анализа: клиент / аккаунт / подписка / заказ.

### Шаг 2 — CAC (Customer Acquisition Cost)
Рассчитать стоимость привлечения клиента:
- **Blended CAC:** общие расходы на маркетинг и продажи / кол-во новых клиентов.
- **CAC по каналам:** разбивка по каждому каналу привлечения.
- Включить: реклама, контент, SEO, продажи (зарплаты), инструменты.
- Определить органическую долю (клиенты с CAC ≈ 0).
- Динамика CAC: растёт / стабилен / снижается.

### Шаг 3 — LTV (Lifetime Value)
Рассчитать пожизненную ценность клиента. Выбрать методологию:
- **Простой метод:** ARPU × Gross Margin × Avg Lifetime (1/churn).
- **Когортный метод:** фактическая выручка с когорты за время жизни.
- **DCF-метод:** дисконтированные будущие денежные потоки от клиента.
- Обосновать выбор метода и указать допущения.

### Шаг 4 — LTV/CAC Ratio
Рассчитать соотношение и интерпретировать:
- **LTV/CAC < 1:** бизнес теряет деньги на каждом клиенте.
- **LTV/CAC 1-3:** на грани окупаемости, нужна оптимизация.
- **LTV/CAC 3-5:** здоровая экономика, можно масштабировать.
- **LTV/CAC > 5:** возможно, недоинвестируем в рост.
- Сравнить с бенчмарками по отрасли.

### Шаг 5 — Payback Period
Определить срок окупаемости привлечения клиента:
- **Формула:** CAC / (ARPU × Gross Margin) = месяцев до окупаемости.
- Бенчмарки: SaaS — 12-18 мес, E-commerce — 1-6 мес.
- Влияние payback на cash flow и потребность в оборотном капитале.

### Шаг 6 — Margin Analysis
Анализ маржинальности:
- **Gross Margin:** выручка - COGS.
- **Contribution Margin:** gross margin - переменные расходы на клиента.
- **Net Margin:** contribution margin - аллоцированные постоянные расходы.
- Определить точку безубыточности (break-even).

### Шаг 7 — Sensitivity Analysis
Оценить чувствительность юнит-экономики к изменению ключевых параметров:
- ARPU ±20%.
- Churn rate ±30%.
- CAC ±25%.
- Gross Margin ±10%.
- Построить таблицу сценариев: оптимистичный / базовый / пессимистичный.

### Шаг 8 — Бенчмарк и рекомендации
- Сравнить все метрики с отраслевыми бенчмарками.
- Определить узкие места (что ограничивает экономику).
- Сформировать рекомендации по улучшению:
  - Как снизить CAC (каналы, конверсия).
  - Как увеличить LTV (retention, upsell, pricing).
  - Как улучшить маржу (COGS, операционная эффективность).

## Валидация (Quality Gate)

- [ ] CAC рассчитан (blended + по каналам)
- [ ] LTV рассчитан с указанием методологии и допущений
- [ ] LTV/CAC ratio рассчитан и интерпретирован
- [ ] Payback period определён в месяцах
- [ ] Margin analysis проведён (gross / contribution)
- [ ] Sensitivity analysis с минимум 3 сценариями
- [ ] Метрики сравнены с отраслевыми бенчмарками
- [ ] Формулы и допущения явно указаны
- [ ] Рекомендации конкретны и привязаны к метрикам

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$unit-economics` является входом для:
- **Стратег / Медиатор** — экономика единицы как фундамент для стратегии роста.
- `$tam-sam-som` — revenue potential = SOM × ARPU.
- `$cohort-analysis` — LTV по когортам для уточнения модели.
- `$rfm-analysis` — LTV по сегментам.
- `$trend-analysis` — тренды влияют на будущий CAC и churn.

При передаче — использовать `$handoff` с ключевыми метриками и формулами.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| LTV без учёта churn | Завышение в разы | Всегда учитывать churn в LTV |
| CAC без продаж | Маркетинг — не весь CAC | Включать зарплаты sales, инструменты, overhead |
| Один метод LTV | Разные методы дают разный результат | Рассчитать 2+ методами, сравнить |
| Без payback period | Не видно нагрузки на cash flow | Всегда считать payback в месяцах |
| Без бенчмарков | Непонятно, хорошие ли цифры | Сравнивать с отраслевыми стандартами |
| Статичная модель | Метрики меняются каждый квартал | Рекомендовать регулярный пересчёт |
| Без sensitivity | Ложная уверенность в одном сценарии | Всегда 3 сценария минимум |
