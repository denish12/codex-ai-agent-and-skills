# Матрица Ансоффа — Стратегия роста бизнеса

## Когда использовать
- При выборе **стратегии роста** — куда направить ресурсы для масштабирования.
- При оценке **новых рыночных возможностей** — выход на новые рынки или сегменты.
- При планировании **расширения продуктовой линейки** — новые продукты для существующих или новых клиентов.
- При принятии решений о **диверсификации** — оценка рисков и потенциала.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Компания / бизнес-единица | ✅ | Объект, для которого определяется стратегия роста |
| Текущие продукты | ✅ | Перечень существующих продуктов / услуг с выручкой и маржой |
| Текущие рынки | ✅ | Перечень текущих рынков, сегментов, географий с долей и динамикой |
| Стратегические цели роста | ⬚ | Целевые показатели (выручка, доля, прибыль) и горизонт |
| Доступные ресурсы | ⬚ | Бюджет, компетенции, временные рамки |
| Конкурентный контекст | ⬚ | Положение на рынке, сильные стороны, бренд |
| Риск-аппетит | ⬚ | Консервативный / умеренный / агрессивный |

> Если обязательные поля не предоставлены — **запросить** у пользователя, не генерировать предположения.

## Источники данных
1. Внутренняя отчётность — выручка и маржа по продуктам и рынкам.
2. Маркетинговые исследования — размер рынков, потенциал роста, клиентские сегменты.
3. Конкурентный анализ — доли рынка, стратегии роста конкурентов.
4. Данные о клиентах — LTV, churn, cross-sell потенциал, удовлетворённость.
5. Отраслевые прогнозы — тренды, новые ниши, технологические сдвиги.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `tam-sam-som` | TAM/SAM/SOM для оценки новых рынков (шаги 2, 4) | Если нет данных о размере целевого рынка |
| `competitive-analysis` | Доли конкурентов, барьеры входа, моаты (шаги 1, 2) | Если нет конкурентного контекста |
| `swot-analysis` | Сильные/слабые стороны, возможности/угрозы (шаг 0) | Если не определена стратегическая позиция |
| `web-research` | Верификация рыночных данных, трендов | Если источники данных отсутствуют или устарели (>12 мес) |

## Протокол

### Шаг 0 — Подготовка контекста
1. Определить текущий портфель продуктов и рынков.
2. Зафиксировать baseline по каждой паре «продукт × рынок»:

   | Пара | Выручка | Доля рынка | Маржа | Динамика YoY |
   |------|---------|:----------:|:-----:|:------------:|
   | [Продукт A × Рынок 1] | $XXM | XX% | XX% | +/-XX% |

3. Определить стратегические цели роста и горизонт (1-3 года).
4. Оценить доступные ресурсы и ограничения.
5. Установить риск-аппетит организации (консервативный / умеренный / агрессивный).

> Каждый факт о рынке — с указанием источника и даты. Данные старше 12 месяцев — пометить ⚠️ и верифицировать через `web-research`.

### Шаг 1 — Проникновение на рынок (Market Penetration)
*Существующие продукты → Существующие рынки. Риск: низкий.*

1. Проанализировать потенциал увеличения доли на текущих рынках с текущими продуктами.
2. Оценить возможности:
   - Увеличение частоты потребления существующими клиентами.
   - Привлечение клиентов конкурентов (switching).
   - Конверсия не-потребителей в текущем сегменте.
3. Определить инструменты: ценообразование, промо, дистрибуция, лояльность.
4. По каждой инициативе заполнить карточку:

   | Поле | Значение |
   |------|----------|
   | Инициатива | [Название] |
   | Механика | [Как работает] |
   | Потенциал | +$XXM / +XX% доли |
   | Инвестиции | $XM |
   | Срок | X мес |
   | Ожидаемый ROI | XX% |
   | Риск (1-5) | 1-2 |

### Шаг 2 — Развитие рынка (Market Development)
*Существующие продукты → Новые рынки. Риск: средний.*

1. Идентифицировать новые рынки для существующих продуктов:
   - Новые географии (города, страны, регионы).
   - Новые клиентские сегменты (возраст, доход, B2B vs B2C).
   - Новые каналы дистрибуции (онлайн, партнёры, франшиза).
2. Для каждого нового рынка оценить:
   - Размер и потенциал — использовать `tam-sam-som` если данных нет.
   - Барьеры входа (регуляторные, культурные, конкурентные) — использовать `competitive-analysis` для оценки.
   - Необходимая адаптация продукта (локализация, кастомизация).
3. По каждой инициативе заполнить карточку (аналогично шагу 1, риск 2-3).

### Шаг 3 — Развитие продукта (Product Development)
*Новые продукты → Существующие рынки. Риск: средний.*

1. Идентифицировать новые продукты для существующих рынков:
   - Расширение текущей продуктовой линейки (новые SKU, тарифы).
   - Апгрейды и следующие поколения продуктов.
   - Комплементарные продукты и сервисы.
   - Кросс-продажи (cross-sell) и апсейлы (up-sell).
2. Для каждого нового продукта оценить:
   - Соответствие потребностям текущих клиентов (product-market fit).
   - Синергия с текущими компетенциями и инфраструктурой.
   - Время до вывода на рынок (time-to-market).
3. По каждой инициативе заполнить карточку (аналогично шагу 1, риск 2-3).

### Шаг 4 — Диверсификация (Diversification)
*Новые продукты → Новые рынки. Риск: высокий.*

1. Рассмотреть варианты диверсификации:
   - **Связанная (related)**: новый продукт на новом рынке, но с синергией с текущим бизнесом.
   - **Несвязанная (unrelated)**: полностью новое направление без синергии.
2. Для каждого варианта оценить:
   - Стратегическое обоснование (зачем выходить за рамки текущего бизнеса).
   - Потенциальная синергия (технологии, бренд, дистрибуция, клиенты).
   - Требуемые компетенции — есть ли они внутри или нужна M&A / партнёрство.
   - Размер инвестиций и срок окупаемости.
3. Рассмотреть альтернативы: партнёрство, JV, лицензирование vs органический рост.
4. По каждой инициативе заполнить карточку (аналогично шагу 1, риск 4-5).

### Шаг 5 — Сравнительный портфельный анализ
*Сквозная оценка всех инициатив из шагов 1-4.*

1. Собрать все инициативы в единую таблицу скоринга:

   | # | Квадрант | Инициатива | Потенциал (1-10) | Риск (1-5) | ROI | Реализуемость (1-10) | Синергия (1-10) | **Score** |
   |---|----------|-----------|:----------------:|:----------:|:---:|:--------------------:|:---------------:|:---------:|
   | 1 | MP | [Название] | X | X | XX% | X | X | **XX** |

2. Формула скоринга (веса адаптировать под риск-аппетит):

   | Риск-аппетит | wПотенциал | wРиск (инверт.) | wROI | wРеализуемость | wСинергия |
   |:------------:|:----------:|:---------------:|:----:|:--------------:|:---------:|
   | Консервативный | 0.20 | 0.30 | 0.25 | 0.15 | 0.10 |
   | Умеренный | 0.25 | 0.20 | 0.25 | 0.15 | 0.15 |
   | Агрессивный | 0.35 | 0.10 | 0.20 | 0.15 | 0.20 |

   `Score = wПотенциал × Потенциал + wРиск × (6 - Риск) + wROI × norm(ROI) + wРеализуемость × Реализуемость + wСинергия × Синергия`

3. Ранжировать инициативы по Score. Топ-N отобрать в портфель.
4. Проверить портфель на баланс: не менее 2 квадрантов представлены.

### Шаг 6 — Сценарный анализ
1. Для топ-3 инициатив построить 3 сценария:

   | Сценарий | Потенциал | Вероятность | Ключевое допущение |
   |----------|-----------|:-----------:|---------------------|
   | Оптимистичный | +XX% от базы | XX% | [Что должно сбыться] |
   | Базовый | база | XX% | [Текущие тренды] |
   | Пессимистичный | -XX% от базы | XX% | [Что может пойти не так] |

2. Рассчитать взвешенный ожидаемый результат (EV) для каждой инициативы.
3. Определить kill-условия: при каком сценарии инициативу стоит свернуть.

### Шаг 7 — Распределение ресурсов и дорожная карта
1. Распределить бюджет между квадрантами (% от общего бюджета роста).
2. Типичные пропорции (ориентир, не догма):

   | Риск-аппетит | Проникновение | Развитие рынка | Развитие продукта | Диверсификация |
   |:------------:|:-------------:|:--------------:|:-----------------:|:--------------:|
   | Консервативный | 50-60% | 20-25% | 15-20% | 0-5% |
   | Умеренный | 35-45% | 20-25% | 20-25% | 10-15% |
   | Агрессивный | 20-30% | 20-25% | 25-30% | 20-30% |

3. Сформулировать дорожную карту с конкретными вехами, ответственными и KPI.
4. Определить gates: точки принятия решений go/no-go по каждой инициативе.

### Шаг 8 — Синтез и выводы
1. Сформулировать рекомендуемый путь роста (1-2 абзаца стратегической логики).
2. Определить ключевые KPI для отслеживания прогресса.
3. Зафиксировать критические решения и сроки.
4. Подготовить план действий с ответственными и вехами.

## Пример — SaaS-платформа аналитики

**Контекст:** B2B SaaS, продукт — платформа бизнес-аналитики. Выручка $12M ARR, 200 клиентов в SMB-сегменте РФ, маржа 72%, рост +25% YoY. Цель: $30M ARR за 2 года. Риск-аппетит: умеренный.

| | Существующие продукты | Новые продукты |
|--|:---------------------:|:--------------:|
| **Существующие рынки** | **MP: Upsell до Enterprise-тарифа** | **PD: AI-модуль прогнозной аналитики** |
| (SMB РФ) | Потенциал: +$3M, Риск: 1, ROI: 180% | Потенциал: +$5M, Риск: 3, ROI: 90% |
| | Переключить 40 топ-клиентов на тариф Pro | Дополнение к базовому продукту, кросс-селл |
| **Новые рынки** | **MD: Выход в СНГ (KZ, UZ)** | **D: White-label для банков** |
| | Потенциал: +$4M, Риск: 3, ROI: 65% | Потенциал: +$8M, Риск: 5, ROI: 45% |
| | TAM СНГ = $50M, SOM = $4M (через партнёров) | Связанная диверсификация, нужна интеграция |

**Скоринг (умеренный):** MP = 8.2, PD = 7.1, MD = 6.4, D = 5.0 → Портфель: MP + PD + MD.
**Бюджет:** Проникновение 40%, Развитие продукта 30%, Развитие рынка 25%, Диверсификация 5% (исследование).
**Kill-условие для MD:** если SOM < $1.5M после 6 мес — свернуть.

## Валидация (Quality Gate)

- [ ] Все 4 квадранта проанализированы с конкретными инициативами
- [ ] Baseline зафиксирован: выручка, доля, маржа по каждой паре «продукт × рынок»
- [ ] Для каждой инициативы заполнена карточка: потенциал, риск, ROI, инвестиции, срок
- [ ] Скоринг проведён по единой модели с учётом риск-аппетита
- [ ] Сценарный анализ для топ-3 инициатив (оптимистичный / базовый / пессимистичный)
- [ ] Портфель сбалансирован: не менее 2 квадрантов представлены
- [ ] Распределение ресурсов обосновано и привязано к риск-аппетиту
- [ ] Дорожная карта имеет вехи, ответственных и go/no-go gates
- [ ] Kill-условия определены для инициатив с риском ≥ 3
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

## Handoff
Результат передаётся:
- **Стратег / Медиатор** — для интеграции в стратегический план и бюджет.
- **`unit-economics`** — для детального расчёта экономики по выбранным инициативам.
- **`competitive-analysis`** — для углублённого анализа конкурентов на новых рынках (шаг 2).
- **`tam-sam-som`** — для уточнения оценок рынка, если использовались приблизительные данные.

Формат передачи: заполненный шаблон вывода + таблица скоринга + сценарный анализ.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Фокус только на проникновении | Потенциал текущего рынка конечен, компания упускает возможности | Оценивать все 4 квадранта, портфель минимум из 2 квадрантов |
| Диверсификация без обоснования | Самый рискованный квадрант без стратегической логики | Диверсификация только при чётком обосновании + связанная > несвязанная |
| Одна стратегия вместо портфеля | Все ресурсы в одном направлении — нет fallback | Комбинировать стратегии с разным уровнем риска и горизонтом |
| Игнорирование рисков | Оптимистичные планы без проверки → провал | Каждая инициатива — со скорингом, сценариями и kill-условиями |
| Отсутствие baseline-метрик | Рост «от неизвестного» невозможно измерить | Начинать с фиксации baseline: выручка, доля, маржа по парам |
| Смешение развития рынка и продукта | Непонятно, где источник роста, размывается фокус | Чётко: новый рынок (MD) vs новый продукт (PD), не смешивать |
| Оценка «на глаз» без модели | Субъективные приоритеты, невоспроизводимый результат | Единый скоринг с весами, адаптированными под риск-аппетит |
| Данные без источников | Невозможно верифицировать, быстро устаревают | Каждый факт — с источником и датой; >12 мес = ⚠️ |
