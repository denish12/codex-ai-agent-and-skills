# Стратегия голубого океана — Создание незанятого рыночного пространства

## Когда использовать
- При поиске **новых рыночных пространств** за пределами существующей конкуренции.
- При необходимости **радикально дифференцироваться** от конкурентов через инновацию ценности.
- При разработке **нового продукта или сервиса** с нуля — без привязки к текущим отраслевым стандартам.
- При подготовке **стратегии роста** — когда красный океан исчерпан или нерентабелен.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Отрасль / рынок | ✅ | Текущий рынок (красный океан) для анализа |
| Продукт / сервис | ✅ | Текущее предложение или концепция нового |
| Конкуренты | ✅ | 3-5 ключевых игроков для построения стратегической канвы |
| Клиентские сегменты | ⬚ | Текущие клиенты и их потребности |
| Факторы конкуренции | ⬚ | Известные параметры, по которым конкурируют в отрасли (5-8) |
| Ценовой диапазон | ⬚ | Текущие ценовые уровни в отрасли |
| Данные о не-клиентах | ⬚ | Кто не покупает и почему |

> Если обязательные поля не предоставлены — **запросить** у пользователя, не генерировать предположения.

## Источники данных
1. Конкурентный анализ — предложения, цены, позиционирование ключевых игроков.
2. Клиентские исследования — потребности, pain points, jobs-to-be-done.
3. Отраслевые отчёты — стандартные факторы конкуренции, отраслевые нормы.
4. Не-клиенты — почему не покупают, барьеры входа в потребление.
5. Смежные отрасли — альтернативные решения тех же задач.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `competitive-analysis` | Факторы конкуренции, позиционирование, моаты (шаг 0, 1) | Если нет данных о конкурентах и факторах конкуренции |
| `tam-sam-som` | Размер пространства не-клиентов, потенциал нового рынка (шаг 3, 6) | Для оценки TAM голубого океана |
| `web-research` | Верификация данных о не-клиентах, смежных отраслях | Если данные отсутствуют или устарели (>12 мес) |
| `ansoff-matrix` | Стратегия диверсификации, если Blue Ocean = новый продукт + новый рынок | После формирования предложения — для интеграции в портфель роста |
| `swot-analysis` | Внутренние компетенции для оценки реализуемости (шаг 7) | Если не ясно, есть ли ресурсы на реализацию |

## Протокол

### Шаг 0 — Подготовка контекста
1. Определить текущий «красный океан» — отрасль и её ключевых игроков (3-5).
2. Идентифицировать 5-8 факторов конкуренции, по которым игроки соревнуются.
3. Собрать данные о текущих клиентах и их удовлетворённости.
4. Зафиксировать отраслевые «священные коровы» — допущения, которые никто не оспаривает.

> Каждый факт о рынке — с указанием источника и даты. Данные старше 12 месяцев — пометить ⚠️ и верифицировать через `web-research`.

### Шаг 1 — Стратегическая канва (Strategy Canvas) — AS-IS
1. Определить оси: горизонтальная — факторы конкуренции, вертикальная — уровень предложения (1-5, где 1 = низкий, 5 = высокий).
2. Построить кривые ценности для каждого ключевого конкурента.
3. Построить кривую ценности для анализируемой компании / продукта.
4. Выявить кластеры: где все игроки одинаковы (зона конвергенции), где есть дифференциация.
5. Зафиксировать: по каким факторам отрасль конкурирует избыточно (over-served), по каким — недостаточно (under-served).

### Шаг 2 — Карта полезности для покупателя (Buyer Utility Map)
1. Определить 6 стадий покупательского цикла:

   | Стадия | Описание |
   |--------|----------|
   | Покупка | Процесс выбора и приобретения |
   | Доставка | Получение продукта / активация сервиса |
   | Использование | Основной опыт эксплуатации |
   | Дополнения | Сопутствующие товары / сервисы |
   | Обслуживание | Поддержка, ремонт, обновления |
   | Утилизация | Завершение использования, миграция |

2. Для каждой стадии — оценить 6 рычагов полезности:

   | | Покупка | Доставка | Использование | Дополнения | Обслуживание | Утилизация |
   |--|:------:|:--------:|:-------------:|:----------:|:------------:|:----------:|
   | **Производительность** | | | ★ | | | |
   | **Простота** | | | | | | |
   | **Удобство** | ★ | | | | | |
   | **Риск** | | | | | | |
   | **Развлечение** | | | | | | |
   | **Экологичность** | | | | | | |

   ★ = фокус отрасли. Пустые ячейки = белые пятна (возможности).

3. Определить: где отрасль фокусируется (≤ 3-4 ячейки), а где — белые пятна (остальные 32-33 ячейки).
4. Выбрать **минимум 2 белых пятна** с наибольшим потенциалом для создания новой ценности.

### Шаг 3 — Три уровня не-клиентов
1. **Первый уровень — «скоро уйдут»**: клиенты на границе рынка, используют минимально, ищут альтернативы.
   - Почему они на границе? Что их не устраивает?
2. **Второй уровень — «отказники»**: сознательно выбрали не пользоваться продуктами отрасли.
   - Какие потребности не удовлетворены? Какие барьеры?
3. **Третий уровень — «неисследованные»**: никогда не рассматривали продукты отрасли как вариант.
   - Какую потребность они решают иначе? Какое предложение их привлечёт?
4. Для каждого уровня оценить:

   | Уровень | Размер | Ключевой барьер | Потенциал конверсии | Источник данных |
   |---------|:------:|-----------------|:-------------------:|-----------------|
   | 1. «Скоро уйдут» | [N / $XM] | [Барьер] | Высокий / Средний / Низкий | [Источник] |
   | 2. «Отказники» | [N / $XM] | [Барьер] | ... | [Источник] |
   | 3. «Неисследованные» | [N / $XM] | [Барьер] | ... | [Источник] |

5. Если данных нет — использовать `tam-sam-som` для оценки размера и `web-research` для исследования барьеров.

### Шаг 4 — Фреймворк ERRC (Eliminate-Reduce-Raise-Create)
1. **Eliminate (Упразднить)**: Какие факторы, принятые в отрасли как данность, можно полностью убрать?
   - Факторы, не создающие ценности для покупателя.
   - Факторы, существующие по инерции или для имитации конкурентов.
2. **Reduce (Снизить)**: Какие факторы можно снизить значительно ниже отраслевого стандарта?
   - Факторы с избыточным уровнем предложения (over-engineering).
3. **Raise (Повысить)**: Какие факторы нужно поднять значительно выше отраслевого стандарта?
   - Факторы, где текущий уровень не удовлетворяет покупателей.
4. **Create (Создать)**: Какие факторы, никогда не предлагавшиеся в отрасли, нужно создать?
   - Новые источники ценности из анализа не-клиентов и Buyer Utility Map.
5. Для каждого действия заполнить карточку:

   | Действие | Фактор | Было (AS-IS) | Стало (TO-BE) | Эффект на затраты | Эффект на ценность | Обоснование |
   |----------|--------|:------------:|:-------------:|:-----------------:|:------------------:|-------------|
   | Eliminate | [A] | Среднее | — | -$XXK/год | 0 | [Почему не нужен] |
   | Reduce | [B] | Высокое | Низкое | -$XXK/год | -незначительно | [Почему избыточно] |
   | Raise | [C] | Низкое | Очень высокое | +$XXK/год | +значительно | [Почему важно] |
   | Create | [D] | — | Высокое | +$XXK/год | +значительно | [Какую проблему решает] |

6. Проверить баланс: **экономия (E + R) должна компенсировать затраты (Ra + C)**, формируя инновацию ценности — одновременное снижение издержек и рост ценности для покупателя.

### Шаг 5 — Новая кривая ценности (Strategy Canvas — TO-BE)
1. Построить новую кривую ценности с учётом ERRC.
2. Сравнить с кривыми конкурентов — должна быть видимая дивергенция.
3. Проверить три характеристики хорошей стратегии:

   | Критерий | Тест | Результат |
   |----------|------|:---------:|
   | **Фокус** | Кривая не повторяет форму конкурентов, ресурсы сконцентрированы | Да / Нет |
   | **Дивергенция** | ≥ 3 фактора отличаются на ≥ 2 уровня от среднего по отрасли | Да / Нет |
   | **Убедительный слоган** | Можно описать одной фразой, понятной покупателю | «[Слоган]» |

4. **Если хотя бы один критерий = Нет — вернуться к шагу 4 и усилить ERRC.**

### Шаг 6 — Ценовой коридор масс (Price Corridor of the Mass)
1. Определить три ценовых кластера:

   | Кластер | Продукты / альтернативы | Ценовой диапазон | Размер аудитории |
   |---------|------------------------|:----------------:|:----------------:|
   | Та же форма | Прямые конкуренты в отрасли | $XX — $XX | [N] |
   | Та же функция | Альтернативные решения той же задачи | $XX — $XX | [N] |
   | Та же цель | Продукты с той же конечной целью | $XX — $XX | [N] |

2. Определить верхнюю и нижнюю границы ценового коридора масс.
3. Выбрать ценовой уровень исходя из уровня защитимости:
   - **Высокая защитимость** (патенты, сетевой эффект, эксклюзивные ресурсы) → верхняя граница коридора.
   - **Средняя защитимость** (трудно копируемые компетенции) → середина коридора.
   - **Низкая защитимость** (легко копируется) → нижняя граница коридора.

### Шаг 7 — Оценка и сценарный анализ
1. Оценить экономику голубого океана:

   | Метрика | Значение | Источник |
   |---------|:--------:|----------|
   | TAM нового пространства (не-клиенты) | $XXM | `tam-sam-som` / оценка |
   | Экономия от E + R (годовая) | -$XXK | Расчёт по ERRC |
   | Инвестиции в Ra + C (годовые) | +$XXK | Расчёт по ERRC |
   | **Нетто-эффект на unit economics** | ±$XX | Экономия − Инвестиции |
   | Целевая цена | $XX | Ценовой коридор масс |
   | Целевая маржа | XX% | Цена − себестоимость |
   | Время до копирования конкурентами | X-X лет | Оценка защитимости |
   | Инвестиции на реализацию (total) | $XM | Ресурсная оценка |
   | Срок до breakeven | X мес | Финансовая модель |

2. Сценарный анализ:

   | Сценарий | TAM | Конверсия не-клиентов | Срок копирования | Выручка (3 года) | Вероятность |
   |----------|:---:|:--------------------:|:----------------:|:----------------:|:-----------:|
   | Оптимистичный | $XXM | XX% | > X лет | $XXM | XX% |
   | Базовый | $XXM | XX% | X лет | $XXM | XX% |
   | Пессимистичный | $XXM | XX% | < X лет | $XXM | XX% |

3. Определить kill-условия: при каком сценарии не запускать / свернуть.
4. Оценить реализуемость:

   | Ресурс | Есть | Нужно | Gap | Способ закрытия |
   |--------|:----:|:-----:|:---:|-----------------|
   | Компетенции | [Какие] | [Какие] | [Что отсутствует] | Найм / партнёрство / обучение |
   | Технологии | [Какие] | [Какие] | [Что отсутствует] | Разработка / лицензирование |
   | Бюджет | $XM | $XM | $XM | Реаллокация / инвестиции |
   | Время | — | X мес | — | [Ограничения] |

### Шаг 8 — Синтез и дорожная карта
1. Сформулировать стратегическое предложение голубого океана (1-2 абзаца).
2. Зафиксировать целевую бизнес-модель (цена, каналы, ключевые метрики).
3. Подготовить дорожную карту:

   | Период | Действие | Ключевая метрика | Go/No-go gate | Ответственный |
   |--------|----------|:----------------:|---------------|---------------|
   | Месяц 1-3 | [Что] | [KPI] | [Условие] | [Кто] |
   | Месяц 4-6 | [Что] | [KPI] | [Условие] | [Кто] |
   | Месяц 7-12 | [Что] | [KPI] | [Условие] | [Кто] |

4. Определить KPI для мониторинга: конверсия не-клиентов, retention, NPS, unit economics.

## Пример — Онлайн-фитнес vs традиционные спортзалы

**Контекст:** Рынок фитнес-услуг РФ. Красный океан: сети спортзалов (World Class, X-Fit, DDX). Задача — найти голубой океан.

### Strategy Canvas — AS-IS

| Фактор | World Class | X-Fit | DDX | Среднее отрасли |
|--------|:----------:|:-----:|:---:|:--------------:|
| Оборудование | 5 | 4 | 3 | 4.0 |
| Локация (престиж) | 5 | 3 | 2 | 3.3 |
| Персональные тренеры | 4 | 4 | 3 | 3.7 |
| Групповые программы | 4 | 4 | 4 | 4.0 |
| Бассейн / спа | 5 | 3 | 1 | 3.0 |
| Цена (инвертировано) | 1 | 3 | 4 | 2.7 |
| Гибкость расписания | 2 | 2 | 3 | 2.3 |
| Сообщество / геймификация | 1 | 1 | 1 | 1.0 |

**«Священные коровы»:** физическое присутствие, дорогое оборудование, фиксированное расписание.

### ERRC-сетка

| Eliminate | Reduce |
|:---------:|:------:|
| Престижная локация (-$XXK/мес аренды) | Оборудование: с 4.0 до 1.5 (только базовое + резинки) |
| Бассейн / спа (-$XXK/мес) | Персональные тренеры: с 3.7 до 2.0 (AI + запись) |

| Raise | Create |
|:-----:|:------:|
| Гибкость расписания: с 2.3 до 5.0 (24/7, любое место) | Сообщество + геймификация: 0 → 5.0 (лидерборды, челленджи) |
| Цена (доступность): с 2.7 до 5.0 (×5 дешевле) | AI-персонализация: 0 → 4.5 (адаптивные программы) |

**Баланс:** Экономия (E+R) = -$2.8M/год. Инвестиции (Ra+C) = +$1.1M/год. **Нетто: -$1.7M/год затрат** → инновация ценности ✅.

### Три уровня не-клиентов

| Уровень | Кто | Размер | Барьер |
|---------|-----|:------:|--------|
| 1. «Скоро уйдут» | Купили абонемент, ходят 1-2 раза/мес | ~2M чел. | Далеко, неудобное расписание |
| 2. «Отказники» | Знают про залы, считают дорого/неуютно | ~8M чел. | Цена, страх осуждения |
| 3. «Неисследованные» | Жители малых городов без спортзалов | ~15M чел. | Нет инфраструктуры |

### Проверка стратегии
- **Фокус:** ✅ — ресурсы на AI, сообщество, доступность. Не размазаны на оборудование и локации.
- **Дивергенция:** ✅ — 5 факторов отличаются на ≥ 2 уровня.
- **Слоган:** «Персональный тренер в кармане за 500₽/мес».

**Ценовой коридор масс:** Та же форма (залы) = 2000-8000₽/мес. Та же функция (YouTube, приложения) = 0-500₽/мес. Та же цель (здоровье) = 500-3000₽/мес. → Цена = 499₽/мес (нижняя граница, защитимость средняя: AI-модели копируемы за 1-2 года).

## Валидация (Quality Gate)

- [ ] Стратегическая канва AS-IS содержит минимум 3 конкурента и 5-8 факторов с числовыми уровнями
- [ ] Buyer Utility Map 6×6 заполнена, выявлено минимум 2 белых пятна
- [ ] Три уровня не-клиентов проанализированы с оценкой размера и барьеров
- [ ] ERRC-сетка заполнена: минимум 1 действие в каждом квадранте с эффектом на затраты и ценность
- [ ] Баланс ERRC проверен: экономия (E+R) компенсирует инвестиции (Ra+C)
- [ ] Новая кривая ценности визуально дивергирует от конкурентных
- [ ] Три характеристики стратегии пройдены (фокус, дивергенция, слоган)
- [ ] Ценовой коридор масс определён (3 кластера: форма, функция, цель)
- [ ] Экономика голубого океана обоснована: TAM, unit economics, breakeven
- [ ] Сценарный анализ проведён (оптимистичный / базовый / пессимистичный)
- [ ] Kill-условия определены
- [ ] Реализуемость оценена: компетенции, технологии, бюджет, время
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

## Handoff
Результат передаётся:
- **Стратег / Медиатор** — для интеграции в стратегию роста или бизнес-план нового продукта.
- **`ansoff-matrix`** — если Blue Ocean = диверсификация (новый продукт + новый рынок), интегрировать в портфель роста.
- **`unit-economics`** — для детального расчёта экономики нового предложения.
- **`tam-sam-som`** — для уточнения размера голубого океана, если использовались приблизительные данные.
- **`competitive-analysis`** — для мониторинга реакции конкурентов и скорости копирования.

Формат передачи: заполненный шаблон вывода + ERRC с экономикой + сценарный анализ + дорожная карта.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| ERRC без стратегической канвы | Нет базы для сравнения, действия «в вакууме» | Сначала AS-IS канва, потом ERRC, потом TO-BE |
| Фокус только на Create | Повышает затраты без оптимизации — нет инновации ценности | Баланс: E + R снижают затраты, Ra + C повышают ценность |
| Игнорирование не-клиентов | Теряется главный источник голубого океана | Все три уровня обязательны с оценкой размера и барьеров |
| Кривая = конкурент + 1 фактор | Это не голубой океан, а инкрементальная инновация | Дивергенция: ≥ 3 фактора отличаются на ≥ 2 уровня |
| Нереализуемое предложение | Красивая стратегия без ресурсов бесполезна | Оценка реализуемости: компетенции, технологии, бюджет, время |
| Отсутствие убедительного слогана | Сложно коммуницировать, значит нет фокуса | Одна фраза, понятная покупателю — или вернуться к ERRC |
| ERRC без экономического расчёта | Непонятно, работает ли инновация ценности | Каждое действие — с эффектом на затраты и ценность |
| Buyer Utility Map пропущена | Упускаются скрытые возможности в 32+ ячейках | Матрица 6×6 обязательна, минимум 2 белых пятна |
| Цена «от себестоимости» | Не учитывает альтернативы и массовость | Ценовой коридор масс: 3 кластера (форма, функция, цель) |
