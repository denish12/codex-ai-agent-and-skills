# PEST/PESTEL-анализ — Анализ макросреды

## Когда использовать
- При оценке **макроокружения** перед выходом на новый рынок или запуском нового продукта.
- При **стратегическом планировании** — для понимания внешних факторов, влияющих на бизнес.
- При анализе **регуляторных и политических рисков** в конкретных регионах или отраслях.
- При подготовке **инвестиционных решений** — оценка странового и отраслевого риска.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Объект анализа | ✅ | Компания, отрасль, рынок или регион |
| Географический фокус | ✅ | Страна / регион / глобальный охват |
| Горизонт анализа | ⬚ | Период прогноза (по умолчанию — 3-5 лет) |
| Режим | ⬚ | PEST (4 фактора) или PESTEL (6 факторов, по умолчанию) |
| Отрасль | ⬚ | Конкретная отрасль для фокусировки факторов |
| Стратегические цели | ⬚ | Контекст для оценки влияния факторов |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. Государственные статистические агентства — макроэкономические показатели, демография.
2. Центральные банки и регуляторы — ставки, регуляторные инициативы.
3. Отраслевые ассоциации — отраслевые отчёты, прогнозы.
4. Международные организации — World Bank, IMF, WTO, OECD.
5. Технологические обзоры — Gartner, Forrester, McKinsey, BCG.
6. Экологические и правовые базы — ESG-рейтинги, реестры законодательства.

> Каждый факт — с указанием источника и даты. Данные старше 12 месяцев — пометить ⚠️ и верифицировать через `web-research`.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `swot-analysis` | Внешние Opportunities / Threats формируются из PESTEL-факторов (шаг 8) | PESTEL → O/T блок SWOT |
| `tam-sam-som` | Макро-ограничения на размер рынка: регуляторные, экономические (шаги 1-2, 6) | Если PESTEL выявил ограничения на SAM/SOM |
| `competitive-analysis` | Регуляторные барьеры входа для конкурентов (шаг 6) | Если правовые факторы влияют на конкурентный ландшафт |
| `ansoff-matrix` | Макро-риски для стратегий роста: новые рынки, диверсификация (шаг 8) | Для оценки рисков расширения |
| `web-research` | Верификация макроданных, свежие отчёты | Если данные отсутствуют или устарели (>12 мес) |
| `trend-analysis` | Тренды для обоснования динамики факторов (↑/↓/→) | Если нужна глубокая проработка тренда |

## Протокол

### Шаг 0 — Подготовка контекста
1. Определить объект анализа и географический фокус.
2. Уточнить режим: PEST (4 фактора) или PESTEL (6 факторов).
3. Зафиксировать горизонт анализа и стратегический контекст.
4. Определить ключевые вопросы, на которые должен ответить анализ.
5. Собрать доступные данные из источников.

### Шаг 1 — Политические факторы (Political)
1. Оценить политическую стабильность и форму правления.
2. Проанализировать текущую и планируемую государственную политику, влияющую на отрасль.
3. Оценить торговую политику: тарифы, санкции, торговые соглашения.
4. Проанализировать налоговую политику и государственное стимулирование.
5. Оценить коррупционные риски и качество институтов.
6. Для каждого фактора заполнить карточку:

   | Поле | Значение |
   |------|----------|
   | Фактор | [Описание] |
   | Влияние (1-5) | [На сколько сильно влияет на бизнес] |
   | Вероятность (1-5) | [Вероятность реализации / усиления] |
   | Тренд | ↑ усиливается / ↓ ослабевает / → стабилен |
   | Характер | 🟢 Возможность / 🔴 Риск / 🟡 Двойственный |
   | Горизонт | Краткосрочный (<1г) / Среднесрочный (1-3г) / Долгосрочный (3-5г) |
   | Источник | [URL / название, дата] |

   **Характер определяется по правилу:**
   - 🟢 **Возможность** — фактор создаёт конкурентное преимущество, снижает барьеры, открывает рынок.
   - 🔴 **Риск** — фактор ограничивает, увеличивает затраты, создаёт угрозу.
   - 🟡 **Двойственный** — зависит от стратегии: может быть и риском, и возможностью.

### Шаг 2 — Экономические факторы (Economic)
1. Проанализировать макроэкономические показатели: ВВП, инфляция, безработица.
2. Оценить валютные риски и динамику курсов.
3. Проанализировать процентные ставки и доступность капитала.
4. Оценить покупательскую способность и структуру потребления.
5. Проанализировать цикличность экономики и фазу делового цикла.
6. Для каждого фактора — карточка (аналогично шагу 1).

### Шаг 3 — Социальные факторы (Social)
1. Проанализировать демографические тренды: возрастная структура, урбанизация, миграция.
2. Оценить культурные нормы и ценности, влияющие на потребление.
3. Проанализировать уровень образования и квалификацию рабочей силы.
4. Оценить изменения в образе жизни и потребительских предпочтениях.
5. Проанализировать социальное неравенство и его влияние на рынки.
6. Для каждого фактора — карточка (аналогично шагу 1).

### Шаг 4 — Технологические факторы (Technological)
1. Оценить уровень технологического развития в регионе / отрасли.
2. Проанализировать темпы технологических изменений и дизрупций.
3. Оценить инвестиции в R&D — государственные и частные.
4. Проанализировать уровень цифровизации и интернет-проникновения.
5. Оценить влияние автоматизации и AI на отрасль.
6. Для каждого фактора — карточка (аналогично шагу 1).

### Шаг 5 — Экологические факторы (Environmental, PESTEL)
1. Проанализировать экологическое законодательство и регуляторные требования.
2. Оценить климатические риски и их влияние на бизнес.
3. Проанализировать ESG-тренды и давление стейкхолдеров.
4. Оценить доступность и стоимость ресурсов (энергия, сырьё, вода).
5. Проанализировать углеродный след и требования к устойчивому развитию.
6. Для каждого фактора — карточка (аналогично шагу 1).

### Шаг 6 — Правовые факторы (Legal, PESTEL)
1. Проанализировать регуляторную среду: лицензирование, сертификация, комплаенс.
2. Оценить защиту интеллектуальной собственности и патентное право.
3. Проанализировать трудовое законодательство и его влияние на кадровую стратегию.
4. Оценить защиту данных и приватности (GDPR, локальные аналоги).
5. Проанализировать антимонопольное регулирование и его влияние на рынок.
6. Для каждого фактора — карточка (аналогично шагу 1).

### Шаг 7 — Scoring и приоритизация
1. Рассчитать итоговый балл для каждого фактора: `Score = Влияние × Вероятность`.
2. Определить ТОП-факторы по каждой категории (Score ≥ 12).
3. Построить тепловую карту:

   | | Вероятность 1 | 2 | 3 | 4 | 5 |
   |--|:---:|:---:|:---:|:---:|:---:|
   | **Влияние 5** | 5 | 10 | 15 🟡 | 20 🔴 | 25 🔴 |
   | **4** | 4 | 8 | 12 🟡 | 16 🔴 | 20 🔴 |
   | **3** | 3 | 6 | 9 | 12 🟡 | 15 🟡 |
   | **2** | 2 | 4 | 6 | 8 | 10 |
   | **1** | 1 | 2 | 3 | 4 | 5 |

   🔴 Score ≥ 16: критический — немедленное реагирование.
   🟡 Score 12-15: значимый — мониторинг и план действий.
   Без маркера < 12: фоновый — отслеживать тренд.

4. Выделить критические факторы, требующие немедленного реагирования.

### Шаг 8 — Сценарный анализ
1. Определить 2-3 ключевых неопределённости (факторы с высоким Score и неоднозначным трендом).
2. Построить 3 сценария:

   | Сценарий | Ключевые допущения | Влияние на бизнес | Вероятность | Стратегическая реакция |
   |----------|-------------------|-------------------|:-----------:|----------------------|
   | Оптимистичный | [Какие факторы улучшаются] | [Что это даёт] | XX% | [Что делать] |
   | Базовый | [Текущие тренды продолжаются] | [Что ожидать] | XX% | [Что делать] |
   | Пессимистичный | [Какие факторы ухудшаются] | [Чем грозит] | XX% | [Что делать] |

3. Для пессимистичного сценария — определить trigger points: при каких условиях активировать contingency plan.

### Шаг 9 — Синтез и рекомендации
1. Сформулировать общий профиль макросреды (благоприятная / нейтральная / враждебная).
2. Определить ключевые драйверы изменений на горизонте анализа.
3. Сформулировать стратегические рекомендации:

   | # | Фактор | Тип | Рекомендация | Приоритет | Срок |
   |---|--------|:---:|-------------|:---------:|------|
   | 1 | [Фактор] | 🔴 Риск | [Что делать для митигации] | P1 | [Когда] |
   | 2 | [Фактор] | 🟢 Возм. | [Как использовать] | P2 | [Когда] |

4. Определить периодичность пересмотра (ежеквартально для волатильных рынков, ежегодно для стабильных).

## Пример — EdTech-платформа, рынок РФ, горизонт 3 года

**Контекст:** Онлайн-платформа корпоративного обучения, B2B SaaS, РФ, горизонт 2026-2029.

## Валидация (Quality Gate)

- [ ] Все категории (P/E/S/T или P/E/S/T/E/L) проработаны с минимум 3 факторами
- [ ] Каждый фактор: влияние (1-5), вероятность (1-5), тренд (↑/↓/→), характер (🟢/🔴/🟡), горизонт
- [ ] Характер обоснован: возможность / риск / двойственный — по правилу из протокола
- [ ] Scoring корректен: Score = Влияние × Вероятность, ТОП-факторы (≥12) выделены
- [ ] Тренды обоснованы с указанием причин и источников
- [ ] Отсутствуют дублирования между категориями
- [ ] Сценарный анализ: 3 сценария с допущениями, влиянием, вероятностью и реакцией
- [ ] Trigger points определены для пессимистичного сценария
- [ ] Рекомендации конкретны, приоритизированы (P1/P2/P3) и привязаны к факторам
- [ ] Анализ учитывает заданный горизонт и географический фокус
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

## Handoff
Результат `$pest-analysis` является входом для:
- **Стратег / Медиатор** — макросреда для стратегических решений и оценки рисков.
- **`swot-analysis`** — внешние Opportunities и Threats формируются из PESTEL-факторов.
- **`tam-sam-som`** — макро-ограничения на размер рынка (регуляторные, экономические).
- **`ansoff-matrix`** — макро-риски для стратегий роста на новых рынках.
- **`competitive-analysis`** — регуляторные барьеры входа для конкурентов.

Формат передачи: сводная таблица факторов + профиль макросреды + сценарный анализ + рекомендации. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Перечисление факторов без scoring | Список без приоритетов бесполезен | Каждый фактор — Score = Влияние × Вероятность |
| Игнорирование трендов | Статичный снимок не показывает динамику | Тренд ↑/↓/→ + горизонт + причина |
| Смешение категорий | Правовой фактор в «Политических» путает анализ | Строго разделять по PESTEL-категориям |
| Фокус только на негативных факторах | Теряются возможности | Характер: 🟢 возможность / 🔴 риск / 🟡 двойственный |
| Отсутствие географической привязки | Факторы различаются по странам | Всегда указывать регион / страну |
| Одинаковые баллы для всех факторов | Нет дифференциации, нет приоритетов | Использовать полную шкалу 1-5, обосновывать каждый балл |
| Сценарии без trigger points | Непонятно, когда активировать contingency | Trigger: конкретное условие → конкретное действие |
| Факторы без источников | Невозможно верифицировать, субъективные оценки | Каждый факт — с источником и датой; >12 мес = ⚠️ |
