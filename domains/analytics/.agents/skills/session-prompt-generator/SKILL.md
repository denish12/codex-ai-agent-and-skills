# Session Prompt Generator — Генерация промпта для следующей сессии

## Когда использовать
- При **завершении сессии** в Full Pipeline (`/analyze`) — перед межсессионным `$handoff`.
- При **необходимости передать проект** другому оператору — формирование самодостаточного промпта.
- При **запросе пользователя** «сгенерируй промпт для продолжения» — в любой момент.
- При **аварийном завершении** сессии — сохранение максимума контекста для восстановления.

> **Ключевой принцип:** следующая сессия начинается с **НУЛЯ** контекста. Промпт должен содержать всё необходимое, чтобы новая сессия продолжила работу без потерь.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Состояние `$board` | ✅ | Текущая доска задач со всеми статусами |
| Завершённые deliverables | ✅ | Полные артефакты текущей и предыдущих сессий |
| Текущий гейт | ✅ | Где остановились (ID гейта из `$board`) |
| Следующий гейт | ✅ | Куда переходим (ID следующего гейта) |
| Открытые вопросы | ⬚ | Блокеры, неразрешённые конфликты, вопросы к пользователю |
| Номер текущей сессии | ✅ | Какая сессия завершается (1-5) |
| Название проекта | ✅ | Название аналитического проекта |

## Протокол

### Шаг 0 — Сбор состояния
1. Прочитать текущее состояние `$board` — все гейты, статусы, deliverables.
2. Собрать все завершённые артефакты текущей сессии.
3. Загрузить кумулятивный индекс из предыдущих сессий (если есть).
4. Определить следующий гейт и его требования из `$gates`.

### Шаг 1 — Определение критических данных
1. Выделить данные, которые **невозможно восстановить** без предыдущего контекста:
   - Ответы пользователя на интервью (из INT-01).
   - Собранные данные исследований (из RES-xx).
   - Результаты анализа (из AN-xx).
   - Стратегические рекомендации (из ST-xx).
   - Вердикт Медиатора (из MED-01).
2. Выделить данные, которые нужны следующему агенту для работы.
3. Пометить данные, которые могут быть восстановлены повторным поиском (но лучше передать).

### Шаг 2 — Определение агентов и скилов для следующей сессии
1. По `$board` определить, какие гейты предстоят в следующей сессии.
2. Для каждого гейта определить агента и его скилы (из AGENTS.md).
3. Сформировать список: какие агенты загружать, какие скилы использовать.

### Шаг 3 — Формирование промпта
1. Заполнить шаблон промпта (см. ниже).
2. Включить ПОЛНЫЙ текст критических данных (не ссылки, не резюме).
3. Сформулировать конкретную задачу для следующей сессии.
4. Указать, какие агенты и скилы активировать.

### Шаг 4 — Валидация промпта
1. Проверить: можно ли начать работу, имея ТОЛЬКО этот промпт?
2. Убедиться, что нет ссылок на «предыдущий контекст» или «как обсуждалось ранее».
3. Убедиться, что все критические данные включены полным текстом.
4. Проверить, что задача для следующей сессии конкретна и actionable.

### Шаг 5 — Сохранение
1. Включить промпт в файл `$handoff`: `docs/analytics/session-N-handoff.md`.
2. Показать промпт пользователю для сохранения / копирования.
3. Предупредить: «Этот промпт — вставьте его в начале следующей сессии».

## Структура генерируемого промпта

Промпт следующей сессии должен состоять из следующих блоков:

### Блок 1 — Идентификация проекта
- Название проекта.
- Режим пайплайна (Full / Quick).
- Номер следующей сессии.
- Какие сессии завершены.

### Блок 2 — Инструкция по загрузке
- Прочитать AGENTS.md домена analytics.
- Загрузить агентов, которые нужны в этой сессии.
- Загрузить скилы: `$board`, `$handoff`, `$gates` + специфичные для гейтов.

### Блок 3 — Выполненная работа (резюме)
- Что было сделано в предыдущих сессиях (2-3 предложения на сессию).
- Ключевые решения и их обоснования.

### Блок 4 — Критические данные
- ПОЛНЫЙ ТЕКСТ ключевых артефактов, необходимых для продолжения.
- Данные исследований, выводы анализов, рекомендации.
- Ответы пользователя на интервью (если были).

### Блок 5 — Текущее состояние
- Доска задач (`$board`) в текущем состоянии.
- Какой гейт следующий.
- Открытые вопросы и блокеры.

### Блок 6 — Задача для сессии
- Конкретная формулировка: что нужно сделать в этой сессии.
- Ожидаемый результат (deliverables).
- Ограничения и особенности.

## Правила формирования промпта

### Обязательные правила
1. **Самодостаточность** — промпт должен работать без ЛЮБОГО предыдущего контекста.
2. **Полнота данных** — включать ПОЛНЫЙ текст, не ссылки и не резюме.
3. **Конкретность задачи** — не «продолжи анализ», а «выполни RES-02: исследуй [тема] с позиции Критиков, данные Аналитиков: [данные]».
4. **Структурированность** — чёткие блоки, маркированные заголовками.
5. **Загрузка скилов** — явно указать какие агенты и скилы загрузить.

### Запрещённые элементы
1. Ссылки на «предыдущую сессию» без полного содержания.
2. «Как мы обсуждали» — нет контекста, нечего обсуждать.
3. «См. результаты выше» — нет «выше», всё должно быть в промпте.
4. Сокращённые данные — «основные выводы: ...». Нужны ВСЕ выводы.
5. Предположения о контексте — «вы уже знаете, что...».

## Валидация (Quality Gate)

Промпт считается корректным если:

- [ ] Промпт самодостаточен — можно начать работу без внешнего контекста
- [ ] Название проекта и режим указаны
- [ ] Номер сессии и история сессий указаны
- [ ] Все критические данные включены полным текстом
- [ ] Доска задач (`$board`) включена в текущем состоянии
- [ ] Следующий гейт и задача сформулированы конкретно
- [ ] Агенты и скилы для загрузки перечислены
- [ ] Открытые вопросы и блокеры зафиксированы
- [ ] Нет запрещённых элементов (ссылки, сокращения, предположения)
- [ ] Промпт проверен на читаемость — понятен без дополнительного контекста

## Handoff
Результат `$session-prompt-generator` является входными данными для: следующей сессии (через копирование промпта), файла `$handoff` (`docs/analytics/session-N-handoff.md`).

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Промпт без данных исследований | Следующая сессия потеряет все собранные факты | Включить ПОЛНЫЙ текст данных |
| «Продолжи работу» | Нет контекста, с чего продолжать | Конкретная задача: гейт, действие, ожидаемый результат |
| Ссылки на файлы без содержания | Файлы могут быть недоступны | Включить содержание прямо в промпт |
| Промпт без доски задач | Непонятно где мы в пайплайне | Всегда включать `$board` |
| Сокращённые артефакты | Потеря критических деталей | Полный текст каждого артефакта |
| Пропуск открытых вопросов | Проблемы всплывут снова | Зафиксировать все блокеры и вопросы |
| Нет инструкции по загрузке | Сессия не загрузит нужных агентов | Явно указать agents и skills |
| Промпт >50% «воды» | Размытие критических данных | Структурированность: блоки, таблицы, конкретика |
