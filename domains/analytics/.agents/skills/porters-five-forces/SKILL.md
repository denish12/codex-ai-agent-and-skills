# 5 сил Портера — Анализ конкурентных сил отрасли

## Когда использовать
- При оценке **привлекательности отрасли** для входа или инвестиций.
- При разработке **конкурентной стратегии** — понимание расстановки сил на рынке.
- При анализе **изменений в отрасли** — как новые тренды меняют баланс сил.
- При подготовке **M&A решений** — оценка конкурентной среды целевой отрасли.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Отрасль / сегмент | ✅ | Чёткое определение анализируемой отрасли и её границ |
| Географический фокус | ✅ | Страна / регион / глобальный рынок |
| Позиция компании | ⬚ | Текущая или планируемая роль в отрасли |
| Ключевые конкуренты | ⬚ | Список основных игроков для контекста |
| Горизонт анализа | ⬚ | Период оценки (по умолчанию — текущее состояние + 2-3 года) |
| Цель анализа | ⬚ | Вход в рынок, стратегия, инвестиции, M&A |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. Отраслевые отчёты — IBISWorld, Statista, отраслевые ассоциации.
2. Финансовая отчётность — годовые отчёты игроков, SEC filings.
3. Конкурентная разведка — рыночные доли, ценообразование, продуктовые линейки.
4. Данные о поставщиках и покупателях — концентрация, switching costs.
5. Регуляторные базы — барьеры входа, лицензирование, патенты.

> Каждый факт — с указанием источника и даты. Данные старше 12 месяцев — пометить ⚠️ и верифицировать через `web-research`.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `competitive-analysis` | Данные о конкурентах, доли рынка, моаты (шаг 5) | Если нет данных о конкурентном ландшафте |
| `pest-analysis` | Макрофакторы, влияющие на силы: регуляторные барьеры, технологические тренды (шаги 1, 4) | Если нужен контекст макросреды |
| `tam-sam-som` | Размер и рост рынка для оценки rivalry (шаг 5) | Если нет данных о темпах роста отрасли |
| `blue-ocean-strategy` | Стратегия за пределами 5 сил, если отрасль непривлекательна (шаг 7) | Если средний балл ≥ 3.5 — рассмотреть Blue Ocean |
| `web-research` | Верификация отраслевых данных, свежие отчёты | Если данные отсутствуют или устарели (>12 мес) |
| `swot-analysis` | 5 сил → Threats и Opportunities для SWOT | После анализа — для интеграции в стратегию |

## Протокол

### Шаг 0 — Подготовка контекста
1. Чётко определить границы отрасли:

   | Параметр | Определение |
   |----------|-------------|
   | Продуктовые границы | [Что входит в отрасль, что нет] |
   | Географические границы | [Страна / регион / глобально] |
   | Ценовой сегмент | [Premium / Mid / Value / Все] |
   | Фаза жизненного цикла | Рост / Зрелость / Спад |

2. Идентифицировать ключевых игроков: лидеры, претенденты, нишевые.
3. Зафиксировать цель анализа и горизонт.

### Шаг 1 — Угроза новых участников (Threat of New Entrants)
Оценить каждый подфактор по шкале 1-5 (1 = высокий барьер / низкая угроза, 5 = низкий барьер / высокая угроза):

| Подфактор | Оценка (1-5) | Обоснование | Источник |
|-----------|:------------:|-------------|----------|
| Экономия масштаба | X | [Минимальный эффективный масштаб] | [Источник] |
| Капиталоёмкость | X | [Объём начальных инвестиций] | [Источник] |
| Доступ к каналам дистрибуции | X | [Сложность выхода на полку/платформу] | [Источник] |
| Патенты и IP | X | [Наличие защиты] | [Источник] |
| Регуляторные барьеры | X | [Лицензии, сертификации] | [Источник] |
| Эффект обучения | X | [Кривая опыта действующих игроков] | [Источник] |

**Итоговый балл силы = среднее подфакторов**, округлённое до 0.5.

Дополнительно: вероятность появления новых игроков на горизонте анализа, ожидаемая реакция действующих.

**Итоговый балл = среднее подфакторов.**

### Шаг 4 — Угроза товаров-заменителей (Threat of Substitutes)
Оценить каждый подфактор (1 = низкая угроза, 5 = высокая угроза):

| Подфактор | Оценка (1-5) | Обоснование | Источник |
|-----------|:------------:|-------------|----------|
| Наличие субститутов из других отраслей | X | [Какие, сколько] | [Источник] |
| Соотношение цена/качество субститута | X | [Лучше/хуже/равно] | [Источник] |
| Switching costs на субститут | X | [Легко переключиться?] | [Источник] |
| Склонность покупателей к переключению | X | [Готовы ли менять] | [Источник] |
| Технологические тренды | X | [Создают ли новые субституты] | [Источник] |

> **Важно:** субституты — это продукты **из другой отрасли**, решающие **ту же потребность**. Uber и такси = конкуренты (одна отрасль). Zoom и авиакомпании для бизнес-встреч = субституты (разные отрасли, одна потребность).

**Итоговый балл = среднее подфакторов.**

| Стратегическая важность | X | [Готовы ли игроки воевать за рынок] | [Источник] |

**Итоговый балл = среднее подфакторов.**

### Шаг 6 — Общая привлекательность отрасли
1. Рассчитать средний балл по 5 силам.
2. Определить доминирующую силу — та, которая сильнее всего влияет на прибыльность.
3. Оценить привлекательность:

   | Средний балл | Привлекательность | Рекомендация |
   |:------------:|-------------------|-------------|
   | 1.0-2.0 | 🟢 Высокая | Входить / инвестировать |
   | 2.1-3.0 | 🟡 Средняя | Входить с чёткой стратегией дифференциации |
   | 3.1-3.5 | 🟠 Низкая | Входить только с сильным конкурентным преимуществом |
   | 3.6-5.0 | 🔴 Непривлекательная | Не входить, или рассмотреть `$blue-ocean-strategy` |

## Пример — SaaS CRM для SMB, рынок РФ

**Контекст:** Рынок SaaS CRM для малого и среднего бизнеса, РФ, горизонт 2026-2029. Цель: оценка привлекательности для входа.

### Scoring подфакторов

**1. Угроза новых участников — 3.7/5**

| Подфактор | Оценка | Обоснование |
|-----------|:------:|-------------|
| Экономия масштаба | 3 | Средний порог: облачная инфраструктура масштабируется |
| Капиталоёмкость | 2 | Низкая: MVP за $200K-500K, облако |
| Доступ к каналам | 4 | Сложно: marketplace переполнен, SEO конкурентен |
| Патенты и IP | 5 | Нет: SaaS CRM не защищён патентами |
| Регуляторные барьеры | 2 | Низкие: ФЗ-152 (перс. данные), но стандартный |
| Эффект обучения | 4 | Средний: amoCRM и Битрикс24 = 10+ лет на рынке |

**2. Сила поставщиков — 2.0/5**

| Подфактор | Оценка | Обоснование |
|-----------|:------:|-------------|
| Концентрация | 2 | Низкая: AWS/Yandex Cloud/VK Cloud — 3+ провайдера |
| Уникальность | 1 | Облачная инфраструктура = commodity |
| Switching costs | 3 | Средние: миграция между облаками ~2-4 недели |
| Прямая интеграция | 2 | Низкая: облачные провайдеры не делают CRM |
| Субституты ресурса | 2 | Есть: on-premise как альтернатива |

**3. Сила покупателей — 3.8/5**

| Подфактор | Оценка | Обоснование |
|-----------|:------:|-------------|
| Концентрация | 2 | Низкая: тысячи SMB, ни один не > 1% выручки |
| Switching costs | 4 | Низкие: миграция CRM за 1-2 недели, данные экспортируемы |
| Ценовая чувствительность | 5 | Высокая: SMB считает каждый рубль |
| Информированность | 4 | Высокая: G2, отзывы, сравнения легко доступны |
| Обратная интеграция | 1 | Нет: SMB не будет писать свою CRM |
| Стандартизированность | 4 | Высокая: базовый CRM = commodity |

**4. Угроза субститутов — 3.4/5**

| Подфактор | Оценка | Обоснование |
|-----------|:------:|-------------|
| Наличие | 4 | Notion, Excel, Telegram-боты, Google Sheets |
| Цена/качество | 3 | Хуже по функционалу, но бесплатно |
| Switching costs | 4 | Низкие: с Excel на CRM и обратно — просто |
| Склонность | 3 | Средняя: SMB привыкли к «бесплатному» |
| Техно-тренды | 3 | AI-ассистенты могут заменить часть CRM-функций |

**5. Rivalry — 4.2/5**

| Подфактор | Оценка | Обоснование |
|-----------|:------:|-------------|
| Кол-во конкурентов | 5 | amoCRM, Битрикс24, Мегаплан, HubSpot, Notion + 20 мелких |
| Рост отрасли | 3 | Умеренный: +12% YoY (не хватает для всех) |
| Дифференциация | 3 | Средняя: базовые фичи одинаковы, дифференциация через UX/AI |
| Фиксированные затраты | 4 | Высокие R&D, давление на цены |
| Барьеры выхода | 4 | Высокие: клиентская база, репутация, команда |
| Стратегическая важность | 5 | Высокая: CRM = ядро экосистемы для всех игроков |

### Сводка

| Сила | Балл | Тренд (2-3г) | Драйвер |
|------|:----:|:------------:|---------|
| Новые участники | 3.7 | ↑ 4.0 | AI снижает стоимость MVP |
| Поставщики | 2.0 | → 2.0 | Облачный рынок стабилен |
| Покупатели | 3.8 | ↑ 4.0 | Растёт информированность, больше альтернатив |
| Субституты | 3.4 | ↑ 3.8 | AI-ассистенты наступают |
| Rivalry | 4.2 | → 4.2 | Консолидация компенсирует рост |
| **Средний** | **3.4** | **3.6** | |

**Привлекательность:** 🟠 Низкая (3.4). Через 2-3 года: 🔴 Непривлекательная (3.6).
**Доминирующая сила:** Rivalry (4.2) — интенсивная конкуренция, 5+ серьёзных игроков.

### Рекомендации

| Сила | Действие | Приоритет |
|------|----------|:---------:|
| Rivalry (4.2) | Дифференциация через AI — единственный незанятый фактор | P1 |
| Покупатели (3.8) | Увеличить switching costs: глубокие интеграции, custom workflows | P1 |
| Субституты (3.4) | Позиционировать CRM как «больше, чем Excel»: аналитика, автоматизация | P2 |
| Новые участники (3.7) | Строить бренд и community — барьер, который не купишь | P2 |

**Вердикт:** Входить только с сильной дифференциацией (AI-first CRM). Без уникального преимущества — рынок слишком конкурентен. Рассмотреть `$blue-ocean-strategy` для поиска незанятого пространства.

## Валидация (Quality Gate)

- [ ] Все 5 сил проанализированы с подфакторами (минимум 5 на каждую силу)
- [ ] Каждый подфактор оценён 1-5 с обоснованием и источником
- [ ] Итоговый балл силы = среднее подфакторов (не субъективный)
- [ ] Границы отрасли чётко определены (продукт, гео, сегмент, фаза)
- [ ] Общая привлекательность рассчитана, доминирующая сила определена
- [ ] Динамика сил на горизонте 2-3 года с драйверами и trigger points
- [ ] Субституты — из другой отрасли (не путать с конкурентами)
- [ ] Стратегические рекомендации привязаны к конкретным силам и приоритизированы
- [ ] Вердикт: входить / инвестировать / защищаться / уходить / Blue Ocean
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

## Handoff
Результат `$porters-five-forces` является входом для:
- **Стратег / Медиатор** — конкурентная среда для стратегических решений.
- **`competitive-analysis`** — rivalry-данные для детального конкурентного анализа.
- **`swot-analysis`** — 5 сил → Threats (высокие силы) и Opportunities (низкие силы).
- **`blue-ocean-strategy`** — если отрасль непривлекательна (≥ 3.5) — поиск Blue Ocean.
- **`ansoff-matrix`** — привлекательность текущей vs новых отраслей для стратегии роста.

Формат передачи: сводная таблица сил с баллами + подфакторы + динамика + рекомендации. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Слишком широкие границы | «Технологическая отрасль» — 100 разных рынков | Конкретный сегмент: «SaaS CRM для SMB в РФ» |
| Балл силы «на глаз» | Субъективная оценка, невоспроизводимая | Балл = среднее подфакторов (каждый 1-5 с обоснованием) |
| Игнорирование динамики | Статичный анализ устаревает | Прогноз каждой силы на 2-3 года + trigger points |
| Путаница конкуренты / субституты | Uber и такси = конкуренты; Zoom и авиакомпании = субституты | Субституты — другая отрасль, та же потребность |
| Без стратегических импликаций | Анализ ради анализа | Каждая сила → конкретное стратегическое действие |
| Одинаковые баллы для всех сил | Нет дифференциации | Полная шкала 1-5, доминирующая сила |
| Подфакторы без данных | Оценки повисают в воздухе | Каждый подфактор — с источником |
| Привлекательность без вердикта | Число без рекомендации бесполезно | Чёткий вердикт: входить / защищаться / Blue Ocean |
