# TAM/SAM/SOM — Оценка размера рынка

## Когда использовать
- При выходе на новый рынок или запуске нового продукта — для обоснования потенциала.
- При подготовке инвестиционных материалов (pitch deck, бизнес-план) — инвесторы требуют размер рынка.
- При стратегическом планировании — выбор между рынками, вертикалями или географиями.
- При пересмотре стратегии — если рынок изменился (регуляция, технологии, конкуренция).

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Продукт / сервис | ✅ | Что именно предлагаем рынку |
| Целевая география | ✅ | Страны / регионы для оценки |
| Бизнес-модель | ✅ | B2B / B2C / B2B2C / SaaS / Marketplace и т.д. |
| Ценовая модель | ✅ | Подписка / транзакция / лицензия / freemium |
| Средний чек (ACV) | ⬚ | Если известен; иначе будет оценён |
| Целевые сегменты | ⬚ | Если уже определены (размер компании, отрасль, демография) |
| Конкуренты | ⬚ | Известные игроки на рынке |
| Горизонт прогноза | ⬚ | На сколько лет (по умолчанию — 3-5 лет) |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — отраслевые отчёты (Statista, Grand View Research, Mordor Intelligence, IBISWorld, McKinsey, Gartner).
2. **Финансовые данные** — публичная отчётность конкурентов (SEC filings, годовые отчёты).
3. **Статистические базы** — данные госстатистик (Росстат, Eurostat, Census Bureau, World Bank).
4. **Отраслевые ассоциации** — профильные объединения, торговые палаты.
5. **Данные от пользователя** — внутренняя аналитика, CRM, текущие продажи.

> Для каждой цифры указать источник и дату. Данные старше 24 месяцев — пометить ⚠️ и верифицировать через `web-research`. Экстраполяции — помечать как 🔮 Assumed.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `web-research` | Отраслевые отчёты, размеры рынков, CAGR (шаги 2-4) | Основной инструмент — вызывать для каждого уровня воронки |
| `pest-analysis` | Макро-ограничения на TAM: регуляторные, экономические (шаг 3) | Если регуляция или макросреда ограничивают рынок |
| `competitive-analysis` | Доли конкурентов для обоснования capture rate SOM (шаг 4) | Если неизвестна конкурентная структура рынка |
| `icp-buyer-persona` | Фильтры SAM: ICP определяет целевой сегмент (шаг 3) | Если целевые сегменты не определены |
| `ansoff-matrix` | TAM новых рынков для стратегий Market Development и Diversification | После расчёта — для оценки стратегий роста |
| `blue-ocean-strategy` | TAM нового рыночного пространства (не-клиенты) | Если основной рынок мал — оценить Blue Ocean TAM |

## Протокол

### Шаг 0 — Сбор контекста
1. Проверить наличие всех обязательных входных данных. Если чего-то не хватает — запросить у пользователя.
2. Определить единицу измерения (USD, EUR, RUB) и временной горизонт.
3. Чётко сформулировать, какую проблему решает продукт.
4. Определить категорию рынка (индустрия → подкатегория → ниша).
5. Зафиксировать границы рынка: что включаем, что исключаем.

### Шаг 1 — TAM (Total Addressable Market)
Оценить **двумя методами** для перекрёстной проверки:

**Top-down:**
1. Найти общий объём индустрии (из отраслевых отчётов).
2. Применить фильтр по категории продукта.
3. Применить фильтр по географии.
4. Зафиксировать: источник, год, методологию отчёта.

**Bottom-up:**
1. Определить общее количество потенциальных клиентов в категории.
2. Умножить на средний чек (ACV) × частоту покупок в год.
3. `TAM (bottom-up) = Кол-во клиентов × ACV × Частота`.
4. Зафиксировать: источник данных о кол-ве клиентов, обоснование ACV.

**Сверка методов:**

| Условие | Действие |
|---------|----------|
| Разброс < 30% | Использовать среднее. Нормальное расхождение |
| Разброс 30-100% | Исследовать причину. Зафиксировать оба числа, обосновать выбор финального |
| Разброс > 100% | 🔴 Проблема: пересмотреть границы рынка или источники. Не усреднять — найти ошибку |

### Шаг 2 — SAM (Serviceable Addressable Market)
Применить фильтры к TAM:

| # | Фильтр | Описание | Множитель | Обоснование | Источник |
|---|--------|----------|:---------:|-------------|----------|
| 1 | География | Где реально работаем | ×XX% | [Почему] | [Источник] |
| 2 | Сегмент | B2B: размер, отрасль; B2C: демография | ×XX% | [Почему] | [Источник] |
| 3 | Регуляция | Лицензии, сертификации, запреты | ×XX% | [Почему] | [Источник] |
| 4 | Технологии | Язык, платформа, интеграции | ×XX% | [Почему] | [Источник] |

`SAM = TAM × Фильтр₁ × Фильтр₂ × ... × ФильтрN`

> Если есть `$icp-buyer-persona` — использовать ICP-критерии как фильтры SAM (идеальный + приемлемый).

### Шаг 3 — SOM (Serviceable Obtainable Market)
Определить реалистичную долю SAM:

1. Оценить capture rate:

   | Стадия | Типичный capture rate | Обоснование |
   |--------|:---------------------:|-------------|
   | Pre-product (идея) | 0.5-1% | Нет продукта, нет клиентов |
   | Стартап (MVP, <$1M ARR) | 1-3% | Первые клиенты, PMF ищется |
   | Растущий ($1M-10M ARR) | 3-5% | PMF найден, масштабирование |
   | Зрелый (>$10M ARR) | 5-15% | Устоявшийся бренд и каналы |
   | Лидер рынка | 15-30% | Доминирующая позиция |

2. Учесть конкурентную интенсивность:
   - Если есть `$competitive-analysis` — использовать доли конкурентов для обоснования.
   - Если нет — оценить по количеству игроков и их размеру.

3. `SOM = SAM × Capture Rate`.
4. Рассчитать для 3 горизонтов: год 1, год 3, год 5.

### Шаг 4 — Анализ чувствительности
Оценить, как изменится SOM при вариации ключевых параметров:

| Параметр | Вариация | Влияние на SOM |
|----------|:--------:|:--------------:|
| ACV | ±20% | SOM ±20% |
| Capture rate | ±50% | SOM ±50% |
| Размер целевого сегмента | ±30% | SAM/SOM ±30% |

3 сценария:

| Сценарий | TAM | SAM | SOM (год 3) | Ключевое допущение |
|----------|:---:|:---:|:-----------:|---------------------|
| Оптимистичный | $XB | $XM | $XM | [Что должно сбыться] |
| Базовый | $XB | $XM | $XM | [Текущие тренды] |
| Пессимистичный | $XB | $XM | $XM | [Что может пойти не так] |

### Шаг 5 — Динамика и рост (YoY)
1. Определить CAGR рынка (историческая + прогнозная).
2. Факторы роста: технологии, регуляция, поведение потребителей.
3. Факторы торможения: насыщение, замещение, кризисы.
4. Прогноз TAM/SAM/SOM на горизонте 3-5 лет.

## Пример — B2B SaaS: платформа корпоративного обучения, РФ

**Контекст:** AI-powered платформа корпоративного обучения, B2B SaaS, подписка per seat, РФ. ACV $6K/год. Горизонт: 2026-2031.

### TAM

**Top-down:**
- Рынок EdTech РФ = $4.2B (Smart Ranking, 2026) ✅
- Корпоративное обучение = 43% EdTech = $1.8B (ВШЭ, 2026) ✅
- **TAM (top-down) = $1.8B**

**Bottom-up:**
- Компании в РФ с 50+ сотрудниками: ~85 000 (Росстат, 2025) ✅
- Средний бюджет на обучение: $18K/год (HH.ru Research, 2026) ⚠️ (1 источник)
- **TAM (bottom-up) = 85 000 × $18K = $1.53B**

**Сверка:** разброс 15% ($1.53B vs $1.8B) — нормальный. **TAM = $1.65B** (среднее).

### SAM

| # | Фильтр | Множитель | Обоснование | Источник |
|---|--------|:---------:|-------------|----------|
| 1 | Только SaaS-решения (не тренеры, не офлайн) | ×35% | 35% бюджетов идёт на digital | ВШЭ, 2026 ✅ |
| 2 | Компании 50-2000 чел. (наш sweet spot) | ×60% | Ent >2000 — SAP/Oracle; <50 — не покупают | CRM + бенчмарк |
| 3 | IT, финтех, ритейл, производство | ×70% | Остальные отрасли — низкая цифровизация | Росстат ⚠️ |

**SAM = $1.65B × 0.35 × 0.60 × 0.70 = $243M**

### SOM

| Горизонт | Capture Rate | SOM | Обоснование |
|:--------:|:-----------:|:---:|-------------|
| Год 1 | 1.5% | $3.6M | Стартап, 50 клиентов, $6K ACV |
| Год 3 | 4% | $9.7M | PMF найден, 160 клиентов |
| Год 5 | 8% | $19.4M | Лидер в нише AI-обучение |

### Анализ чувствительности (SOM год 3)

| Сценарий | TAM | SAM | SOM | Допущение |
|----------|:---:|:---:|:---:|-----------|
| Оптимистичный | $2.1B | $310M | $15.5M | CAGR 25% + госсубсидии на 50% |
| Базовый | $1.65B | $243M | $9.7M | CAGR 18%, текущая конкуренция |
| Пессимистичный | $1.3B | $182M | $5.5M | Инфляция давит бюджеты, capture 3% |

### Динамика

| Год | TAM | CAGR | SAM | SOM |
|:---:|:---:|:----:|:---:|:---:|
| 2026 | $1.65B | — | $243M | $3.6M |
| 2027 | $1.95B | 18% | $287M | $5.7M |
| 2029 | $2.71B | 18% | $399M | $15.9M |
| 2031 | $3.77B | 18% | $555M | $44.4M |

**Факторы роста:** дефицит кадров (HR-бюджеты ↑), AI-персонализация, госпрограмма «Кадры».
**Факторы торможения:** инфляция (8-10%), конкуренция (Яндекс, Skillbox), регуляторные барьеры (ФЗ-152).

## Валидация (Quality Gate)

- [ ] TAM рассчитан двумя методами (top-down + bottom-up)
- [ ] Разброс между методами зафиксирован и обработан по правилу (<30% / 30-100% / >100%)
- [ ] SAM имеет чёткие фильтры, каждый обоснован с множителем и источником
- [ ] SOM основан на реалистичном capture rate с обоснованием стадией компании
- [ ] SOM рассчитан для 3 горизонтов (год 1 / 3 / 5)
- [ ] Анализ чувствительности: 3 сценария с вариацией ACV, capture rate, сегмента
- [ ] CAGR и прогноз на 3-5 лет с факторами роста и торможения
- [ ] TAM ≥ SAM ≥ SOM (нет внутренних противоречий)
- [ ] Единицы измерения и валюта консистентны
- [ ] Все цифры имеют источник; >24 мес = ⚠️; экстраполяции = 🔮
- [ ] Конфликтующие источники зафиксированы с объяснением выбора

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$tam-sam-som` является входом для:
- **Стратег / Медиатор** — размер рынка как фактор при выборе стратегии.
- **`unit-economics`** — размер рынка для калькуляции потенциальной выручки.
- **`competitive-analysis`** — рыночные доли конкурентов для бенчмаркинга.
- **`ansoff-matrix`** — TAM новых рынков для стратегий Market Development.
- **`bcg-matrix`** — рост рынка для классификации бизнес-единиц.
- **`blue-ocean-strategy`** — TAM нового пространства (не-клиенты).

Формат передачи: воронка TAM/SAM/SOM + фильтры + чувствительность + динамика + допущения. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Только top-down | Одна методология даёт искажённую картину | Всегда два метода: top-down + bottom-up + сверка |
| TAM = вся индустрия | Завышение в десятки раз, потеря доверия | Чётко ограничить категорию продукта |
| SOM без capture rate | Фантазия, а не прогноз | Обосновать capture rate стадией компании и конкурентной средой |
| Старые данные | Рынки меняются быстро | Источники ≤ 24 мес; старше = ⚠️ + верификация |
| Без анализа чувствительности | Ложная точность — одно число вместо диапазона | 3 сценария: ACV ±20%, capture ±50%, сегмент ±30% |
| SAM = TAM | Нет фильтрации, бесполезная оценка | Фильтры: гео, сегмент, регуляция, технологии — каждый с множителем |
| Без указания допущений | Невозможно оценить надёжность оценки | Каждое допущение — с влиянием на результат |
| Top-down и bottom-up расходятся > 100% | Один из методов ошибочен, нельзя усреднять | Найти источник ошибки, не усреднять |
