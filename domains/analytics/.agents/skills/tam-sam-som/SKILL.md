# TAM/SAM/SOM — Оценка размера рынка

## Когда использовать
- При выходе на новый рынок или запуске нового продукта — для обоснования потенциала.
- При подготовке инвестиционных материалов (pitch deck, бизнес-план) — инвесторы требуют размер рынка.
- При стратегическом планировании — выбор между рынками, вертикалями или географиями.
- При пересмотре стратегии — если рынок изменился (регуляция, технологии, конкуренция).

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Продукт / сервис | ✅ | Что именно предлагаем рынку |
| Целевая география | ✅ | Страны / регионы для оценки |
| Бизнес-модель | ✅ | B2B / B2C / B2B2C / SaaS / Marketplace и т.д. |
| Ценовая модель | ✅ | Подписка / транзакция / лицензия / freemium |
| Средний чек (ACV) | ⬚ | Если известен; иначе будет оценён |
| Целевые сегменты | ⬚ | Если уже определены (размер компании, отрасль, демография) |
| Конкуренты | ⬚ | Известные игроки на рынке |
| Горизонт прогноза | ⬚ | На сколько лет (по умолчанию — 3-5 лет) |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — отраслевые отчёты (Statista, Grand View Research, Mordor Intelligence, IBISWorld, McKinsey, Gartner).
2. **Финансовые данные** — публичная отчётность конкурентов (SEC filings, годовые отчёты).
3. **Статистические базы** — данные госстатистик (Росстат, Eurostat, Census Bureau, World Bank).
4. **Отраслевые ассоциации** — профильные объединения, торговые палаты.
5. **Данные от пользователя** — внутренняя аналитика, CRM, текущие продажи.

> Для каждой цифры указать источник. Если данные экстраполированы — явно отметить как оценку.

## Протокол

### Шаг 0 — Сбор контекста
Проверить наличие всех обязательных входных данных. Если чего-то не хватает — запросить у пользователя. Определить единицу измерения (USD, EUR, RUB) и временной горизонт.

### Шаг 1 — Определение продукта и категории
- Чётко сформулировать, какую проблему решает продукт.
- Определить категорию рынка (индустрия → подкатегория → ниша).
- Зафиксировать границы рынка: что включаем, что исключаем.

### Шаг 2 — TAM (Total Addressable Market)
Оценить двумя методами для перекрёстной проверки:
- **Top-down:** от общего объёма индустрии → фильтр по категории → фильтр по географии.
- **Bottom-up:** количество потенциальных клиентов × средний чек × частота покупок.
- Указать разброс между методами и финальную оценку.

### Шаг 3 — SAM (Serviceable Addressable Market)
Применить фильтры к TAM:
- Географические ограничения (где реально работаем).
- Сегментные ограничения (B2B: размер компании, отрасль; B2C: демографика, платёжеспособность).
- Регуляторные ограничения (лицензии, сертификации, запреты).
- Технологические ограничения (язык, платформа, интеграции).
- Обосновать каждый фильтр и его влияние на размер рынка.

### Шаг 4 — SOM (Serviceable Obtainable Market)
Определить реалистичную долю SAM, которую можно захватить:
- Текущая рыночная позиция и ресурсы.
- Конкурентная интенсивность (сколько игроков, их доли).
- Реалистичный capture rate (обычно 1-5% для стартапа, 10-20% для зрелого бизнеса).
- Временной горизонт достижения (1 год / 3 года / 5 лет).

### Шаг 5 — Верификация источников
- Проверить каждый источник данных на актуальность (не старше 2 лет).
- Сопоставить цифры из разных источников.
- Выявить и зафиксировать расхождения.
- Отметить допущения и ограничения.

### Шаг 6 — Анализ чувствительности
Оценить, как изменится SOM при вариации ключевых параметров:
- ACV ±20%.
- Capture rate ±50%.
- Размер целевого сегмента ±30%.
- Построить таблицу «оптимистичный / базовый / пессимистичный» сценарий.

### Шаг 7 — Динамика и рост (YoY)
- Определить CAGR рынка (историческая + прогнозная).
- Факторы роста: технологии, регуляция, поведение потребителей.
- Факторы торможения: насыщение, замещение, кризисы.
- Прогноз TAM/SAM/SOM на горизонте 3-5 лет.

## Валидация (Quality Gate)

- [ ] TAM рассчитан двумя методами (top-down + bottom-up) с указанием разброса
- [ ] SAM имеет чёткие фильтры, каждый обоснован
- [ ] SOM основан на реалистичном capture rate с обоснованием
- [ ] Все цифры имеют указание источника
- [ ] Экстраполяции и допущения явно отмечены
- [ ] Проведён анализ чувствительности (мин. 3 сценария)
- [ ] Указан CAGR и прогноз на 3-5 лет
- [ ] Единицы измерения и валюта консистентны по всему документу
- [ ] Нет внутренних противоречий между TAM → SAM → SOM (каждый ≤ предыдущего)

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$tam-sam-som` является входом для:
- **Стратег / Медиатор** — размер рынка как фактор при выборе стратегии.
- `$unit-economics` — размер рынка для калькуляции потенциальной выручки.
- `$competitive-analysis` — рыночные доли конкурентов для бенчмаркинга.
- `$trend-analysis` — рыночная динамика как контекст для трендов.

При передаче — использовать `$handoff` с указанием финальных цифр TAM/SAM/SOM и ключевых допущений.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Только top-down | Одна методология даёт искажённую картину | Всегда два метода: top-down + bottom-up |
| TAM = вся индустрия | Завышение в десятки раз, потеря доверия | Чётко ограничить категорию продукта |
| SOM без capture rate | Фантазия, а не прогноз | Обосновать capture rate бенчмарками |
| Старые данные | Рынки меняются быстро | Источники не старше 2 лет |
| Без анализа чувствительности | Ложная точность | Всегда 3 сценария: опт / база / песс |
| SAM = TAM | Нет фильтрации, бесполезная оценка | Применить все релевантные фильтры |
| Без указания допущений | Невозможно оценить надёжность | Явно фиксировать каждое допущение |
