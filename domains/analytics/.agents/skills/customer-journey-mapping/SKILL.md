# Карта путешествия клиента — Анализ точек контакта от осведомлённости до адвокации

## Когда использовать
- При проектировании нового продукта / сервиса — для понимания полного пути клиента.
- При оптимизации воронки конверсии — выявление узких мест и точек оттока.
- При запуске омниканальной стратегии — согласование опыта через все каналы.
- При росте churn rate — диагностика моментов разочарования и потерь.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Продукт / сервис | ✅ | Что анализируем |
| Целевая персона | ✅ | Для кого строим карту (из `$icp-buyer-persona` или описание) |
| Бизнес-модель | ✅ | B2B / B2C / SaaS / Marketplace и т.д. |
| Текущие каналы | ✅ | Онлайн и офлайн точки контакта |
| Известные проблемы | ⬚ | Где уже есть жалобы / низкая конверсия |
| Данные аналитики | ⬚ | GA, Hotjar, CRM-воронки, NPS, отзывы |
| Конкуренты | ⬚ | Для бенчмаркинга опыта |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — бенчмарки CX по отрасли, лучшие практики customer journey.
2. **Отзывы и обратная связь** — NPS-данные, отзывы на площадках, тикеты поддержки.
3. **Аналитика продукта** — воронки конверсии, тепловые карты, сессионные записи.
4. **CRM-данные** — длина цикла продажи, причины отказов, точки повторных покупок.
5. **Интервью / опросы** — если предоставлены пользователем.

> Для каждого этапа указать источники, подтверждающие паттерн поведения. Данные старше 12 месяцев — пометить ⚠️.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `icp-buyer-persona` | Профиль персоны, цели, контекст (шаг 1) | Если персона не определена |
| `jtbd-analysis` | Jobs-to-be-done на каждом этапе пути (шаги 2-7) | Для глубокого понимания мотивации на каждом этапе |
| `cohort-analysis` | Retention curves, точки churn (шаги 6-7) | Если нужны количественные данные по удержанию |
| `competitive-analysis` | Бенчмаркинг CX конкурентов (шаг 9) | Для сравнения клиентского опыта с конкурентами |
| `rfm-analysis` | Сегменты по частоте/сумме для этапа Retention (шаг 6) | Для понимания разных паттернов удержания |
| `web-research` | Бенчмарки конверсии, CX-практики по отрасли | Если нет данных или нужны отраслевые ориентиры |

## Протокол

### Шаг 0 — Сбор контекста
1. Проверить наличие всех обязательных входных данных.
2. Определить тип карты:

   | Тип | Описание | Когда использовать |
   |-----|----------|-------------------|
   | **As-Is** | Текущее состояние пути клиента | По умолчанию. Диагностика проблем |
   | **To-Be** | Целевое состояние после оптимизации | После as-is, на основе gap-анализа |

3. Определить наличие количественных данных (конверсии, NPS, churn). Если данных нет — работать с качественной моделью + отраслевые бенчмарки.

### Шаг 1 — Определение персоны
1. Зафиксировать ключевые характеристики:

   | Параметр | Значение |
   |----------|----------|
   | Персона | [Имя / роль] |
   | Цель | [Чего хочет достичь] |
   | Контекст | [В какой ситуации обращается к продукту] |
   | Техническая грамотность | Низкая / Средняя / Высокая |
   | Триггер входа | [Что запускает путешествие] |
   | Успешное завершение | [Что считается результатом] |

2. Если персона не определена — вызвать `$icp-buyer-persona`.

### Шаг 2 — Awareness (Осведомлённость)
1. Как клиент впервые узнаёт о проблеме / продукте.
2. Каналы первого касания (поиск, соцсети, рекомендации, реклама, контент).
3. Эмоциональное состояние: любопытство, неосознанная потребность, срочная боль.
4. Действия клиента на этом этапе.
5. Болевые точки и барьеры.
6. **Метрики этапа:** охват, CTR, стоимость привлечения (CPA), brand awareness.

### Шаг 3 — Consideration (Рассмотрение)
1. Как клиент сравнивает варианты и углубляет исследование.
2. Точки контакта: сайт, демо, отзывы, case studies, сравнительные обзоры.
3. Критерии принятия решения для данной персоны.
4. Вопросы и сомнения клиента.
5. Болевые точки и барьеры.
6. **Метрики этапа:** конверсия визит → лид, время на сайте, bounce rate, вовлечённость с контентом.

### Шаг 4 — Decision (Принятие решения)
1. Финальные действия перед покупкой / подпиской.
2. Точки контакта: прайсинг, триал, demo-call, контракт.
3. Кто ещё влияет на решение (ЛПР, stakeholders, influencers).
4. Типичные возражения и как они разрешаются.
5. Болевые точки и барьеры.
6. **Метрики этапа:** конверсия лид → trial/demo, длина цикла продажи, win rate.

### Шаг 5 — Purchase / Onboarding (Покупка / Активация)
1. Процесс оформления покупки или подписки.
2. Первый опыт использования (FTUE — First Time User Experience).
3. Ожидания vs реальность: момент «aha-moment» или разочарования.
4. Точки контакта: оплата, welcome-серия, онбординг, поддержка.
5. Болевые точки и барьеры.
6. **Метрики этапа:** конверсия trial → paid, time-to-value, activation rate, CSAT оnboarding.

### Шаг 6 — Retention (Удержание)
1. Регулярное использование и формирование привычки.
2. Триггеры повторных покупок или расширения (upsell / cross-sell).
3. Моменты разочарования (frustration moments): баги, медленная поддержка, скрытые расходы.
4. Точки контакта: email-коммуникация, в-продуктовые уведомления, саппорт, обновления.
5. Болевые точки и барьеры.
6. **Метрики этапа:** retention rate (M1, M3, M6, M12), churn rate, NPS, expansion revenue.
7. Если есть данные — привязать к `$cohort-analysis` (точки drop-off).

### Шаг 7 — Advocacy (Адвокация)
1. Переход от лояльного клиента к промоутеру.
2. Действия: рекомендации коллегам, отзывы, UGC, участие в community.
3. Триггеры адвокации: исключительный опыт, чувство принадлежности, rewards.
4. Барьеры адвокации: нет механизма, неудобно, нечем поделиться.
5. Точки контакта: реферальная программа, community, события, social sharing.
6. **Метрики этапа:** NPS promoters %, referral rate, review rate, UGC volume, viral coefficient.

### Шаг 8 — Канальный маппинг
1. Для каждого этапа — определить активные каналы и их роль:

   | Этап | Онлайн-каналы | Офлайн-каналы | Основной канал | Согласованность |
   |------|:-------------:|:-------------:|:--------------:|:---------------:|
   | Awareness | [Какие] | [Какие] | [Какой] | ✅ / ⚠️ |
   | Consideration | [Какие] | [Какие] | [Какой] | ✅ / ⚠️ |
   | Decision | [Какие] | [Какие] | [Какой] | ✅ / ⚠️ |
   | Purchase | [Какие] | [Какие] | [Какой] | ✅ / ⚠️ |
   | Retention | [Какие] | [Какие] | [Какой] | ✅ / ⚠️ |
   | Advocacy | [Какие] | [Какие] | [Какой] | ✅ / ⚠️ |

2. Выявить разрывы: где клиент «теряется» при переходе между каналами.
3. Оценить омниканальную согласованность: одинаков ли опыт в разных каналах.
4. Отметить петли и альтернативные пути (клиент возвращается с Decision на Consideration, пропускает этапы и т.д.).

### Шаг 9 — Moments of Truth
Выделить критические моменты, определяющие дальнейшее поведение клиента:

| # | Момент | Этап | Описание | Текущее состояние | Риск (1-5) | Влияние на метрику |
|---|--------|------|----------|-------------------|:----------:|-------------------|
| 1 | Первое впечатление | Awareness → Consideration | Соответствует ли messaging ожиданиям | [Как сейчас] | X | Конверсия визит → лид |
| 2 | Первый успех (aha-moment) | Onboarding | Достигает ли клиент цели быстро | [Как сейчас] | X | Activation rate, M1 retention |
| 3 | Первая проблема | Retention | Качество поддержки и скорость решения | [Как сейчас] | X | Churn rate, NPS |

> Допустимо добавить больше Moments of Truth (до 5), если для данного продукта они критичны.

### Шаг 10 — Gap-анализ и рекомендации
1. Для каждого этапа — сравнить ожидания клиента и реальный опыт:

   | Этап | Ожидание клиента | Реальность (as-is) | Gap | Severity |
   |------|------------------|--------------------|-----|:--------:|
   | [Этап] | [Чего ожидает] | [Что получает] | [В чём разрыв] | 🔴/🟡/🟢 |

2. Приоритизировать по матрице Impact × Effort:

   | | Low Effort | High Effort |
   |--|:----------:|:-----------:|
   | **High Impact** | 🎯 Quick Wins (P1) | 💎 Big Bets (P2) |
   | **Low Impact** | 🔧 Fill-ins (P3) | ❌ Avoid |

3. Сформировать action plan:

   | Приоритет | Этап | Gap | Действие | Целевая метрика | Ожидаемый эффект | Effort |
   |:---------:|------|-----|----------|-----------------|:----------------:|:------:|
   | P1 | [этап] | [разрыв] | [что делать] | [метрика] | +X% | Low |

## Пример — B2B SaaS HR-платформа

**Контекст:** HR-платформа для автоматизации рекрутинга, B2B SaaS, подписка. Персона: HR-директор, компания 50-200 чел.

### Карта пути клиента

| Этап | Действия | Точки контакта | Эмоции | Боли | Метрики |
|------|----------|----------------|:------:|------|---------|
| **Awareness** | Ищет решение для закрытия 15+ вакансий | Google, LinkedIn, рекомендации коллег | 😰 Перегрузка | Не знает с чего начать сравнение | CTR 2.1%, CPA $45 |
| **Consideration** | Сравнивает 3-4 платформы, читает отзывы на G2 | Сайт, G2, демо-видео, блог | 🤔 Осторожность | Непонятно, чем отличаемся от конкурентов | Визит→лид: 4%, bounce: 62% 🔴 |
| **Decision** | Запрашивает демо, показывает CEO | Demo-call, прайсинг, trial | 😟 Страх ошибки | Нет ROI-калькулятора, CEO не понимает ценность | Лид→trial: 28%, цикл: 21 день |
| **Purchase** | Подключает trial, импортирует вакансии | Welcome email, онбординг, чат поддержки | 😊→😐 | Импорт из Excel сломан, 3 письма без ответа | Trial→paid: 18% 🔴, TTV: 5 дней |
| **Retention** | Использует 3 раза/нед, но не upsell | Email-дайджест, in-app, саппорт | 😊 Привычка | Нет интеграции с HH.ru, вручную дублирует | M1: 72%, M6: 48%, NPS: 32 🟡 |
| **Advocacy** | Молча пользуется, не рекомендует | — | 😐 Нейтрально | Нет реферальной программы, нечем поделиться | Referral: 2% 🔴 |

### Канальный маппинг

| Этап | Онлайн | Офлайн | Основной | Согласованность |
|------|--------|--------|----------|:---------------:|
| Awareness | Google Ads, LinkedIn | HR-конференции | LinkedIn | ✅ |
| Consideration | Сайт, G2, блог | — | Сайт | ⚠️ Сайт не отражает фичи с G2 |
| Decision | Demo-call (Zoom) | — | Demo-call | ✅ |
| Purchase | Онбординг в продукте | — | In-app | ⚠️ Welcome email не связан с онбордингом |
| Retention | Email, in-app | — | In-app | ✅ |
| Advocacy | — | — | Нет канала | 🔴 Канал отсутствует |

### Moments of Truth

| # | Момент | Этап | Текущее | Риск | Влияние |
|---|--------|------|---------|:----:|---------|
| 1 | Первый визит на сайт | Consideration | Bounce 62%, messaging размытый | 4 | Визит→лид |
| 2 | Первый импорт вакансий (aha-moment) | Purchase | Excel-импорт сломан у 30% | 5 | Trial→paid |
| 3 | Первый тикет в поддержку | Retention | Ответ >24ч, 3 письма без ответа | 4 | Churn, NPS |

### Gap-анализ → Action Plan

| Приоритет | Этап | Gap | Действие | Метрика | Эффект | Effort |
|:---------:|------|-----|----------|---------|:------:|:------:|
| P1 🎯 | Purchase | Excel-импорт сломан | Починить парсер + добавить drag-n-drop | Trial→paid | +8% | Low |
| P1 🎯 | Consideration | Bounce 62%, нет дифференциации | Редизайн hero-секции: «Закрывайте вакансии в 2× быстрее» | Bounce rate | -15pp | Low |
| P2 💎 | Retention | Нет интеграции с HH.ru | API-интеграция с HH.ru | M6 retention | +10pp | High |
| P2 💎 | Advocacy | Нет реферальной программы | Запустить «пригласи HR-коллегу → месяц бесплатно» | Referral rate | +5% | Med |
| P3 🔧 | Decision | Нет ROI-калькулятора | Интерактивный калькулятор на сайте | Лид→trial | +3% | Low |

## Валидация (Quality Gate)

- [ ] Все 6 этапов пути покрыты (Awareness → Consideration → Decision → Purchase → Retention → Advocacy)
- [ ] Для каждого этапа: действия, эмоции, точки контакта, боли, возможности
- [ ] Метрики указаны для каждого этапа (реальные или бенчмарки с пометкой)
- [ ] Retention и Advocacy проанализированы отдельно
- [ ] Выделены минимум 3 Moments of Truth с оценкой риска и влиянием на метрику
- [ ] Канальный маппинг: онлайн + офлайн каналы на каждом этапе, согласованность оценена
- [ ] Петли и альтернативные пути отмечены (клиент не всегда идёт линейно)
- [ ] Карта привязана к конкретной персоне (не абстрактному клиенту)
- [ ] Gap-анализ с приоритизацией Impact × Effort
- [ ] Action plan с целевыми метриками и ожидаемым эффектом
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$customer-journey-mapping` является входом для:
- **Стратег / Медиатор** — понимание клиентского пути для стратегических решений.
- **`icp-buyer-persona`** — обогащение персон данными о реальном поведении и болях.
- **`rfm-analysis`** — точки для увеличения частоты и суммы покупок (этап Retention).
- **`unit-economics`** — оптимизация CAC через улучшение конверсии на ключевых этапах.
- **`cohort-analysis`** — точки churn для углублённого когортного анализа.
- **`jtbd-analysis`** — jobs на каждом этапе для проработки value proposition.

Формат передачи: карта пути + Moments of Truth + gap-анализ + action plan. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Карта без персоны | Абстрактный путь, не привязан к реальному поведению | Всегда привязывать к конкретной персоне |
| Только онлайн-каналы | Пропущена половина пути клиента | Канальный маппинг: онлайн + офлайн на каждом этапе |
| Нет эмоций | Сухая схема без понимания клиента | Фиксировать эмоциональное состояние на каждом этапе |
| Линейный путь | Клиенты петляют, возвращаются, пропускают этапы | Отмечать петли и альтернативные пути |
| Без gap-анализа | Карта ради карты, нет actionable-выводов | Всегда завершать gap-анализом + Impact × Effort |
| Идеализированный путь | Рисуем «как хотим», а не «как есть» | Начинать с as-is, потом строить to-be |
| Слишком много деталей | Карта нечитаема, теряется фокус | 5-7 точек контакта на этап максимум |
| Retention + Advocacy в одном | Удержание ≠ рекомендация, разные механики и метрики | Анализировать как отдельные этапы |
| Нет метрик | Качественная карта без цифр — половина ценности | Конверсия, NPS, time-to-value, churn на каждом этапе |
