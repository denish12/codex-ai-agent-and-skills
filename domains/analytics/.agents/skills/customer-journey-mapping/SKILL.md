# Карта путешествия клиента — Анализ точек контакта от осведомлённости до адвокации

## Когда использовать
- При проектировании нового продукта / сервиса — для понимания полного пути клиента.
- При оптимизации воронки конверсии — выявление узких мест и точек оттока.
- При запуске омниканальной стратегии — согласование опыта через все каналы.
- При росте churn rate — диагностика моментов разочарования и потерь.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Продукт / сервис | ✅ | Что анализируем |
| Целевая персона | ✅ | Для кого строим карту (из `$icp-buyer-persona` или описание) |
| Бизнес-модель | ✅ | B2B / B2C / SaaS / Marketplace и т.д. |
| Текущие каналы | ✅ | Онлайн и офлайн точки контакта |
| Известные проблемы | ⬚ | Где уже есть жалобы / низкая конверсия |
| Данные аналитики | ⬚ | GA, Hotjar, CRM-воронки, NPS, отзывы |
| Конкуренты | ⬚ | Для бенчмаркинга опыта |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — бенчмарки CX по отрасли, лучшие практики customer journey.
2. **Отзывы и обратная связь** — NPS-данные, отзывы на площадках, тикеты поддержки.
3. **Аналитика продукта** — воронки конверсии, тепловые карты, сессионные записи.
4. **CRM-данные** — длина цикла продажи, причины отказов, точки повторных покупок.
5. **Интервью / опросы** — если предоставлены пользователем.

> Для каждого этапа указать источники, подтверждающие паттерн поведения.

## Протокол

### Шаг 0 — Сбор контекста
Проверить наличие всех обязательных входных данных. Определить тип карты: текущее состояние (as-is) или целевое (to-be). По умолчанию — as-is.

### Шаг 1 — Определение персоны
- Зафиксировать ключевые характеристики персоны: цели, контекст, уровень технической грамотности.
- Определить начальную точку (триггер входа в путешествие).
- Определить конечную точку (что считается успешным завершением).

### Шаг 2 — Awareness (Осведомлённость)
- Как клиент впервые узнаёт о продукте / проблеме.
- Каналы первого касания (поиск, соцсети, рекомендации, реклама).
- Эмоциональное состояние: любопытство, неосознанная потребность, срочная боль.
- Действия клиента на этом этапе.
- Болевые точки и барьеры.

### Шаг 3 — Consideration (Рассмотрение)
- Как клиент сравнивает варианты и углубляет исследование.
- Точки контакта: сайт, демо, отзывы, case studies, сравнительные обзоры.
- Критерии принятия решения для данной персоны.
- Вопросы и сомнения клиента.
- Болевые точки и барьеры.

### Шаг 4 — Decision (Принятие решения)
- Финальные действия перед покупкой / подпиской.
- Точки контакта: прайсинг, триал, demo-call, контракт.
- Кто ещё влияет на решение (ЛПР, stakeholders).
- Типичные возражения и как они разрешаются.
- Болевые точки и барьеры.

### Шаг 5 — Purchase / Onboarding (Покупка / Активация)
- Процесс оформления покупки или подписки.
- Первый опыт использования (FTUE — First Time User Experience).
- Ожидания vs реальность: момент «aha-moment» или разочарования.
- Точки контакта: оплата, welcome-серия, онбординг, поддержка.
- Болевые точки и барьеры.

### Шаг 6 — Retention / Advocacy (Удержание / Адвокация)
- Регулярное использование и формирование привычки.
- Триггеры повторных покупок или расширения (upsell / cross-sell).
- Моменты восторга (delight moments) и разочарования (frustration).
- Переход в адвокацию: рекомендации, отзывы, UGC.
- Болевые точки и барьеры.

### Шаг 7 — Moments of Truth
Выделить критические моменты, определяющие дальнейшее поведение клиента:
- **Момент истины #1:** первое впечатление (соответствие ожиданиям).
- **Момент истины #2:** первый успех (достижение цели с продуктом).
- **Момент истины #3:** первая проблема (качество поддержки и реакции).
- Для каждого момента — текущее состояние, риск и рекомендация.

### Шаг 8 — Gap-анализ и рекомендации
- Выявить разрывы между ожиданиями клиента и реальным опытом.
- Приоритизировать проблемы по влиянию на бизнес (конверсия, retention, NPS).
- Сформировать конкретные рекомендации с указанием этапа и точки контакта.

## Валидация (Quality Gate)

- [ ] Все 5 этапов пути покрыты (Awareness → Advocacy)
- [ ] Для каждого этапа указаны: действия, эмоции, точки контакта, боли, возможности
- [ ] Выделены минимум 3 момента истины (Moments of Truth)
- [ ] Карта привязана к конкретной персоне (не абстрактному клиенту)
- [ ] Болевые точки конкретны и actionable
- [ ] Проведён gap-анализ с приоритизацией
- [ ] Рекомендации привязаны к конкретным этапам и точкам контакта
- [ ] Источники данных указаны для ключевых паттернов поведения

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$customer-journey-mapping` является входом для:
- **Стратег / Медиатор** — понимание клиентского пути для стратегических решений.
- `$icp-buyer-persona` — обогащение персон данными о реальном поведении.
- `$rfm-analysis` — точки для увеличения частоты и суммы покупок.
- `$unit-economics` — оптимизация CAC через улучшение конверсии на ключевых этапах.

При передаче — использовать `$handoff` с указанием ключевых findings и приоритетных gap'ов.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Карта без персоны | Абстрактный путь, не привязан к реальному поведению | Всегда привязывать к конкретной персоне |
| Только онлайн-каналы | Пропущена половина пути клиента | Включать офлайн: звонки, встречи, мероприятия |
| Нет эмоций | Сухая схема без понимания клиента | Фиксировать эмоциональное состояние на каждом этапе |
| Линейный путь | Клиенты петляют, возвращаются, пропускают этапы | Отмечать петли и альтернативные пути |
| Без gap-анализа | Карта ради карты, нет actionable-выводов | Всегда завершать gap-анализом и рекомендациями |
| Идеализированный путь | Рисуем «как хотим», а не «как есть» | Начинать с as-is, потом строить to-be |
| Слишком много деталей | Карта нечитаема, теряется фокус | 5-7 точек контакта на этап максимум |
