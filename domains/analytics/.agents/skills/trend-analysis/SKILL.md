# Анализ трендов — Идентификация и оценка рыночных трендов

## Когда использовать
- При стратегическом планировании — понять, куда движется рынок на горизонте 1-5 лет.
- При поиске новых возможностей — выявить emerging-тренды до того, как они станут мейнстримом.
- При оценке рисков — определить тренды, которые могут подорвать текущую бизнес-модель.
- При подготовке инвестиционных материалов — тренды как обоснование рыночного тайминга.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Домен / отрасль | ✅ | В какой области ищем тренды |
| Горизонт | ✅ | Временной горизонт: 1 год / 3 года / 5 лет |
| Продукт / бизнес | ⬚ | Для оценки влияния трендов на конкретный бизнес |
| Фокус | ⬚ | Технологии / Поведение потребителей / Регуляция / Всё |
| География | ⬚ | Глобально или конкретные регионы |
| Известные тренды | ⬚ | Что уже отслеживается (для расширения списка) |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — свежие отчёты аналитиков (McKinsey, BCG, Deloitte, CB Insights, a16z).
2. **Технологические источники** — Gartner Hype Cycle, TechCrunch, Wired, MIT Tech Review.
3. **Отраслевые отчёты** — профильные аналитические агентства и ассоциации.
4. **Академические публикации** — исследования университетов и research labs.
5. **Социальные сигналы** — обсуждения на Reddit, HN, Twitter, профильные конференции.
6. **Патентные базы** — USPTO, EPO (для технологических трендов).

> Для каждого тренда — минимум 2 независимых источника. Данные старше 12 месяцев для быстро меняющихся областей — пометить ⚠️.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `pest-analysis` | Макрофакторы (P/E/S/T/E/L) как структурированный источник трендов (шаг 2) | Если нужен системный обзор макросреды |
| `web-research` | Основной инструмент сбора данных по трендам (шаги 2-4) | На каждом шаге — для поиска и верификации |
| `competitive-analysis` | Технологические и стратегические тренды конкурентов (шаг 3) | Для понимания, кто уже адаптируется |
| `tam-sam-som` | Размер рынков, создаваемых трендами, CAGR (шаг 6) | Для количественной оценки impact |
| `blue-ocean-strategy` | Тренды, создающие новые рыночные пространства | Если emerging-тренд = потенциальный Blue Ocean |
| `cohort-analysis` | Поведенческие тренды в retention / engagement (шаг 3) | Для трендов в потребительском поведении |

## Протокол

### Шаг 0 — Сбор контекста
1. Проверить наличие обязательных данных.
2. Определить широту поиска: узкий домен или широкий обзор.
3. Зафиксировать горизонт и географию.
4. Определить слои анализа:

   | Слой | Примеры трендов | Источники |
   |------|----------------|-----------|
   | Технологии | AI, blockchain, IoT, quantum | Gartner, TechCrunch, патенты |
   | Поведение потребителей | Remote work, sustainability, wellness | McKinsey, Reddit, опросы |
   | Регуляция | GDPR, AI Act, ESG-требования | Consultant Plus, EUR-Lex |
   | Экономика | Инфляция, рецессия, рост EM | IMF, World Bank, ЦБ |
   | Демография | Старение, урбанизация, миграция | UN, Росстат, Eurostat |

### Шаг 1 — Макро-тренды (Mega Trends)
Через веб-поиск выявить 3-5 глобальных макро-трендов, наиболее релевантных для домена:
- Технологические сдвиги.
- Демографические изменения.
- Экономические факторы.
- Регуляторные изменения.
- Социальные сдвиги.

> Если есть `$pest-analysis` — использовать факторы со Score ≥ 12 как кандидатов в макро-тренды.

### Шаг 2 — Отраслевые тренды (Industry Trends)
Углубиться в конкретную отрасль, выявить 5-10 трендов:
- Технологические инновации внутри отрасли.
- Новые бизнес-модели и подходы.
- Изменения в поведении целевой аудитории.
- Конкурентная динамика (консолидация, новые игроки, disruption).

### Шаг 3 — Валидация (множественные источники)
Для каждого тренда проверить:

| Проверка | Критерий | Результат |
|----------|----------|:---------:|
| Минимум 2 независимых источника | Не из одной аналитической компании | ✅ / ❌ |
| Количественные данные | Adoption rate, инвестиции, рост рынка | ✅ / ⚠️ |
| Географическая широта | Наблюдается не только в 1 регионе | ✅ / ⚠️ |
| Актуальность источников | ≤ 12 мес для быстрых областей | ✅ / ⚠️ |
| Hype-фильтр | Реальный adoption, а не только buzz | ✅ / ❌ |

**Если ❌ по любому критерию — тренд помечается как «неподтверждённый» и переносится в Watch-list.**

### Шаг 4 — Оценка силы тренда
Классифицировать каждый тренд по стадии:

| Стадия | Adoption | Признаки | Пример |
|--------|:--------:|----------|--------|
| **Emerging** | <5% | Ранние сигналы, R&D, пилоты, венчурные инвестиции | Квантовые вычисления для бизнеса |
| **Growing** | 5-25% | Набирает обороты, первые коммерческие продукты, рост инвестиций | AI-генерация контента |
| **Mature** | >25% | Массовый adoption, стабильный рост, commoditization | Cloud computing |
| **Declining** | Снижается | Замещается новыми трендами, снижение инвестиций | On-premise CRM |

Для каждого — evidence (данные, подтверждающие стадию).

### Шаг 5 — Оценка влияния и scoring
Если указан конкретный продукт / бизнес — оценить влияние каждого тренда:

| Критерий | Вес | Шкала | Описание |
|----------|:---:|:-----:|----------|
| Impact | 0.35 | 1-10 | Насколько сильно повлияет на бизнес |
| Likelihood | 0.30 | 1-10 | Вероятность реализации в заданном горизонте |
| Readiness | 0.20 | 1-10 | Насколько бизнес готов к тренду (инверт.: 10 = не готов) |
| Speed | 0.15 | 1-10 | Скорость наступления (10 = внезапный) |

`Trend Score = Σ(вес × оценка)`

| Порог | Категория | Действие |
|:-----:|-----------|----------|
| ≥ 7.0 | 🔴 **Must-act** | Немедленные действия, бюджет, команда |
| 5.0-6.9 | 🟡 **Must-watch** | Мониторинг, триггер для действия |
| 3.0-4.9 | 🔵 **Experiment** | Пилотные эксперименты, low-cost тесты |
| < 3.0 | ⚪ **Ignore** | Нерелевантно для данного бизнеса |

**Влияние на бизнес-метрики** (для топ-5 трендов):

| Тренд | Метрика | Текущее | Прогноз (с трендом) | Δ |
|-------|---------|:-------:|:-------------------:|:-:|
| [Тренд 1] | TAM | $XB | $XB | +X% |
| [Тренд 1] | Маржа | XX% | XX% | ±Xpp |
| [Тренд 2] | CAC | $XX | $XX | ±X% |

### Шаг 6 — Timeline и Adoption
Для каждого ключевого тренда (Score ≥ 5.0):

| Элемент | Описание |
|---------|----------|
| Текущий adoption | XX% в [регионе] |
| Прогноз adoption | XX% через [N лет] |
| Ключевые milestone'ы | [Событие 1 → год], [Событие 2 → год] |
| Tipping point | [Что должно произойти для массового перехода] |
| Факторы ускорения | [Что ускорит тренд] |
| Факторы замедления | [Что замедлит] |
| Window of opportunity | [Когда нужно войти, чтобы не опоздать] |

### Шаг 7 — Рекомендации
На основе scoring и timeline:

| Категория | Тренды | Конкретные действия |
|-----------|--------|---------------------|
| 🔴 **Must-act** | [Тренды с Score ≥ 7.0] | [Действие + бюджет + срок + ответственный] |
| 🟡 **Must-watch** | [Score 5.0-6.9] | [Что мониторить + триггер для перехода в Must-act] |
| 🔵 **Experiment** | [Score 3.0-4.9] | [Эксперимент + бюджет + метрика успеха] |
| ⚪ **Ignore** | [Score < 3.0] | [Почему нерелевантно + условие пересмотра] |

## Валидация (Quality Gate)

- [ ] Выявлены минимум 3 макро-тренда и 5 отраслевых трендов
- [ ] Для каждого тренда — минимум 2 независимых источника; не прошедшие валидацию — в Watch-list
- [ ] Каждый тренд классифицирован по стадии (Emerging / Growing / Mature / Declining) с adoption %
- [ ] Trend Score рассчитан по формуле (Impact × 0.35 + Likelihood × 0.30 + Readiness × 0.20 + Speed × 0.15)
- [ ] Тренды категоризированы: must-act (≥7.0) / must-watch (5.0-6.9) / experiment (3.0-4.9) / ignore (<3.0)
- [ ] Для must-act трендов: влияние на конкретные бизнес-метрики (TAM, маржа, CAC, retention)
- [ ] Timeline и tipping points определены для трендов с Score ≥ 5.0
- [ ] Hype-фильтр пройден: отфильтрованы тренды без реального adoption
- [ ] Рекомендации конкретны: действие + бюджет + срок (must-act), триггер (must-watch), эксперимент (experiment)
- [ ] Источники актуальны (≤ 12 мес для быстрых областей); устаревшие помечены ⚠️

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$trend-analysis` является входом для:
- **Стратег / Медиатор** — тренды как контекст для стратегических решений.
- **`tam-sam-som`** — рост рынка на основе трендов (CAGR, драйверы).
- **`competitive-analysis`** — тренды, определяющие конкурентную динамику.
- **`pest-analysis`** — обогащение макрофакторов трендовыми данными.
- **`jtbd-analysis`** — новые Jobs, порождённые трендами.
- **`blue-ocean-strategy`** — emerging-тренды как источник новых рыночных пространств.

Формат передачи: матрица трендов с Trend Score + влияние на метрики + рекомендации по категориям. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Один источник на тренд | Риск принять hype за тренд | Минимум 2 независимых источника; 1 = ⚠️ |
| Без классификации стадии | Непонятно, как реагировать | Emerging / Growing / Mature / Declining + adoption % |
| Тренды без количественных данных | Невозможно оценить масштаб | Adoption rate, инвестиции, рост рынка |
| Только технологические тренды | Пропущены регуляторные и социальные | 5 слоёв: техно, поведение, регуляция, экономика, демография |
| Без timeline | Непонятно, когда действовать | Timeline + tipping point + window of opportunity |
| Все тренды = must-act | Нет приоритизации, паралич | Scoring → 4 категории с порогами |
| Устаревшие источники | Тренды меняются быстро | ≤ 12 мес для быстрых областей |
| Impact без метрик | «Сильно повлияет» — не actionable | Влияние на TAM (±X%), маржу (±Xpp), CAC (±X%) |
| Hype без substance | Buzz ≠ тренд | Hype-фильтр: реальный adoption, 2+ источника, >1 региона |
