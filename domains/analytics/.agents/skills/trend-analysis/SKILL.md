# Анализ трендов — Идентификация и оценка рыночных трендов

## Когда использовать
- При стратегическом планировании — понять, куда движется рынок на горизонте 1-5 лет.
- При поиске новых возможностей — выявить emerging-тренды до того, как они станут мейнстримом.
- При оценке рисков — определить тренды, которые могут подорвать текущую бизнес-модель.
- При подготовке инвестиционных материалов — тренды как обоснование рыночного тайминга.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Домен / отрасль | ✅ | В какой области ищем тренды |
| Горизонт | ✅ | Временной горизонт: 1 год / 3 года / 5 лет |
| Продукт / бизнес | ⬚ | Для оценки влияния трендов на конкретный бизнес |
| Фокус | ⬚ | Технологии / Поведение потребителей / Регуляция / Всё |
| География | ⬚ | Глобально или конкретные регионы |
| Известные тренды | ⬚ | Что уже отслеживается (для расширения списка) |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — свежие отчёты аналитиков (McKinsey, BCG, Deloitte, CB Insights, a16z).
2. **Технологические источники** — Gartner Hype Cycle, TechCrunch, Wired, MIT Tech Review.
3. **Отраслевые отчёты** — профильные аналитические агентства и ассоциации.
4. **Академические публикации** — исследования университетов и research labs.
5. **Социальные сигналы** — обсуждения на Reddit, HN, Twitter, профильные конференции.
6. **Патентные базы** — USPTO, EPO (для технологических трендов).

> Для каждого тренда — минимум 2 независимых источника, подтверждающих его существование.

## Протокол

### Шаг 0 — Сбор контекста
Проверить наличие обязательных данных. Определить широту поиска: узкий домен или широкий обзор. Зафиксировать горизонт и географию.

### Шаг 1 — Определение домена и горизонта
- Чётко определить границы исследуемого домена.
- Зафиксировать горизонт: short-term (1 год), mid-term (2-3 года), long-term (5+ лет).
- Определить слои анализа: технологии, потребительское поведение, регуляция, экономика, общество.

### Шаг 2 — Макро-тренды (Mega Trends)
Через веб-поиск выявить глобальные макро-тренды, влияющие на домен:
- Технологические сдвиги (AI, blockchain, IoT, quantum, etc.).
- Демографические изменения (старение, урбанизация, миграция).
- Экономические факторы (инфляция, рецессия, рост EM).
- Регуляторные изменения (GDPR, AI Act, ESG-требования).
- Социальные сдвиги (remote work, sustainability, wellness).
- Отобрать 3-5 макро-трендов, наиболее релевантных для домена.

### Шаг 3 — Отраслевые тренды (Industry Trends)
Углубиться в конкретную отрасль:
- Технологические инновации внутри отрасли.
- Новые бизнес-модели и подходы.
- Изменения в поведении целевой аудитории.
- Конкурентная динамика (консолидация, новые игроки, disruption).
- Отобрать 5-10 отраслевых трендов.

### Шаг 4 — Валидация (множественные источники)
Для каждого тренда проверить:
- Минимум 2 независимых источника подтверждают тренд.
- Есть количественные данные (размер рынка, adoption rate, инвестиции).
- Тренд наблюдается не только в одном регионе / компании.
- Источники актуальны (не старше 12 месяцев для быстро меняющихся областей).
- Исключить hype без substance (шум без реального adoption).

### Шаг 5 — Оценка силы тренда
Классифицировать каждый тренд по стадии:
- **Emerging** — зарождающийся, ранние сигналы, низкий adoption (<5%).
- **Growing** — растущий, набирает обороты, средний adoption (5-25%).
- **Mature** — зрелый, массовый adoption (>25%), стабильный рост.
- **Declining** — затухающий, замещается новыми трендами.
- Для каждого — evidence (данные, подтверждающие стадию).

### Шаг 6 — Оценка влияния на бизнес
Если указан конкретный продукт / бизнес — оценить влияние каждого тренда:
- **Impact** (1-10): насколько сильно повлияет на бизнес.
- **Likelihood** (1-10): вероятность реализации в заданном горизонте.
- **Readiness** (1-10): насколько бизнес готов к этому тренду.
- Классифицировать: угроза / возможность / нейтральный.

### Шаг 7 — Timeline и Adoption
Для каждого ключевого тренда:
- Ожидаемый timeline массового adoption.
- Ключевые milestone'ы (регуляция, технологическая зрелость, killer app).
- Факторы ускорения и замедления.
- Tipping points: что должно произойти для массового перехода.

### Шаг 8 — Рекомендации
На основе анализа сформировать:
- **Must-watch** — тренды, которые нужно мониторить непрерывно.
- **Must-act** — тренды, требующие немедленных действий.
- **Experiment** — тренды для пилотных экспериментов.
- **Ignore** — тренды, нерелевантные для данного бизнеса.
- Конкретные шаги для каждой категории.

## Валидация (Quality Gate)

- [ ] Выявлены минимум 3 макро-тренда и 5 отраслевых трендов
- [ ] Для каждого тренда — минимум 2 независимых источника
- [ ] Каждый тренд классифицирован по стадии (Emerging/Growing/Mature/Declining)
- [ ] Оценено влияние на бизнес (если указан продукт)
- [ ] Определён timeline и ключевые milestone'ы для топ-трендов
- [ ] Рекомендации конкретны: must-watch / must-act / experiment / ignore
- [ ] Источники актуальны (не старше 12 месяцев для быстрых областей)
- [ ] Отфильтрован hype без substance

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$trend-analysis` является входом для:
- **Стратег / Медиатор** — тренды как контекст для стратегических решений.
- `$tam-sam-som` — рост рынка на основе трендов (CAGR, драйверы).
- `$competitive-analysis` — тренды, определяющие конкурентную динамику.
- `$jtbd-analysis` — новые Jobs, порождённые трендами.
- `$unit-economics` — влияние трендов на CAC, LTV, маржу.

При передаче — использовать `$handoff` с ранжированной матрицей трендов и рекомендациями.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Один источник на тренд | Риск принять hype за тренд | Минимум 2 независимых источника |
| Без классификации стадии | Непонятно, как реагировать | Emerging / Growing / Mature / Declining |
| Тренды без количественных данных | Невозможно оценить масштаб | Adoption rate, объём инвестиций, размер рынка |
| Только технологические тренды | Пропущены регуляторные и социальные сдвиги | Анализировать все слои: техно, регуляция, социум |
| Без timeline | Непонятно, когда действовать | Для каждого тренда — ожидаемый горизонт |
| Все тренды = must-act | Нет приоритизации, паралич | Чёткая категоризация: watch / act / experiment / ignore |
| Устаревшие источники | Тренды меняются быстро | Источники не старше 12 месяцев |
