# SWOT-анализ — Анализ сильных и слабых сторон, возможностей и угроз

## Когда использовать
- При оценке **стратегического положения** компании, продукта или проекта перед принятием ключевых решений.
- При подготовке к **стратегическому планированию** — как входные данные для формирования стратегии.
- При необходимости **системно оценить** внутренние возможности и внешние условия одновременно.
- При проведении **конкурентного анализа** — сравнение SWOT нескольких игроков.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Объект анализа | ✅ | Компания, продукт, проект или подразделение |
| Отрасль / рынок | ✅ | Контекст рынка и индустрии |
| Период анализа | ⬚ | Горизонт планирования (по умолчанию — 1-3 года) |
| Конкуренты | ⬚ | Список основных конкурентов для контекста |
| Стратегические цели | ⬚ | Текущие цели и приоритеты организации |
| Ограничения | ⬚ | Бюджет, ресурсы, регуляторные рамки |

## Источники данных
1. Внутренняя отчётность — финансы, операционные метрики, HR-данные.
2. Маркетинговые исследования — доля рынка, узнаваемость бренда, NPS.
3. Конкурентный анализ — бенчмарки, публичные отчёты конкурентов.
4. Макросреда — отраслевые отчёты, регуляторные изменения, технологические тренды.
5. Обратная связь стейкхолдеров — интервью, опросы, CRM-данные.

## Протокол

### Шаг 0 — Подготовка контекста
1. Уточнить объект анализа, границы и горизонт.
2. Определить ключевых стейкхолдеров и их перспективы.
3. Собрать доступные данные из источников (шаг 1-5 из «Источники данных»).
4. Зафиксировать допущения и ограничения анализа.

### Шаг 1 — Сильные стороны (Strengths, внутренние)
1. Провести аудит внутренних ресурсов: финансовые, человеческие, технологические, репутационные.
2. Оценить уникальные компетенции и конкурентные преимущества.
3. Проверить устойчивость каждой сильной стороны (легко ли её скопировать?).
4. Ранжировать по степени влияния на стратегические цели (высокое / среднее / низкое).
5. Для каждой — указать доказательную базу (метрика, факт, бенчмарк).

### Шаг 2 — Слабые стороны (Weaknesses, внутренние)
1. Провести аудит внутренних дефицитов: ресурсы, процессы, компетенции, культура.
2. Определить разрывы относительно конкурентов и лучших практик.
3. Оценить критичность каждой слабости (блокирует стратегию или ограничивает?).
4. Определить причинно-следственные связи между слабостями.
5. Ранжировать по степени влияния и срочности устранения.

### Шаг 3 — Возможности (Opportunities, внешние)
1. Сканировать внешнюю среду: рыночные тренды, технологии, регуляторика, демография.
2. Оценить доступность каждой возможности (ресурсы для реализации).
3. Определить окно возможности (timing) и вероятность реализации.
4. Оценить потенциальный эффект на бизнес (выручка, доля, позиция).
5. Ранжировать: привлекательность = эффект x вероятность / затраты.

### Шаг 4 — Угрозы (Threats, внешние)
1. Идентифицировать внешние риски: конкуренция, регуляторика, макроэкономика, технологии.
2. Оценить вероятность реализации каждой угрозы.
3. Оценить потенциальный ущерб при реализации.
4. Определить скорость наступления (внезапная / постепенная).
5. Ранжировать: критичность = ущерб x вероятность.

### Шаг 5 — Перекрёстная матрица (Cross-impact)
1. **SO-стратегии** (Strengths + Opportunities): как использовать сильные стороны для захвата возможностей.
2. **WO-стратегии** (Weaknesses + Opportunities): как преодолеть слабости через возможности.
3. **ST-стратегии** (Strengths + Threats): как использовать сильные стороны для нейтрализации угроз.
4. **WT-стратегии** (Weaknesses + Threats): как минимизировать слабости и избежать угроз.
5. Для каждой пары — сформулировать конкретную стратегическую инициативу.

### Шаг 6 — Приоритизация
1. Оценить каждую инициативу из перекрёстной матрицы по: влияние, реализуемость, срочность.
2. Построить матрицу приоритизации (Impact vs Effort).
3. Выделить Quick Wins, стратегические проекты и «парковку».
4. Сформировать ТОП-5 приоритетных действий с ответственными и сроками.

### Шаг 7 — Синтез и выводы
1. Сформулировать стратегический вердикт: общая оценка позиции.
2. Определить критические факторы успеха.
3. Зафиксировать ключевые риски и план митигации.
4. Подготовить actionable-рекомендации с горизонтом реализации.

## Валидация (Quality Gate)

- [ ] Каждый квадрант SWOT содержит минимум 3 фактора
- [ ] Факторы подкреплены доказательной базой (метрики, факты, источники)
- [ ] Внутренние факторы (S/W) отделены от внешних (O/T)
- [ ] Перекрёстная матрица содержит конкретные стратегические инициативы, а не общие фразы
- [ ] Приоритизация основана на объективных критериях (влияние, реализуемость, срочность)
- [ ] ТОП-5 действий имеют ответственных и сроки
- [ ] Нет дублирования факторов между квадрантами
- [ ] Анализ учитывает временной горизонт

## Handoff
Результат -> Стратег / Медиатор для интеграции в стратегический план, портфельный анализ или дорожную карту.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Слишком общие формулировки | «Сильная команда» не даёт actionable-инсайтов | Конкретика: «3 senior ML-инженера с опытом NLP > 5 лет» |
| Смешение внутренних и внешних факторов | Размывает границу анализа | S/W — только то, что контролируется; O/T — внешнее |
| SWOT без перекрёстной матрицы | Список без стратегии бесполезен | Всегда строить SO/WO/ST/WT и формулировать инициативы |
| Отсутствие приоритизации | Всё важно = ничего не важно | Ранжировать по Impact x Feasibility |
| Игнорирование доказательной базы | Анализ на «ощущениях» необъективен | Каждый фактор — с метрикой или источником |
| Одноразовый SWOT | Устаревает за 3-6 месяцев | Пересматривать регулярно, привязать к OKR-циклу |
