# SWOT-анализ — Анализ сильных и слабых сторон, возможностей и угроз

## Когда использовать
- При оценке **стратегического положения** компании, продукта или проекта перед принятием ключевых решений.
- При подготовке к **стратегическому планированию** — как входные данные для формирования стратегии.
- При необходимости **системно оценить** внутренние возможности и внешние условия одновременно.
- При проведении **конкурентного анализа** — сравнение SWOT нескольких игроков.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Объект анализа | ✅ | Компания, продукт, проект или подразделение |
| Отрасль / рынок | ✅ | Контекст рынка и индустрии |
| Период анализа | ⬚ | Горизонт планирования (по умолчанию — 1-3 года) |
| Конкуренты | ⬚ | Список основных конкурентов для контекста |
| Стратегические цели | ⬚ | Текущие цели и приоритеты организации |
| Ограничения | ⬚ | Бюджет, ресурсы, регуляторные рамки |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. Внутренняя отчётность — финансы, операционные метрики, HR-данные.
2. Маркетинговые исследования — доля рынка, узнаваемость бренда, NPS.
3. Конкурентный анализ — бенчмарки, публичные отчёты конкурентов.
4. Макросреда — отраслевые отчёты, регуляторные изменения, технологические тренды.
5. Обратная связь стейкхолдеров — интервью, опросы, CRM-данные.

> Каждый факт — с указанием источника и даты. Данные старше 12 месяцев — пометить ⚠️.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `pest-analysis` | Внешние факторы → O/T: возможности и угрозы из PESTEL (шаги 3-4) | Если O/T не определены — PESTEL даёт структурированный вход |
| `competitive-analysis` | S/W относительно конкурентов: доли, фичи, моаты (шаги 1-2) | Если нет данных о конкурентных преимуществах/слабостях |
| `porters-five-forces` | Угрозы (T) из 5 сил: новые участники, субституты, rivalry (шаг 4) | Если нужна структурированная оценка конкурентных угроз |
| `ansoff-matrix` | SO-стратегии → стратегии роста (шаг 5) | После перекрёстной матрицы — для проработки SO-инициатив |
| `web-research` | Верификация внешних факторов, отраслевые данные | Если данные отсутствуют или устарели (>12 мес) |
| `blue-ocean-strategy` | WO-стратегии → создание нового пространства | Если слабости блокируют конкуренцию в текущей отрасли |

## Протокол

### Шаг 0 — Подготовка контекста
1. Уточнить объект анализа, границы и горизонт.
2. Определить ключевых стейкхолдеров и их перспективы.
3. Собрать доступные данные из источников.
4. Зафиксировать допущения и ограничения анализа.

### Шаг 1 — Сильные стороны (Strengths, внутренние)
1. Провести аудит внутренних ресурсов: финансовые, человеческие, технологические, репутационные.
2. Оценить уникальные компетенции и конкурентные преимущества.
3. Проверить устойчивость каждой сильной стороны (легко ли скопировать?).
4. Для каждой — scoring и доказательная база:

   | # | Сильная сторона | Влияние (1-5) | Устойчивость (1-5) | Score | Доказательство | Источник |
   |---|----------------|:-------------:|:-------------------:|:-----:|-----------------|----------|
   | S1 | [Конкретная формулировка] | X | X | XX | [Метрика / факт] | [Источник] |

   `Score = Влияние × Устойчивость`. Топ-факторы: Score ≥ 12.

> **Правило формулировки:** не «сильная команда», а «3 senior ML-инженера с опытом NLP >5 лет, публикации в EMNLP 2024-2025».

### Шаг 2 — Слабые стороны (Weaknesses, внутренние)
1. Провести аудит внутренних дефицитов: ресурсы, процессы, компетенции, культура.
2. Определить разрывы относительно конкурентов и лучших практик.
3. Оценить критичность и срочность:

   | # | Слабая сторона | Критичность (1-5) | Срочность (1-5) | Score | Доказательство | Источник |
   |---|---------------|:------------------:|:---------------:|:-----:|-----------------|----------|
   | W1 | [Конкретная формулировка] | X | X | XX | [Метрика / gap] | [Источник] |

   `Score = Критичность × Срочность`. Топ-факторы: Score ≥ 12.

4. Определить причинно-следственные связи между слабостями (W1 вызывает W3?).

### Шаг 3 — Возможности (Opportunities, внешние)
1. Сканировать внешнюю среду: рыночные тренды, технологии, регуляторика, демография.
2. Если есть `$pest-analysis` — использовать факторы-возможности (🟢) оттуда.
3. Оценить каждую возможность:

   | # | Возможность | Потенциал (1-5) | Вероятность (1-5) | Окно | Score | Источник |
   |---|-------------|:---------------:|:-----------------:|------|:-----:|----------|
   | O1 | [Описание] | X | X | [Когда закроется] | XX | [Источник] |

   `Score = Потенциал × Вероятность`. Топ-факторы: Score ≥ 12.

### Шаг 4 — Угрозы (Threats, внешние)
1. Идентифицировать внешние риски: конкуренция, регуляторика, макроэкономика, технологии.
2. Если есть `$pest-analysis` — использовать факторы-риски (🔴) оттуда.
3. Если есть `$porters-five-forces` — использовать силы с баллом ≥ 3.5.
4. Оценить каждую угрозу:

   | # | Угроза | Ущерб (1-5) | Вероятность (1-5) | Скорость | Score | Источник |
   |---|--------|:-----------:|:-----------------:|----------|:-----:|----------|
   | T1 | [Описание] | X | X | Внезапная / Постепенная | XX | [Источник] |

   `Score = Ущерб × Вероятность`. Топ-факторы: Score ≥ 12.

### Шаг 5 — Перекрёстная матрица (Cross-impact)
Для каждого квадранта — конкретные стратегические инициативы:

1. **SO-стратегии** (Strengths + Opportunities): как использовать сильные стороны для захвата возможностей.
2. **WO-стратегии** (Weaknesses + Opportunities): как преодолеть слабости через возможности.
3. **ST-стратегии** (Strengths + Threats): как использовать сильные стороны для нейтрализации угроз.
4. **WT-стратегии** (Weaknesses + Threats): как минимизировать слабости и избежать угроз.

Для каждой инициативы:

| # | Тип | Факторы | Инициатива | Ожидаемый эффект |
|---|:---:|---------|------------|------------------|
| 1 | SO | S1 + O2 | [Конкретное действие] | [Результат: +$XM, +X% доли] |

> Минимум 2 инициативы на квадрант. Каждая — конкретное действие, не общая фраза.

### Шаг 6 — Приоритизация инициатив
Оценить каждую инициативу по скоринговой модели:

| Критерий | Вес | Шкала |
|----------|:---:|:-----:|
| Impact (влияние на бизнес) | 0.35 | 1-10 |
| Feasibility (реализуемость) | 0.30 | 1-10 |
| Urgency (срочность) | 0.20 | 1-10 |
| Risk (инвертированный риск провала) | 0.15 | 1-10 |

`Priority Score = Σ(вес × оценка)`

Матрица Impact × Effort:

| | Low Effort | High Effort |
|--|:----------:|:-----------:|
| **High Impact** | 🎯 Quick Wins (P1) | 💎 Strategic Bets (P2) |
| **Low Impact** | 🔧 Fill-ins (P3) | ❌ Avoid |

ТОП-5 действий — с ответственными, сроками и KPI.

### Шаг 7 — Синтез и выводы
1. Сформулировать стратегический вердикт: общая оценка позиции.
2. Определить критические факторы успеха (CSF).
3. Зафиксировать ключевые риски и план митигации.
4. Подготовить actionable-рекомендации с горизонтом.
5. Определить периодичность пересмотра (привязать к OKR-циклу, обычно каждые 3-6 мес).

## Пример — EdTech-платформа корпоративного обучения, РФ

**Контекст:** B2B SaaS, платформа корпоративного обучения. 50 клиентов, $2M ARR, рынок РФ. Горизонт: 1-3 года.

### Матрица SWOT

**Strengths:**

| # | Фактор | Влияние | Устойч. | Score | Доказательство |
|---|--------|:-------:|:-------:|:-----:|----------------|
| S1 | AI-персонализация обучения (единственные на рынке РФ) | 5 | 4 | **20** | Патентная заявка, 3 ML-инженера |
| S2 | NPS 72 (vs среднее по отрасли 45) | 4 | 3 | **12** | Опрос Q1 2026, N=180 |
| S3 | Time-to-deploy 3 дня (vs конкуренты 2-4 недели) | 4 | 4 | **16** | CRM: среднее по последним 20 внедрениям |

**Weaknesses:**

| # | Фактор | Критичн. | Срочн. | Score | Доказательство |
|---|--------|:--------:|:------:|:-----:|----------------|
| W1 | Нет интеграции с 1С (table stakes на рынке РФ) | 5 | 5 | **25** | 30% отказов в CRM ссылаются на 1С |
| W2 | Команда продаж 3 чел. (vs 15+ у Skillbox B2B) | 4 | 4 | **16** | Headcount report |
| W3 | Нет мобильного приложения | 3 | 3 | **9** | 15% запросов в поддержку |

**Opportunities:**

| # | Фактор | Потенциал | Вероятн. | Score | Источник |
|---|--------|:---------:|:--------:|:-----:|----------|
| O1 | Госпрограмма «Кадры» — субсидии на корп. обучение | 5 | 4 | **20** | Минтруд, мар 2026 |
| O2 | Дефицит кадров → рост бюджетов на upskilling (+28% YoY) | 5 | 5 | **25** | HH.ru Research, 2026 |
| O3 | Уход иностранных LMS (SAP SuccessFactors, Cornerstone) | 4 | 4 | **16** | Коммерсантъ, фев 2026 |

**Threats:**

| # | Фактор | Ущерб | Вероятн. | Score | Источник |
|---|--------|:-----:|:--------:|:-----:|----------|
| T1 | Яндекс Практикум запускает B2B-направление | 5 | 4 | **20** | TechCrunch Russia, янв 2026 |
| T2 | ФЗ о перс. данных — ужесточение хранения | 4 | 5 | **20** | Consultant Plus, текущая редакция |
| T3 | Инфляция 8-10% — давление на бюджеты HR | 3 | 5 | **15** | ЦБ РФ, прогноз 2026-2027 |

### Перекрёстная матрица

| # | Тип | Факторы | Инициатива | Эффект |
|---|:---:|---------|------------|--------|
| 1 | SO | S1 + O2 | AI-модуль «ROI на обучение»: показать компаниям, что upskilling окупается | +$1M ARR за 12 мес |
| 2 | SO | S3 + O3 | Экспресс-миграция с SAP/Cornerstone: deploy за 3 дня + импорт данных | 30 новых enterprise-клиентов |
| 3 | WO | W1 + O1 | Интеграция с 1С → доступ к госсубсидиям (1С обязателен для отчётности) | -30% отказов, +субсидии |
| 4 | WO | W2 + O2 | Нанять 5 sales + партнёрская программа (бюджеты растут — есть кому продавать) | ×3 pipeline |
| 5 | ST | S1 + T1 | Усилить AI как моат: Яндекс не может скопировать за 6 мес | Удержать дифференциацию |
| 6 | ST | S2 + T3 | Использовать высокий NPS для реферальной программы (снизить CAC при инфляции) | CAC -20% |
| 7 | WT | W1 + T2 | Интеграция с 1С + миграция на РФ-серверы = compliance bundle | Закрыть оба блокера |

### Приоритизация

| # | Инициатива | Тип | Impact | Feasib. | Urgency | Risk inv. | **Score** | Зона |
|---|------------|:---:|:------:|:-------:|:-------:|:---------:|:---------:|:----:|
| 7 | 1С + РФ-серверы (compliance bundle) | WT | 9 | 7 | 10 | 6 | **8.3** | 🎯 P1 |
| 3 | 1С → госсубсидии | WO | 8 | 7 | 8 | 7 | **7.7** | 🎯 P1 |
| 2 | Экспресс-миграция с SAP | SO | 9 | 8 | 7 | 6 | **7.8** | 🎯 P1 |
| 5 | Усилить AI как моат | ST | 8 | 6 | 8 | 5 | **7.0** | 💎 P2 |
| 4 | Нанять 5 sales + партнёры | WO | 7 | 5 | 6 | 6 | **6.1** | 💎 P2 |

### ТОП-5 действий

1. **1С + РФ-серверы** (Score 8.3) — CTO, Q2 2026. KPI: 0% compliance-отказов.
2. **Экспресс-миграция с SAP** (Score 7.8) — Head of Sales, Q2-Q3 2026. KPI: 30 enterprise-клиентов.
3. **Интеграция 1С → госсубсидии** (Score 7.7) — Product Lead, Q2 2026. KPI: -30% отказов.
4. **AI-моат** (Score 7.0) — ML Team Lead, Q3-Q4 2026. KPI: 2 новых AI-фичи.
5. **Sales expansion** (Score 6.1) — VP Sales, Q3 2026. KPI: pipeline ×3.

## Валидация (Quality Gate)

- [ ] Каждый квадрант SWOT содержит минимум 3 фактора с числовым scoring
- [ ] Scoring: S (Влияние × Устойчивость), W (Критичность × Срочность), O (Потенциал × Вероятность), T (Ущерб × Вероятность)
- [ ] Факторы подкреплены доказательной базой (метрики, факты, источники)
- [ ] Внутренние факторы (S/W) отделены от внешних (O/T)
- [ ] Перекрёстная матрица содержит минимум 2 инициативы на квадрант (SO/WO/ST/WT)
- [ ] Инициативы конкретны: действие + ожидаемый эффект (не общие фразы)
- [ ] Приоритизация по скоринговой модели (Impact, Feasibility, Urgency, Risk)
- [ ] ТОП-5 действий с ответственными, сроками и KPI
- [ ] Нет дублирования факторов между квадрантами
- [ ] Анализ учитывает временной горизонт, определена периодичность пересмотра
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

## Handoff
Результат `$swot-analysis` является входом для:
- **Стратег / Медиатор** — стратегическая позиция для формирования стратегии.
- **`ansoff-matrix`** — SO-инициативы → стратегии роста.
- **`blue-ocean-strategy`** — WO-инициативы → создание нового рыночного пространства.
- **`competitive-analysis`** — S/W для конкурентного бенчмаркинга.
- **`pest-analysis`** — O/T как вход для углублённого макроанализа.

Формат передачи: матрица SWOT с scoring + перекрёстная матрица + приоритизированные инициативы + ТОП-5. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Слишком общие формулировки | «Сильная команда» не даёт actionable-инсайтов | Конкретика: «3 senior ML-инженера с опытом NLP > 5 лет» |
| Смешение внутренних и внешних | Размывает границу анализа | S/W — только контролируемое; O/T — внешнее |
| SWOT без перекрёстной матрицы | Список без стратегии бесполезен | SO/WO/ST/WT с конкретными инициативами (min 2 на квадрант) |
| Отсутствие приоритизации | Всё важно = ничего не важно | Скоринг: Impact × Feasibility × Urgency × Risk |
| Факторы без доказательств | Анализ на «ощущениях» необъективен | Каждый фактор — с метрикой или источником |
| Одноразовый SWOT | Устаревает за 3-6 месяцев | Привязать к OKR-циклу, пересматривать каждые 3-6 мес |
| Факторы без scoring | Нельзя сравнить S1 и S3 объективно | Числовой scoring per quadrant: Влияние × Устойчивость и т.д. |
| Инициативы без эффекта | «Усилить продажи» — не actionable | Конкретный эффект: «+$1M ARR за 12 мес», «-30% отказов» |
