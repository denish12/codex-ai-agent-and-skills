# Конкурентный анализ — Бенчмаркинг конкурентов

## Когда использовать
- При выходе на новый рынок — для понимания конкурентного ландшафта.
- При разработке стратегии позиционирования — дифференциация от конкурентов.
- При ценообразовании — бенчмаркинг цен и пакетов конкурентов.
- При обнаружении новых конкурентов или изменении конкурентной динамики.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Продукт / сервис | ✅ | Что анализируем в конкурентном контексте |
| Рынок / ниша | ✅ | В какой категории конкурируем |
| Известные конкуренты | ⬚ | Список, если есть; иначе будут определены |
| Ключевые фичи продукта | ⬚ | Для построения матрицы сравнения |
| Текущее ценообразование | ⬚ | Текущая ценовая модель для бенчмарка |
| Целевая аудитория | ⬚ | Из `$icp-buyer-persona`, если доступно |
| Фокус анализа | ⬚ | Функционал / Цены / Позиционирование / Полный |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — сайты конкурентов, обзоры, рейтинги (G2, Capterra, TrustRadius, Product Hunt).
2. **Финансовые данные** — публичная отчётность, раунды финансирования (Crunchbase, PitchBook).
3. **Маркетинговые материалы** — лэндинги, блоги, case studies конкурентов.
4. **Социальные сигналы** — отзывы клиентов, обсуждения на Reddit, HN, профильных форумах.
5. **Аналитические отчёты** — Gartner Magic Quadrant, Forrester Wave, отраслевые обзоры.

> Для каждого конкурента — минимум 2 независимых источника информации.

## Протокол

### Шаг 0 — Сбор контекста
Проверить наличие обязательных входных данных. Определить фокус и глубину анализа. Запросить недостающее у пользователя.

### Шаг 1 — Идентификация конкурентов
Классифицировать конкурентов по трём категориям:
- **Прямые** — решают ту же проблему для того же сегмента тем же способом.
- **Непрямые** — решают ту же проблему другим способом или для другого сегмента.
- **Аспирационные** — лидеры рынка, на которых стоит равняться (даже если в другой категории).
- Итого: 3-5 прямых + 2-3 непрямых + 1-2 аспирационных.

### Шаг 2 — Матрица функционала
Построить сравнительную таблицу по ключевым фичам:
- Определить 10-20 ключевых функций / критериев.
- Сгруппировать по категориям (Core, Advanced, Integrations, Support).
- Оценить каждого конкурента: есть / нет / частично / лучше всех.
- Выделить уникальные фичи каждого конкурента.

### Шаг 3 — Анализ ценообразования
- Собрать публичные тарифы всех конкурентов.
- Привести к единой модели для сравнения ($/мес, $/пользователь, $/транзакция).
- Определить ценовые стратегии: premium / value / freemium / usage-based.
- Выявить скрытые расходы (onboarding, support, integrations).

### Шаг 4 — Карта позиционирования
Построить позиционирование по двум ключевым осям:
- Выбрать оси на основе приоритетов рынка (цена vs функционал, простота vs мощность, etc.).
- Разместить всех конкурентов на карте.
- Выявить пустые зоны (незанятые позиции).
- Определить кластеры конкурентов.

### Шаг 5 — Сильные и слабые стороны
Для каждого конкурента:
- **Сильные стороны** — чем лучше (технология, бренд, цена, UX, экосистема).
- **Слабые стороны** — чем хуже (ограничения, жалобы, технический долг).
- **Источники** — откуда данные (отзывы, тесты, публичная информация).

### Шаг 6 — Конкурентные рвы (Moats)
Определить устойчивые преимущества каждого конкурента:
- Сетевые эффекты, переключающие издержки, бренд.
- Технологический барьер, патенты, эксклюзивные данные.
- Экономия масштаба, экосистема интеграций.
- Оценить прочность рва: слабый / средний / сильный.

### Шаг 7 — Стратегические рекомендации
На основе анализа — рекомендации по позиционированию:
- Где дифференцироваться (незанятые ниши, слабые стороны конкурентов).
- Какие фичи критичны для конкуренции (table stakes).
- Ценовая стратегия на фоне рынка.
- Риски: что конкуренты могут скопировать / улучшить.

## Валидация (Quality Gate)

- [ ] Определены минимум 3 прямых + 2 непрямых конкурента
- [ ] Матрица функционала содержит минимум 10 критериев
- [ ] Ценообразование приведено к единой модели для сравнения
- [ ] Построена карта позиционирования по 2 осям
- [ ] Для каждого конкурента указаны сильные и слабые стороны с источниками
- [ ] Определены конкурентные рвы (moats) для ключевых конкурентов
- [ ] Рекомендации по позиционированию конкретны и actionable
- [ ] Нет устаревших данных (все данные о ценах и фичах актуальны)

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$competitive-analysis` является входом для:
- **Стратег / Медиатор** — конкурентный ландшафт для стратегических решений.
- `$tam-sam-som` — рыночные доли конкурентов для оценки SOM.
- `$icp-buyer-persona` — позиционирование влияет на выбор целевых персон.
- `$jtbd-analysis` — конкурирующие решения для каждого Job.

При передаче — использовать `$handoff` с матрицей конкурентов и ключевыми рекомендациями.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Только прямые конкуренты | Пропускаем угрозы из смежных рынков | Включать непрямых и аспирационных |
| Копирование фич лидера | «Гонка за фичами» без стратегии | Дифференцироваться, а не копировать |
| Анализ без источников | Субъективные оценки без доказательств | Каждый тезис подкреплять данными |
| Устаревшие цены | Конкуренты меняют прайсинг часто | Проверять актуальность на сайте конкурента |
| Фокус только на фичах | Фичи — не единственная ось конкуренции | Анализировать UX, бренд, экосистему, поддержку |
| Слишком много конкурентов | Потеря фокуса, поверхностный анализ | 5-8 ключевых, остальные — краткий обзор |
| Без рекомендаций | Анализ ради анализа | Завершать actionable-стратегией |
