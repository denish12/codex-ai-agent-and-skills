# Конкурентный анализ — Бенчмаркинг конкурентов

## Когда использовать
- При выходе на новый рынок — для понимания конкурентного ландшафта.
- При разработке стратегии позиционирования — дифференциация от конкурентов.
- При ценообразовании — бенчмаркинг цен и пакетов конкурентов.
- При обнаружении новых конкурентов или изменении конкурентной динамики.

## Вход

| Поле | Обязательно | Описание |
|------|:-----------:|----------|
| Продукт / сервис | ✅ | Что анализируем в конкурентном контексте |
| Рынок / ниша | ✅ | В какой категории конкурируем |
| Известные конкуренты | ⬚ | Список, если есть; иначе будут определены |
| Ключевые фичи продукта | ⬚ | Для построения матрицы сравнения |
| Текущее ценообразование | ⬚ | Текущая ценовая модель для бенчмарка |
| Целевая аудитория | ⬚ | Из `$icp-buyer-persona`, если доступно |
| Фокус анализа | ⬚ | Функционал / Цены / Позиционирование / Полный |

> Если обязательные поля не предоставлены — **запросить у пользователя** до начала анализа. Не додумывать.

## Источники данных
1. **Веб-поиск** — сайты конкурентов, обзоры, рейтинги (G2, Capterra, TrustRadius, Product Hunt).
2. **Финансовые данные** — публичная отчётность, раунды финансирования (Crunchbase, PitchBook).
3. **Маркетинговые материалы** — лэндинги, блоги, case studies конкурентов.
4. **Социальные сигналы** — отзывы клиентов, обсуждения на Reddit, HN, профильных форумах.
5. **Аналитические отчёты** — Gartner Magic Quadrant, Forrester Wave, отраслевые обзоры.

> Для каждого конкурента — минимум 2 независимых источника информации. Каждый факт — с указанием источника и даты. Данные старше 12 месяцев — пометить ⚠️ и верифицировать через `web-research`.

### Связь с другими скилами
| Скил | Что берём | Когда вызывать |
|------|-----------|----------------|
| `web-research` | Данные о конкурентах, цены, фичи, отзывы (шаги 1-6) | Основной инструмент сбора — вызывать на каждом шаге |
| `swot-analysis` | Сильные/слабые стороны нашего продукта для сравнения (шаг 5) | Если нет внутренней оценки S/W |
| `tam-sam-som` | Размер рынка, доли для оценки масштаба конкурентов (шаг 1) | Если неизвестен размер рынка |
| `blue-ocean-strategy` | Незанятые позиции, ERRC для дифференциации (шаг 4, 7) | Если анализ выявил перенасыщенную зону |
| `icp-buyer-persona` | Целевые персоны для оценки конкурентного overlap | Если нет данных о целевой аудитории |

## Протокол

### Шаг 0 — Сбор контекста
1. Проверить наличие обязательных входных данных.
2. Определить фокус анализа:

   | Фокус | Глубина | Шаги |
   |-------|---------|------|
   | Функционал | Матрица фич + качество реализации | 0-2, 5, 7 |
   | Цены | Тарифы + модель + скрытые расходы | 0-1, 3, 7 |
   | Позиционирование | Карта + messaging + аудитория | 0-1, 4, 7 |
   | Полный | Всё вышеперечисленное | 0-7 |

3. Запросить недостающие данные у пользователя.
4. Определить горизонт: текущий snapshot + динамика за последние 12 месяцев.

### Шаг 1 — Идентификация конкурентов
1. Классифицировать конкурентов по трём категориям:
   - **Прямые** — решают ту же проблему для того же сегмента тем же способом (3-5).
   - **Непрямые** — решают ту же проблему другим способом или для другого сегмента (2-3).
   - **Аспирационные** — лидеры рынка, на которых стоит равняться (1-2).
2. Для каждого конкурента собрать карточку:

   | Поле | Описание |
   |------|----------|
   | Название | Юрлицо / бренд |
   | Тип | Прямой / Непрямой / Аспирационный |
   | Год основания | Зрелость |
   | Размер | Сотрудники, выручка (если публично) |
   | Финансирование | Раунды, инвесторы (если стартап) |
   | Целевой сегмент | Кому продают |
   | Ключевая фича | Главная дифференциация |
   | Динамика (12 мес) | Растёт ↗ / Стабилен → / Снижается ↘ |

### Шаг 2 — Матрица функционала
1. Определить 10-20 ключевых функций / критериев.
2. Сгруппировать по категориям (Core, Advanced, Integrations, Support).
3. Оценить каждого конкурента по шкале качества реализации:

   | Оценка | Значение |
   |:------:|----------|
   | ★★★ | Лучший в классе — конкурентное преимущество |
   | ★★ | Хорошая реализация — на уровне рынка |
   | ★ | Базовая реализация — есть, но слабая |
   | — | Отсутствует |

4. Подсчитать score: ★★★ = 3, ★★ = 2, ★ = 1, — = 0. Итого по каждому конкуренту.
5. Выделить уникальные фичи каждого конкурента (есть только у одного).
6. Определить table stakes — фичи, которые есть у всех (обязательный минимум).

### Шаг 3 — Анализ ценообразования
1. Собрать публичные тарифы всех конкурентов.
2. Привести к единой модели для сравнения:

   | Конкурент | Модель | $/мес (начальная) | $/мес (средний) | $/мес (enterprise) | Free tier | Стратегия |
   |-----------|--------|:-----------------:|:---------------:|:-----------------:|:---------:|-----------|
   | [A] | Per seat | $XX | $XX | По запросу | Да / Нет | Premium |

3. Определить ценовые стратегии: premium / value / freemium / usage-based.
4. Выявить скрытые расходы (onboarding, support, integrations, минимальный контракт).
5. Рассчитать ценовой индекс: `наша цена / средняя по конкурентам × 100`.

### Шаг 4 — Карта позиционирования
1. Выбрать оси на основе приоритетов рынка:

   | Популярные пары осей | Когда использовать |
   |---------------------|--------------------|
   | Цена × Функционал | Классическое сравнение value vs features |
   | Простота × Мощность | Для продуктов с разным UX-подходом |
   | Вертикализация × Горизонтальность | Нишевые vs универсальные решения |
   | Self-serve × Sales-led | Модель дистрибуции |

2. Разместить всех конкурентов на карте (оценка 1-10 по каждой оси).
3. Выявить кластеры конкурентов (зоны скопления).
4. Выявить пустые зоны (незанятые позиции — потенциал для дифференциации).
5. Определить рекомендуемую позицию для нашего продукта.

### Шаг 5 — Сильные и слабые стороны
Для каждого конкурента:

| Аспект | Сильные стороны | Слабые стороны | Источник |
|--------|----------------|----------------|----------|
| Технология | [+] | [-] | [Источник, дата] |
| UX / дизайн | [+] | [-] | [Источник, дата] |
| Бренд / репутация | [+] | [-] | [Источник, дата] |
| Цена / value | [+] | [-] | [Источник, дата] |
| Экосистема / интеграции | [+] | [-] | [Источник, дата] |
| Поддержка / сервис | [+] | [-] | [Источник, дата] |

> Каждый тезис подкреплять данными: отзыв, рейтинг, факт. Не «хороший UX», а «4.7/5 на G2 (234 отзыва), +0.3 vs среднее по категории».

### Шаг 6 — Конкурентные рвы (Moats) и угрозы
1. Определить устойчивые преимущества каждого конкурента:

   | Тип рва | Описание | Примеры |
   |---------|----------|---------|
   | Сетевые эффекты | Ценность растёт с числом пользователей | Маркетплейсы, соцсети |
   | Switching costs | Высокая стоимость миграции | ERP, CRM с глубокой интеграцией |
   | Бренд | Узнаваемость и доверие | Enterprise-бренды |
   | Технологический барьер | Патенты, уникальная технология | AI-модели, проприетарные данные |
   | Экономия масштаба | Себестоимость падает с ростом | Инфраструктурные продукты |
   | Экосистема | Lock-in через интеграции и плагины | Платформы с app store |

2. Оценить прочность рва: слабый / средний / сильный.
3. **Threat score** — ранжировать конкурентов по совокупной угрозе:

   | Критерий | Вес | Шкала |
   |----------|:---:|:-----:|
   | Overlap по аудитории | 0.25 | 1-10 |
   | Overlap по функционалу | 0.20 | 1-10 |
   | Прочность рва | 0.20 | 1-10 |
   | Динамика роста | 0.20 | 1-10 |
   | Ресурсы (финансирование, команда) | 0.15 | 1-10 |

   `Threat Score = Σ(вес × оценка)`. Ранг: **≥ 7.0 High**, **4.0-6.9 Medium**, **< 4.0 Low**.

### Шаг 7 — Динамика и тренды (12 месяцев)
1. Для каждого ключевого конкурента зафиксировать изменения за последние 12 месяцев:

   | Конкурент | Изменения фич | Изменения цен | Новые сегменты | Финансирование | Стратегический разворот |
   |-----------|:-------------:|:-------------:|:--------------:|:--------------:|:----------------------:|
   | [A] | +3 major фичи | Повышение на 15% | Вход в enterprise | Series C, $50M | Нет |

2. Определить тренды: куда движется рынок в целом.
3. Выявить потенциальные новые угрозы: стартапы, big tech, смежные рынки.

### Шаг 8 — Стратегические рекомендации
На основе анализа:
1. **Позиционирование** — рекомендуемая позиция на карте, messaging.
2. **Table stakes** — фичи, обязательные для конкуренции (есть у всех прямых конкурентов).
3. **Дифференциация** — уникальное преимущество, основанное на слабостях конкурентов и пустых зонах.
4. **Ценообразование** — рекомендуемая стратегия и уровень на фоне рынка.
5. **Risky bets** — что конкуренты могут скопировать / улучшить, на что ставить.
6. **Рекомендуемые действия:**

   | # | Действие | Цель | Против кого | Приоритет | Срок |
   |---|----------|------|-------------|:---------:|------|
   | 1 | [Что делать] | [Какой эффект] | [Конкурент] | P1/P2/P3 | [Когда] |

## Пример — CRM для SMB-сегмента

**Контекст:** Новая CRM-платформа для малого бизнеса, рынок РФ. Ценовая модель: per seat. Фокус: полный анализ.

### Конкурентный ландшафт

| Конкурент | Тип | Сегмент | Год | Размер | Динамика |
|-----------|-----|---------|-----|--------|:--------:|
| amoCRM | Прямой | SMB-Mid | 2009 | ~600 чел. | → |
| Битрикс24 | Прямой | SMB-Ent | 2012 | ~1500 чел. | ↗ |
| Мегаплан | Прямой | SMB | 2008 | ~200 чел. | ↘ |
| HubSpot | Аспирационный | SMB-Ent (глоб.) | 2006 | 7000+ чел. | ↗ |
| Notion | Непрямой | SMB (PM→CRM) | 2013 | ~500 чел. | ↗ |

### Матрица функционала (фрагмент)

| Критерий | Наш продукт | amoCRM | Битрикс24 | Мегаплан | HubSpot |
|----------|:-----------:|:------:|:---------:|:--------:|:-------:|
| Воронка продаж | ★★ | ★★★ | ★★ | ★ | ★★★ |
| Email-интеграция | ★★ | ★★ | ★★ | ★ | ★★★ |
| Телефония | ★ | ★★★ | ★★ | ★★ | ★ |
| AI-ассистент | ★★★ | — | ★ | — | ★★ |
| Мобильное приложение | ★★ | ★★ | ★ | ★ | ★★★ |
| Автоматизации | ★★ | ★★ | ★★★ | ★ | ★★★ |
| **Score** | **12** | **12** | **11** | **6** | **15** |

**Table stakes:** воронка продаж, email, мобильное приложение.
**Наша уникальная фича:** AI-ассистент (★★★, только у нас на таком уровне).

### Threat Score

| Конкурент | Overlap аудитории | Overlap фич | Ров | Динамика | Ресурсы | **Score** | **Ранг** |
|-----------|:-----------------:|:-----------:|:---:|:--------:|:-------:|:---------:|:--------:|
| amoCRM | 9 | 8 | 7 | 5 | 6 | **7.2** | 🔴 High |
| Битрикс24 | 7 | 7 | 8 | 7 | 8 | **7.3** | 🔴 High |
| Мегаплан | 8 | 5 | 3 | 3 | 3 | **4.8** | 🟡 Medium |
| HubSpot | 4 | 9 | 9 | 8 | 10 | **7.6** | 🔴 High |
| Notion | 3 | 3 | 6 | 8 | 7 | **4.9** | 🟡 Medium |

**Главные угрозы:** HubSpot (score 7.6, сильный ров + ресурсы), Битрикс24 (7.3, экосистема), amoCRM (7.2, максимальный overlap).

### Рекомендации

| # | Действие | Против кого | Приоритет |
|---|----------|-------------|:---------:|
| 1 | Усилить AI-ассистент как главный дифференциатор | amoCRM, Битрикс24 (нет AI) | P1 |
| 2 | Ценовая стратегия: value (ниже amoCRM, выше Мегаплан) | amoCRM | P1 |
| 3 | Интеграция с 1С (table stakes в РФ, у нас пока нет) | Битрикс24 | P2 |
| 4 | Позиционирование: «AI-first CRM для малого бизнеса» | Все | P1 |

## Валидация (Quality Gate)

- [ ] Определены минимум 3 прямых + 2 непрямых конкурента с карточками
- [ ] Матрица функционала содержит минимум 10 критериев с оценкой качества (★/★★/★★★)
- [ ] Table stakes и уникальные фичи явно определены
- [ ] Ценообразование приведено к единой модели для сравнения
- [ ] Построена карта позиционирования по 2 осям с кластерами и пустыми зонами
- [ ] Для каждого конкурента указаны сильные и слабые стороны с источниками и датами
- [ ] Конкурентные рвы определены для ключевых конкурентов с оценкой прочности
- [ ] Threat score рассчитан, конкуренты ранжированы по уровню угрозы
- [ ] Динамика за 12 мес зафиксирована: изменения фич, цен, стратегии
- [ ] Рекомендации конкретны, приоритизированы и привязаны к конкурентам
- [ ] Для каждого конкурента — минимум 2 независимых источника
- [ ] Источники данных указаны; данные старше 12 мес помечены ⚠️

> Если валидация не пройдена — доработать до повторного прохождения, а не передавать дальше.

## Handoff
Результат `$competitive-analysis` является входом для:
- **Стратег / Медиатор** — конкурентный ландшафт для стратегических решений.
- **`tam-sam-som`** — рыночные доли конкурентов для оценки SOM.
- **`bcg-matrix`** — относительная доля рынка (наша доля / доля лидера) для классификации БЕ.
- **`blue-ocean-strategy`** — факторы конкуренции для стратегической канвы AS-IS.
- **`ansoff-matrix`** — барьеры входа на новые рынки (шаг 2, развитие рынка).
- **`icp-buyer-persona`** — позиционирование влияет на выбор целевых персон.
- **`jtbd-analysis`** — конкурирующие решения для каждого Job.

Формат передачи: конкурентный ландшафт + матрица функционала + threat score + ключевые рекомендации. При передаче — использовать `$handoff`.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Только прямые конкуренты | Пропускаем угрозы из смежных рынков | Включать непрямых и аспирационных |
| Копирование фич лидера | «Гонка за фичами» без стратегии | Дифференцироваться, а не копировать; фокус на слабостях конкурентов |
| Анализ без источников | Субъективные оценки без доказательств | Каждый тезис — с данными: рейтинг, отзыв, факт |
| Устаревшие цены | Конкуренты меняют прайсинг часто | Верифицировать на сайте конкурента, помечать ⚠️ если >6 мес |
| Фокус только на фичах | Фичи — не единственная ось конкуренции | Анализировать UX, бренд, экосистему, поддержку, moats |
| Слишком много конкурентов | Потеря фокуса, поверхностный анализ | 5-8 ключевых, остальные — краткий обзор |
| Без рекомендаций | Анализ ради анализа | Завершать actionable-стратегией с приоритетами |
| Бинарная матрица фич (✅/⬚) | Не видно качества: «есть» ≠ «хорошо реализовано» | Шкала ★/★★/★★★ с score |
| Статичный snapshot | Не видно, куда движутся конкуренты | Динамика за 12 мес: фичи, цены, финансирование, стратегия |
